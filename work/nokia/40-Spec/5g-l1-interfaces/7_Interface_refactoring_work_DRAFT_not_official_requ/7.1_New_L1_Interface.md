## 7.1.1 Sounding Reference Signal Receive Procedure

### 7.1.1.1 General procedure information

#### 7.1.1.1.1 Overview

**7.1.1.1.1.0-1**  (ID: `12563532`)

Overview
**Messages:**

* Request messages:

[UlData\_SrsReceiveReq](https://dn-prod.ext.net.nokia.com/rm/resources/BI_6u5lQCqOEfCNjfqfyUlvmw?oslc_config.context=https%3A%2F%2Fdn-prod.ext.net.nokia.com%2Fgc%2Fconfiguration%2F9)

*UlData\_SrsPosReceiveReq*

* Response messages:

*UlData\_SrsReceiveRespPs*

*UlData\_SrsReceiveRespBwvPs*

*UlData\_SrsReceiveRespBwvPs*

*UlData\_SrsReceiveRespRtBfPs*

*UlData\_SrsPosReceiveRespPs*

**Purpose:**

Request 5G-L1-UL to receive one or multiple SRS for one or multiple SRS users.

This procedure shall be done per subcell~~­~~~~­~~. This means that IE numOfSubcell in UlData\_SrsReceiveReq is always set to 1, and 5G-L1-UL sends response messages for each subcell separately.

**Precondition:**

Subcell exists and L1 uplink addresses have been negotiated for it ([11513672](https://dn-prod.ext.net.nokia.com/rm/resources/BI_FRyW5vvxEe-AqvopbP1qhQ)).

**Addressing:**

The *UlData\_SrsReceiveReq is sent to the*srsReceiveReqAddress address obtained in *UlData\_AddressResp message.*\*\*

The address to send the *UlData\_*SrsReceiveRespPs response message is given in the request.\*\*

The address to send the *UlData\_*SrsReceiveRespPs response message is given in the request.\*\*

The address to send the *UlData\_*SrsReceiveRespBmPs response message is given in the request.\*\*

**Parallel Processing:**

List all the cases that parallel message processing is not expected or list all the cases that parallel message processing shall be supported.
