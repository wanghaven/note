### 3.1.2.3 UL Cell Parameter Reconfiguration

#### 3.1.2.3.1 Procedure

**3.1.2.3.1.0-3**  (ID: `11513562`)

Procedure
**Expected Exceptions:**

If any reconfigurable parameter from list:

For Loner:  
a) rxScalingFactor

For Loki/Thor:  
a) rxScalingFactor  
b) [CB013507-A/B] actUlNrEcidTAdv [End CB013507-A/B]  
c) [CNI-142789] firstPrachRootSeqIndex [End CNI-142789].

d) [CNI-147062-A] actPucchF1InterfEnhancement for Loki [End CNI-147062-A] and [CB014386-B] Thor [End CB014386-B]

e) [CB014094-A] srsPortPowerThreshold, srsSinrThreshold [End CB014094-A].

f) [CB014093-B] zeroCorrelationZoneConfig.[End CB014093-B]

is not valid or its online modification was not successful, then in the reported UCell\_ParameterReconfigurationResp 5G-L1-UL shall:  
  - set the IE status = EStatus5G\_NotOk (1)  
  - set the IE cause and IE diagnosticInformation according to [11513511](https://dn-prod.ext.net.nokia.com/rm/resources/BI_FRxIyfvxEe-AqvopbP1qhQ?oslc_config.context=https://dn-prod.ext.net.nokia.com/gc/configuration/9) and according to the first parameter that caused rejection of reconfiguration

[PR835728] If the subcell is marked as secondary by ulSubCellCaps field ([11513449](https://dn-prod.ext.net.nokia.com/rm/resources/BI_FRwhwfvxEe-AqvopbP1qhQ)) in UlCell\_SetupReq then 5G-L1-UL shall skip validation and reconfiguration of firstPrachRootSeqIndex  [CB014093-B] and zeroCorrelationZoneConfig.[End CB014093-B] [End PR835728]

**Unexpected Exceptions:**

In case of unexpected exception, no response message is returned.
