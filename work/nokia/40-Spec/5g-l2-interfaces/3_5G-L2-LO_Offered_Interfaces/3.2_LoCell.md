## 3.2.2 Procedures

### 3.2.2.1 Cell Group Setup

#### 3.2.2.1.0-1 Cell Group Setup Overview (ID: 10983144)

**3.2.2.1.0-1.0-1**  (ID: `13238347`)

Cell Group Setup
Messages:

LoCell\_CellGroupSetupReq/Resp

Purpose:

To setup the common parameters of a NRCELLGRP (see 5G\_UP\_2720\_replaced([11391897](https://dn-prod.ext.net.nokia.com/rm/resources/BI_ufbP1fvkEe-AqvopbP1qhQ))) in 5G-L2-LO.

Precondition:

Address distribution in LoCnfg interface is done.

Addressing:

The request is sent to the LoCell address.

The response is sent to the address where the request was sent from.

Relevant UP MSCs:

5G\_L1L2\_FLOW\_81\_replaced([11177749](https://dn-prod.ext.net.nokia.com/rm/resources/BI_BeU56_vREe-AqvopbP1qhQ)): L2-RT cell group setup

---

#### 3.2.2.1.0-2 Cell Group Setup Description (ID: 10983150)

**3.2.2.1.0-2.0-1**  (ID: `13238348`)

Cell Group Setup
Upon reception of the request, 5G-L2-LO shall update the cell group context for the common parameters share in one NRCELLGRP, which are provided in the message and all related cell resources.

The new values shall be valid for all UEs in the cell group.

---

#### 3.2.2.1.0-3 Cell Group Setup Message validation (ID: 10983154)

**3.2.2.1.0-3.0-1**  (ID: `13238358`)

Cell Group Setup
Upon reception of the LoCell\_CellGroupSetupReq message, 5G-L2-LO shall check the validity of the request message.

5G-L2-LO shall reply with LoCell\_CellGroupSetupResp message and set the status IE = EStatus5G\_Ok if the request message is valid and the cell group was setup successfully.

For exception handling, see 5G\_L2\_IF\_750\_replaced([10983162](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2V2yovu6Ee-AqvopbP1qhQ)).

#### 3.2.2.1.0-5 Cell Group Setup Exceptions (ID: 10983162)

**3.2.2.1.0-5.0-1**  (ID: `13238409`)

Cell Group Setup
Expected Exceptions:

Generic exception handling rules can be found in the chapter 5G\_UP\_1795\_replaced([11374912](https://dn-prod.ext.net.nokia.com/rm/resources/BI_ueKrmfvkEe-AqvopbP1qhQ)).

Exception handling rules specifically for object setup can be found in the requirements 5G\_UP\_1805\_replaced([11375008](https://dn-prod.ext.net.nokia.com/rm/resources/BI_ueLSpvvkEe-AqvopbP1qhQ)) and 5G\_UP\_1806\_replaced([11375035](https://dn-prod.ext.net.nokia.com/rm/resources/BI_ueLSqfvkEe-AqvopbP1qhQ)).

5G-L2-LO shall reply with LoCell\_CellGroupSetupResp message, set the status IE = EStatus5G\_NotOk, and set an appropriate cause value if:

* nrCellGrpId IE is not valid or it already exists or;
* the request message is not valid or;
* the resources were not allocated successfully.

For available cause values, see 5G\_UP\_1846\_replaced([11375790](https://dn-prod.ext.net.nokia.com/rm/resources/BI_uePkGPvkEe-AqvopbP1qhQ)).

In case of failure, if 5G-L2-LO is not able to do a proper internal rollback for all sub-systems, and the state of the 5G-L2-LO is inconsistent, 5G-L2-LO shall indicate this in the response message, set status IE = "EStatus5G\_NotOk" and set the needForRollback IE = TRUE. This is to indicate that rollback is expected from 5G-CP-RT, and that 5G-CP-RT should not presume that no resources were allocated by the 5G-L2-LO.

Unexpected Exceptions:

In case of an unexpected exception, 5G-L2-LO shall not send response message.

Error is logged for both expected and unexpected exception.

### 3.2.2.2 Cell Group Delete

#### 3.2.2.2.0-1 Cell Group Delete Overview (ID: 10983203)

**3.2.2.2.0-1.0-1**  (ID: `13254452`)

Cell Group Delete
Messages:

LoCell\_CellGroupDeleteReq/Resp

Purpose:

Cell Group and all cells (and subcells) belonging to a NRCELLGRP (see 5G\_UP\_2720\_replaced([11391897](https://dn-prod.ext.net.nokia.com/rm/resources/BI_ufbP1fvkEe-AqvopbP1qhQ))) shall be deleted by 5G-L2-LO.

Precondition:

Cell group exists.

Addressing:

The request is sent to the LoCell address.

The response is sent to the address where the request was sent from.

Relevant UP MSCs:

5G\_L1L2\_FLOW\_93\_replaced([11178229](https://dn-prod.ext.net.nokia.com/rm/resources/BI_BeXWMvvREe-AqvopbP1qhQ)): L2-RT cell delete

---

#### 3.2.2.2.0-2 Cell Group Delete Description (ID: 10983208)

**3.2.2.2.0-2.0-1**  (ID: `13254456`)

Cell Group Delete
Upon reception of the request, 5G-L2-LO shall delete the cell group context and all related cell resources (including all the subcells).

If the cell group context does not exist, 5G-L2-LO shall consider the procedure is successful and return a response accordingly.

---

**3.2.2.2.0-3.0-1**  (ID: `13254459`)

5G-L2-LO shall reply with LoCell\_CellGroupDeleteResp message if the request message is valid, the cell group identified and deleted successfully.

For exception handling, see 5G\_L2\_IF\_758\_replaced([10983220](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2V2ysPu6Ee-AqvopbP1qhQ)).

**3.2.2.3.0-2.0-1**  (ID: `10983273`)

Cell Setup

5G-L2-LO shall add a new NRCELL object for PM, see 5G\_L2\_1517\_replaced([11192164](https://dn-prod.ext.net.nokia.com/rm/resources/BI_28iOzPvTEe-AqvopbP1qhQ)).

#### 3.2.2.3.0-3 Cell Setup Message Validation (ID: 10983278)

**3.2.2.3.0-3.0-1**  (ID: `10983283`)

Cell Setup
Upon reception of the LoCell\_CellSetupReq message, 5G-L2-LO shall check the validity of the request message.

Upon reception of the message LoCell\_CellSetupReq, 5G-L2-LO DL receiver shall check the value range for physCellId, dlSubcellList, ulSubcellList and sulConfig , if any of them is out of range, the message is invalid.

5G-L2-LO shall reply with LoCell\_CellSetupResp message and set the status IE = Ok if the request message is valid and the cell was setup successfully.

For exception handling, see 5G\_L2\_IF\_768\_replaced([10983288](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2V3ZvPu6Ee-AqvopbP1qhQ)).

### 3.2.2.4 Cell Delete

#### 3.2.2.4.0-1 Cell Delete Overview (ID: 10983453)

Cell Delete
Messages:

LoCell\_CellDeleteReq/Resp

Purpose:

To delete a cell from 5G-L2-LO.

Precondition:

Address distribution is done.

Addressing:

The request is sent to the LoCell address.

The response is sent to the address where the request was sent from.

Relevant UP MSCs:

5G\_L1L2\_FLOW\_88\_replaced([11178176](https://dn-prod.ext.net.nokia.com/rm/resources/BI_BeXWLPvREe-AqvopbP1qhQ)): L2-RT cell delete and L1 subcells delete

#### 3.2.2.4.0-2 Cell Delete Description (ID: 10983458)

**3.2.2.4.0-2.0-1**  (ID: `13254464`)

Cell Delete
Upon reception of the request, 5G-L2-LO shall delete the cell context and all related resources.

5G-L2-LO shall also remove the related NRCELL object from PM, see 5G\_L2\_1517\_replaced([11192164](https://dn-prod.ext.net.nokia.com/rm/resources/BI_28iOzPvTEe-AqvopbP1qhQ)).

**[5GC001804-A] For actions required for PCMD see 5G\_L2\_2120\_replaced([11219675](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2_Sn2_vTEe-AqvopbP1qhQ)).** [End 5GC001804-A]\*\*\*\*

**[Before 5GC000559-C]**

If the cell context does not exist, 5G-L2-LO shall consider the procedure is successful and return a response accordingly.

**[End 5GC000559-C]**

#### 3.2.2.4.0-3 Cell Delete Message validation (ID: 10983464)

**3.2.2.4.0-3.0-1**  (ID: `13254465`)

Cell Delete
Upon reception of the LoCell\_CellDeleteReq message, 5G-L2-LO shall check the validity of the request message.

5G-L2-LO shall reply with LoCell\_CellDeleteResp message if the request message is valid, the cell was identified and deleted successfully.

For exception handling, see 5G\_L2\_IF\_784\_replaced([10983469](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2V4n5vu6Ee-AqvopbP1qhQ)).

#### 3.2.2.4.0-4 Cell Delete Exceptions (ID: 10983469)

**3.2.2.4.0-4.0-1**  (ID: `13254466`)

Exception handling rules specifically for object deletion can be found in the requirements 5G\_UP\_1815\_replaced([11375190](https://dn-prod.ext.net.nokia.com/rm/resources/BI_ueMgwfvkEe-AqvopbP1qhQ)) and 5G\_UP\_1816\_replaced([11375218](https://dn-prod.ext.net.nokia.com/rm/resources/BI_ueMgxPvkEe-AqvopbP1qhQ)).

#### 3.2.2.5.0-4 Cell Reconfiguration Exceptions (ID: 10983521)

Expected Exceptions:
