#### 4.5.2.1.0-1 Address Distribution Overview (ID: 10989499)

**4.5.2.1.0-1.0-1**  (ID: `13434097`)

Address Distribution
Messages:

HiCnfgDu\_AddressDistributionReq/Resp

Purpose:

To distribute addresses to 5G-L2-HI(DU) and to enquire 5G-L2-HI(DU) addresses.

Precondition:

5G-L2-HI has been started.

Addressing:

The request is sent to the HiCnfgDu address.

The response is sent to the address where the request was sent from.

Relevant UP MSCs:

5G\_L1L2\_FLOW\_72\_replaced([11177073](https://dn-prod.ext.net.nokia.com/rm/resources/BI_BeRPmfvREe-AqvopbP1qhQ)): L2RT and L2NRT Instance Configuration

#### 4.5.2.1.0-2 Address Distribution Description (ID: 10989506)

**4.5.2.1.0-2.0-2**  (ID: `10989517`)

Address Distribution
Upon reception of the request, 5G-L2-HI shall store the given addresses and then return its own dynamic addresses in the related HiCnfgDu\_AddressDistributionResp.

5G-L2-HI shall create a new NRDU object instance for PM, see 5G\_L2\_2776\_replaced([11233420](https://dn-prod.ext.net.nokia.com/rm/resources/BI_3Aqg0PvTEe-AqvopbP1qhQ))/5G\_L2\_2778\_replaced([11233560](https://dn-prod.ext.net.nokia.com/rm/resources/BI_3ArH6vvTEe-AqvopbP1qhQ)).

[Before CNI-109641-A]

In case rd1514TeIdMappingLogicFlag is set to ‘1’,

* 5G-L2-HI shall perform the actions described in 5G\_L2\_IF\_971\_replaced([10986749](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2WRpYvu6Ee-AqvopbP1qhQ)).
* Populate TEID mapping table, numTeidMappingEntries and default queue IE.

In case rd1514TeIdMappingLogicFlag is set to ‘0’,

Set  numTeidMappingEntries IE to '0'.

[CNI-109641-A]

Due to common code for Classical and Cloud, rd1514TeIdMappingLogicFlag is removed (there is one common TEID format that can be found under 5G\_L2\_15625\_replaced([11221929](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2_h4avvTEe-AqvopbP1qhQ))).

[End CNI-109641-A]

#### 4.5.2.1.0-3 Address Distribution Message validation (ID: 10989524)

**4.5.2.1.0-3.0-1**  (ID: `10989530`)

Address Distribution
Upon reception of the HiCnfgDu\_AddressDistributionReq message, 5G-L2-HI shall check the validity of the request message.

5G-L2-HI shall reply with HiCnfgDu\_AddressDistributionResp message and set the status IE = Ok if the request message is valid and the addresses and pools were configured successfully.

Note:

1. 5G-L2-Hi shall reject any procedure until the address distribution procedure is completed.
2. 5G-L2-HI cannot validate that distributed address/parameter values are correct. Only ranges can be checked where applicable.
3. [CB007431-A] l2HiInstanceId value in HiCnfgCu\_AddressDistributionReq and HiCnfgDu\_AddressDistributionReq shall be the same for a given L2-HI container on Classical. [End CB007431-A]

For exception handling, see 5G\_L2\_IF\_1160\_replaced([10989535](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2WmZgvu6Ee-AqvopbP1qhQ)).

#### 4.5.2.2.0-4 Configuration Parameters Exceptions (ID: 10989619)

**4.5.2.2.0-4.0-4**  (ID: `13420069`)

Configuration Parameters
**Expected Exceptions:**

Generic exception handling rules can be found in the chapter 5G\_UP\_1795\_replaced([11374912](https://dn-prod.ext.net.nokia.com/rm/resources/BI_ueKrmfvkEe-AqvopbP1qhQ)).

For any other expected exceptions,

5G-L2-HI shall reply with HiCnfgDu\_ConfigParamResp message, set the status IE = NotOk, and set an appropriate cause value if:

* the request message is not valid,
* the parameters were not configured successfully.

Set status IE: EStatus5G\_NotOk with cause IE: ECause5G\_InvalidParameter (2) if:

* nrCellIdsForCountersReporting IE is out of range,
* The number of PMQAP profiles configured are more than 40 or out of range,
* **[CNI-150183] ( PmCountersInfo--downlinkThptHistParams AND PmCountersInfo--uplinkThptHistParams ) structures not present  AND IE cfgThptDistUpdate is present,**
* ( PmCountersInfo--downlinkThptHistParams OR PmCountersInfo--uplinkThptHistParams ) structures is present  AND IE cfgThptDistUpdate is not present,
* IE minRange is present AND IE scale set to linear,
* IE minRange is not present AND IE scale set to logarithmic,
* nrPmqapId provided in HistogramPmqap does not exist in the pmqapProfileList **[End CNI-150183]**

For available cause values, see 5G\_UP\_1846\_replaced([11375790](https://dn-prod.ext.net.nokia.com/rm/resources/BI_uePkGPvkEe-AqvopbP1qhQ)).

**Unexpected Exceptions:**

In case of unexpected exception, no response message is returned.

*Note:* Currently 5G-L2-HI always responds with OK to the request.\*\*

In case of an unexpected exception, 5G-L2-HI shall not send response message.

Error is logged for both expected and unexpected exception.
