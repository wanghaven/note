#### 4.4.2.1.0-2 UPUE VM Load Measurement Description (ID: 10989300)

**4.4.2.1.0-2.0-2**  (ID: `10989312`)

UPUE VM Load Measurement
On classical BTS only, upon reception of the request, 5G-L2-HI shall store the Load measurement configuration and start the load measurement as per 5G\_UP\_ALG\_13836.

[CB007078-E]

5G-L2-HI shall process the request as idempotent message. It means that upon reception of the request with parameters already configured, 5G-L2-HI shall process it and recognize that nothing has changed, in other cases 5G-L2-HI shall override the values and start using the new ones.

In both situations, 5G-L2-HI shall send the related response message with OK status.

[End CB007078-E]

---

#### 4.4.2.1.0-3 UPUE VM Load Measurement Message validation (ID: 10989319)

**4.4.2.1.0-3.0-1**  (ID: `13407802`)

UPUE VM Load Measurement
Upon reception of the HiMeasCu\_LoadMeasSetupReq message, 5G-L2-HI shall check the validity of the request message.

5G-L2-HI shall reply with HiMeasCu\_LoadMeasSetupResp message and set the status IE = Ok if the request message is valid and measurement was configured successfully.

For exception handling, see 5G\_L2\_IF\_1141\_replaced([10989324](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2WkkZfu6Ee-AqvopbP1qhQ)).

#### 4.4.2.1.0-4 UPUE VM Load Measurement Exceptions (ID: 10989324)

**4.4.2.1.0-4.0-1**  (ID: `13407804`)

UPUE VM Load Measurement
Expected Exceptions:

[5GC002184-H]

Generic exception handling rules can be found in the chapter 5G\_UP\_1795\_replaced([11374912](https://dn-prod.ext.net.nokia.com/rm/resources/BI_ueKrmfvkEe-AqvopbP1qhQ)).

5G-L2-HI shall reply with HiMeasCu\_LoadMeasSetupResp message, set the status IE = NotOk, and set an appropriate cause value if the request message is not valid or if the measurement was not configured successfully.

L2-HI CU reporting used resources
Messages:

HiMeasCu\_GetResourceUsageReq/Resp

Purpose:

To report used resources in 5G-L2-HI (CU).
