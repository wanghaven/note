#### 4.9.2.3.0-2 Cell Setup Description (ID: 10991606)

**4.9.2.3.0-2.0-1**  (ID: `10991612`)

Cell Setup
Upon reception of the message, 5G-L2-HI shall build the Cell list with parameter values given in the message.

5G-L2-HI shall register the cell and its capabilities towards the BTC via add() as defined in 5G\_L2\_11514\_replaced([11231417](https://dn-prod.ext.net.nokia.com/rm/resources/BI_3AbQSPvTEe-AqvopbP1qhQ)).

#### 4.9.2.3.0-3 Cell Setup Message validation: (ID: 10991618)

**4.9.2.3.0-3.0-1**  (ID: `10991625`)

Cell Setup
Upon reception of the HiCellDu\_CellSetupReq message, 5G-L2-HI shall check the validity of the request message to ensure that each IE is within the range.

5G-L2-HI shall reply with HiCellDu\_CellSetupResp message and set the status IE = Ok if the request message is valid.

For exception handling, see 5G\_L2\_IF\_3721\_replaced([10987369](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2WWh4Pu6Ee-AqvopbP1qhQ)).

---

#### 4.9.2.3.0-4 Cell Setup Expected Exceptions: (ID: 10991630)

**4.9.2.3.0-4.0-1**  (ID: `10991635`)

Cell Setup
Generic exception handling rules can be found in the chapter 5G\_UP\_1795\_replaced([11374912](https://dn-prod.ext.net.nokia.com/rm/resources/BI_ueKrmfvkEe-AqvopbP1qhQ)).

Exception handling rules specifically for object setup can be found in the requirements 5G\_UP\_1805\_replaced([11375008](https://dn-prod.ext.net.nokia.com/rm/resources/BI_ueLSpvvkEe-AqvopbP1qhQ)) and 5G\_UP\_1806\_replaced([11375035](https://dn-prod.ext.net.nokia.com/rm/resources/BI_ueLSqfvkEe-AqvopbP1qhQ)).

5G-L2-HI shall reply with HiCellDu\_CellSetupResp message, set the status IE = NotOk, and set cause as ECause5G\_InvalidParameter if:

* nrCellGrpId IE is out of range or;
* nrCellIdentity IE is out of range or;
* physCellId IE is out of range

For available cause values, see 5G\_UP\_1846\_replaced([11375790](https://dn-prod.ext.net.nokia.com/rm/resources/BI_uePkGPvkEe-AqvopbP1qhQ)).

#### 4.9.2.4.0-2 Cell Delete Description (ID: 10991686)

**4.9.2.4.0-2.0-1**  (ID: `10991690`)

Cell Delete
Upon reception of the message, 5G-L2-HI shall delete the Cell list with parameter values given in the message.

5G-L2-HI shall de-register the cell towards the BTC via remove() as defined in 5G\_L2\_11514\_replaced([11231417](https://dn-prod.ext.net.nokia.com/rm/resources/BI_3AbQSPvTEe-AqvopbP1qhQ)).

#### 4.9.2.4.0-3 Cell Delete Message validation: (ID: 10991696)

**4.9.2.4.0-3.0-1**  (ID: `10991702`)

Cell Delete
Upon reception of the HiCellDu\_CellDeleteReq message, 5G-L2-HI shall check the validity of the request message to ensure that each IE is within the range.

5G-L2-HI shall reply with HiCellDu\_CellDeleteResp message, if the request message is valid, the cell is identified and deleted/deregistered successfully.

For exception handling, see 5G\_L2\_IF\_3721\_replaced([10987369](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2WWh4Pu6Ee-AqvopbP1qhQ)).

---

#### 4.9.2.4.0-4 Cell Delete Expected Exceptions: (ID: 10991708)

**4.9.2.4.0-4.0-1**  (ID: `10991714`)

Cell Delete
Generic exception handling rules can be found in the chapter 5G\_UP\_1795\_replaced([11374912](https://dn-prod.ext.net.nokia.com/rm/resources/BI_ueKrmfvkEe-AqvopbP1qhQ)).

Exception handling rules specifically for object deletion can be found in the requirements 5G\_UP\_1815\_replaced([11375190](https://dn-prod.ext.net.nokia.com/rm/resources/BI_ueMgwfvkEe-AqvopbP1qhQ)) and 5G\_UP\_1816\_replaced([11375218](https://dn-prod.ext.net.nokia.com/rm/resources/BI_ueMgxPvkEe-AqvopbP1qhQ)).

5G-L2-HI shall reply with HiCellDu\_CellDeleteResp message also if:

 nrCellGrpId IE or nrCellIdentity IE is not known (object is not identified).

5G-L2-HI shall not send response message if:

 the request message is not valid or;

 the resources were not deleted successfully.

If value of nrCellGrpId and nrCellIdentity are not in Range, 5G-L2-HI shall not reply with HiCellDu\_CellDeleteResp.
