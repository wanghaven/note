Precondition:

In case of positioning dedicated L2RT pool deployment (see 5G\_L2\_19071\_replaced([11140277](https://dn-prod.ext.net.nokia.com/rm/resources/BI_23iaCPvTEe-AqvopbP1qhQ))):

#### 2.14.2.1.0-2 Positioning cell setup Description (ID: 10982215)

**2.14.2.1.0-2.0-2**  (ID: `10982225`)

Positioning cell setup
This procedure may be executed in two different deployment scenarios:

1. L2RT pool dedicated for positioning deployment (see 5G\_L2\_19071\_replaced([11140277](https://dn-prod.ext.net.nokia.com/rm/resources/BI_23iaCPvTEe-AqvopbP1qhQ)))
2. L2RT pool not dedicated for positioning. In that case, the positioning cell context is integrated with regular cell (NRCELL) context.

Upon reception of the PsPos\_PosCellSetupReq, 5G-L2-PS shall:

In case of L2RT positioning dedicated pool:

* Perform Uplink Addressing Procedure with L1, see 5G\_L1\_IF\_173\_replaced([11513659](https://dn-prod.ext.net.nokia.com/rm/resources/BI_FRyW4_vxEe-AqvopbP1qhQ))
* Create positioning cell context
* [CB010680-GIF] Establish slot synchronization, see 5G\_L1L2\_FLOW\_6567\_replaced([11177486](https://dn-prod.ext.net.nokia.com/rm/resources/BI_BeTrxPvREe-AqvopbP1qhQ))[End CB010680-GIF]

otherwise:

* Create positioning cell context

See also 5G\_L2\_20155\_replaced([11147153](https://dn-prod.ext.net.nokia.com/rm/resources/BI_24JeC_vTEe-AqvopbP1qhQ)).

5G-L2-PS shall reply with PsPos\_PosCellSetupResp message. See also 'Exceptions'.

#### 2.14.2.1.0-3 Positioning cell setup Message validation (ID: 10982231)

**2.14.2.1.0-3.0-4**  (ID: `12763256`)

Positioning cell setup
Upon reception of the PsPos\_PosCellSetupReq message, 5G-L2-PS shall check the validity of the request message. Following checks shall be executed:

* In case of L2RT pool **not dedicated for positioning (see 5G\_L2\_IF\_6752\_replaced([10982215](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2VwE-vu6Ee-AqvopbP1qhQ))):**
* nrPosCellIdentity shall be equal to nrCellIdentity of the existing cell context,
* if numOfSuperSubcells > 0 in existing cell context, number of elements on ulPosSubcellList shall be equal to numOfSuperSubcells,
* if numOfSuperSubcells = 0 in existing cell context, there shall be one element on ulPosSubcellList
* the ulPosSubcell.subcellType shall not be equal to **C4S**[CB013273-A] or C8S **[End CB013273-A]**\*\*\*\*
* for every element on the ulPosSubcellList there should exist corresponding subcell context with subcellId equal to posSubcellId
* In case of L2RT pool dedicated for positioning:
* there shall be one element on ulPosSubcellList
* the ulPosSubcell.subcellType shall be equal to **[CB013273-A] or C8S**[End CB013273-A]\*\*\*\*
* **[Before CB016080] Optional parameters l2SubPoolId, firstCellSlotId, numCellSlots,**[CB010680-GIF] mandatory parameters oneWayHwDelayDl, frameStartSub10ms, frameStartSFN **[CB010680-GIF] shall be present.**[CB016080] Mandatory parameters l2SubPoolId, firstCellSlotId, numCellSlots, **[Before CB014054-G] oneWayHwDelayDl,** [CB014054-G] optional parameter oneWayHwDelayDl*[End CB014054-G] frameStartSub10ms, frameStartSFN shall be present**[End CB016080].****\*******\****\*
* l2SubPoolId shall be equal to one of l2SubPoolId received in Address Distribution procedure (see 5G\_L2\_IF\_8\_replaced([10966028](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2UL-sfu6Ee-AqvopbP1qhQ)))
* firstCellSlotId and firstCellSlotId+numCellSlots parameters received in this message shall be in range of [firstCellSlotId, firstCellSlotId+numCellSlots) received in Address Distribution procedure

**[CB014054-G] *Note - oneWayHwDelayDl will be always present in the PsPos\_PosCelSetupReq message. As part of CB014054-G the onewayHwDelayDl IE in the PsCell\_CellSetupReq is changed to optional, which made to change this to optional here.  CB014054-G enables FDD mixed mode FH. \*\*[End CB014054-G]***\*

#### 2.14.2.3.0-2 positioning measurement setup Description (ID: 10982460)

**2.14.2.3.0-2.0-3**  (ID: `12700891`)

positioning measurement setup
1. "ranMeasId" is a unique identifier for one message PsPos\_MeasSetupReq at one L2 pool.

1. Trps located at one L2 pool will be filled into one PsPos\_MeasSetupReq message, and trps belong to one positioning cell will be filled into one posCellMeasReq structure and trps belonging to subcell will be filled into one trpsInSubcell structure.
2. L2PS will reply one PsPos\_MeasSetupResp message for one PsPos\_MeasSetupReq message.
3. If partial trps been setup successfully, the "status" IE in message PsPos\_MeasSetupResp will be filled with EStatus5G\_OK, and trps been setup successfully will be filled int IE "posCellMeasSetSuc".

5.If the posMeasType[].size is more than 2 , L2 PS will print log seen as "wrong config" from cprt/OAM, refer to 5G\_L2\_IF\_6512\_replaced([10982522](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2Vx6MPu6Ee-AqvopbP1qhQ))

The maximum number of TRPs requested to be measured in single subcell equals MAX\_NUM\_OF\_TRPS\_IN\_SUBCELL is equal to 4 **[CB013273-A] or 8 [End CB013273-A]. See 5G\_UP\_13358\_replaced([11374827](https://dn-prod.ext.net.nokia.com/rm/resources/BI_ueKEh_vkEe-AqvopbP1qhQ)) for reference.**

#### 2.14.2.3.0-3 positioning measurement setup Message validation (ID: 10982478)

**2.14.2.3.0-3.0-3**  (ID: `10982495`)

positioning measurement setup
Upon reception of the PsPos\_MeasSetupReq message, 5G-L2-PS shall check the validity of the request message.

5G-L2-PS shall reply with PsPos\_MeasSetupResp message and set the status IE = EStatus5G\_Ok if the request message is valid and new measurement context setup successfully,set the status IE = EStatus5G\_NotOk Only when parameter validation failed for all the TRPs. If partial TRPs validate failed still need to setup measurement for the successfully validated TRPs.

[CB007880-D]

If any configuration of the subcells included in posCellMeasReq but not all included in posCellMeas[] cannot be matched with the information of the subcells(ulPosSubcellList) configured, those posCell included in the struct of posCellMeasReq shall be considered as unmeasured posCell in 5G-L2-PS, then this case shall be considered as partial validate failed and set the status IE = EStatus5G\_Ok

[End CB007880-D]

**2.14.2.6.0-2.0-1**  (ID: `10982926`)

positioning measurement report
