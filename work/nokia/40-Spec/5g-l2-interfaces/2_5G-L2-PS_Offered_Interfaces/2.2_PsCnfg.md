### 2.2.2.1 Address Distribution

#### 2.2.2.1.0-1 Address Distribution Overview (ID: 10966033)

**2.2.2.1.0-1.0-1**  (ID: `12728072`)

Address Distribution
**Messages:**

PsCnfg\_AddressDistributionReq/Resp

**Purpose:**

To distribute addresses to 5G-L2-PS and to enquire 5G-L2-PS addresses.

**Precondition:**

5G-L2-PS has been started.

**Addressing:**

The request is sent to the PsCnfg address.

The response is sent to the address where the request was sent from.

**Relevant UP MSCs:**

5G\_L1L2\_FLOW\_72\_replaced([11177073](https://dn-prod.ext.net.nokia.com/rm/resources/BI_BeRPmfvREe-AqvopbP1qhQ)): L2RT and L2NRT Instance Configuration

#### 2.2.2.1.0-2 Address Distribution Description (ID: 10966038)

**2.2.2.1.0-2.0-2**  (ID: `10966048`)

Address Distribution
5G-L2-PS receives a set of addresses of other components in the PsCnfg\_AddressDistributionReq message. The request also contains address to be used to exchange LoCtrl addresses with 5G-L2-LO.

The request also contains elements to make a L2-RT [Before CB009851-F]instance[CB009851-F]pool[End CB009851-F] decides of its deloyment [5G\_L2\_IF\_14\_replaced([10966124](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2UMlxPu6Ee-AqvopbP1qhQ))]. Any modifications of these parameters means a reception of a new PsCnfg\_AddressDistributionReq message and it requires a L2-RT instance restart.

#### 2.2.2.1.0-3 Address Distribution Message validation (ID: 10966052)

**2.2.2.1.0-3.0-2**  (ID: `10966062`)

Address Distribution
Message validation:

Upon reception of the PsCnfg\_AddressDistributionReq message, 5G-L2-PS shall check the validity of the request message.

5G-L2-PS shall reply with PsCnfg\_AddressDistributionResp message and set the status IE = Ok if the request message is valid and the addresses and pools were configured successfully.

Note: 5G-L2-PS cannot validate that distributed address/parameter values are correct. Only ranges can be checked where applicable.

For exception handling, see 5G\_L2\_IF\_5929\_replaced([10966067](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2UL-ufu6Ee-AqvopbP1qhQ)).

#### 2.2.2.1.0-4 Address Distribution Exceptions (ID: 10966067)

**2.2.2.1.0-4.0-4**  (ID: `10966087`)

Address Distribution
Expected Exceptions:

Generic exception handling rules can be found in the chapter 5G\_UP\_1795\_replaced([11374912](https://dn-prod.ext.net.nokia.com/rm/resources/BI_ueKrmfvkEe-AqvopbP1qhQ)).

5G-L2-PS shall reply with PsCnfg\_AddressDistributionResp message, set the status IE = NotOk, and set an appropriate cause value if:

* the request message is not valid or;
* the addresses were not configured successfully.

For available cause values, see 5G\_UP\_1846\_replaced([11375790](https://dn-prod.ext.net.nokia.com/rm/resources/BI_uePkGPvkEe-AqvopbP1qhQ)).

Expected Exception 1:

If maxNumOfCAUserL2SubPool in bbPoolInfos as 5G\_L2\_IF\_1555\_replaced([10966229](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2UNM1Pu6Ee-AqvopbP1qhQ)) is greater than the internal R&D parameter rdMaxActiveCaUe as 5G\_L2\_6836\_replaced([11184745](https://dn-prod.ext.net.nokia.com/rm/resources/BI_27tIXPvTEe-AqvopbP1qhQ)), one single cell in the L2 subpool can not support maxNumOfCAUserL2SubPool for all. Details can be referred to 5G\_L2\_15817\_replaced([11159207](https://dn-prod.ext.net.nokia.com/rm/resources/BI_25PqNfvTEe-AqvopbP1qhQ)).

Expected Exception 2:

If maxNumOfCAUserL2SubPool in bbPoolInfos as 5G\_L2\_IF\_1555\_replaced([10966229](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2UNM1Pu6Ee-AqvopbP1qhQ)) is greater than 40 for TDD FR1 cell, an L2PS warning log is given. And then TDD L2SubPool's maximun number of active CA UE is 40.

Expected Exception 3:

If slowPrbPoolingTddFr1Ecpri72eDlAllowed == true or slowPrbPoolingTddFr1Ecpri72eUlAllowed == true ulSubcellDeactForARMAllowed has to be false otherwise set the status IE = NotOk and cause to InvalidParameter. Refer to 5G\_UP\_ALG\_27702.

[CB010680-C]

Expected Exception 4:

5G-L2-PS shall validate that bbPoolInfos structure is present when positioningDedicatedPoolDeployment equals TRUE.

[End CB010680-C]

Unexpected Exceptions:

In case of an unexpected exception, 5G-L2-PS shall not send response message.

Error is logged for both expected and unexpected exception.
