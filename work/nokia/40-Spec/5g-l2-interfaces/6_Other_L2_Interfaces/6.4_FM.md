## 6.4.0-2 FM Overview (ID: 10994774)

**6.4.0-2.0-1**  (ID: `10994780`)

FM
Description: FM interface (WA: valid regardless of the OAM solution (Megazone/SOAM/cOAM/...)) - informative information

Purpose: Fault collection/management

Used by: Fault Subscriber (OAM)

Service provider: LOM FaultService (Fault Manager)

Addressing: See 5G\_L1L2\_FLOW\_55\_replaced([11176847](https://dn-prod.ext.net.nokia.com/rm/resources/BI_BeQoc_vREe-AqvopbP1qhQ))/5G\_L1L2\_FLOW\_58\_replaced([11176868](https://dn-prod.ext.net.nokia.com/rm/resources/BI_BeQodvvREe-AqvopbP1qhQ))/5G\_L1L2\_FLOW\_61\_replaced([11176950](https://dn-prod.ext.net.nokia.com/rm/resources/BI_BeQofvvREe-AqvopbP1qhQ))/5G\_L1L2\_FLOW\_63\_replaced([11176998](https://dn-prod.ext.net.nokia.com/rm/resources/BI_BeRPgvvREe-AqvopbP1qhQ)) and the table below.

See 5G\_UP\_2724\_replaced([11391923](https://dn-prod.ext.net.nokia.com/rm/resources/BI_ufbP2fvkEe-AqvopbP1qhQ)) for U-Plane fault reporting mechanism.

Messages (SysCom), informative:

|  |  |  |
| --- | --- | --- |
| Message name | Purpose | Addressing |
| LomFm\_AvailInd | To announce a new Fault Manager instance is created. | Fault Manager broadcasts the message via AaSysMb (MessageBroker) service. Fault Subscribers (e.g. OAM FM Service) are subscribers of this message and get aware of the original sender address. |
| FaultManagerInitReq FaultManagerInitResp | To provide Fault Subscriber address to the Fault Manager. | Fault Subscriber knows the address based on the LomFm\_AvailInd message. The response is sent to the address where the request was sent from. |
| FaultReq FaultAckMsg   Note that naming of the FaultReq does not follow common naming rules according to which the name should have postfix Ind. | To propagate a fault set/reset indication to the currently registered Fault Subscribers. | Fault Manager knows the Fault Subscriber addresses based on the FaultManagerInitReq messages. |
| FaultBlockAllReq FaultBlockAllResp | To block fault forwarding to Fault Subscriber.   Note: This service is currently not in use, open if will be needed in future releases. | Fault Subscriber knows the address based on the LomFm\_AvailInd message. The response is sent to the address where the request was sent from. |
