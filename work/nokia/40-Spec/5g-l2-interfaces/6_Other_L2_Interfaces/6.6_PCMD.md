## 6.6.1 PCMD Interface Details

### 6.6.1.0-1 PCMD Interface Details Identities used in communication with the BTS Trace Controller (ID: 10994963)

**6.6.1.0-1.0-3**  (ID: `10994982`)

PCMD Interface Details
In 5G-L2-PS and 5G-L2-LO, resource refers to an UE.

In 5G-L2-HI, resource refers to a DRB.

|  |  |  |  |
| --- | --- | --- | --- |
| Logical entity | Parameter | Description | First Feature |
| A cell | cellId | Cell identifier. Maps to nrCellIdentity received in the related cell setup request. This parameter is required only in case of 5G-L2-PS, 5G-L2-LO and L2-HI-DU. | 5GC001804-A |
| An UE | ueResourceId | UE resource identifier with a continuous range (from first to last), which implicitly limits maximum number of UE resources (last-first+1) handled by a TC proxy instance.   Assigned locally by a U-Plane L2 system component. Unique within the related TC proxy.   Note that a TC Slave instance is configured with sum of UE resources from all related TC proxies.   5G-L2-PS shall assign ueResourceId per UE per (PCell/PSCell/SCell) cell. 5G-L2-LO shall assign ueResourceId per UE per [Before CB008911-B]PCell[CB008911-B](PCell/SCell) cell[End CB008911-B] (per a 5G-L2-LO instance).  5G-L2-HI CU shall assign ueResourceId per DRB within its own proxy. 5G-L2-HI DU shall assign ueResourceId per DRB within its own proxy. ueResourceId is independent allocated between L2-HI CU and L2-HI DU.   For actual formulas see 5G\_L2\_IF\_1514\_replaced([10994988](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2XIlBPu6Ee-AqvopbP1qhQ)). | 5GC001804-A |
| An UE | cookie | Call specific trace id assigned by C-Plane. Received (optionally) in the related UE resource setup request. If not given, PCMD activation for the UE resource is not possible. | 5GC001804-A |
| An UE | crnti | Received in the related UE resource setup request. This parameter is required only in case of 5G-L2-PS and 5G-L2-LO. | 5GC001804-A |
| A DRB | drbId | DRB identifier from 38.331. Maps to drbIdentity received in the related UE or DRB setup request. Used in 5G-L2-LO and L2-HI only. | 5GC001804-A |

Note that when several system component instances are served by a single TC Slave instance, this instance is configured to support maximum number of UE resources over all the system component instances. The system component specific ranges for ueResourceIds are however independent from each others (are defined per a TC Proxy instance) and may overlap. [5G\_UP\_2758\_replaced([11392311](https://dn-prod.ext.net.nokia.com/rm/resources/BI_ufdFDfvkEe-AqvopbP1qhQ))]

### 6.6.2.0-2 API-based PCMD Interface Trace Controller (TC) Slave Service (ID: 10995187)

**6.6.2.0-2.0-1**  (ID: `10995192`)

API-based PCMD Interface
TC proxy (SlaveServiceProxyEm)

---

**6.6.2.0-2.0-1.0-1**  (ID: `10995198`)

API-based PCMD Interface
- SlaveServiceProxyEm is a lightweight component running within an U-Plane application (PCMD provider) queue context.

* SlaveServiceProxyEm message handler is plugged to the message dispatcher of the application queue. This way the SlaveServiceProxyEm receives SlaveServiceEm commands through application queue to execute the corresponding application callback on application queue context.
* Inversely, when an application queue is having a trace record to send, it calls the corresponding SlaveServiceProxyEm instance to forward the record to the corresponding SlaveServiceEm instance.
* It must be ensured that SlaveServiceEm is created before creating SlaveServiceProxyEm.
* One SlaveServiceProxyEm instance shall be created per each application queue which may later register a cell to TC.

---

**6.6.2.0-2.0-2**  (ID: `10995204`)

API-based PCMD Interface
TC slave (SlaveServiceEm)

---

**6.6.2.0-2.0-2.0-1**  (ID: `10995208`)

API-based PCMD Interface
- SlaveServiceEm takes care of the communication towards the TC Master.

* SlaveServiceEm accumulates trace records before forwarding them to TC Master.
* SlaveServiceEm is having its own queue context/queue Id and shall be configured to use a single CPU queue.
* An U-Plane application creates the SlaveServiceEm at startup during local\_init phase and configures it with adequate queue/event sizes.
* Typically one SlaveServiceEm instance per provider shall be created, but applications on the same HW board may share the same instance for better performance.

For PCMD components and interfaces see 5G\_UP\_2749\_replaced([11392211](https://dn-prod.ext.net.nokia.com/rm/resources/BI_ufdFAvvkEe-AqvopbP1qhQ)).
