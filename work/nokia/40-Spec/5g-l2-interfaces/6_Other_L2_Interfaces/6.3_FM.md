## 6.3.0-3 FM' Fault Reporter Library for L2 Applications (ID: 10994737)

**6.3.0-3.0-1**  (ID: `10994743`)

FM'
A simple fault reporter library shall be implemented and shall be used by the L2 applications.

LOM Fault reporter library shall not be used by the 5G gNB L2 applications.

This library shall support the following functionality:

1. Maintain an L2 internal mapping for external fault IDs(defined in NIDD) to internal fault IDs. Internal Fault IDs shall be used as an index while searching faults during fault filtering.
2. Filter the faults as explained in 5G\_UP\_2727\_replaced([11392026](https://dn-prod.ext.net.nokia.com/rm/resources/BI_ufb25_vkEe-AqvopbP1qhQ)).
3. Send the fault to LOM Faultservice as a syscom message(LOM\_FaultInd) as defined in the Fault management interface at  <http://esling16.emea.nsn-net.net:8020/>
4. Fault reporter shall not use any blocking operations and shall be kept as minimal as possible.

This fault reporter can be extended in future for any other filtering criteria as its implemented within the L2 application and not from LOM.

A fault reporter object shall be created per core. With this model, if an L2 application raises a fault using the fault reporter in one core and raises the same fault from another core, the fault may not get filtered even though the filtering criteria is met. This does not cause a problem as the LOM FaultService will filter this fault if it meets the filtering criteria.

In case of a fault flooding situation, the same fault is sent again only till the fault is raised for the first time in that core. Once a fault has been raised it will get filtered on that core as long as filtering criteria is met.

Note: The LOM\_FaultInd message is sent using SACK serialization as defined in LTE LOM implementation and does not support the MulTi or protobuf serialization. This can be extended in future after LOM supports any of these new serialization techniques.

CP2 reference: 5G\_UP\_2724\_replaced([11391923](https://dn-prod.ext.net.nokia.com/rm/resources/BI_ufbP2fvkEe-AqvopbP1qhQ))
