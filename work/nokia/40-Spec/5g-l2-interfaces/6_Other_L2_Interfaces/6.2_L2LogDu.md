### 6.2.2.1 DU Streaming Activation or Deactivation

#### 6.2.2.1.0-1 DU Streaming Activation or Deactivation Overview (ID: 10994248)

**6.2.2.1.0-1.0-2**  (ID: `10994261`)

DU Streaming Activation or Deactivation
**Messages:**

L2LogDu\_StreamActivationReq

L2LogDu\_StreamActivationResp

L2LogDU\_StreamDeactivationReq

L2LogDU\_StreamDeactivationResp

**Purpose:**

To configure 5G-L2 component to start or stop L2 streaming with capability to filter per Cell.

This is relevant to the following 5G-L2 components:

5G-L2-PS in DU

5G-L2-LO in DU

5G-L2-HI in DU

**[CB006063-A, CB006063-B]vDU platform configuration shall have same behaviour and managed same messages**[End CB006063-A, CB006063-B]\*\*\*\*

**Precondition:**

At least one L2 Instance exists.

**Addressing:**

The request is sent to the L2Log address of 5G-L2 component.

The response is sent to the address where the request was sent from.

**UP MSCs:**

5G\_L1L2\_FLOW\_770\_replaced([11205641](https://dn-prod.ext.net.nokia.com/rm/resources/BI_BgmKwvvREe-AqvopbP1qhQ)), 5G\_L1L2\_FLOW\_771\_replaced([11205714](https://dn-prod.ext.net.nokia.com/rm/resources/BI_BgmKz_vREe-AqvopbP1qhQ))

#### 6.2.2.1.0-2 DU Streaming Activation or Deactivation Description: (ID: 12516930)

**6.2.2.1.0-2.0-2**  (ID: `12516932`)

DU Streaming Activation or Deactivation
Description:

Upon reception of the request, L2 instance ( 5G-L2-PS, 5G-L2-LO, 5G-L2-HI) shall start or stop streaming of requested L2 logs. TraceType given in the request defines which L2 logs are to be configured. See 5G\_L1L2\_FLOW\_770\_replaced([11205641](https://dn-prod.ext.net.nokia.com/rm/resources/BI_BgmKwvvREe-AqvopbP1qhQ)), 5G\_L1L2\_FLOW\_771\_replaced([11205714](https://dn-prod.ext.net.nokia.com/rm/resources/BI_BgmKz_vREe-AqvopbP1qhQ))  for details.

This is relevant to the following 5G-L2 components:

5G-L2-PS in DU

5G-L2-LO in DU

5G-L2-HI in DU

[CB010680-I] In 5G-L2-PS component, the message is also used to enable tracing for positioning cells (NRPOSCELL). 5G-L2-PS shall start streaming for all positioning cells matching the requested physCellId.  
Note: Since physCellId is not unique for positioning cells, single physCellId on the PCIList may start streaming for multiple positioning cells. [End CB010680-I]

DU Overload Status Indication to Client.
L2-HI DU informs when CPU overload state is detected.

When message is sent with OvlStatus "ON" suspend of streaming is expected to be triggered by client.

If after sending msg actTraceOvlProt is set to "FALSE", then all above action are immediately aborted.

Suspended streamings must be resumed with L2LogDu\_StreamActivationReq.

Messages:  
L2LogDu\_VonrMuteBinUpdateInd

Purpose:  
To identify a call that has experienced VoNR mute from the collected logs to simplify and expedite troubleshooting.

Precondition:  
At least one L2 instance exists, vonrPmCount structure ([10983375](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2V4Az_u6Ee-AqvopbP1qhQ)) was present in LoCell\_CellSetupReq message and rdEnableVonrMuteBinUpdateReporting ([12680535](https://dn-prod.ext.net.nokia.com/rm/resources/BI_G7jNtlAqEfCeeMlEb49MrQ)) is set to True.

Addressing:  
L2-LO sends the message to Syscom Drain (AASYSCOM\_EM\_DRAIN\_CPID)

---

#### 6.2.2.8.0-2 DU VoNR Mute Detection Description (ID: 12676578)

**6.2.2.8.0-2.0-1**  (ID: `12676998`)

DU VoNR Mute Detection
The message will be sent only if VoNR mute occurs. The message contains UeId, Mute type and Bin information values to simplify the identification process and troubleshooting of calls that experienced VoNR mute. 5G-L2-LO updates counters and reports the DL mute duration for VoNR bearer. When the counter are updated, 5G-L2-LO shall collect required data and then send a message to drain sicad when debug tracing is configured.
