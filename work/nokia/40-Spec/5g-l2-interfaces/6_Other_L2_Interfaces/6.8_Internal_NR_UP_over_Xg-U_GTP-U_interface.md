# 6.8 Internal NR UP' (over Xg-U GTP-U interface)

## 6.8.0-1 Internal NR UP' (over Xg-U GTP-U interface) Document Owner Zablotny, Bartosz (Nokia - PL/Wroclaw) (ID: 10995262)

## 6.8.1 Description

### 6.8.1.0-1 Description (ID: 10995272)

**6.8.1.0-1.0-1**  (ID: `10995278`)

Description
The NRUP protocol has been adapted to be used for communication between primary gNB and external gNB for the inter gNB carrier aggregation

[CB008140-E]

NR UP' is a nokia proprietry interface protocol for which the versioning has to be maintained to ensure the "Xg-U protocol version" compatibility for inter-gNB communication over Xg-U Gtp-U interface.

Each interface message has to be versioned for any change that alters the Frame structure.

[End CB008140-E]

---

### 6.8.1.0-2 Description NRUP' Version (ID: 10995284)

**6.8.1.0-2.0-5**  (ID: `12780336`)

Description
The NRUP' protocol version is maintained as a master version for any change to the NRUP' protocol across the frame formats.

NRUP' version to frame format version mapping:

|  |  |  |  |  |
| --- | --- | --- | --- | --- |
| NRUP' version | Frame format version |  | Comments | First Feature |
| DUD | DDDS |  |  |  |
| 0x00 | 1.0 | 1.0 | NRUP' version IE was not created |  |
| 0x01 | 2.0 | 2.0 | NRUP' version IE added as the first octet to all the frame formats supported over NRUP' protocol | CB008140-HE |
| 3.0 | Added NrCellIdentity/BurstInfo/BurstStartInfo IE | CB010636-E |  |  |
| 4.0 | Added Estimated Throughput Bare | CB010936-A |  |  |
| 0x02 | 3.0 | 5.0 | Added L2Lo SN Index/RLC SN Sub Range Index IE | CNI-138886 |
| 0x03 | 4.0 | 6.0 | Removed not used optional parameters related to lost NR-U Sequence. Changed timestamp format. | CB015504-A |

Note: If there is a change done to the NRUP' extension header format - the NRUP' version as defined above can not be used to track the change. A new GTP-U extension header versioning mechanism needs to be defined for that purpose in chapter 13.11 ([11552434](https://dn-prod.ext.net.nokia.com/rm/resources/BI_tZw_nvv1Ee-AqvopbP1qhQ)).

## 6.8.3 DU DL DATA DELIVERY STATUS

### 6.8.3.0-1 DU DL DATA DELIVERY STATUS Overview (ID: 10995351)

**6.8.3.0-1.0-1**  (ID: `10995356`)

DU DL DATA DELIVERY STATUS
Messages:

Du DL Data Delivery Status (GTP-U PDU)

Purpose:

To provide DL Data Delivery status towards 5G-L2-HI on Prime gNB

Precondition:

UE Context exists

Addressing:

This message frame shall be packed in GTP-U PDU with NRUP RLC Container and send through TrswData interface (Refer 5G\_UP\_1931\_replaced([11376719](https://dn-prod.ext.net.nokia.com/rm/resources/BI_ueSnZ_vkEe-AqvopbP1qhQ))) to the 5G-L2-HI on primary gNB using the TRSW Receiver EQID from LoCnfg\_AddressDistributionReq (Refer 5G\_L2\_IF\_738\_replaced([10983064](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2V1kj_u6Ee-AqvopbP1qhQ))).

Relevant UP MSCs:

N/A

### 6.8.3.0-2 DU DL DATA DELIVERY STATUS Description (ID: 10995362)

**6.8.3.0-2.0-1**  (ID: `10995368`)

DU DL DATA DELIVERY STATUS
