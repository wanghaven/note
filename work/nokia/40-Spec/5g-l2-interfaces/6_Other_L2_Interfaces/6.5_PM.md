# 6.5 PM

## 6.5.0-2 PM Description (ID: 10994798)

**6.5.0-2.0-1**  (ID: `10994804`)

PM
Description: PM interface (WA: valid regardless of the OAM solution (Megazone/SOAM/cOAM/...))

Purpose: PM counter collection

Used by: PM Collector (OAM)

Service provider: PM Agent / PM Provider (5G-L2-PS, 5G-L2-LO, 5G-L2-HI)

Addressing: See 5G\_L1L2\_FLOW\_58\_replaced([11176868](https://dn-prod.ext.net.nokia.com/rm/resources/BI_BeQodvvREe-AqvopbP1qhQ))/ 5G\_L1L2\_FLOW\_61\_replaced([11176950](https://dn-prod.ext.net.nokia.com/rm/resources/BI_BeQofvvREe-AqvopbP1qhQ))/ 5G\_L1L2\_FLOW\_63\_replaced([11176998](https://dn-prod.ext.net.nokia.com/rm/resources/BI_BeRPgvvREe-AqvopbP1qhQ)), and the table below.

See 5G\_UP\_2732\_replaced([11392091](https://dn-prod.ext.net.nokia.com/rm/resources/BI_ufcd8fvkEe-AqvopbP1qhQ)) for PM collection mechanism in 5G-L2 components.

Note that PM counters are currently not supported in 5G-L1-UL or 5G-L1-DL.

The PM Agent message API and the PM interface are described in LOM PM Agent Documentation:

<https://bstat.wrgitlab-pages.int.net.nokia.com/lom_pm_docs/>

Note: the link does not work with Windows Explorer, please use another browser.

For URI construction see 5G\_L2\_IF\_1490\_replaced([10994810](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2XHW5_u6Ee-AqvopbP1qhQ)) and the following document:

<http://svne1.access.nsn.com/isource/svnroot/BTS_I_RTPM/trunk/Documentation/RTPM.pdf>

Higher level Use Cases are given in PM EFS (for SOAM) in Sphinx database:

<http://oam.emea.nsn-net.net/btsom_efs/Performance_Monitoring.html>

Note that W/A is 5G reuses the LTE RAT specific Use Cases as such.

Messages (SysCom), informative:

|  |  |  |
| --- | --- | --- |
| Message name | Purpose | Addressing |
| PMCounterAvailInd | To announce PM counters which can be reported by this PM Provider. | PM Provider broadcasts the message via AaSysMb (MessageBroker) service. PM Collector is a subscriber of this message and gets aware of the original sender address. |
| PMReportingConfigReq PMReportingConfigResp | To (re)configure periodicity and timing which PM Provider shall use to report PM counter values. | PM Collector knows PM Provider address based on the PMCounterAvailInd message. The response is sent to the address where the request was sent from. |
| PMCounterValuesReport | To provide PM counter values for the reporting period. The message is send periodically according to configuration received in PMReportingConfigReq. | PM Provider knows the address based on the PMReportingConfigReq message. |

## 6.5.0-3 PM PM URI in 5G User Plane (ID: 10994810)

**6.5.0-3.0-6**  (ID: `10994847`)

PM
A PM URI is constructed at creation of a new PM object instance based on Network Profiles given in 5G\_UP\_ALG\_6255.

For PM Object Model in U-Plane see 5G\_UP\_2718\_replaced([11391877](https://dn-prod.ext.net.nokia.com/rm/resources/BI_ufbP0_vkEe-AqvopbP1qhQ)).

OAM will replace any dummy values (for (primary) PLMN, MRBTS, and/or NRCELL) provided by an U-Plane component with a proper value in all URI descriptions. Note that 5G-L2-PS/5G-L2-LO does not know nrCellId, thus always provides the nrCellIdentity as part of the HOME\_CELL\_NR. For more information on URI see 5G PM URI library usage guideline:

<https://confluence.ext.net.nokia.com/display/~mrozowsk/5G+and+SRAN+PM+URI+library+usage+guideline>

pm::uri uri;

1. NRBTS level counters (PLMN-MRBTS-NRBTS)

uri/PLMN(dummy\_mcc, dummy\_mnc, 2)/MRBTS(dummy\_id)/NRBTS(nrBtsId);

1. NRDU level counters (PLMN-MRBTS-NRBTS-NRDU)

uri/PLMN(dummy\_mcc, dummy\_mnc, 2)/MRBTS(dummy\_id)/NRBTS(nrBtsId)/NRDU(nrDuId);

1. NRCELL level counters (PLMN-MRBTS-NRBTS-NRCELL)

uri/HOME\_CELL\_NR(PLMN(dummy\_mcc, dummy\_mnc, 2), nrCellIdentity, 0)/NRCELL(nrCellId);

where

* A dummy nrCellId=0 can be provided when a real (3GPP based) nrCellIdentity is provided.
* A dummy nrCellIdentity=0 can be provided when a real (Nokia Object Id) nrCellId is provided.
* NRBTS level per PLMN counters (PLMN-MRBTS-NRBTS, MCC-MNC)

uri/PLMN(dummy\_mcc, dummy\_mnc, 2)/MRBTS(dummy\_id)/NRBTS(nrBtsId)/PLMN(secondary\_mcc, secondary\_mnc, secondary\_mnc\_length);

1. NRBTS level per PLMN per Slice counters (PLMN-MRBTS-NRBTS, MCC-MNC, SST-SD)

uri/PLMN(dummy\_mcc, dummy\_mnc, 2)/MRBTS(dummy\_id)/NRBTS(nrBtsId)/PLMN(secondary\_mcc, secondary\_mnc, secondary\_mnc\_length)/SNSSAI(sst, sd)

Real network slice id (sst, sd) is provided. sd=0xFFFFFF is used to indicate no slice differentiator.

1. NRCELL level per PLMN counters (PLMN-MRBTS-NRBTS-NRCELL, MCC-MNC)

uri/HOME\_CELL\_NR(PLMN(dummy\_mcc, dummy\_mnc, 2), nrCellIdentity, 0)/NRCELL(nrCellId)/PLMN(secondary\_mcc, secondary\_mnc, secondary\_mnc\_length)

where

* A dummy nrCellId=0 can be provided when a real (3GPP based) nrCellIdentity is provided.
* A dummy nrCellIdentity=0 can be provided when a real (Nokia Object Id) nrCellId is provided.
* NRCELL level per NRRESOURCEGROUP counters (PLMN-MRBTS-NRBTS-NRCELL, MRBTS-NRBTS-NRCELL-NRRESOURCEGROUP)

uri/HOME\_CELL\_NR(PLMN(mcc, mnc, mnc\_length), nrCellIdentity, lcrid\_length)/NRCELL(nrCellId)/NRRESOURCEGROUP(nrResourceGroupId)

where

* A dummy nrCellId=0 can be provided by counter provider application only when real (3GPP based) nrCellIdentity is provided.
* A dummy nrCellIdentity=0 can be provided only when real (Nokia Object Id) nrCellId is provided.
* A dummy Cell primary PLMN (mcc=0, mnc=0, mnc\_length=2) can be provided.
* A dummy lcrid\_length=0 can be provided.
* NRCELLGRP level counters (PLMN-MRBTS-NRBTS-NRCELLGRP)

uri/PLMN(dummy\_mcc, dummy\_mnc, 2)/MRBTS(dummy\_id)/NRBTS(nrBtsId)/NRCELLGRP(nrCellGrpId);

[CB009352-B]

1. NRCUUP level counters (PLMN-MRBTS-NRBTS-NRCUUP)

uri/PLMN(dummy\_mcc, dummy\_mnc, 2)/MRBTS(dummy\_id)/NRBTS(nrBtsId)/NRCUUP(nrCuUpId);

Unlike nrDuId which is retrieved from CP-RT via  HiCnfgDu\_ConfigParamReq.nrDuId, nrCuUpId need not be retrieved from CP-NRT in HiCnfgCu\_AddressDistributionReq and 5G-L2-HI shall consider nrCuUpId as 1 since max instances of NRCUUP MO is 1 and maximum/minimum value of NRBTS.NRCUUP.nrCuUpId is 1 (as per NIDD)

[End CB009352-B]

[CB013684-D]

1. NRBTS level per SSBPM counters (PLMN-MRBTS-NRBTS, SSBPM)

uri/HOME\_CELL\_NR(PLMN(mcc, mnc, mnc\_length), nrCellIdentity, lcrid\_length)/NRCELL(nrCellId)/SSBPM(nrSsbpmId);

where

* A dummy nrCellId=0 can be provided by counter provider application only when real (3GPP based) nrCellIdentity is provided.
* A dummy nrCellIdentity=0 can be provided only when real (Nokia Object Id) nrCellId is provided.
* A dummy Cell primary PLMN (mcc=0, mnc=0, mnc\_length=2) can be provided.

- A dummy lcrid\_length=0 can be provided.

- Real nrSsbpmId (instance id of SSBPM) needs to be provided.

## 6.5.1 PM Collection Mechanism in L2RT

### 6.5.1.0-1 PM Collection Mechanism in L2RT L2CounterClient and Bank Switch Time (ID: 10994859)

**6.5.1.0-1.0-1**  (ID: `10994865`)

PM Collection Mechanism in L2RT
L2CounterClient (a L2RT application process run in the NRT core) terminates the OAM interface for PM.

Bank Switch Time:

* In L2RT, bank is switched every 1024 NR-subframes (1.024s). Based on SFN information (no timer).

L2RT applications on RT EM cores collect counter values to an active bank. Collected values are delivered to L2CounterClient via shared memory. L2CounterClient has a (100ms) timer to check, if bank switch has happened based on a flag in each bank. PM counters from 5G-L2-PS and 5G-L2-LO are handled by a single L2CounterClient instance.

For more information on implementation see: <https://confluence.ext.net.nokia.com/display/5GINDIA/User+Guide+for+Performance+Management+in+U-Plane>

CP2 reference: 5G\_UP\_2733\_replaced([11392097](https://dn-prod.ext.net.nokia.com/rm/resources/BI_ufcd8vvkEe-AqvopbP1qhQ)), 5G\_UP\_2735\_replaced([11392113](https://dn-prod.ext.net.nokia.com/rm/resources/BI_ufcd9PvkEe-AqvopbP1qhQ))

### 6.5.1.0-2 PM Collection Mechanism in L2RT Interface between L2CounterClient and L2RT applications (ID: 10994871)

**6.5.1.0-2.0-5**  (ID: `10994905`)

PM Collection Mechanism in L2RT
L2RT applications shall provide the PM Measurement object identifiers to the L2CounterClient, so that L2CounterClient shall use this Measurement object identifier to peg the counters against the Measurement object.

L2RT supports NRCELL level counters and nrCellIdentity is the Measurement object identifier as well as NRCELL level counters collected per PLMN where nrCellIdentity together with PLMN ID is the Measurement object identifier and NRCELL level counters collected per PMQAP profile where nrCellIdentity together with PMQAP ID is the Measurement object identifier. See 5G\_L2\_1517\_replaced([11192164](https://dn-prod.ext.net.nokia.com/rm/resources/BI_28iOzPvTEe-AqvopbP1qhQ)) (5G-L2-PS) and 5G\_L2\_2115\_replaced([11218485](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2_KsCvvTEe-AqvopbP1qhQ)) (5G-L2-LO).

## 6.5.2 PM Collection Mechanism in L2NRT

### 6.5.2.0-1 PM Collection Mechanism in L2NRT L2HICounterClient and Bank Switch Time (ID: 10994916)

**6.5.2.0-1.0-1**  (ID: `10994921`)

PM Collection Mechanism in L2NRT
L2HICounterClient (a L2NRT application process run in the NRT core) terminates the OAM interface for PM.

Bank Switch Time:

* In 5G-L2-HI, bank is switched every 1s (timer based).

L2NRT applications on RT EM cores collect counter values to an active bank. Collected values are delivered to L2HICounterClient via a messaging interface (instead of a shared memory) upon request of the L2HICounterClient once per every 1s.

For more information on implementation see: <https://confluence.ext.net.nokia.com/display/5GINDIA/User+Guide+for+Performance+Management+in+U-Plane>

CP2 reference: 5G\_UP\_2733\_replaced([11392097](https://dn-prod.ext.net.nokia.com/rm/resources/BI_ufcd8vvkEe-AqvopbP1qhQ)), 5G\_UP\_2735\_replaced([11392113](https://dn-prod.ext.net.nokia.com/rm/resources/BI_ufcd9PvkEe-AqvopbP1qhQ))
