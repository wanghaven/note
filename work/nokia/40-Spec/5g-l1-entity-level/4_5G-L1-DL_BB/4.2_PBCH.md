##### 4.2.7.4.3 PSS/SSS

**4.2.7.4.3.0-2**  (ID: `12707835`)

PSS/SSS
For **synchronization signals, logical antenna ports may be mapped to either the one or the other polarization of the X-Pol-Beam or to both. An R&D parameter is required which allows to switch between these three configurations. This R&D parameter *rdSyncSignalsTxMode is defined in requirement  [11445204](https://dn-prod.ext.net.nokia.com/rm/resources/BI_e_vNuPvvEe-AqvopbP1qhQ).***

Logical antenna port used for synchronization signals is logical antenna port 4000.

The mapping of the logical antenna ports for synchronization signals to polarizations shall be done as given in below table.

|  |  |  |  |
| --- | --- | --- | --- |
| *rdSyncSignalsTxMode* | Logical antenna port(s) for synchronization signals | Polarization |  |
|  |  |  |  |
| 0 | 4000 | polarization +45° |  |
| 1 | 4000 | polarization -45° |  |
| 2 | **4000** | polarizations +/-45° |  |

If *rdSyncSignalsTxMode is set to 2, multiplexing (with a potential sign inversion) should be applied to the synchronization signals. This multiplexing operation is described in the section 3.3 of 5GMax algorithm specification document [11481180](https://dn-prod.ext.net.nokia.com/rm/resources/BI_fDcB_vvvEe-AqvopbP1qhQ).*

If *rdSyncSignalsTxMode is set to 2, 5G-L1-DL uses the IE* PrecodingVectorIndex retrieved from *DlData\_SsBlockSendReq message to select the appropriate Precoding Vector (PV) specified in the section 3.3 of 5GMax algorithm specification document [11481180](https://dn-prod.ext.net.nokia.com/rm/resources/BI_fDcB_vvvEe-AqvopbP1qhQ).*\*\*

* If *PrecodingVectorIndex = 0; 5G-L1-DL selects its corresponding PV*
* If *PrecodingVectorIndex = 1; 5G-L1-DL selects the other PV*

The mapping between the *PrecodingVectorIndex and the respective precoding vector is handled internally in 5G-L1-DL.*

During a given SSB period (let's denote *n), L2-PS fills the IE* PrecodingVectorIndex with the same value (either 0 or 1, see [11173773](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kuV1ffvSEe-AqvopbP1qhQ)) which means 5G-L1-DL uses the same PV during this period. For next SSB period (i.e. *n+1), the value of* PrecodingVectorIndex changes and 5G-L1-DL uses the other PV.\*\*\*\*
