## 11.7.2 Startup

### 11.7.2.0-15 Startup 5G-L1-HI shall use SCS value from environment variable L1_DEFAULT_SCS, which is passed to POD container using l1-configmap.yaml (ID: 11477185)

### 11.7.2.1 LDPC procesing mode

#### 11.7.2.1.0-7 LDPC procesing mode 5G-L1-HI enable HW FEC usage in DL when: Hardware FEC was found on startup and L1_DEFAULT_HW_LDPC_ENC = {HW_FEC_ENABLED = 1} (ID: 11477219)

### 11.7.2.2 ISA Support

#### 11.7.2.2.0-6 ISA Support 5G-L1-Hi shall throw fatal error, if generic DSP mode cannot be used. (ID: 11477246)

## 11.7.3 Trace and Debug

### 11.7.3.1 IQ Snapshots configured by R&amp;D parameters

**11.7.3.1.0-9**  (ID: `11477289`)

IQ Snapshots configured by R&D parameters
When trigger condition for activated event occurred in given slot, 5G-L1-HI should capture relevant scheduling information and IQ data from that slot, as described in the table.

|  |  |  |
| --- | --- | --- |
| Type of event | Scheduling information to capture | IQ data to capture |
| PRACH | PRACH | PRACH streams |
| PUCCH or PUSCH | PUCCH and PUSCH | PUxCH streams |
| DL event | SSB/PBCH, PDSCH, PDCCH, CSI-RS and NR-Slot Type Configuration | DL streams |

---

### 11.7.3.2 IQ Snapshots configured by Command Line Interface

#### 11.7.3.2.0-1 IQ Snapshots configured by Command Line Interface 5G-L1-HI shall support CLI handling which is provided by L1. CLI is a separate NRT thread that works as a server for telnet connections and libcli is used for telnet communication and parsing methods, after interaction CLI communicates with vl1 and TRBL service through syscom. Instruction: http://vl1.docs.l1.nsn-net.net/debug/snapshot.html (ID: 11477297)

### 11.7.3.3 IQ Snapshots file handling

#### 11.7.3.3.0-7 IQ Snapshots file handling 5G-L1-HI shall handle AATRBL_TRANSF_STOP_NOTIF_MSG. Upon reception, 5G-L1-HI can delete previously stored files. (ID: 11477328)
