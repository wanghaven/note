# 8.5 DL OFDM phase offset compensation

**8.5.0-1**  (ID: `13261206`)

DL OFDM phase offset compensation
For PBCH [CB013976-C] /PDCCH/PDSCH/CSI-RS [End CB013976-C] transmission in FR1 FDD, 5G-L1-DL (FHE-CPRI-GW) shall apply the phase compensation to the baseband signal related to an OFDM symbol within a subframe (per 3GPP TS 38.211 section 5.3) multiplied by a phase compensation coefficient.

The phase compensation coefficients for each symbol are pre-calculated and stored into a constant array in L1-DL (FHE-CPRI-GW) and the index phaseCompensationLutIndex is used to get the phase compensation coefficients.

The phaseCompensationLutIndex is calculated as follows (for details, see [11274097: Generation of Phase compensation LUT](https://dn-prod.ext.net.nokia.com/rm/resources/BI_TpFZJPvcEe-AqvopbP1qhQ?oslc_config.context=https%3A%2F%2Fdn-prod.ext.net.nokia.com%2Fgc%2Fconfiguration%2F9)):

Input:

* arfcn: provided in API\_BBB\_CREATE\_EAXC\_AXC\_ROUTING\_REQ\_MSG message with direction = "EBbbEaxcDirection\_Downlink".

Definition:

* p(𝑙): phaseCompensationLutIndex, with OFDM symbol 𝑙 = 0..13
* PDL = arfcn

Algorithm:

offset = 0 // offset is used to accumulate amount of samples over 14 symbols within a slot.

FOR  𝑙 = 0 to 13  // loop over 14 consecutive symbols within slot (1ms).

IF 𝑙 =0 OR 𝑙 =7: // these symbols are longer than others

NCP = 32+288 // length of the cyclic prefix

ELSE:

NCP = 288 // length of the cyclic prefix

offset = offset + NCP // Amount of samples in cyclic prefix is added.

p(𝑙) = (PDL  x offset) % 12288 // index is calculated at symbol start after cyclic prefix

offset = offset + 4096 // Length of the symbol is added

Output:

* The phaseCompensationLutIndex p(𝑙) with 𝑙 = 0..13:
