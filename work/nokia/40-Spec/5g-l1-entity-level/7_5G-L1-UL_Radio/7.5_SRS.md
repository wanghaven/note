### 7.5.2.1 SRS usages

**7.5.2.1.0-3**  (ID: `12552892`)

SRS usages
The following types of SRS and respective usage are supported in 5G-L1-UL (Radio):

|  |  |  |  |
| --- | --- | --- | --- |
| SRS type | Allocation | Usage |  |
| Thor | Odin |  |  |
| Aperiodic | UL slots | SU-MIMO | [CB012115-SR-HA] SU-MIMO [End CB012115-SR-HA] |
| Periodic | Special slots | Beam management SU-MIMO Real time beamforming | [CB012115-SR-HA] Beam management SU-MIMO [End CB012115-SR-HA] |

**Note: 5G-L1-UL (Radio) has not explicit information regarding actual SRS type. Current SRS type can be inferred via IF activation flag and frame/slot info.**

**Beam management: SRS is used as side information for the selection of the beam(s) for a specific UE.**

**Real time beamforming: SRS is used as side information for the selection of the beam(s) for a specific UE for each beamforming subband. This is the same concept as DL subband beamforming in SFS level.**

##### 7.5.3.1.2.3 Blanked PRBs

**7.5.3.1.2.3.0-1**  (ID: `11469151`)

Blanked PRBs
5G-L1-UL (Radio) shall support the following information elements from class CombiningLinkConf of HLAPI interface (see L1L\_SW\_4811):

- num\_of\_ul\_blanked\_prbs\_high\_edge: Indicates the number of UL blanked PRBs [0,268] at high edge, ending at PRB 272. The value 0 is for disabling the feature, this parameter shall be taken into account for periodic wideband SRS configuration (5G\_CP\_ARC\_ALG\_11077) and for the short-term beamforming subband (5G\_UP\_ALG\_13924\_replaced([11107540](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kog6QPvSEe-AqvopbP1qhQ))).

- num\_of\_ul\_blanked\_prbs\_low\_edge: Indicates the number of UL blanked PRBs on the low edge, starting from PRB 0.

The total number of blanked PRBs shall be determined as num\_of\_ul\_blanked\_prbs\_low\_edge+num\_of\_ul\_blanked\_prbs\_high\_edge.

Both parameters shall be set to 0 for disabling PRB blanking feature.

these parameters shall be taken into account for periodic wideband SRS configuration and for real-ime beamforming configuration.

* CoMah,+45: independent covariance matrix in horizontal and polarization +45°
* CoMav,-45: independent covariance matrix in vertical and polarization -45°
* CoMav,+45: independent covariance matrix in vertical and polarization +45°

**Algorithms:**

5G-L1-UL (Radio) compute the independent covariance matrices, using the HW element in the table below, according to the following steps:

* For horizontal independent covariance matrix:
* The horizontal covariance matrices for each RBG in each row is computed by multiplying the channel estimation of the row with its conjugate transpose.
* The horizontal independent covariance matrix (CoMah,-45 / CoMah,+45) is computed by averaging the covariance matrices over RBGs per SRS\_BM resource, followed by averaging the covariance matrices over rows with same polarization.
* For vertical independent covariance matrix:
* The vertical covariance matrices for each RBG in each column is computed by multiplying the channel estimation of the column with its conjugate transpose.
* The vertical independent covariance matrix (CoMav,-45 / CoMav,+45) is computed by averaging the covariance matrices over RBGs per SRS SRS\_BM resource, followed by averaging the covariance matrices over columns with same polarization.

**Output:**

* CoMah,-45 /CoMah,+45 horizontal independent covariance matrix of polarization -45° and polarization +45°
* CoMav,-45 /CoMav,+45 vertical independent covariance matrix of polarization -45° and polarization +45°

##### 7.5.5.4.6.3 Channel estimation replacement for interfered subband

**7.5.5.4.6.3.0-1**  (ID: `11469890`)

Channel estimation replacement for interfered subband
Description and rationale: 5G-L1-UL (radio) shall replace the CE results of those BF subbands that are detected as interfered with the CE result from the closest valid BF subband. Here valid means the BF subband is identified as low/no interference.

Input:

1. SRS channel estimation results (after pilot separation), see 5G\_L1\_15810\_replaced([11469801](https://dn-prod.ext.net.nokia.com/rm/resources/BI_fB37vPvvEe-AqvopbP1qhQ))

Algorithm:

1. Go through all BF subband indices that have high interference. For each of them, find the closest valid BF subband and copy the raw CE result.
2. If there is no valid BF subband in current SRS subband, then skip the procedure.

Output:

Updated SRS CE result per BF subband per SRS port. Size is NTRX by 1 where NTRX = numOfColumnTRX  x numOfRowTRX x 2 (UlArrayState in L1L\_SW\_216)

Note:

1. If all BF subbands are detected as high interfered, it is expected that the CE results in this period will be dropped because the SINR would be low and SRS port will be DTX = 1.
2. In case of more than one closest valid BF subbands, 5G-L1-UL (radio) shall select the one with smaller BF subband index for CE result replacement.

**Rank selection:**

The rank selection (also loosely called “RI selection”) based on SRS depends on the SRS SINR, averaged over all receive antenna, and the SRS boost, as described in Section §2.9.1 of 5GMax Algorithm Specification ([11469927](https://dn-prod.ext.net.nokia.com/rm/resources/BI_fB5w5fvvEe-AqvopbP1qhQ)). The RI algorithm consists of the following steps:

* Firstly, if SRS SINR is below the threshold *alpha, rank is selected as 1, otherwise a new threshold is computed based on SRS SINR:*

[image-0][image\_desc]The image contains a mathematical equation. The equation is:
$Thr = \frac{\alpha \* SINR\_{SRS}}{(\alpha + SINR\_{SRS})^2}$[/image\_desc][image\_path]https://storage.googleapis.com/dng\_files/images/HTML\_pukkola\_5G\_L1\_Entity\_Level\_file\_8495.001.png[/image\_path][/image-0]

-Secondly, the MMSE smoothed channel estimates of 6 subcarriers, equally spaced over the SRS bandwidth, are used for RI selection. A correlation matrix is computed for each of the chosen MMSE smoothed channel estimates:

[image-1][image\_desc]The image contains a mathematical equation. The equation is:
$R\_{subc} = H\_{subc}^H H\_{subc}$[/image\_desc][image\_path]https://storage.googleapis.com/dng\_files/images/HTML\_pukkola\_5G\_L1\_Entity\_Level\_file\_8495.002.png[/image\_path][/image-1]

Note that each **Rsubc is of size 2x2 (2 SRS ports).**

-Finally, the **Rank selection decides the preferred transmission rank for a UE as follows:**

- If [spectral flatness squared (eigenvalue geometric mean over eigenvalue arithmetic mean squared, i.e., determinant(**Rsubc )/trace2(**Rsubc )) of every                               **Rsubc is greater than or equal to *Thr] AND [SINR is larger than \*alpha], THEN rank=2.*****\***

- Otherwise, rank=1.

**Quantities needed for RI selection:**

**1. Signal and noise (+interference) power:**

Antenna specific SRS SNR is computed based on receive antenna specific signal power (Section  §2.6.8 of 5GMax Algorithm Specification) and noise power (Section §2.6.5 of 5GMax Algorithm Specification).

**2. SRS power boost:**

The SRS power boost value is defined as:

[image-2][image\_desc]The image contains a mathematical equation. The equation is:
$P\_{\text{boost\_SRS}} = \frac{\text{SRS transmit power per subcarrier}}{\text{PUSCH data transmit power per subcarrier}}$[/image\_desc][image\_path]https://storage.googleapis.com/dng\_files/images/HTML\_pukkola\_5G\_L1\_Entity\_Level\_file\_8495.003.png[/image\_path][/image-2]

RI selection threshold is UE specific and provided to 5G-L1-UL (Radio) by 5G-L1-UL (BB) using parameter *ulRiSelectionThreshold in* UlDataFH\_srsSuMimoReceiveReq message. See [11517212](https://dn-prod.ext.net.nokia.com/rm/resources/BI_FSJjSPvxEe-AqvopbP1qhQ) for info on *ulRiSelectionThreshold.*\*\*

**7.5.9.3.1.0-2**  (ID: `11470363`)

General
For L1 periodic-SRS capacity purposes, it is defined an **L1 BM resource as reception of the SRS signal using a cyclic shift of a comb of a subband within one symbol.**

For a given SRS configuration (c-SRS, b-SRS), an L1 BM resource is completely defined by the triplet (srsBmSubbandId, transmissionCombId, BmCyclicHsifts) from srsBm structure in *UlDataFH\_SrsBmReceiveReq message ([11517354](https://dn-prod.ext.net.nokia.com/rm/resources/BI_FSKxZ_vxEe-AqvopbP1qhQ)).*

One L1 BM resource corresponds to one one-port SRS for BM. When UE transmits the SRS from multiple antenna ports then each antenna port is allocated different cyclic shift, so it corresponds to multiple L1 BM resources.

For the mixed case, a dynamic load between SU-MIMO UEs and  L1 BM resources is allowed. The exchange rate between both is 1 SU-MIMO UE=20 L1 BM resources \*.

Using previous rule, to add 1 additional SU-MIMO UEs the Max. Num. of L1 BM resources is reduced by 20.

For example, for frame structure 5GC001070, when the baseline is 2 SU-MIMO UEs with 136 L1 BM resources. Following capacity numbers are allowed:

- 2 SU-MIMO UE with 136 L1 BM resources.

- 3 SU-MIMO UEs with 116 L1 BM resources.

- 4 SU-MIMO UEs with 96 L1 BM resources.

- ....

[CB012115-SR-HA]

Note: This requirement applies to all 5G-L1-UL(Radio) platforms, Thor and Odin.

[End CB012115-SR-HA]

### 7.5.10.0-2 R&amp;D parameters rdEnableSrsDataForwarding (ID: 11470407)

**7.5.10.0-2.0-4**  (ID: `12566152`)

R&D parameters
*rdEnableSrsDataForwarding*

This parameter is defined in 5G-L1-UL (radio) to indicate whether 5G-L1-UL (Radio) forwards the raw SRS IQ data over fronthaul for troubleshooting purpose.

Value range: u16

LSB+0 bit: value 1 means raw SRS IQ data of first SRS symbol forwarding, can be online modifiable.

LSB+1 bit: value 1 means raw SRS IQ data of the second SRS symbol forwarding, can be online modifiable.

LSB 4~7 bits: indicates the ccid of the cell for SRS IQ data forwarding, can be online modifiable.

LSB 14 bit: value 1 means starting SRS IQ data forwarding, can be online modifiable.

LSB+15 bit: value 1 means dedicated eAxC streams are created during carrier setup for raw SRS IQ data forwarding, can not be online modifiable, refer to L1L\_SW\_4990 for details.

Other bits are reserved and set to 0.

For example:

* 0x0000: raw SRS IQ data forwarding deactivated.
* 0x8000: dedicated eAxC streams for raw SRS IQ data forwarding are created during cell setup.
* 0xC011: starting the raw SRS IQ data forwarding for the cell with ccid 1, 5G-L1-UL (Radio) starts forwarding the raw SRS IQ data of first SRS symbol over fronthaul.
* 0xC023: starting the SRS IQ data forwarding for the cell with ccid 2, 5G-L1-UL (Radio) starts forwarding the raw SRS IQ data of both first and second SRS symbols over fronthaul.
* 0x8023: stop raw SRS IQ data forwarding.
