## 7.1.6 Latency optimization for eCPRI 7-2e for low load ("ping")

**7.1.6.0-2**  (ID: `13391411`)

Latency optimization for eCPRI 7-2e for low load ("ping")
When latency optimization is activated in UlDataFH\_PuschReceiveReq (see description in [11517052](https://dn-prod.ext.net.nokia.com/rm/resources/BI_FSIVKfvxEe-AqvopbP1qhQ?oslc_config.context=https%3A%2F%2Fdn-prod.ext.net.nokia.com%2Fgc%2Fconfiguration%2F9)), 5G-L1-UL(Radio) shall select the symbol-layer mapping indicated in ([13375735](https://dn-prod.ext.net.nokia.com/rm/resources/BI_wG1lcPwwEfCPDo8cGBoqIw?oslc_config.context=https%3A%2F%2Fdn-prod.ext.net.nokia.com%2Fgc%2Fconfiguration%2F9)) to generate the symbol reordering target.

IF ulSubCellType=C16\_8

5G-L1-UL(Radio) shall use the same pattern as subcelltype C8\_8 in 5G-L1-UL(BB).

ELSIF ulSubCellType=C16\_4

5G-L1-UL(Radio) shall use the same pattern as subcelltype C8\_4 in 5G-L1-UL(BB).

ELSE

symbol-layer mapping follows default behavior.

IF latency optimization is deactivated

symbol-layer mapping follows default behavior.

Note: The symbol mask is used by FH to allocate layer-PRB and beta coefficients to the different eAxC-IDs.

Note: Due to constraints in the number of the symbol masks handled by 5G-L1-UL(Radio), extra PRBs might be delivered to 5G-L1-UL(BB). The loss of FH traffic efficiency is assumed.
