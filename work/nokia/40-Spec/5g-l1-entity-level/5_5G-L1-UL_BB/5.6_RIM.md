#### 5.6.3.1.1 Remote Interference (RI) detection

**5.6.3.1.1.0-2**  (ID: `12763964`)

Remote Interference (RI) detection
On Loki and Thor HW, in FR1-TDD-eCPRI [CB010315-A]  and FR1-TDD-CPRI [End CB010315-A], 5G-L1-UL shall estimate RSSI per symbol and per sub-band for remote interference management (RIM).

5G-L1-UL is informed by 5G-L2-PS that it shall estimate RSSI for RIM on a given slot by the message *UlData\_RimReceiveReq (see: [11514602](https://dn-prod.ext.net.nokia.com/rm/resources/BI_FR4difvxEe-AqvopbP1qhQ)).*

The symbols and sub-bands on which RSSI shall be estimated are independent from the multiplexed channels of the slot and on the scheduling of PRBs.

The message *UlData\_RimReceiveReq contains the definition of frequency domain regions on which RSSI shall be measured:*

* the starting symbol and number of symbols,
* the number of sub-bands,
* the position and number of PRBs per sub-band.

[CB010315-A] For FR1-TDD-CPRI 5G-L1-UL shall get all the IQ data (all the antenna data), so there is no need to additional actions  (FCP creating) for getting IQ data for un-scheduled resources. The message structures: UlData\_RimReceiveReq->Subcells->unscheduledRegionInfo is not used for FR1-TDD-CPRI [End CB010315-A]

**5.6.3.3.1.1.0-1**  (ID: `11465722`)

eCPRI configuration
A common eCPRI configuration shall be used for non-scheduled PRB on which RSSI shall be estimated for RIM. It is provided in message UlData\_RimReceiveReq.

For scheduled PRB L1 uses the eCPRI configuration given in channel request messages UlData\_PuschReceiveReq, UlData\_PucchReceiveReq or UlData\_SrsReceiveReq.

##### 5.6.3.3.1.2 Spatial stream allocation for RI detection

**5.6.3.3.1.2.0-1**  (ID: `11465732`)

Spatial stream allocation for RI detection
RSSI measurements shall be performed on 2 eAxC streams.

[CB006991] If RIM framework-0 is not activated, eAxC index 0 and 1 shall be used for both C2 and C4 cell type.

If RIM framework-0 is activated and RIM-RS is requested in the same slot:

* For a C2 subcell, eAxC 0 and 1 are used.
* For a C4 subcell, eAxC 2 and 3 are used.

[End CB006991].

The number of eAxC streams and eAxC indexes to be used are given in message UlData\_RimReceiveReq.

##### 5.6.3.3.1.3 Spatial stream allocation for RIM-RS detection

**5.6.3.3.1.3.0-2**  (ID: `11465746`)

Spatial stream allocation for RIM-RS detection
**Stream configuration for RIM-RS**

L1 shall fetch data for RIM-RS detection from two dedicated eAxC streams:

* in case of a C4 subcell, eAxC indexes 2 and 3 are used,
* in case [ a C2 subcell, eAxC indexes 0 and 1 are used.

The number of eAxC  and eAxC indexes to be used are provided in message UlData\_RimReceiveReq.

On symbols and PRBs which are scheduled for PUSCH, **[Before CB006991-C-UPD00x] long PUCCH** [End CB006991-C-UPD00x] and SRS, eAxC streams used for RIM-RS detection are shared with other channels.\*\*\*\*

If RSSI measurements for RIM and RIM-RS detection are requested on the same slot, L1 shall fetch data from the same two eAxC streams for both operations.

**[Before CNI-108168] (part of requirement moved to [11465994](https://dn-prod.ext.net.nokia.com/rm/resources/BI_fBb22_vvEe-AqvopbP1qhQ)**

**If RIM-RS detection is requested over a non-delay-managed symbol, data are precombined using RIM beam method. The beam pattern ID to be used is given in message *UlData\_RimReceiveReq. RIM-RS detection is done on those precombined data.***

L1 shall get weights and pattern ID of full sector beam from IE *UlBeams of message* UlCell\_SetupReq.\*\*

Note: Periodic SRS reception on NDM-SRS symbols is not supported if RIM-RS detection is requested.

**[End CNI-108168]**

##### 5.6.3.3.1.5 RIM over NDM-SRS symbols for precise positioning

**5.6.3.3.1.5.0-1**  (ID: `11465766`)

RIM over NDM-SRS symbols for precise positioning
In non-beamforming deployment, if SRS for precise positioning is enabled (message *UlCell\_SetupReq contains parameter* ulEcpriFdBeamforming=false and parameter *srsPosAct ([11513449](https://dn-prod.ext.net.nokia.com/rm/resources/BI_FRwhwfvxEe-AqvopbP1qhQ?oslc_config.context=https%3A%2F%2Fdn-prod.ext.net.nokia.com%2Fgc%2Fconfiguration%2F9)) =*true - see [11512445](https://dn-prod.ext.net.nokia.com/rm/resources/BI_FRpM_PvxEe-AqvopbP1qhQ) and [11513501](https://dn-prod.ext.net.nokia.com/rm/resources/BI_FRxIx_vxEe-AqvopbP1qhQ), 5G-L1-UL (BB) shall support RIM operations on NDM-SRS symbols.\*\*\*\*

In that case, I/Q data for RI detection and RIM-RS detection shall be fetched from two of the NDM-SRS eAxCs. The selection os these two eAxCs is aligned on delay-managed case described in [11161420](https://dn-prod.ext.net.nokia.com/rm/resources/BI_ktUhwfvSEe-AqvopbP1qhQ):

* If only RI detection is requested, eAxC SRS[0] and SRS[1] shall be considered.
* If RIM-RS detection is requested or both RI and RIM-RS detection are requested in the same special slot, the two last NDM-SRS streams shall be considered.

Position of NDM-SRS streams is given in [11513484](https://dn-prod.ext.net.nokia.com/rm/resources/BI_FRxIxPvxEe-AqvopbP1qhQ).

**5.6.3.3.2.0-1**  (ID: `12765224`)

CPRI FH configuration
For FR1-TDD-CPRI 5G-L1-UL shall use:

* for RI RSSI measurement - two spatial streams

**5.6.3.4.1.1.1.0-2**  (ID: `11465798`)

RI detection over NDM-SRS
L1 shall support RIM on NDM-SRS symbols. In this case, RSSI estimation is performed in the same timing as SRS data processing, during the next DL slots following periodic SRS reception.

**[Before CNI-108168] (part moved to [11465865](https://dn-prod.ext.net.nokia.com/rm/resources/BI_fBbPy_vvEe-AqvopbP1qhQ)) L1-HI shall perform data precombining prior to RSSI estimation.** [End CNI-108168]\*\*\*\*

If RIM is needed on a NDM-SRS symbol, all the PRBs shall be considered as scheduled. Unscheduled regions in UlData\_RimReceiveReq shall not overlap with NDM-SRS symbols.

If RIM is needed on both DM and NDM symbols within the same slot, 5G-L1-UL will report RSSI measurements using two messages *UlData\_RimReceiveRespPs - one for DM symbols, the second one for NDM symbols.*

Parity from CB008249: This requirement also applies to Thor for FR1-TDD-eCPRI.

###### 5.6.3.4.1.1.2 RIM-RS detection over NDM-SRS symbols

**5.6.3.4.1.1.2.0-2**  (ID: `11465813`)

RIM-RS detection over NDM-SRS symbols
L1 shall support RIM-RS detection over non-delay-managed (NDM) symbols.

**[Before CNI-108168] (part moved to [11465994](https://dn-prod.ext.net.nokia.com/rm/resources/BI_fBb22_vvEe-AqvopbP1qhQ)) In this case, L1 shall perform data pre-combining prior to RIM-RS detection.** [End CNI-108168]\*\*\*\*

###### 5.6.3.4.3.1 Generation of UlDataFH_RimReceiveReq

**5.6.3.4.3.1.0-1**  (ID: `11465825`)

Generation of UlDataFH\_RimReceiveReq
On reception of message *UlData\_RimReceiveReq from L2-PS (see [11514602](https://dn-prod.ext.net.nokia.com/rm/resources/BI_FR4difvxEe-AqvopbP1qhQ)), 5G-L1-UL(BB) shall build the message* UlDataFH\_RimReceiveReq (see [11517567](https://dn-prod.ext.net.nokia.com/rm/resources/BI_FSL_lPvxEe-AqvopbP1qhQ)) and send it to Fronthaul for transmission to 5G-L1-UL (Radio) (see [11517563](https://dn-prod.ext.net.nokia.com/rm/resources/BI_FSL_k_vxEe-AqvopbP1qhQ)).\*\*

5G-L1-UL (BB) shall derive the content of message *UlDataFH\_RimReceiveReq from the content of message* UlData\_RimReceiveReq.\*\*

As message *UlDataFH\_RimReceiveReq fits in a single eCPRI packet, fragmentation of this message is not needed.*

---

###### 5.6.3.4.3.2 Reception of UlDataFH_RimReceiveRespPs

**5.6.3.4.3.2.0-1**  (ID: `11465835`)

Reception of UlDataFH\_RimReceiveRespPs
5G-L1-UL (BB) shall support the reception of message *UlDataFH\_RimReceiveRespPs from 5G-L1-UL (Radio) via eCPRI-7-2e FH interface.*

As this message fits in a single eCPRI packet, 5G-L1-UL (BB) shall support the reception of one message fragment per subcell and per slot.

On reception of this message, 5G-L1-UL (BB) shall build message *UlData\_RimReceiveRespPs (see [11514630](https://dn-prod.ext.net.nokia.com/rm/resources/BI_FR4djfvxEe-AqvopbP1qhQ)) and send it to L2-PS.*

5G-L1-UL (BB) shall derive the content of message *UlData\_RimReceiveRespPs from the content of* UlDataFH\_RimReceiveRespPs.\*\*

---

###### 5.6.3.4.3.3 Loss of message UlDataFH_RimReceiveRespPs

**5.6.3.4.3.3.0-1**  (ID: `11465845`)

Loss of message UlDataFH\_RimReceiveRespPs
If 5G-L1-UL (BB) has received a message *UlData\_RimReceiveReq from L2-PS for the current subcell and slot but has not received message* UlDataFH\_RimReceiveRespPs from 5G-L1-UL (Radio) when the end of FH reception window [*Ta4\_min\_cp..*Ta4\_max\_cp] is reached, message *UlDataFH\_RimReceiveRespPs shall be considered as lost.*\*\*\*\*

This situation can also be caused by the loss of the request message *UlDataFH\_RimReceiveReq.*

In that case, message *UlData\_RimReceiveRespPs is not sent to L2-PS.*

*Ta4\_min\_cp and* Ta4\_max\_cp are defined in "eCPRI LL Split U-Plane Application Layer Specification" (see [11470496](https://dn-prod.ext.net.nokia.com/rm/resources/BI_fB_3hvvvEe-AqvopbP1qhQ)).\*\*

### 5.6.4.9 RIM-RS detection algorithm

#### 5.6.4.9.1 Overview

**5.6.4.9.1.0-2**  (ID: `12713554`)

Overview
**RIM-RS detection algorithm**

RIM-RS detection algorithm is described in section 2.10.2 of "5GMax algorithm specification".

This algorithm shall be executed for each candidate RIM-RS sequence on each resource unit.

The main steps of RIM-RS detection algorithm are:

* Combination of the received data over both antenna streams,
* Correlation of the combined signal with the candidate root sequence in frequency domain,
* Conversion of the correlated sequence in time domain by IFFT,
* Determination of the power delay profile of the correlated signal in time domain, and search for the peak value.

The peak value is then compared to the threshold[Before CB014153-C] defined by R&D parameter *rdRimRsDetectionThreshold [CB014153-C] defined by ri*mRsDetectionThreshold which is set in message **UlCell\_SetupReq or**UlCell\_ParameterReconfigurationReq after uplink subcell's setup**. [End CB014153-C].****\***\*

* If the peak value is greater or equal to the threshold, the candidate RIM-RS sequence is considered as detected on this resource unit.
* If the peak is below the threshold, the candidate RIM-RS sequence is considered as not detected on this resource unit.

[CB010315-B, CB010315-C] The requirement is also applicable for FR1-TDD-CPRI [End CB010315-B, CB010315-C]
