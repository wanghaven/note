# 9.4 UL OFDM phase offset compensation

**9.4.0-1**  (ID: `13346441`)

UL OFDM phase offset compensation
For symbol reception in FR1 FDD, 5G-L1-UL (FHE-CPRI-GW) shall apply the phase compensation to effectively reverse the phase shift introduced by the RU. This operation is related to an OFDM symbol within a subframe (per 3GPP TS 38.211 section 5.3).

The phase compensation coefficients for each symbol are pre-calculated and stored into a constant array in L1-UL (FHE-CPRI-GW) and the index phaseCompensationLutIndex is used to get the phase compensation coefficients.

The phaseCompensationLutIndex is calculated as follows (for details, see [11274097: Generation of Phase compensation LUT](https://dn-prod.ext.net.nokia.com/rm/resources/BI_TpFZJPvcEe-AqvopbP1qhQ?oslc_config.context=https%3A%2F%2Fdn-prod.ext.net.nokia.com%2Fgc%2Fconfiguration%2F9)):

Input:

* arfcn: provided in API\_BBB\_CREATE\_EAXC\_AXC\_ROUTING\_REQ\_MSG message with direction = "EBbbEaxcDirection\_Uplink".

Definition:

* p(𝑙): phaseCompensationLutIndex, with OFDM symbol 𝑙 = 0..13
* PUL= arfcn

Algorithm:

offset = 0

FOR  𝑙 = 0 to 13

IF 𝑙 =0 OR 𝑙 =7:

NCP = 32+288

ELSE:

NCP = 288

offset = offset + NCP

p(𝑙) = (PUL x offset) % 12288

offset = offset + 4096

Output:

* The phaseCompensationLutIndex p(𝑙) with 𝑙 = 0..13:
