# 4.28 DRB Inactive State Transition Handling

## 4.28.0-2 DRB Inactive State Transition Handling (ID: 11228450)

**4.28.0-2.0-1**  (ID: `11228456`)

DRB Inactive State Transition Handling
On request to perform DRB suspend to switch from "active" to "inactive", 5G-L2-HI shall

* Suspend transmission of DL PDCP PDUs for the DRB over source F1-U tunnel by setting f1DataSendSuspended = true
* Initialize all PDCP state variables.  Set TX\_NEXT and RX\_NEXT = 0
* [Before PR725228] Discard all stored PDCP PDUs and SDUs, if any. [End PR725228]
* Retain the UL F1-U TEID and release the DL F1-U TEID.
* Mark the bearer as “inactive”.

[CB008225-B-UPD003]

* Disable all timers related to the DRB.
* If configured, disable following discarding mechanisms for the DRB

- DL PDCP SDU discarding based on queuing delay. Refer 5G\_UP\_ALG\_5938\_replaced([11224024](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kyFF8vvSEe-AqvopbP1qhQ)).

- DL PDCP SDU discarding based on discard timer. Refer 5G\_UP\_ALG\_7930\_replaced([11224150](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kyFtBfvSEe-AqvopbP1qhQ)).

[End CB008225-B-UPD003]

[CB008225-B-UPD006]

* If configured, disable monitoring of the bearer for any further activity. Refer 5G\_UP\_ALG\_6206\_replaced([11228422](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kyiY8_vSEe-AqvopbP1qhQ)).

[End CB008225-B-UPD006]

[CB008225-C]

On reception of the DL PDCP SDU for the bearer marked as “inactive”, 5G-L2-HI shall

* Buffer the DL PDCP SDU
* If the received DL PDCP SDU is the first packet received after the bearer is marked as “inactive” then 5G-L2-HI shall send a DL Data Notification Message to C Plane.

[End CB008225-C]

[CB008225-D]

On request to perform DRB resume for the bearer masked as ”inactive”, 5G-L2-HI shall perform following actions:

* Update the security keys if provided
* Initialize the F1-U flow control parameter. Refer 5G\_UP\_ALG\_5451\_replaced([11218932](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kxnL6vvSEe-AqvopbP1qhQ)).
* Enable flow control over the F1-U tunnel by setting f1DataSendSuspended = false
* Resume the DL and UL data transfer
* [PR725228] Any un-acked PDCP PDUs have to be transmitted first followed by the PDCP SDUs. [End PR725228]
* Mark the bearer as "active".

[CB008225-B-UPD003]

* Enable all timers related to the DRB.
* If configured, enable following discarding mechanisms for the DRB

- DL PDCP SDU discarding based on queuing delay. Refer 5G\_UP\_ALG\_5938\_replaced([11224024](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kyFF8vvSEe-AqvopbP1qhQ)).

- DL PDCP SDU discarding based on discard timer. Refer 5G\_UP\_ALG\_7930\_replaced([11224150](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kyFtBfvSEe-AqvopbP1qhQ)).

[End CB008225-B-UPD003]

[CB008225-B-UPD006]

* If configured, enable performing the bearer inactivity detection. Refer 5G\_UP\_ALG\_6204\_replaced([11228392](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kyhx8PvSEe-AqvopbP1qhQ)).

[End CB008225-B-UPD006]

[End CB008225-D]

On request of bearer release of a bearer marked as ”inactive”, 5G-L2-HI shall release the bearer by releasing all resources allocated for the bearer.
