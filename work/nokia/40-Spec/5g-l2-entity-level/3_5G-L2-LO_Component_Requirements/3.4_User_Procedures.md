# 3.4 User Procedures

## 3.4.1 Msg3

### 3.4.1.0-1 Msg3 Reception of Random Access Msg3 with UL CCCH in contention based Random Access procedure (ID: 11215932)

**3.4.1.0-1.0-4**  (ID: `12493000`)

Msg3
Upon reception of a RA Msg3 (UlData\_PuschReceiveRespLo, UL CCCH (SRB0), 5G\_RAN\_UP\_3907) 5G-L2-LO shall:

**[CNI-133878]If the UE is being deleted which is triggered by [12461603: Release Temporary UE due CP Overload](https://dn-prod.ext.net.nokia.com/rm/resources/BI_ofc6sQroEfCT35inaBGo7g?oslc_config.context=https://dn-prod.ext.net.nokia.com/gc/configuration/9), it's exception and 5G-L2-LO shall follow the legacy exception handling.**[End CNI-133878]\*\*\*\*

If UE have already got the valid C-RNTI previously, L2-LO will drop the UL CCCH data after receive UlData\_PuschReceiveRespLo from L1-UL.

Else:

- create a temporary UE context and store the corresponding:

* temporary C-RNTI (IE rnti of UlData\_PuschReceiveRespLo)
* RLC TMD PDU containing the UL CCCH data (corresponds to MAC CE CRI which is later sent in Msg4)

Note, 5G-L2-LO also forwards the RLC TMD PDU to C-Plane as described in 5G\_L2\_1983\_replaced([11216627](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2-_s4PvTEe-AqvopbP1qhQ)).

* start temporary UE context timer as described in 5G\_L2\_9992\_replaced([11215978](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2-7bd_vTEe-AqvopbP1qhQ))
* send LoCtrl\_UlMacPduReceiveInd with isUlCcchData set to TRUE and temporaryCrnti set to the temporary C-RNTI to 5G-L2-PS.

For relevant MSC see 5G\_L1L2\_FLOW\_178\_replaced([11186265](https://dn-prod.ext.net.nokia.com/rm/resources/BI_Be_oRvvREe-AqvopbP1qhQ))

Msg3

[End PR633523]

### 3.4.1.0-3 Msg3 Temporary UE context supervision (ID: 11215978)

**3.4.1.0-3.0-3**  (ID: `11215998`)

Msg3
Temporary UE context supervision

5G-L2-LO shall start the temporary UE context supervision timer rrmL2LoTempUeTimer\_mSec during temporary UE context establishment (upon reception of RACH Msg3). The rrmL2LoTempUeTimer\_mSec timer shall be initialized to value of IE t300Timer from LoCell\_CellsetupReq/LoCell\_CellReconfigurationReq (refer to 5G\_L2\_IF\_770\_replaced([10983299](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2V3Zvvu6Ee-AqvopbP1qhQ))/5G\_L2\_IF\_3361\_replaced([10983526](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2V5O6_u6Ee-AqvopbP1qhQ))).

If this timer expires, the temporary UE context shall be released.

The timer shall be stopped, if the UE context setup is triggered by C-Plane (upon reception of LoUser\_UserSetupReq).

[Before PR780035] the max temp UE number is 80 [PR780035] the max temp UE number is 512 [End PR780035]

For relevant MSC see 5G\_L1L2\_FLOW\_178\_replaced([11186265](https://dn-prod.ext.net.nokia.com/rm/resources/BI_Be_oRvvREe-AqvopbP1qhQ))

## 3.4.2 Msg4

### 3.4.2.0-1 Msg4 Reception of CCCH data send request (SRB0) (ID: 11216009)

**3.4.2.0-1.0-2**  (ID: `11216023`)

Msg4
Reception of CCCH data send request (SRB0)

Upon reception of message LoUser\_CcchDataSendReq carrying the DL CCCH data (SRB0), 5G-L2-LO shall

* [5GC000723-J] ignore this message if the UE Context corresponding to the C-RNTI received (IE crnti) cannot be found, otherwise [End 5GC000723-J]
* store the RRC payload (RLC TMD PDU),
* create the message LoCtrl\_CcchDataInd (see 5G\_L2\_IF\_882\_replaced([10984791](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2WC_4Pu6Ee-AqvopbP1qhQ))) and send it to 5G-L2-PS.
* [5GC000723-J] set the IE isRrcReject of LoCtrl\_CcchDataInd to the value of IE isRrcReject received in LoUser\_CcchDataSendReq. [End 5GC000723-J]

The IE sizeOfRrcMsg of LoCtrl\_CcchDataInd shall be set to the size of the RRC payload size received in LoUser\_CcchDataSendReq plus the size of the required MAC sub-header (R/F/LCID/L sub-header).

Size of the MAC sub-header (see 5G\_L2\_1970\_replaced([11215653](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2-5mSPvTEe-AqvopbP1qhQ))):

* [Before PR418986] 2 bytes
* [PR418986] 2 or 3 bytes

For relevent MSC, see 5G\_L1L2\_FLOW\_178\_replaced([11186265](https://dn-prod.ext.net.nokia.com/rm/resources/BI_Be_oRvvREe-AqvopbP1qhQ)).

## 3.4.3 Usersetup

### 3.4.3.1 Initial

#### 3.4.3.1.0-1 Initial Reception of User Setup Request (ID: 11216041)

**3.4.3.1.0-1.0-1**  (ID: `11216046`)

Initial
Upon reception of message LoUser\_UserSetupReq with ueAddCause 'UE add w/o RACH', 5G-L2-LO shall not care about missing RACH procedure and continue SRB1 setup.

---

### 3.4.3.2 RrcReEstablishment

#### 3.4.3.2.0-1 RrcReEstablishment User setup procedure with cause RRCREESTABLISHMENT (ID: 11216059)

**3.4.3.2.0-1.0-1**  (ID: `11216065`)

RrcReEstablishment
When 5G-L2-LO receives "LoUser\_UserSetupReq" message with the ueAddCause equal to "RRCREESTABLISHMENT " with corresponding unique “ueIdDu”, "nrcellidentity" and "CRNTI" following procedure should be performed:

- L2-LO should create new UE context according to received ueIdDu, nrCellIdentity and CRNTI for the target cell included in the LoUser\_UserSetupReq message. This context is permanent as no Rach procedure is expected for this UE.

- L2-LO shall wait for LoCtrl\_StartFlowControlReq to start flow Control according to 5G\_L2\_1927\_replaced([11216918](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2_BiEPvTEe-AqvopbP1qhQ))

---

### 3.4.3.3 HandOver

#### 3.4.3.3.0-1 HandOver User setup procedure with handover(not include intra cell HO) (ID: 11216078)

**3.4.3.3.0-1.0-1**  (ID: `11216084`)

HandOver
When 5G-L2-LO receives "LoUser\_UserSetupReq" message with the ueAddCause equal to

"Intra DU Handover" [5GC000704-A] or "inter DU Handover" or "Handover" [End 5GC000704-A]

with corresponding unique “ueIdDu”, "nrCellIdentity" and "CRNTI" following procedure should be performed

 L2-LO should create a new UE context according to received ueIdDu, nrCellIdentity and CRNTI for the target cell included in the LoUser\_UserSetupReq message

 user context of HO source cell deletion procedure:

a. Upon reception of LoUser\_UserDeleteReq message, using the corresponding ueIdDu1, nrCellIdentity and CRNTI, identify the user context and delete the user context.

 New user context shall be used for resume:

a. Upon complete RACH, the data in DL and UL shall resume on the new user context on the HO target cell.

---

#### 3.4.3.3.0-2 HandOver User setup procedure with intra cell handover (ID: 11216091)

**3.4.3.3.0-2.0-1**  (ID: `11216097`)

HandOver
When 5G-L2-LO receives "LoUser\_UserSetupReq" message with the ueAddCause equal to "Intra Cell Handover"

* L2-LO shall create a new UE context for target cell using the same nrCellIdentity as source cell but different ueIdDu generated by 5G-CP-RT, both ueIdDu and CRNTI for the target cell included in the LoUser\_UserSetupReq message from CP-RT will be used for differentiating the user context between HO source cell and HO target cell.

---

#### 3.4.3.4.0-1 CA User setup procedure with cause SCellAddition (ID: 11216109)

**3.4.3.4.0-1.0-1**  (ID: `11216117`)

CA

- L2-LO shall wait for LoCtrl\_StartFlowControlReq to start flow Control according to 5G\_L2\_1927\_replaced([11216918](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2_BiEPvTEe-AqvopbP1qhQ))

## 3.4.4 Usermodify

### 3.4.4.0-1 Usermodify User modify procedure with PCMD cookie (ID: 11216133)

**3.4.4.0-1.0-2**  (ID: `11216144`)

Usermodify
When 5G-L2-LO receives "LoUser\_UserModifyReq" message with "hasCookie" set to "true" and "ueIdDu", "nrCellIdentity" and "cRnti" corresponding to existing UE context:
