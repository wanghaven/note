### 3.7.1.0-2 RLC Flow Control Providing flow control status information (ID: 11216925)

**3.7.1.0-2.0-1**  (ID: `11216930`)

RLC Flow Control
Flow Control DDDS Report Mode

---

**3.7.1.0-2.0-1.0-1**  (ID: `11216940`)

RLC Flow Control
Flow Control Periodical DDDS Report Mode Selection:

At reception of LoCell\_CellSetupReq, Flow Control Periodical DDDS Report Mode been selected  based on the parameter actFlowControlLoadBalance "TURE/FALSE"  in the given cell.

Slot Counts used for Flow Control Periodical DDDS report decided by SCS, Rad Parameter "DlRlcFlowCtrlddsReportPeriod".

i.e. DlRlcFlowCtrlddsReportPeriod=5u(5ms)

|  |  |  |  |  |
| --- | --- | --- | --- | --- |
| DlRlcFlowCtrlddsReportPeriod | SCS | slotsPerMS | numOfSlotsPerReportPeriod | numOfDddsReportSlots |
| 5ms | KHZ\_15 | 1 | 5 | 5 |
| 5ms | KHZ\_30 | 2 | 10 | 10 |
| 5ms | KHZ\_120 | 8 | 40 | 40 |

-actFlowControlLoadBalance = TURE, Flow Control Periodical DDDS Report with Load Balance Mode.

The DL RLC Flow Control Periodical DDDS Report shall be balanced sent in each slot during the period.

-actFlowControlLoadBalance = FALSE, Flow Control Periodical DDDS Report without Load Balance Mode.

The DL RLC Flow Control Periodical DDDS Reports shall only be sent on the specific slot, which is upon receiving LoCtrl\_StartFlowControlReq message from L2-PS or LoUser\_BearerSetupReq from CP, for each DRB, and repeat once per Rad Parameter "DlRlcFlowCtrlddsReportPeriod".

For example: 10 Bears been Setup, and 5/3/2 bears been setup on the Slot#0/3/4 respectively.

- Flow Control Enable scenario, The bear counts on Flow Control DDDS Reports shall be balanced on each slot(total bears/slotCounts = 10/5=2),

- Flow Control Disable scenario, The bear counts on Flow Control DDDS Reports depend on the slot#, which bear been setup on.

**3.7.1.0-2.0-2.0-6**  (ID: `12679147`)

RLC Flow Control
5G-L2-LO shall construct and send the feedback message LoData\_UeDataDeliveryStatusInd to the 5G-L2-HI which contains the following:

The message contains status information

* Estimated throughput (eTh) – [PR822466] If NRCELL.drainRateCalcAdjFlowCtrl (see [10983303](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2V3Zv_u6Ee-AqvopbP1qhQ?oslc_config.context=https://dn-prod.ext.net.nokia.com/gc/configuration/9?oslc_config.context=https://dn-prod.ext.net.nokia.com/gc/configuration/9) and [10983526](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2V5O6_u6Ee-AqvopbP1qhQ?oslc_config.context=https://dn-prod.ext.net.nokia.com/gc/configuration/9?oslc_config.context=https://dn-prod.ext.net.nokia.com/gc/configuration/9)) is set to Off, then [End PR822466]

Set from the estimated throughput received from 5G-L2-PS in LoCtrl\_PduMuxReq or LoCtrl\_DlUuStatusUpdateInd. As long as the "estimated throughput" is not received from 5G-L2-PS, it is set based on 'Minimum throughput for flow control' received in LoUser\_UserSetupReq or in LoUser\_BearerSetupReq.

[PR822466]Else if NRCELL.drainRateCalcAdjFlowCtrl (see [10983303](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2V3Zv_u6Ee-AqvopbP1qhQ?oslc_config.context=https://dn-prod.ext.net.nokia.com/gc/configuration/9?oslc_config.context=https://dn-prod.ext.net.nokia.com/gc/configuration/9) and [10983526](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2V5O6_u6Ee-AqvopbP1qhQ?oslc_config.context=https://dn-prod.ext.net.nokia.com/gc/configuration/9?oslc_config.context=https://dn-prod.ext.net.nokia.com/gc/configuration/9)) is set to CalcRateAdj, then

5G-L2-LO shall set eTh according to [11192830](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kvnn0fvSEe-AqvopbP1qhQ?oslc_config.context=https://dn-prod.ext.net.nokia.com/gc/configuration/9) (5G\_UP\_ALG\_15751).[End PR822466]

* L2-LO Queueing time for RLC PDU (l2loQueueingDelayRlcPdu)  – Refer 5G\_L2\_14925\_replaced([11216234](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2-9QovvTEe-AqvopbP1qhQ?oslc_config.context=https://dn-prod.ext.net.nokia.com/gc/configuration/9?oslc_config.context=https://dn-prod.ext.net.nokia.com/gc/configuration/9?oslc_config.context=https://dn-prod.ext.net.nokia.com/gc/configuration/9))
* l2loElapsedTimeSinceLastSentPdu in case bufferThresholdLevlReached is set to BufferEmpty, is set to amount of time, in unit of ms, since the last RLC PDU has been sent out in L2-LO.The maximum value shall not exceed 255
* Highest 5G-L2-HI Sequence Number (SN) submitted to MAC – The Highest 5G-L2-HI sequence number among the completely submitted RLC PDUs (new transmission only considered, not including retransmission) within the current feedback period. In case there has been no data submitted to MAC yet after bearer setup, set the highest 5G-L2-HI sequence number to an invalid value 0xFFFFFFFF. In case no new RLC PDU were submitted to MAC after the last LoData\_UeDataDeliveryStatusInd, set the highest 5G-L2-HI sequence number to the highest 5G-L2-HI sequence number sent in the last LoData\_UeDataDeliveryStatusInd.
* RLC PDU drain rate(rlcPduDrainRate) = [PR822466] If NRCELL.drainRateCalcAdjFlowCtrl (see [10983303](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2V3Zv_u6Ee-AqvopbP1qhQ?oslc_config.context=https://dn-prod.ext.net.nokia.com/gc/configuration/9?oslc_config.context=https://dn-prod.ext.net.nokia.com/gc/configuration/9) and [10983526](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2V5O6_u6Ee-AqvopbP1qhQ?oslc_config.context=https://dn-prod.ext.net.nokia.com/gc/configuration/9?oslc_config.context=https://dn-prod.ext.net.nokia.com/gc/configuration/9)) is set to Off, then

5G-L2-LO shall set rlcPduDrainRate to [End PR822466] (Amount of RLC PDU in bytes sent to MAC between the current LoData\_UeDataDeliveryStatusInd and previous LoData\_UeDataDeliveryStatusInd)/(time difference in ms between current LoData\_UeDataDeliveryStatusInd and previous LoData\_UeDataDeliveryStatusInd)

[PR822466]Else if NRCELL.drainRateCalcAdjFlowCtrl (see [10983303](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2V3Zv_u6Ee-AqvopbP1qhQ?oslc_config.context=https://dn-prod.ext.net.nokia.com/gc/configuration/9?oslc_config.context=https://dn-prod.ext.net.nokia.com/gc/configuration/9) and [10983526](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2V5O6_u6Ee-AqvopbP1qhQ?oslc_config.context=https://dn-prod.ext.net.nokia.com/gc/configuration/9?oslc_config.context=https://dn-prod.ext.net.nokia.com/gc/configuration/9)) is set to CalcRateAdj, then

5G-L2-LO shall set rlcPduDrainRate to (Amount of RLC PDU in bytes sent to MAC between the current LoData\_UeDataDeliveryStatusInd and previous LoData\_UeDataDeliveryStatusInd)/Max(50ms, time difference in ms between current LoData\_UeDataDeliveryStatusInd and previous LoData\_UeDataDeliveryStatusInd).

Note: to reduce CPU load, time elapsed form the last scheduling is approximated by rdDlRlcFlowCtrlddsReportPeriod × NrOfDddsRaportsWithDR0, where NrOfDddsRaportsWithDR0 is incremented prior to each sent LoData\_UeDataDeliveryStatusInd with drain rate 0 and reset to 0 at each LoData\_UeDataDeliveryStatusInd with drain rate larger than 0.[End PR822466]

* bufferThresholdLevlReached – an enum value that indicates which buffer level was reached. bufferThresholdLevlReached (stored in an internal variable rrmbufferThresholdLevlReached) shall be set as follows:
* 3 Buffer is empty when current RLC buffer size at 5G-L2-LO is 0. Current RLC buffer size at 5G-L2-LO shall consider all RLC Data PDU's including RLC Data PDU's for retransmission.
* 2 (‘Higher Buffer threshold reached’) if current RLC buffer size at 5G-L2-LO > ((actOptDataFlowRateEst==true)? max (maxThput, rlcPduDrainRate) : maxThput) \* (RTT + Max Allowed Delay),
* 1 (‘Lower Buffer threshold reached’) if current RLC buffer size at 5G-L2-LO < maxThput/N\_low\_watermark \*RTT,
* 0 Else

where

* maxThput is received in LoCtrl\_PduMuxReq or LoCtrl\_DlUuStatusUpdateInd message,
* RTT is given by the parameter rdInternalFlowControlRTT, cf. 5G\_UP\_ALG\_2607\_replaced([11118832](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kqGOqvvSEe-AqvopbP1qhQ?oslc_config.context=https://dn-prod.ext.net.nokia.com/gc/configuration/9?oslc_config.context=https://dn-prod.ext.net.nokia.com/gc/configuration/9?oslc_config.context=https://dn-prod.ext.net.nokia.com/gc/configuration/9)),
* Max Allowed Delay is given by the parameter rdInternalFlowControlMaxDelay,
* N\_low\_watermark (default=4) is given by the parameter rdNlowWatermark, cf. 5G\_L2\_2082\_replaced([11220204](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2_W5QvvTEe-AqvopbP1qhQ?oslc_config.context=https://dn-prod.ext.net.nokia.com/gc/configuration/9?oslc_config.context=https://dn-prod.ext.net.nokia.com/gc/configuration/9?oslc_config.context=https://dn-prod.ext.net.nokia.com/gc/configuration/9)),
* actOptDataFlowRateEst is a parameter in msg LoUser\_UserSetupReq.drbAddReqLo, refer to 5G\_L2\_IF\_2544\_replaced([10983749](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2V7EFfu6Ee-AqvopbP1qhQ?oslc_config.context=https://dn-prod.ext.net.nokia.com/gc/configuration/9?oslc_config.context=https://dn-prod.ext.net.nokia.com/gc/configuration/9?oslc_config.context=https://dn-prod.ext.net.nokia.com/gc/configuration/9)).

If actCsiMeasEthFlowControl= ‘PastTputBased’, then rmbufferThresholdLevlReached  IE shall be set to ‘0’, regardless RLC buffer size.

The variable rrmbufferThresholdLevlReached shall be calculated at each reception of a LoCtrl\_PduMuxReq or LoCtrl\_DlUuStatusUpdateInd message.

Note: a L2 /MAC TTI trace of hitting either upper or the lower buffer threshold is highly recommended.

* pollSnSubmittedToMac - The highest RLC sequence number among the complete submitted RLC PDUs with pollBitIncluded = True (new transmission and retransmission) within the current feedback period.

In case there has been no complete RLC PDU(s) with pollBitIncluded = True submitted to MAC within the current feedback period, it is set to INVALID(0xFFFFFFFF).

Note: In case there are multiple RLC SN with poll bit set submitted to MAC, pollSnSubmittedToMac shall be set to the highest RLC SN with pollBitIncluded = True which has been submitted to MAC.

### 3.7.1.0-5 RLC Flow Control Handling of Nack and DTX flood (ID: 11217086)

**3.7.1.0-5.0-1**  (ID: `11217090`)

RLC Flow Control
5G-L2-LO shall provide support for handling of Nack and DTX flood in the flow control

---

**3.7.1.0-5.0-1.0-1**  (ID: `11217098`)

RLC Flow Control
[Before 01016677]

In case of massive occurrence of HARQ Nack and Dtx indications, 5G-L2-LO shall mediate between 5G-L2-HI and 5G-L2-PS in handling of massively deteriorated radio condition.

The parameter highNackDtxInd in the LoCtrl\_PduMuxReq message from 5G-L2-PS according 5G\_L2\_IF\_863\_replaced([10984505](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2WAjrfu6Ee-AqvopbP1qhQ)) is 'true' if cumulated Nack+Dtx to Ack ratio over a certain time interval exceeds a sustainable level.

### 3.7.6.1 Carrier Aggregation for the bearers with RLC UM Mode

#### 3.7.6.1.0-1 Carrier Aggregation for the bearers with RLC UM Mode RLC SN assignment for the segmented RLC SDU under UM Mode (ID: 12408103)

**3.7.6.1.0-1.0-2**  (ID: `12814451`)

Carrier Aggregation for the bearers with RLC UM Mode
The SN Full Range shall be considered as default setting before the **L2LO SN Index is indicated from 5G-L2-HI(refer to [12346362](https://dn-prod.ext.net.nokia.com/rm/resources/TX_Qc3-sASWEfCJ0Mh_3UUzdQ)).**

5G-L2-Hi shall indicate 5G-L2-Lo instance to switch to the RLC SN indicated by the pair {RLC SN Sub Range Index, **L2LO SN Index}.**

For the RLC UM bearer configured with SN Sub Range:

Once the RLC SN Sub Range Index info received from 5G-L2-HI via [Before CB015504-C] LoData\_DlRlcPduSendReq [CB015504-C] LoData\_DlDrbRlcPduSendReq [End CB015504-C](intra-gNB CA Ref:[10985810](https://dn-prod.ext.net.nokia.com/rm/resources/TX_0IDTwPu5Ee-AqvopbP1qhQ)) or (inter-gNB CA Ref: [10995309](https://dn-prod.ext.net.nokia.com/rm/resources/TX_0xutcPu6Ee-AqvopbP1qhQ)), 5G-L2-LO instances shall update the local pair {RLC SN Sub Range Index, **L2LO SN Index} info immediately.**

Once the {RLC SN Sub Range Index, **L2LO SN Index} updated, 5G-L2-LO instances(both Pcell and Scell) shall switch the RLC SN immediately to the starting RLC SN of {RLC SN Sub Range Index,** L2LO SN Index} for the new segmented RLC SDU according to the table([11193282](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kvrSN_vSEe-AqvopbP1qhQ)).\*\*\*\*

For the pending segments of the RLC SDU, the RLC SN shall align with the 1st segment of the RLC SDU(already sent to MAC before the {RLC SN Sub Range Index, **L2LO SN Index} be updated).**

In case RLC SN of the current {RLC SN Sub Range Index, **L2LO SN Index} exhausted before the new RLC SN Sub Range Index indication from 5G-L2-HI, 5G-L2-LO shall switch to the starting RLC SN of the {RLC SN Sub Range Index+1,** L2LO SN Index} according to the table([11193282](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kvrSN_vSEe-AqvopbP1qhQ)), and feedback to 5G-L2-HI via immediate DDDS with new RLC SN Sub Range Index.\*\*\*\*

For the RLC UM bearer configured with SN Full Range:

In case last Scell removal, then the SN Sub Range shall be switch to SN Full Range for the Pcell 5G-L2-LO instance.

Once the RLC UM bearer configured with SN Full Range, The lowest RLC SN of the {RLC SN Sub Range Index + 2, **L2LO SN Index} shall be used as the starting RLC SN.**

Note:

When L2LO with SN Full Range and RLC SN within the range{4032~4095}(RLC SN 12 bits), 5G-L2-Lo report RLC SN Sub Range Index = 41 to 5G-L2-Hi.
