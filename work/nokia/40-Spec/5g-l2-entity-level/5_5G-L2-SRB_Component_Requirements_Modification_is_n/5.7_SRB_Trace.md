# 5.7 SRB Trace

**5.7.0-3**  (ID: `11241119`)

SRB Trace
5G-L2-SRB shall log information of every PDCP PDU in UL

Following additional information will be sent to Emil tracing for each SrbSgnl\_UlSduReceiveInd message to trace L2-SRB when a SDU is constructed from the UL PDCP PDU(s) and sent to CP-UE.

The following information will be added as additional parameters to 5G\_L2\_IF\_1311\_replaced([10992969](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2W7Jrvu6Ee-AqvopbP1qhQ))

|  |  |
| --- | --- |
| RX\_RCVD\_COUNT | Indicates the received COUNT for the PDCP PDU |
| RX\_NEXT COUNT | Indicates the next expected to be received COUNT |
| RX\_DELIV COUNT | Indicates the next to be delivered COUNT |
| RX\_REORD\_COUNT | Indicates the most recent COUNT that triggered reordering |
| T-Reorder timer status | Indicates if t-Reorder timer is currently running or expired |

In addition to the above, L2-SRB shall perform the following textual logging in the below mentioned conditions.

- When for a SrbSgnl\_UlPduReceiveInd message received at L2-SRB, a SrbSgnl\_UlSduReceiveInd is not sent to CP-UE for various reasons listed

1. When a PDU is discarded with the possible PDU Discard causes viz.

2. When received PDU triggers reordering

|  |  |
| --- | --- |
| RX\_RCVD\_COUNT | Indicates the SN for the PDCP PDU |
| RX\_NEXT COUNT | Indicates the next expected SN |
| RX\_DELIV COUNT | Indicates the next to be delivered SN |
| T-Reorder timer status | Indicates the current status of the t-Reorder timer. Possible values are: - Timer never started and not running - Timer started but expired - Timer is running - Timer is stopped |
| RX\_REORD COUNT | Indicates the SN that started the t-Reorder timer |
| ueIdCu | Identifies the UE ID in gNB-CU |
| Srb Identity | Indicates the SRB Id |
| Bearer Index | Indicates the bearer Index for the SRB ID for this UE |
| PDU Discard Cause | Indicates the PDU Discard cause with possible values: - No Discard - Integrity Verification failure - Duplicate PDU - Invalid PDU - Bearer Release |

Note: All the above mentioned trace and log entries should be post PDU processing, after the PDCP RX state variables are updated and when the PDU is sent to the CP-UE or is buffered.

Note: CP-UE shall not interpret these extra parameters for any functional requirement. These extra parameters are purely for debug and information purposes.
