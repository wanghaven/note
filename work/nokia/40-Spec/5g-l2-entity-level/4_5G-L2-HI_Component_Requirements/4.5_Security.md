## 4.5.1 General

**4.5.1.0-2**  (ID: `11224762`)

General
5G-L2-HI shall maintain unique COUNT value per each direction (uplink and downlink) for each RB instances.

The COUNT is a combination of Hyper Frame number (HFN) and PDCP SN. Refer to 5G\_UP\_IF\_3731\_replaced([11533117](https://dn-prod.ext.net.nokia.com/rm/resources/BI_tWzLKPv1Ee-AqvopbP1qhQ)).

The corresponding COUNT value of the PDCP PDU shall be used by 5G-L2-HI for Ciphering functions as per 5G\_UP\_ALG\_6216\_replaced([11223741](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kyDQwvvSEe-AqvopbP1qhQ)).

Detection of COUNT rollover shall be done as defined in 5G\_UP\_ALG\_6222\_replaced([11223794](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kyDQyvvSEe-AqvopbP1qhQ)). If the rollover has occurred, L2-HI informs CPlane by sending HiUserCu\_PdcpMaxCountInd defined in 5G\_L2\_IF\_1061\_replaced([10988415](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2Wd2rPu6Ee-AqvopbP1qhQ)).

## 4.5.4 Sensitive parameters

### 4.5.4.1 General

#### 4.5.4.1.0-1 General This chapter describes the sensitive data that can be present in various artifacts that may be shared with offshore development teams as part of product maintenance or troubleshooting. This list of items will satisfy the requirements of multiple regulatory and commercial agencies. These items come from multiple technologies (2G, 3G, 4G, 5G etc.) for multiple products. "Sensitive Parameters" defines the list of sensitive data items from Nokia MN RAN BTS products that can be traced back to end users or network topology. This list will be kept up to date with future regulatory and commercial requirements and also due to product development. When a sensitive data is added/modified via features or CNIs, this list should be updated as part of feature FS1TA/CFAM process. Parameters in this document are analysed for their sensitivity as per the regulatory requirements and marked to be scrambled or not. Verification teams can use this document to check for scrambling of sensitive data. (ID: 11224838)

### 4.5.4.2 Sensitive parameters info

**4.5.4.2.0-2**  (ID: `11224856`)

Sensitive parameters info
Sensitive data type:

Device Identifier - Any information element that uniquely identifies the device used by the end user to access the 3GPP network

IP Address - An IP address in any form (binary or text) or format (IPv4 or IPv6)

MAC Address - A Media Access Control address

Network Element Identifier - Any identifier of a node or network element in a 3GPP network

Network Element Name - A human-friendly name associated to a network element

Protocol Container - A message from a higher level protocol carried transparently between two interfaces of the RAN

Security Related - Any information element related to the authentication of the end user or the ciphering or integrity protection of the information exchanged with the network (signalling or data)

Service Area Identity - Any identifier of an area in which the end user has access to a particular service. The term "service" should be considered in a very broad sense; a cell is a service area, as well as a routing area is a service area

Service Area Name - A human-friendly name associated to a service area

Subscriber Identity - Any information element that may uniquely identify the end user

User Data - Data sent or received by the end user or the device used by the end user

Usernames And Passwords - Login information
