## 2.17.1 L2-PS Configuration through R&amp;D Parameters to support Test Phases in 5G18A

### 2.17.1.0-5 L2-PS Configuration through R&amp;D Parameters to support Test Phases in 5G18A Enabling/disabling of continuous scheduling in DL and UL (ID: 12750101)

**2.17.1.0-5.0-2**  (ID: `12750103`)

L2-PS Configuration through R&D Parameters to support Test Phases in 5G18A
[Before CNI-157970]

DL:

rdDlAlwaysScheduled = 1:

The RRC connected UE is added to the DL CS1 independently from its DL BS, according to what specified before 5GC000523-A;

rdDlAlwaysScheduled = 0 (defaut):

The RRC connected UE is added to the DL CS1 only if it is a candidate, according to what specified from 5GC000523-A;

UL:

rdUlAlwaysScheduled = 1:

The RRC connected UE is added to the UL CS1 independently from its UL BS, according to what specified before 5GC000523-A;

rdUlAlwaysScheduled = 0 (defaut):

The RRC connected UE is added to the UL CS1 only if it is a candidate, according to what specified from 5GC000523-A;

[End CNI-157970]

### 2.17.1.0-6 L2-PS Configuration through R&amp;D Parameters to support Test Phases in 5G18A Enabling/Disabling of periodic CSI scheduling on PUCCH and UE periodic CSI-RS scheduling (CSI-RS for channel measurement) (ID: 12762664)

**2.17.1.0-6.0-2**  (ID: `12762665`)

L2-PS Configuration through R&D Parameters to support Test Phases in 5G18A
[Before CNI-137616-B]

rdActivatePerCsi = 1 (default):

* Periodic CSI on PUCCH is scheduled according to the configuration received in PsUser\_UserSetupReq;
* UE Periodic CSI-RS is scheduled according to the configuration received in PsUser\_UserSetupReq;

rdActivatePerCsi = 0:

* Even if a UE is configured with periodic CSI on PUCCH in PsUser\_UserSetupReq, L2-PS does not schedule its reception;
* Even if a UE is configured with periodic CSI-RS resources in PsUser\_UserSetupReq, L2-PS does not schedule its transmission;

If long PUCCH is configured then L2-PS shall ignore rdActivatePerCsi set to 0, i.e. CSI reception and CSI-RS transmission shall always be scheduled as configured.

[End CNI-137616-B]

### 2.17.1.0-7 L2-PS Configuration through R&amp;D Parameters to support Test Phases in 5G18A Enabling/Disabling of aperiodic CSI feature (channel measurement) (ID: 11215130)

**2.17.1.0-7.0-1**  (ID: `11215136`)

L2-PS Configuration through R&D Parameters to support Test Phases in 5G18A
The aperiodic CSI feature is enabled/disabled on L2-PS side by an activation flag (actAperCsiCell) received from CP-RT within PsCell\_CellSetupReq (see 5G\_L2\_IF\_61\_replaced([10967196](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2UTTc_u6Ee-AqvopbP1qhQ))).

In addition to the activation flag at cell level, each UE receives a dedicated flag within PsUser\_UserSetupReq (see 5G\_L2\_IF\_389\_replaced([10974645](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2U9ayPu6Ee-AqvopbP1qhQ))) which used for enabling/disabling the aperiodic CSI scheduling functionality for that particular UE.

**2.17.1.0-10**  (ID: `11215156`)

L2-PS Configuration through R&D Parameters to support Test Phases in 5G18A
Simulating the UE capability twoPUCCH-F0-2-ConsecSymbols

[PR455759] A dedicated R&D parameter rdFddF0F2TwoSymbols shall be supported to simulate the twoPUCCH-F0-2-ConsecSymbols UE capability

When rdFddF0F2TwoSymbols = 0 (default):

L2-PS considers all UEs it manages don't have the  twoPUCCH-F0-2-ConsecSymbols UE capability.

When rdEnableUlDynTransmissionScheme = 1:

L2-PS considers all UEs it manages have the  twoPUCCH-F0-2-ConsecSymbols UE capability.

## 2.17.2 Requirements without Subfeature Links

**2.17.2.0-1**  (ID: `11215170`)

Requirements without Subfeature Links
[Candidate to be added to 5GC000447]

Selection of patternId in analog beamforming, beam blanking test mode

Precondition:

(actDigitalBF == false) and (rdBeamMutingTestModeActivationFlag == true)

If (beamMutingBit[beamId] == 0) and (portMutingBit[port] == 0) Then

patternId = antennaPatternStart +(beamId + mapStartShift ) MOD numOfBrs,

where the variables are defined in the same way as in 5G\_RAN\_UP\_8295 with

Else  patternId = nullPatternId

where nullPatternId=2^15-1, similar to 5G\_RAN\_UP\_4448
