### 6.3.1.0-1 Address Distribution and Deploying Pooling Sequence diagram of pooling deployment (ID: 11244737)

**6.3.1.0-1.0-1**  (ID: `11244747`)

Address Distribution and Deploying Pooling
[image-0][image\_desc]The image is a sequence diagram illustrating the "Address Distribution Req PRB pooling configuration" process. The diagram shows interactions between different components: CP-RT, SGNL, SCH-DL1, SCH-UL1, SCH-DL2, SCH-UL2, and BBRM.
The process begins with CP-RT sending an `addressDistributionReq(...)` to SGNL. If the distribution and pooling data are incorrect, CP-RT sends an `addressDistributionReq(error)`. If the data is valid, SGNL creates L2PS EO, which involves multiple "EO creation" steps directed towards SCH-DL1, SCH-UL1, SCH-DL2, and SCH-UL2.
Concurrently, SGNL saves pooling configuration details in a "GlobalDb", including `poolingMode`, `poolingPeriod`, `prbPoolingAllowed`, and `scheUePoolingAllowed`. SGNL then sends an `AddressDistributionResp(ok)` to CP-RT, indicating that it does not wait for a BBRM response.
If the pooling feature is allowed, SGNL sends a pooling configuration to BBRM via `poolingDeploymentReq(pooling config)`. BBRM then updates its L1subpool list, which includes `L1poolld`, a list of DL L1subpoolld per L1pool, and a list of UL L1subpoolld per L1pool. Finally, BBRM sends a `poolingDeploymentResp(ok)`.
The diagram uses different arrow styles (solid and dashed) to represent synchronous and asynchronous messages, respectively. The components are arranged horizontally, and the timeline progresses vertically downwards.

### 6.3.1.0-2 Address Distribution and Deploying Pooling Reception of Address Distribution Message (ID: 11244751)

**6.3.1.0-2.0-1**  (ID: `11244758`)

Address Distribution and Deploying Pooling
When Address distribution message is received by L2PS-SGNL EO, if the value of  l2RtPoolDeploymentInformations.bbPoolInfos is present, SGN sends the poolingDeploymentReq message to BBRM EO.

BBRM replies by poolingDeploymentResp message.
