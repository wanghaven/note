###### 6.2.1.7.3.0-1 DL To UL Intra Sched CA PUCCH PRB Update Overview (ID: 13398042)

**6.2.1.7.3.0-1.0-1**  (ID: `13398056`)

DL To UL Intra Sched CA PUCCH PRB Update
Messages:  
PsInternal\_DlToUlIntraSchedCaPucchPrbUpdate

Purpose:  
Via this message, DL Scheduler informs DL scheduler of the same 5G-L2-PS  instance about first available PUSCH PRBs for SCell scheduling. The message is sent only in Carrier aggregation scenario in PsPeerCtrl\_ScellUeSchedulingInd, see [10979612](https://dn-prod.ext.net.nokia.com/rm/resources/TX_FTDcoPu5Ee-AqvopbP1qhQ).

Precondition:  
The message is sent only after reception of PsPeerCtrl\_ScellUeSchedulingInd, see [10979612](https://dn-prod.ext.net.nokia.com/rm/resources/TX_FTDcoPu5Ee-AqvopbP1qhQ), from SCell. DL scheduling information, for this SCell, regarding number of PUCCH bits used for SCell scheduling is ready for a given UE.

Addressing:  
The request is sent to UL UP message queue.

Relevant UP MSCs:

None

###### 6.2.1.7.3.0-2 DL To UL Intra Sched CA PUCCH PRB Update Description (ID: 13398044)

**6.2.1.7.3.0-2.0-1**  (ID: `13398058`)

DL To UL Intra Sched CA PUCCH PRB Update
The message is sent from DL to UL Scheduler, only for the SCell, after reception of PsPeerCtrl\_ScellUeSchedulingInd, see [10979612](https://dn-prod.ext.net.nokia.com/rm/resources/TX_FTDcoPu5Ee-AqvopbP1qhQ). The message is introduced to overcome the timing delay of PUCCH PRB update for a SCell in a single slot.

Note: Without this message, the PsInternal\_DlToUlIntraSchedUpdate message is sent before reception of PsPeerCtrl\_ScellUeSchedulingInd and therefore the PUCCH PRB information will not be sent in the processing slot which introduces timing delay.

### 6.2.1.13 Dynamic switching procedures

#### 6.2.1.13.1 Dynamic timing pattern switching

##### 6.2.1.13.1.0-1 Dynamic timing pattern switching Overview (ID: 11244592)

**6.2.1.13.1.0-1.0-1**  (ID: `11244596`)

Dynamic timing pattern switching
Messages:

PsInternal\_DynamicSwitchReq

PsInternal\_DynamicSwitchResp

Purpose:

To trigger the dynamic timing pattern switching on DL scheduler EO (i.e. PS-DL EO) of the other cell of the L2 subpool and SRS Beam management EO (SRS-BM EO) of its proper cell of the L2 subpool.

Precondition:

A DL scheduler of the L2 subpool has detected either the entry into overload condition based on the new load metric or the exit from overload condition based on max CS1 list size conditions.

Addressing:

The request is sent from DL scheduler which has detected the entry/exit into/from overload condition (PS\_DL1 EO) to DL scheduler (PS\_DL2 EO) of the other cell of the L2 subpool and to SRS-BM EO of its proper cell of the L2 subpool

The response is sent from PS-DL2 EO to PS\_DL1 EO

Both messages are sent to DL SchTime message queue.

Relevant UP MSCs:

NA

##### 6.2.1.13.1.0-2 Dynamic timing pattern switching Description (ID: 11244605)

**6.2.1.13.1.0-2.0-1**  (ID: `11244611`)

Dynamic timing pattern switching
Upon reception of the request, DL scheduler (PS\_DL2 EO) of the other cell of the L2 subpool shall inform the UL scheduler (PS\_UL4 EO) and the SRS beam management (SRS-BM2 EO).

DL scheduler (PS\_DL2 EO) shall set the k2Cell of its cell to rrmK2min and if timing pattern = from\_500us\_to\_750us, send the response with k2Cell and status to  DL scheduler (PS\_DL1 EO).

Upon reception of the response, DL scheduler (PS\_DL1 EO) shall inform the UL scheduler (PS\_UL5 EO) and the SRS beam management (SRS-BM1 EO).

Note that if timing pattern = from\_750us\_to\_500us, there is no need to send the response with k2Cell because at the end of the dynamic switching  each cell will be in timing pattern 500us and will use their proper k2Cell (i.e. rrmK2min).
