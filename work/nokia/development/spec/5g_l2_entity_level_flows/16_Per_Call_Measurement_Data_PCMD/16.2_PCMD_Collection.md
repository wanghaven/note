The diagram shows the following components:
- 5G-CP-RT
- 5G-L2-PS / 5G-L2-LO
- BTS Trace Controller (BTC)
- PCell
- Loop (for each SCell)
The sequence of events is as follows:
1. **5G-CP-RT** sends a message "(10) User Setup / User Modify with PCMD cookie" to **5G-L2-PS / 5G-L2-LO**.
2. **5G-L2-PS / 5G-L2-LO** then sends a message "(20) SlaveServiceProxyEm.startCalIPCell()" to **PCell**.
3. **PCell** sends a reference message "Ref (30) BTC start call tracing" to **BTS Trace Controller (BTC)**.
4. A loop is initiated for "for each SCell".
5. Inside the loop, **5G-L2-PS / 5G-L2-LO** sends a message "(40) SlaveServiceProxyEm.startCallSCell()" to the respective **SCell**.
6. Each **SCell** sends a reference message "Ref (30) BTC start call tracing" to **BTS Trace Controller (BTC)**.
7. Finally, a message "PCMD activated for the UE" is displayed at the bottom, indicating the completion of the activation process.

[image-0][image\_desc]The image is a sequence diagram illustrating the process of PCMD trace data provision from 5G-L2-PS/5G-L2-LO to BTC.
The diagram is divided into two main sections: "Loop" and "Alt".
**Section 1: Loop (Periodic Data Reporting)**
\* **Participants:**
\* 5G-L2-PS / 5G-L2-LO
\* BTS Trace Controller (BTC)
\* **Activation:** The process is initiated when "PCMD activated".
\* **Flow:**
1. **(10) Every PCMD reporting period:** This represents a periodic trigger.
2. **(20) SlaveServiceProxyEm.storeRecords():** A message is sent from 5G-L2-PS/5G-L2-LO to BTC.
3. **Ref (30) BTC store PCMD data:** This indicates a reference to an action performed by BTC, which is to store the PCMD data.
**Section 2: Alt (Non-Periodic Data Reporting - Event Base)**
\* **Participants:** Same as above.
\* **Activation:** This section is triggered by "Non-Periodic data reporting (Event base)".
\* **Flow:**
1. **(40) Initial Timing Advance:** This represents an event-based trigger.
2. **(20) SlaveServiceProxyEm.storeRecords():** Similar to the periodic reporting, a message is sent from 5G-L2-PS/5G-L2-LO to BTC.
3. **Ref (30) BTC store PCMD data:** Again, this indicates BTC storing the PCMD data.
