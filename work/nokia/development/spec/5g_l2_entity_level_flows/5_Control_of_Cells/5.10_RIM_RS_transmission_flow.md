Framework 1 RIM RS1 and RIM RS2 transmission flow from 5G-L2-PS perspective.

[image-2][image\_desc]The image is a sequence diagram illustrating the "MSC 5G\_L1L2\_FLOW\_9823 Framework 1 RIM RS1 and RIM RS2 indication flow". The diagram depicts interactions between two entities: "5G-L2-PS" and "5G-L1".
Here's a breakdown of the flow:
1. **Initial Interaction:**
\* "5G-L2-PS" receives an address via "DIData\_AddressResp(rimRsSendReqAddress)" from "5G-L1".
\* "L2-PS" then uses this received address to send "DIData\_RimRsSendReq".
\* "L2-PS" enters "State 2" if a remote interference detection mechanism is active or if "rsTransTrigger" is not equal to "RI trigger".
2. **Loop #1: RIM RS Period:**
\* This loop is triggered when the "RIM RS period happens".
\* Within the "RIM RS1 trans. period", "Cell Setld RIM RS1 transmission window happens".
\* This occurs if "State 2b" is active or if "rsTransTrigger" is set to "constantly trigger" or "RS1 constantly".
3. **Loop #2: RIM RS1 Transmission Repetition:**
\* "Cell Setld RIM RS1 transmission repetition happens".
\* This results in sending "DIData\_RimRsSendReq(RIM RS1)" to "5G-L1".
4. **RIM RS2 Transmission Window:**
\* Within the "RIM RS2 trans. period", "Cell Setld RIM RS2 transmission window happens".
\* This occurs if "State 2b" is active (and at least one RS1 is detected) or if "rsTransTrigger" is set to "constantly trigger" or "RS2 constantly".
5. **Loop #3: RIM RS2 Transmission Repetition:**
\* "Cell Setld RIM RS1 transmission repetition happens".
\* This results in sending "DIData\_RimRsSendReq(RIM RS2)" to "5G-L1".
