## 15.3.2 Description

### 15.3.2.0-1 Description (1) UlData_PuschReceiveReq (ID: 11208173)

**15.3.2.0-1.0-1**  (ID: `11208181`)

Description
k2 slots before every CG slot, the UlData\_PuschReceiveReq is sent to 5G-L1-UL to receive the CG PUSCH, with the following setting:

* rnti = CS-RNTI (it is used for the data scrambling)
* cRnti = C-RNTI

---

### 15.3.2.0-2 Description (2) PUSCH (ID: 11208190)

**15.3.2.0-2.0-1**  (ID: `11208200`)

Description
UE sends to 5G-L1-UL the CG PUSCH related to its CG resource allocated during the CG activation

---

### 15.3.2.0-3 Description (3) UlData_PuschReceiveRespPs (ID: 11208208)

**15.3.2.0-3.0-1**  (ID: `11208217`)

Description

### 15.3.2.0-5 Description (5) UlData_PuschReceiveRespHarqU (ID: 11208245)

**15.3.2.0-5.0-1**  (ID: `11208254`)

Description
5G-L1-UL returns to 5G-L2-PS the UlData\_PuschReceiveRespHarqU message with the following setting:

* rnti = CS-RNTI
* CRC = OK
* DTX = non-DTX

[CB008124-C]

If CG initial transmission over PUSCH.

Counter PRB\_ALLOC\_CG\_ACT\_PUSCH M55308C00032 shall be incremented by the number of PUSCH PRBs, which are allocated from the CG Pool for CG activated UE for configured grant.

Counter UL\_SLOT\_ALLOC\_CG M55308C00033 shall be incremented.

If PUSCH CRC-OK and non-DTX detected by gNB for a CG initial transmission over PUSCH.

Counter CG\_INI\_TX\_PUSCH\_SUCC M55308C00026 shall be incremented.

If PUSCH CRC-OK and non-DTX detected by gNB for a CG retransmission over PUSCH

Counter CG\_RE\_TX\_PUSCH\_SUCC M55308C00029 shall be incremented.

[End CB008124-C]

### 15.3.2.0-8 Description (8) UlData_PuschReceiveRespHarqU (ID: 11208299)

**15.3.2.0-8.0-1**  (ID: `11208308`)

Description
5G-L1-UL returns to 5G-L2-PS the UlData\_PuschReceiveRespHarqU message with the following setting:

* rnti = CS-RNTI
* CRC = NOK
* DTX = non-DTX

[CB008124-C]

If CG initial transmission over PUSCH.

Counter PRB\_ALLOC\_CG\_ACT\_PUSCH M55308C00032 shall be incremented by the number of PUSCH PRBs, which are allocated from the CG Pool for CG activated UE for configured grant.

Counter UL\_SLOT\_ALLOC\_CG M55308C00033 shall be incremented.

If PUSCH CRC-NOK detected by gNB for a CG initial transmission over PUSCH.

Counter CG\_INI\_TX\_PUSCH\_UNSUCC M55308C00027 shall be incremented.

If PUSCH CRC-NOK detected by gNB for a CG retransmission over PUSCH

Counter CG\_RE\_TX\_PUSCH\_UNSUCC M55308C00030 shall be incremented.

[End CB008124-C]

Perform UL Harq ReTx: refer to 5G\_L1L2\_FLOW\_561\_replaced([11200624](https://dn-prod.ext.net.nokia.com/rm/resources/BI_BgNwQPvREe-AqvopbP1qhQ)).

### 15.3.2.0-10 Description (10) UlData_PuschReceiveRespHarqU (ID: 11208332)

**15.3.2.0-10.0-1**  (ID: `11208340`)

Description
5G-L1-UL returns to 5G-L2-PS the UlData\_PuschReceiveRespHarqU message with the following setting:

* rnti = CS-RNTI
* DTX = DTX

[Before CB008124-C]

This is a normal case in which no UL data is ready to be transmitted in the CG slot

[CB008124-C]

If CG initial transmission over PUSCH.

Counter PRB\_ALLOC\_CG\_ACT\_PUSCH M55308C00032 shall be incremented by the number of PUSCH PRBs, which are allocated from the CG Pool for CG activated UE for configured grant.

Counter UL\_SLOT\_ALLOC\_CG M55308C00033 shall be incremented.

If DTX detected by gNB and UE UL Data is not null for a CG initial transmission over PUSCH.

Counter CG\_INI\_TX\_PUSCH\_DTX M55308C00028 shall be incremented.

[End CB008124-C]
