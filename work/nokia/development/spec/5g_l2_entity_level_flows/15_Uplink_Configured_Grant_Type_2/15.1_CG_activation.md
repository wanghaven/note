## 15.1.2 Description

### 15.1.2.0-1 Description (1) DlData_PdcchSendReq (ID: 11207715)

**15.1.2.0-1.0-1**  (ID: `11207724`)

Description
Conditions to trigger the CG activation are fulfilled (refer to 5G\_L2\_11025\_replaced([11175752](https://dn-prod.ext.net.nokia.com/rm/resources/BI_26u38fvTEe-AqvopbP1qhQ))).

A PDCCH candidate to send the DCI 0\_1 CG activation  with CG activation format (refer to 5G\_UP\_ALG\_14810\_replaced([11172938](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kuPHy_vSEe-AqvopbP1qhQ))) is available.

The DlData\_PdcchSendReq message is sent to 5G-L1-DL with the following setting:

* rnti = CS-RNTI
* dciScramblingRnti = C-RNTI
* dciPayload = DCI 0\_1 CG activation with CG activation format (refer to 5G\_UP\_ALG\_14810\_replaced([11172938](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kuPHy_vSEe-AqvopbP1qhQ)))

[CB008124-C]

Counter UL\_CG\_ACT\_ATT M55308C00020 shall be incremented.

[End CB008124-C]

---

### 15.1.2.0-2 Description (2) PDCCH (ID: 11207732)

**15.1.2.0-2.0-1**  (ID: `11207743`)

Description
PDCCH containing the DCI 0\_1 CG activation  with CG activation format (refer to 5G\_UP\_ALG\_14810\_replaced([11172938](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kuPHy_vSEe-AqvopbP1qhQ))) is sent to UE

---

### 15.1.2.0-3 Description (3) UlData_PuschReceiveReq (ID: 11207751)

**15.1.2.0-3.0-1**  (ID: `11207758`)

Description
puschSlotOffset slots (refer to 5G\_L1\_IF\_280\_replaced([11514747](https://dn-prod.ext.net.nokia.com/rm/resources/BI_FR5EovvxEe-AqvopbP1qhQ))) before the first CG slot, the UlData\_PuschReceiveReq is sent to 5G-L1-UL to receive the MAC CE Configured Grant Confirmation from UE, with the following setting:

* rnti = CS-RNTI (it is used as a protection, i.e. only the right UE can have a CRC OK. It allows UE to discriminate the CS-RNTI from the C-RNTI)
* cRnti = C-RNTI (it is used for forwarding to 5G-L2-LO)

---

### 15.1.2.0-4 Description (4) PUSCH (ID: 11207768)

**15.1.2.0-4.0-1**  (ID: `11207778`)

Description
UE sends to 5G-L1-UL the PUSCH that is expected to contain at least the MAC CE Configured Grant Confirmation

---

### 15.1.2.0-5 Description (5) UlData_PuschReceiveRespPs (ID: 11207787)

**15.1.2.0-5.0-1**  (ID: `11207794`)

Description
5G-L1-UL forwards to 5G-L2-PS the UlData\_PuschReceiveRespPs message with rnti = CS-RNTI

---

### 15.1.2.0-6 Description (6) UlData_PuschReceiveRespHarqU (ID: 11207803)

**15.1.2.0-6.0-1**  (ID: `11207812`)

Description
5G-L1-UL forwards to 5G-L2-PS the UlData\_PuschReceiveRespHarqU message with the following setting:

* rnti = CS-RNTI
* CRC = OK
* DTX = non-DTX

---

### 15.1.2.0-7 Description (7) UlData_PuschReceiveRespLo (ID: 11207821)

**15.1.2.0-7.0-1**  (ID: `11207829`)

Description
5G-L1-UL forwards to 5G-L2-LO the UlData\_PuschReceiveRespLo message with the following setting:

* rnti = C-RNTI (5G-L2-LO does not need to know the CS-RNTI to C-RNTI mapping)
* data = TB received from UE and containing the MAC CE Configured Grant Confirmation

---

### 15.1.2.0-8 Description (8) LoCtrl_UlMacPduReceiveInd (ID: 11207839)

**15.1.2.0-8.0-1**  (ID: `11207847`)

Description
5G-L2-LO detects, in the TB received from UE, the MAC CE Configured Grant confirmation.

5G-L2-LO sends the LoCtrl\_UlMacPduReceiveInd with isConfiguredGrantContCe = True to inform 5G-L2-PS that CG activation is completed

From this moment, 5G-L2-PS shall consider that the UE is CG activated.

[CB008124-C]

Counter UL\_CG\_ACT\_ATT\_SUCC M55308C00021 shall be incremented.

[End CB008124-C]
