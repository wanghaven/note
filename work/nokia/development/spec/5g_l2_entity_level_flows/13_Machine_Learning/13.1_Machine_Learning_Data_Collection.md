**13.1.1.2.0-1.0-2**  (ID: `11206570`)

Description
[Before CB008761-L] 5G-CP informs 5G-L2-PS about activated usecase. This includes model tag and inference engine sicad address. Other information may be Machine Learning Usecase specific. For details, please refer specification of given Machine Learning Usecase (see 5G\_L2\_9412\_replaced([11214903](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2-1U2fvTEe-AqvopbP1qhQ))).

[CB008761-L]

5G-CP informs 5G-L2-PS activate data collection procedure for a specific use case.

PsMl\_ActivateUseCaseReq includes [PR805810]actMlScellAct, [End PR805810] usecaseId(usecaseId, please refer to 5G\_L3\_31105\_replaced([11069639](https://dn-prod.ext.net.nokia.com/rm/resources/BI_BDRVgv_KEe-D3seUwaI0tg))), data collection engine sicad address and data collection period which is NIDD from OAM. For details, please refer specification of given Machine Learning Usecase (see 5G\_L2\_9412\_replaced([11214903](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2-1U2fvTEe-AqvopbP1qhQ))).

[PR805810]actMlScellAct is the flag of ML based Scell activation use case, which is always be true in PsMl\_ActivateUseCaseReq.[End PR805810]

usecaseId and data collection engine sicad address are updated immediately.

data collection period is updated after current period pass and next period use new value.

PsMl\_ActivateUseCaseReq for Data collection and PsMl\_ActivateUseCaseReq for Inference are independent.  PsMl\_ActivateUseCaseReq for Data collection may be earlier than PsMl\_ActivateUseCaseReq for Inference or later than PsMl\_ActivateUseCaseReq for Inference. The sequence of the two requests depends on different scenarios.

Scenario 1: cell setup procedure followed by PsMl\_ActivateUseCaseReq for Data collection, then after the model is available, PsMl\_ActivateUseCaseReq for Inference will be triggered. The interval of two proceures may be more than one day.

Scenario 2: feature online activation/deactivation or cell goes into/leaving energy saving state. The two procedures are in parallel.

[End CB008761-L]

### 13.1.1.3 Exceptions

#### 13.1.1.3.0-1 Exceptions Exceptions: (ID: 11206635)

**13.1.1.3.0-1.0-2**  (ID: `11206652`)

Exceptions
[Before CB008761-L]

Expections are specific to usecases. For deatils please refer requirements of given Machine Learning Usecase (see 5G\_L2\_9412\_replaced([11214903](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2-1U2fvTEe-AqvopbP1qhQ))).

[CB008761-L]

Exception 1: DataCollectionRequest and InferenceRequest are empty in PsMl\_ActivateUseCaseReq. Action: This use case won’t be activated and return NOK to CPRT.

Exception 2: Invalid cellIdentity ​in PsMl\_ActivateUseCaseReq. Action: This use case won’t be activated and return NOK to CPRT.

Exception 3: FDD or TDD FR1 w/o BF or TDD FR2 or NSA for Scell activation use case​ in PsMl\_ActivateUseCaseReq. Action: This use case won’t be activated and return NOK to CPRT.

Exception 4: Invalid use case Id (only 1)​ in PsMl\_ActivateUseCaseReq. Action: This use case won’t be activated and return NOK to CPRT.

Exception 5: Invalid period (valid: 0 ~ 2s)​ in PsMl\_ActivateUseCaseReq. Action: This use case won’t be activated and return NOK to CPRT.

Exception 6: Address/modelId in deactivation not exist in L2PS ​in PsMl\_ActivateUseCaseReq. Action: This use case won’t be activated and return NOK to CPRT.

Exception 7: Only 3 types of MIMO mode (CLOSEDLOOPSPATIALMULTIPLEXING\_8X4, OPENLOOPSPATIALMULTIPLEXING\_4X4OR4X2, CLOSEDLOOPSPATIALMULTIPLEXING\_4X4OR4X2) is supported. Action: This use case won’t be activated and return NOK to CPRT in case of other MIMO mode.

[End CB008761-L]

Message definition: 5G\_L2\_IF\_6690\_replaced([10982000](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2Vu22fu6Ee-AqvopbP1qhQ))

**13.1.3.2.0-1.0-1**  (ID: `11206820`)

Description
Data collection period is configured by PsMl\_ActivateUseCaseReq and started after first PsMl\_ActivateUseCaseReq. During the period, Pcell L2-PS collect every input data and output data and store them.

When period timeout, Pcell L2-PS calculate the average data for input and output data and send them in MlDataPushMsg to Data Collection Engine. Different use case have different data.

Input and output data are define in 5G\_UP\_7224\_replaced([11365636](https://dn-prod.ext.net.nokia.com/rm/resources/BI_udUXC_vkEe-AqvopbP1qhQ)).
