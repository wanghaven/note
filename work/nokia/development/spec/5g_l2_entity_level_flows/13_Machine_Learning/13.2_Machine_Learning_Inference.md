### 13.2.1.2 Description

#### 13.2.1.2.0-1 Description (1) PsMl_ActivateUseCaseReq (ID: 11206888)

**13.2.1.2.0-1.0-1**  (ID: `11206896`)

Description
5G-CP informs 5G-L2-PS activate inference procedure for a specific use case.

PsMl\_ActivateUseCaseReq includes usecaseId(usecaseId, please refer to 5G\_L3\_31105\_replaced([11069639](https://dn-prod.ext.net.nokia.com/rm/resources/BI_BDRVgv_KEe-D3seUwaI0tg))

), inference engine sicad address. For details, please refer specification of given Machine Learning Usecase (see 5G\_L2\_9412\_replaced([11214903](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2-1U2fvTEe-AqvopbP1qhQ))).

PsMl\_ActivateUseCaseReq for Data collection and PsMl\_ActivateUseCaseReq for Inference are independent.  PsMl\_ActivateUseCaseReq for Data collection may be earlier than PsMl\_ActivateUseCaseReq for Inference or later than PsMl\_ActivateUseCaseReq for Inference. The sequence of the two requests depends on different scenarios.

Scenario 1: cell setup procedure followed by PsMl\_ActivateUseCaseReq for Data collection, then after the model is available, PsMl\_ActivateUseCaseReq for Inference will be triggered. The interval of two proceures may be more than one day.

Scenario 2: feature online activation/deactivation or cell goes into/leaving energy saving state. The two procedures are in parallel.

#### 13.2.1.2.0-2 Description (2) PsMl_ActivateUseCaseResp (ID: 11206906)

**13.2.1.2.0-2.0-1**  (ID: `11206913`)

Description
5G-L2-PS needs to validate any parameters related with activated usecase. In case of any expected errors NOK result shall be sent to 5G-CP. For details, please refer specification of given Machine Learning Usecase. PsMl\_ActivateUseCaseResp shall contain the same transactionId as corresponding PsMl\_ActivateUseCaseReq.

### 13.2.1.3 Exceptions

**13.2.1.3.0-1**  (ID: `11206930`)

Exceptions
Exception 1: DataCollectionRequest and InferenceRequest are empty in PsMl\_ActivateUseCaseReq. Action: This use case won’t be activated and return NOK to CPRT.

Exception 2: Invalid cellIdentity ​in PsMl\_ActivateUseCaseReq. Action: This use case won’t be activated and return NOK to CPRT.

Exception 3: FDD or TDD FR1 w/o BF or TDD FR2 or NSA for Scell activation use case​ in PsMl\_ActivateUseCaseReq. Action: This use case won’t be activated and return NOK to CPRT.

Exception 4: Invalid use case Id (only 1)​ in PsMl\_ActivateUseCaseReq. Action: This use case won’t be activated and return NOK to CPRT.

Exception 5: Address/modelId in deactivation not exist in L2PS ​in PsMl\_ActivateUseCaseReq. Action: This use case won’t be deactivated and return NOK to CPRT.

Exception 6: Only 3 types of MIMO mode (CLOSEDLOOPSPATIALMULTIPLEXING\_8X4, OPENLOOPSPATIALMULTIPLEXING\_4X4OR4X2, CLOSEDLOOPSPATIALMULTIPLEXING\_4X4OR4X2) is supported. Action: This use case won’t be activated and return NOK to CPRT in case of other MIMO mode.
