### 10.3.1-1.0-1 Descriptions : [Exc.1] When L2-PS receives a EN_DSS_DL_CELL_RESOURCE_AREA_ALLOCATION_REQUEST with Extension_01 and it doesn’t received yet CatMinfo in PsCell_DssCellCrmClientCfgReq received from CP-RT, L2-PS answers EN_DSS_DL_CELL_RESOURCE_AREA_ALLOCATION_FAILURE with EN DSS Failure Cause(Subcarrier Allocation Failure) (ID: 11204198)

### 10.3.1.0-1 Descriptions : (1) step: (ID: 11204107)

**10.3.1.0-1.0-1**  (ID: `11204114`)

Descriptions :
L2-PS receives Cat-M Narrow Band within PsCell\_DssCellCrmClientCfgReq.CatMinfo. from CP-RT when CP-RT receives them from the CRM.

CatMinfo could be not present in the first PsCell\_DssCellCrmClientCfgReq. If actLteNrDssLteCatM (defined in 5G\_L2\_IF\_122\_replaced([10969219](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2UbPTPu6Ee-AqvopbP1qhQ))) set to TRUE, L2-PS stores CatMinfo the first time when it’s received.

### 10.3.1.0-3 Descriptions : (3) step: (ID: 11204138)

**10.3.1.0-3.0-1**  (ID: `11204146`)

Descriptions :
If actLteNrDssLteCatM (defined in 5G\_L2\_IF\_122\_replaced([10969219](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2UbPTPu6Ee-AqvopbP1qhQ))) set to TRUE:

On receipt of EN\_DSS\_DL\_CELL\_RESOURCE\_AREA\_ALLOCATION\_REQUEST, L2-PS gets Cat-M Narrow band reservation for the new 80ms patterns:

Cat-M channels NB reservation which shall avoid or rate-matched by L2-PS:

 Cat-M data MPDXCH:

In all normal subframe symbols [3…13] for PRB area received in PsCell\_DssCellCrmClientCfgReq.CatMinfo.dataCatMPrbAreaList.prbArea[0] (5G\_L2\_IF\_1528\_replaced([10972080](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2UsVCPu6Ee-AqvopbP1qhQ)))

 Cat-M MPBCH:

If PsCell\_DssCellCrmClientCfgReq.CatMinfo.repeatedMPBCH (5G\_L2\_IF\_1527\_replaced([10972068](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2UsVBvu6Ee-AqvopbP1qhQ))) equal to TRUE, in all slots 0 and 9,  for  PRB [23..29] 10Mz, PRB [36..42] 15Mhz and PRB [50..56] 20Mhz , symbols [3…13].

 Cat-M Paging:

The slot pattern (80ms start at SFN%8=0) is received in EN\_DSS\_DL\_CELL\_RESOURCE\_AREA\_ALLOCATION\_REQUEST. DL PRB Allocation Info. Extension\_01. PRB area for Cat-M Paging Subframes (CRM\_3361) and the PRB area in PsCell\_DssCellCrmClientCfgReq.CatMinfo.pagingCatMAreaList.prbArea[0] (5G\_L2\_IF\_1529\_replaced([10972091](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2UsVCvu6Ee-AqvopbP1qhQ))).

 Cat-M SIB:

The slot pattern (80ms start at SFN%8=0) is received in EN\_DSS\_DL\_CELL\_RESOURCE\_AREA\_ALLOCATION\_REQUEST. DL PRB Allocation Info. Extension\_01. PRB area (A..D) for Cat-M SIB1-BR Subframes (CRM\_3361) and the PRB area in PsCell\_DssCellCrmClientCfgReq.CatMinfo.Sib1CatMAreaList.prbArea[0..3] (5G\_L2\_IF\_1530\_replaced([10972103](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2UsVDPu6Ee-AqvopbP1qhQ))) with the following association:

PRB area A for Cat-M SIB1-BR Subframes => Sib1CatMAreaList.prbArea[0]

PRB area B for Cat-M SIB1-BR Subframes => Sib1CatMAreaList.prbArea[1]

PRB area C for Cat-M SIB1-BR Subframes => Sib1CatMAreaList.prbArea[2]

PRB area D for Cat-M SIB1-BR Subframes => Sib1CatMAreaList.prbArea[3]

Note Cat-M NB reserved are not excluded from DL PRB Allocation Info. PRB Area Allocation List.
