### 2.15.2.0-1 Maximum number of UEs per NRCellgroup (under preparation) Common (ID: 11214555)

**2.15.2.0-1.0-1**  (ID: `11214560`)

Maximum number of UEs per NRCellgroup (under preparation) In 5G-L2-PS software the max cell capacity for RRC UE, nbMaxUeRrcPerCellGroup, is managed by one NRcellgroup. The possible number of CSI report for CQI depends on

  * Used FR and Duplex mode

  * Frame Pattern

  * L1 Baseband Block

## 2.15.3 Maximum number of Scheduled UEs per Slot in one 5G Cell without BB Pooling (under preparation)

### 2.15.3.0-1 Maximum number of Scheduled UEs per Slot in one 5G Cell without BB Pooling (under preparation) Common (ID: 11214679)

**2.15.3.0-1.0-1**  (ID: `11214684`)

Maximum number of Scheduled UEs per Slot in one 5G Cell without BB Pooling (under preparation) At 5G-L2-PS level the number of PDSCH/PUSCH Scheduled UEs per slot, rrmMaxNumUesFdm, is depending on FDM.

FDM allows the scheduling of multiple UEs in a single slot by giving each UE different frequency resources. DL FD resources are counted with granularity of RBGs. UL FD resources are counted with granularity of PRBs.

FDM is currently supported in FR1 TDD and FDD only. In FR2, the behavior of FD scheduling is not changed by 5GC000679. Only a single UE is scheduled with full BW allocation as before.

rrmMaxNumUesFdm is the max number of PDSCH/PUSCH scheduled UEs per slot.

if actFdmScheduling is False then rrmMaxNumUesFdm equals to 1.

## 2.15.4 Maximum number of Scheduled UEs per Slot in one 5G Cell with BB Pooling (under preparation)

### 2.15.4.0-1 Maximum number of Scheduled UEs per Slot in one 5G Cell with BB Pooling (under preparation) Common (ID: 11214779)

**2.15.4.0-1.0-1**  (ID: `11214786`)

Maximum number of Scheduled UEs per Slot in one 5G Cell with BB Pooling (under preparation) At 5G-L2-PS level the number of PDSCH/PUSCH Scheduled UEs per slot in one cell with baseband (BB) pooling depends on FDM and L1/L2 pool/subpool capacities in one L2-RT.

The scheduled UEs per slot (UL&DL) are pooled among the cells on same BB card. BB Resource Manager (BBRM) shall distribute dynamically the BB pooled resources to cells that belong in this BB pool.

During each BB pooling period (bbPoolEvalPeriod received in poolingDeploymentReq message), DL/UL scheduler sends the resourceReq (cf. 5G_L2_3804_replaced([11243928](https://dn-prod.ext.net.nokia.com/rm/resources/BI_3BzwVfvTEe-AqvopbP1qhQ))) to BBRM to query available scheduled PDSCH and PUSCH UEs per slot per cell and then BBRM will reply by the resourceResp (cf. 5G_L2_3806_replaced([11243966](https://dn-prod.ext.net.nokia.com/rm/resources/BI_3BzwW_vTEe-AqvopbP1qhQ))) to provide the available scheduled PDSCH and PUSCH UEs per slot per cell. Refer to 5G_UP_ALG_5264_replaced([11210371](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kw4zLvvSEe-AqvopbP1qhQ)) and 5G_L2_842_replaced([11174052](https://dn-prod.ext.net.nokia.com/rm/resources/BI_26g1gPvTEe-AqvopbP1qhQ)) for more details.

actFdmScheduling, actScheUePooling, actConnRrcPooling must be True and pucchModeSelect=longPucch.

rrmMaxNumUesFdm is the max number of PDSCH/PUSCH scheduled UEs per slot that can be reached by a cell out of the total available schedule UE per slot of the L1 subpool, L1 pool during a pooling period. It is received in BbrmScheUeResult (5G_L2_5811_replaced([11241431](https://dn-prod.ext.net.nokia.com/rm/resources/BI_3Bkfz_vTEe-AqvopbP1qhQ))).

## 2.15.7 Maximum number of FR1 CA cell relations per cell

### 2.15.7.0-1 Maximum number of FR1 CA cell relations per cell Maximum number of FR1 CA cell relations per cell in L2-PS (ID: 11214878)

**2.15.7.0-1.0-1**  (ID: `11214885`)

Maximum number of FR1 CA cell relations per cell The maximum number of FR1 CA cell relations per cell in L2-PS is controlled by 5G constant MAX_NUM_REMOTE_CELL. 

This constant defines L2-PS cell can support maximum MAX_NUM_REMOTE_CELL outgoing cell relations setup and MAX_NUM_REMOTE_CELL incoming cell relations setup.

But this constant does not mean the maxmium number of active cell relations per cell in L2-PS can support. The maximum active cell relations per cell can support should be defined by each CA feature's C&P sub-feature. And maximum active cell relations per cell <= MAX_NUM_REMOTE_CELL.

Where:

outgoing cell relations setup per cell: as a PCell, setup cell relation with a SCell

incoming cell relations setup per cell: other cell as PCell setup cell relation with the cell as SCell

active cell relation: if on one PCell-SCell relation, there is at least one UE activated, then it is an active cell relation; otherwise, if there is no UE activated on the PCell-SCell relation, then it is not an active cell relation.

[CB008950-A]

The value of MAX_NUM_REMOTE_CELL is increased from 10 to 21. Value is set by L2-PS in 5G constant mt file.

[End CB008950-A]
