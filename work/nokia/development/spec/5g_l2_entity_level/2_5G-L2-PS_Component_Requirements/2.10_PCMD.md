#### 2.10.1.1.0-2 PCMD reporting PCMD reporting for Carrier Aggregation SCells (ID: 12712008)

**2.10.1.1.0-2.0-1**  (ID: `12712014`)

PCMD reporting
In the full list of supported Trace Records (see: [12454978: PCMD measurements for MAC layer operation](https://dn-prod.ext.net.nokia.com/rm/resources/BI_FK8avgpAEfCT35inaBGo7g?oslc_config.context=https%3A%2F%2Fdn-prod.ext.net.nokia.com%2Fgc%2Fconfiguration%2F9)) one will also find Trace Records with the prefix "CA".

* BTS Trace Controller uses these records to deliver PCMD measurements from SCells for UEs that support Carrier Aggregation.
* Typically, the Trace Records with the prefix "CA" have exactly the same structure as the regular non-CA Trace Records. The only additional field is an NR Cell Identity of a given SCell.
* The Trace Records with the prefix "CA" are not visible as such to 5G-L2-PS. 5G-L2-PS shall use regular Trace Records to report measurements for both PCell and SCells. Later, the BTS Trace Controller shall build the final structure by appending the identity of the given SCell on top of the structure reported by 5G-L2-PS.

### 2.10.1.3 Operation on Cell level

#### 2.10.1.3.0-1 Operation on Cell level 1) At reception of PsCell_CellSetupReq (ID: 11198406)

**2.10.1.3.0-1.0-1**  (ID: `11198410`)

Operation on Cell level
5G-L2-PS shall register the new cell to BTS Trace Controller with the add() function.

* For a cell with no ulSubcells,
* 5G-L2-PS shall indicate the capability to support DL trace records only.
* 5G-L2-PS shall indicate the capability to support SCell trace records only.
* For other cells,
* 5G-L2-PS shall indicate the capability to support both UL and DL trace records.
* 5G-L2-PS shall indicate the capability for PCell and SCell trace records.

Rationale: If no ulSubcells configured, the cell is for DL transmission only and cannot act as PCell. Note that every cell has at least one dlSubcell. Capabilities for PCell and SCell trace records shall be indicated separately via the add() function.

**2.10.1.3.0-3.0-1**  (ID: `11198419`)

Operation on Cell level
5G-L2-PS shall immediately send the last trace records for each UE still in the cell and having PCMD activated, and clean up PCMD related memory reservations of the UEs. Finally, the 5G-L2-PS shall deregister the cell from the BTS Trace Controller with the remove() function.

b) or at end of the ongoing collection period.

#### 2.10.1.4.0-3 Operation on UE level 3) At reception of PsUser_UserModifyReq (ID: 11198458)

**2.10.1.4.0-3.0-1**  (ID: `11198462`)

Operation on UE level
If the cookie is provided and the UE has no cookie configured yet, 5G-L2-PS shall:

1. store the received cookie,
2. assign an ueResourceId [5G\_L2\_IF\_1512\_replaced([10994963](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2XIlAPu6Ee-AqvopbP1qhQ))] for the UE per each cell configured via PsUser\_UserSetupReq,
3. register the new UE to the BTS Trace Controller via startCallPCell() function in PCell (PCell/PsCell) and via startCallSCell() function in each SCell configured for the UE.

If the cookie is already configured for the UE, 5G-L2-PS shall ignore the new value and not perform any further actions.

If the cell configuration of the UE is modified:

* For each newly added cell of the UE, if the UE has a valid cookie value, 5G-L2-PS shall assign an ueResourceId for the UE in the cell and register the cell to BTS Trace Controller via startCallSCell() function.
* For each deleted cell of the UE, if PCMD is activated for the UE in the cell, 5G-L2-PS shall immediately send the last trace records with “isLast” flag set and clean up PCMD related memory reservations of the UE in the cell. In addition, if the UE has a valid cookie value, 5G-L2-PS shall also deregister the UE to BTS Trace Controller via stopCall() function of the cell (regardless of whether PCMD was activated for the UE in the cell or not).

## 2.10.4 PCMD Collection Algorithms in 5G-L2-PS

### 2.10.4.1 Collecting Timing Advance Trace Record with Initial TA

#### 2.10.4.1.0-1 Collecting Timing Advance Trace Record with Initial TA (ID: 11200061)

**2.10.4.1.0-1.0-1**  (ID: `11200066`)

Collecting Timing Advance Trace Record with Initial TA
Event based measurements are usually reported to TC proxy upon receiving the related event, however Initial Timing Advance (InitialTA) measurement (see 5G\_TC\_552) requires specific handling, because it is determined during RA procedure at msg1 reception (see 5G\_UP\_ALG\_3381\_replaced([11152328](https://dn-prod.ext.net.nokia.com/rm/resources/BI_ksfbVvvSEe-AqvopbP1qhQ))), when the pernament UE context may not be yet known.

If PCMD collection for the Timing Advance Trace Record (see 5G\_TC\_551) has been activated for an UE, L2-PS shall report the InitialTA value to TC proxy as soon as the UE (cookie, ueResourceId, CRNTI) is known during or after the RA procedure.

If PCMD collection for the Timing Advance Trace Record is not activated for an UE, but the UE has a valid cookie/ueResourceId (meaning PCMD is not permanently disabled for it), L2-PS shall store the InitialTA value for possible delayed delivery, in case when Timing Advance Trace Record will be activated for the UE later.

L2-PS shall store and report only the first (InitialTA) TA value determined for an UE, and discard any subsequent TA values during lifetime of the UE context.

L2-PS shall report the InitialTA value of an UE to TC proxy once after every Timing Advance Trace Record activation request.

Along with InitialTA, L2-PS shall record the time (timestamp) of InitialTa measurement. From the timestamp, L2-PS shall calculate and report the time which passed since the recording time, when reporting to TC proxy.

For detailed description of 5G-L2-PS behaviour on RT core see 5G\_L2\_1536\_replaced([11200072](https://dn-prod.ext.net.nokia.com/rm/resources/BI_29nz5vvTEe-AqvopbP1qhQ)) and on NRT core 5G\_L2\_1538\_replaced([11200084](https://dn-prod.ext.net.nokia.com/rm/resources/BI_29nz6PvTEe-AqvopbP1qhQ)).

#### 2.10.4.1.0-2 Collecting Timing Advance Trace Record with Initial TA Collection of Timing Advance Trace Record with Initial TA on RT core (ID: 11200072)

**2.10.4.1.0-2.0-1**  (ID: `11200078`)

Collecting Timing Advance Trace Record with Initial TA
Whenever L2-PS application on RT core prepares and sends RAR (msg2) Timing Advance command (see 5G\_UP\_ALG\_3417\_replaced([11152587](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kshQgvvSEe-AqvopbP1qhQ))), it shall deliver the InitialTA from RA msg1 (UlData\_PrachReceiveInd, see 5G\_L1\_IF\_234\_replaced([11514309](https://dn-prod.ext.net.nokia.com/rm/resources/BI_FR2oWvvxEe-AqvopbP1qhQ))) and RAR transmission time (sfn, slot) to L2-PS application on NRT core for collection of the Timing Advance Trace Record. Before delivery, InitialTA value shall be converted to Ts units:

'InitialTA\_converted' = 'InitialTA\_msg1' \* (16/2μ),

where μ is the numerology. Numerology can be determined from cell configuration scs. Depending on PRACH format, either Long PRACH or Short PRACH scs should be used and mapped to numerology as follows (μ, scs): (0, 15KHz), (1, 30KHz), (2, 60KHz), (3, 120KHz), (4, 240KHz).

Converted InitialTA and transmission time are further called in this requirement as "measurements".

Detailed behaviour depends on UE state:

a) During Contention Free RA (any Contention free RA scenario)

Characterization: UE has a permanent context (with a reserved dedicated preamble) known after msg1 reception (identified by msg1 preamble).

* L2-PS application on RT core shall send the measurements to NRT core.

b) During Contention Based RA (any Contention based RA scenario)

Characterization: UE is not known at msg1 reception and a temporary UE context is created for it.

* L2-PS application on RT core shall store the measurements to the temporary UE context created for the UE.
* When L2-PS application on RT core receives msg3, two cases:

b-1) During SgNB addition in NSA or P(s)Cell change in SA/NSA or Beam Recovery or other Contention based RACH procedure, where this UE already exists

Characterization: UE has a permanent context known via contention resolution after msg3 reception (identified by msg3 CRNTI MAC CE).

* After contention resolution is completed, L2-PS application on RT core shall deliver the stored measurements to NRT core for this permanent UE.

b-2) [5GC001756-D] During Contention based Initial Access in SA

Characterization: A new permanent UE context is created for the UE after msg3 reception.

* After permanent UE context creation is completed, L2-PS application on RT core shall deliver the stored measurements to NRT core for this permanent UE.

[End 5GC001756-D]

Note: PCMD support for SA added via 5GC001756-D, until that PCMD is supported in NSA only.

#### 2.10.4.1.0-3 Collecting Timing Advance Trace Record with Initial TA Collection of Timing Advance Trace Record on NRT core (ID: 11200084)

**2.10.4.1.0-3.0-1**  (ID: `11200088`)

Collecting Timing Advance Trace Record with Initial TA
L2-PS application on NRT core shall store only the first initial TA value and corresponding time (further called in this requirement as "measurements") for an UE. Once after each Timing Advance Trace Record (see 5G\_TC\_551) activation request from TC proxy for the UE, L2-PS shall calculate the time difference between the current time and InitialTA transmission time. If the difference is less or equal to value supported by TC (see 5G\_L2\_IF\_1507\_replaced([10995099](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2XJMFPu6Ee-AqvopbP1qhQ))) then L2-PS shall immediately deliver both InitialTA and calculated time difference to TC proxy. Otherwise record shall not be reported.

Detailed behaviour:

1) When L2-PS application on NRT core receives measurements for an UE from RT core,

a) If measurements are not stored for this UE yet, L2-PS application on NRT core shall store the new measurements for the UE.

* In addition, if PCMD collection for Timing Advance Trace Record is activated for the UE, L2-PS application on NRT core shall deliver data (InitialTA and calculated time difference) to TC proxy.

Note that if PCMD collection for Timing Advance Trace Record is not activated for the UE, no further actions at this phase, but L2-PS shall wait for possible later activation.

b) If measurements are already stored for this UE, L2-PS application on NRT core shall ignore/discard the newly received measurements.

2) When L2-PS application on NRT core receives a Timing Advance Trace Record activation request from TC proxy for the UE,

* if measurements are stored for the UE, L2-PS application on NRT core shall deliver data (InitialTA and calculated time difference) to TC proxy.

Note that if measurements are not stored for the UE, no further actions at this phase, but L2-PS shall wait for possible TA measurement delivery after the UE enters the cell via the first RA procedure.

### 2.10.4.3 Data conversion algorithms

#### 2.10.4.3.0-1 Data conversion algorithms Measurement conversion to the basic PMCD time units (ID: 11200113)

**2.10.4.3.0-1.0-1**  (ID: `11200117`)

Data conversion algorithms
When specified, 5G-L2-PS shall convert the measured time in slots to the basic time units used by PCMD:

the basic time unit used by PCMD is equal to 1/32 milliseconds.

Slot duration depends on subcarrier spacing configured within PsCell\_CellGroupSetupReq message content (see: 5G\_L2\_IF\_3117\_replaced([10966509](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2UPCB_u6Ee-AqvopbP1qhQ))).

* If the scs IE was set to 15 kHz, 5G-L2-PS shall multiply the number of slots by 32 (slot duration 1000 μs = 32 units).
* If the scs IE was set to 30 kHz, 5G-L2-PS shall multiply the number of slots by 16 (slot duration 500 μs = 16 units).
* If the scs IE was set to 120 kHz, 5G-L2-PS shall multiply the number of slots by 4 (slot duration 125 μs = 4 units).
