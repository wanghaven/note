# 5.3 Security

## 5.3.0-5 Security 5G-L2-SRB shall support PDCP lock step function for SRB1 (ID: 11241001)

**5.3.0-5.0-2**  (ID: `11241015`)

Security
The PDCP lock step function shall be defined in the following phases

Phase1-No Security:

Downlink

The MAC-I is set to 0.

Uplink

The received MAC-I’s shall be ignored and the messages shall be forwarded to C-Plane.

Phase2-Lock Step:

Downlink

Integrity protection for the message shall be done.

Ciphering shall be done except for the message that cause PDCP lock step activation.

Uplink

Integrity verification shall be done based on corresponding MAC-I.

Deciphering is not performed for the message.

The message plus verification result shall be forwarded to C-Plane.

5G-L2-SRB shall pause processing of next message and wait for indication from C-plane to resume.

Note: Upon reception of the SrbUser\_SecurityDeactivationReq, 5G-L2-SRB shall exit lock step and security function shall be applied as defined in "Phase1-No Security".  Refer 5G\_L2\_IF\_1409\_replaced([10993531](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2W-0Dfu6Ee-AqvopbP1qhQ)) for the handling of SrbUser\_SecurityDeactivationReq message.

[PR632836] If alignSN field is set to TRUE, it means that RRC message has been sent by gNB after entering lock step mode. Reception of SrbUser\_SecurityDeactivationReq means, that security activation failed and UE discarded RRC message, which caused SN desynchronization. Alignment of SN between gNB and UE is needed. 5G-L2-SRB shall decrease SN value by one.[End PR632836]

Phase3-Security Activated:

Downlink

Integrity protection and ciphering is performed.

Uplink

Integrity verification shall be based on corresponding MAC-I.

Deciphering is performed for the message.

The message shall be forwarded only if integrity verification is successful.

C-Plane shall trigger the transition from/to different phases. Below requirements define the interfaces used.

1) "Phase1-No Security": Refer Refer 5G\_L3\_IF\_4659 and 5G\_L3\_IF\_4661.

2) "Phase1-No Security" to "Phase2-Lock Step" : Please refer 5G\_L3\_IF\_4620/5G\_L3\_IF\_5236.

3) "Phase2-Lock Step" Please refer to  5G\_L3\_IF\_4708/5G\_L3\_IF\_4709.

4) "Phase2-Lock Step" to "Phase3-Security Activated" : Please refer 5G\_L3\_IF\_4710/5G\_L3\_IF\_4716.

5) "Phase2-Lock Step" to "Phase1-No Security" : Please refer 5G\_L2\_IF\_1409\_replaced([10993531](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2W-0Dfu6Ee-AqvopbP1qhQ)).

PDCP Lock step is not applicable during mobility.
