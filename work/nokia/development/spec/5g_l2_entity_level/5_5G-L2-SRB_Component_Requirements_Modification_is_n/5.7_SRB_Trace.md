# 5.7 SRB Trace

## 5.7.0-3 SRB Trace 5G-L2-SRB shall log information of every PDCP PDU in UL Following additional information will be sent to Emil tracing for each SrbSgnl_UlSduReceiveInd message to trace L2-SRB when a SDU is constructed from the UL PDCP PDU(s) and sent to CP-UE. The following information will be added as additional parameters to 5G_L2_IF_1311_replaced( 10992969 ) RX_RCVD_COUNT Indicates the received COUNT for the PDCP PDU RX_NEXT COUNT Indicates the next expected to be received COUNT RX_DELIV COUNT Indicates the next to be delivered COUNT RX_REORD_COUNT Indicates the most recent COUNT that triggered reordering T-Reorder timer status Indicates if t-Reorder timer is currently running or expired In addition to the above, L2-SRB shall perform the following textual logging in the below mentioned conditions. - When for a SrbSgnl_UlPduReceiveInd message received at L2-SRB, a SrbSgnl_UlSduReceiveInd is not sent to CP-UE for various reasons listed 1. When a PDU is discarded with the possible PDU Discard causes viz. 2. When received PDU triggers reordering RX_RCVD_COUNT Indicates the SN for the PDCP PDU RX_NEXT COUNT Indicates the next expected SN RX_DELIV COUNT Indicates the next to be delivered SN T-Reorder timer status Indicates the current status of the t-Reorder timer. Possible values are:  - Timer never started and not running  - Timer started but expired  - Timer is running  - Timer is stopped RX_REORD COUNT Indicates the SN that started the t-Reorder timer ueIdCu Identifies the UE ID in gNB-CU Srb Identity Indicates the SRB Id Bearer Index Indicates the bearer Index for the SRB ID for this UE PDU Discard Cause Indicates the PDU Discard cause with possible values:  - No Discard  - Integrity Verification failure  - Duplicate PDU  - Invalid PDU  - Bearer Release Note: All the above mentioned trace and log entries should be post PDU processing, after the PDCP RX state variables are updated and when the PDU is sent to the CP-UE or is buffered. Note: CP-UE shall not interpret these extra parameters for any functional requirement. These extra parameters are purely for debug and information purposes. (ID: 11241119)
