# 5.6 PM Counters

## 5.6.0-2 PM Counters PM Counter management in 5G-L2-SRB (ID: 11241071)

**5.6.0-2.0-3**  (ID: `11241093`)

PM Counters
5G-L2-SRB shall report PM counters with the following aggregation levels, for URI construction see 5G\_L2\_IF\_1490\_replaced([10994810](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2XHW5_u6Ee-AqvopbP1qhQ)).

|  |  |  |  |
| --- | --- | --- | --- |
| Granularity | Aggregation Level | Network Profile | First Feature |
| NRBTS | NRBTS | PLMN-MRBTS-NRBTS | 5GC000578-C |
| NRBTS per PLMN | NRBTS,MNC | PLMN-MRBTS-NRBTS,MCC-MNC | 5GC000738-B |

5G-L2-SRB shall report NRBTS specific PM counters per NRBTS.

* The NRBTS object for NRBTS specific PM counters is created at reception of the first SrbUser\_BearerSetupReq.
* Note that deletion of the NRBTS object is not currently supported.

5G-L2-SRB shall report PLMN specific PM counters per PLMN:

* A new PLMN object instance for PLMN specific PM counters is created at reception of a SrbUser\_BearerSetupReq or a SrbUser\_SecurityConfReq, if an unknown plmnId is associated with the bearer.
* [Before 5GC001630-A] There can be up to two PLMNs per gNB. [End 5GC001630-A]
* [5GC001630-A] There can be up to six PLMNs per gNB. [End 5GC001630-A]
* 5G-L2-SRB shall internally collect and store counter data for PLMN specific PM counters of a bearer until PLMN ID of the bearer is known. When 5G-L2-SRB receives the PLMN ID, it shall report any internally collected counter data of the bearer at once to PM, empty the internal data and start normal counter collection and reporting via PM.

Rationale: C-Plane delivers PLMN ID via SrbUser\_BearerSetupReq if it is known at the time this message is sent, and otherwise via SrbUser\_SecurityConfReq later. In the latter case there may be UL/DL signalling traffic on the bearer before the PLMN ID gets to known on L2-SRB side, thus until that L2-SRB must collect counter data internally to avoid mismatch on PM counter values. Note that L2-SRB behaviour here can be the same regardless of the ongoing signalling procedure or SRB type.

* Note that deletion of a PLMN related PM object instance is not currently supported.

Note:

* For PM object management see 5G\_UP\_ALG\_6250\_replaced([11230118](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kysJ-fvSEe-AqvopbP1qhQ)).
* For more information on 5G MOCs see 5G\_UP\_2718\_replaced([11391877](https://dn-prod.ext.net.nokia.com/rm/resources/BI_ufbP0_vkEe-AqvopbP1qhQ)).
* 5G-L2-SRB reports per each L2-SRB instance. OAM gathers data and combines per NRBTS.
* 5G-L2-SRB uses C-Plane PM collection mechanism described in DOORS Control Plane module [5G\_L3\_77\_replaced([10971161](https://dn-prod.ext.net.nokia.com/rm/resources/BI_A6IAiv_KEe-D3seUwaI0tg)) and 5G\_L3\_78\_replaced([10971182](https://dn-prod.ext.net.nokia.com/rm/resources/BI_A6IAjv_KEe-D3seUwaI0tg))].
