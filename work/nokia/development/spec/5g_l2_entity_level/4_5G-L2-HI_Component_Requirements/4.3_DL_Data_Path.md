a. 5G-L2-HI shall extract the TEID present in the GTP-U header from the GTP-U G-PDU.

 Perform TEID validation as described in 5G_L2_2282_replaced([11222013](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2_ifc_vTEe-AqvopbP1qhQ)).

c. 5G-L2-HI shall validate the GTP-U Extension header "PDU Session Container” present in the GTP-U header of the GTP-U PDU as specified in 5G_L2_17223_replaced([11222757](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2_mJ3vvTEe-AqvopbP1qhQ)).

"PDU Session Container” is defined in 5G_UP_IF_5049_replaced([11547359](https://dn-prod.ext.net.nokia.com/rm/resources/BI_tY75J_v1Ee-AqvopbP1qhQ)).

The "PDU Session Container” contains the "DL PDU SESSION INFORMATION". The "DL PDU SESSION INFORMATION" format is defined in 5G_UP_IF_4288_replaced([11539150](https://dn-prod.ext.net.nokia.com/rm/resources/BI_tXnqivv1Ee-AqvopbP1qhQ)).

d. 5G-L2-HI shall extract the QFI from the GTP-U Extension header "PDU Session Container” present in the GTP-U header of the GTP-U PDU.

e. 5G-L2-HI shall extract the "Mapping Index" from the GTP-U TEID and fetch the PDU session using the "Mapping Index".

GTP-U TEID format is defined in 5G_L2_2200_replaced([11221946](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2_h4bPvTEe-AqvopbP1qhQ)).

f. 5G-L2-HI shall choose the DRB using the QFI obtained in step 2c.

If no DRB is chosen and the default DRB is configured, 5G-L2-HI shall choose the default DRB. Else, in case of absence of the default DRB, 5G-L2-HI shall stop processing and discard the received GTP-U G-PDU.

#### 4.3.3.2.0-4 RLC RLC SDU discard procedures (ID: 11223992)

**4.3.3.2.0-4.0-1**  (ID: `11223999`)

RLC RLC buffer congestion mechanism

---

**4.3.3.2.0-4.0-1.0-6**  (ID: `13417587`)

RLC RLC packets discarding due to buffer congestion is enabled when structure duBufferCongestionDiscardMechanism in HiCnfgDu_ConfigParamReq is present, which contains following parameters:

\- bufferDiscardTrigger (reflects NRBTS.actPdcpRlcBufCongOvlMech)

\- bufferCongDiscardThreshold (reflects NRBTS._NRBTS.rlcBufUsageThreshold)_

\- numSduDiscardForCong (reflects NRBTS.numNonGBRRlcSduDiscard)

[CB014699-D]

Selecting a bearer to discard RLC SDUs depend on:

\- Traffic Type: The type of traffic the bearer carries i.e. critical traffic or non-critical traffic is configured in HiUserDu_BearerSetupReq.qos.mapping.criticalDrbSchedEnabled IE and can be updated via HiUserDu_BearerModifyReq.criticalDrbSchedEnabled. [End CB014699-D]

\- Priority: In case priorityBased bit is not set in HiCnfgDu_ConfigParamReq.duBufferCongestionDiscardMechanism .bufferDiscardTrigger only non-GBR packets are discarded else packets are discarded according to priority as described in 5G_UP_ALG_31442_replaced([11224057](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kyFF9_vSEe-AqvopbP1qhQ))

\- Bearer Type: The bearer can be configured as DC-GBR, MC-GBR,**GBR, non-GBR.**

The value of bufferDiscardTrigger indicates trigger for checking of buffer congestion:

  * The received RLC SDU is for GBR bearer **AND "GBR" bit is set in the bufferCongestionDiscardMechanism parameter** received via HiCnfgDu_ConfigParamReq message.****



**OR**

  * The received RLC SDU is for non-GBR bearer **AND** "NonGBR" bit or "rrcInactive" bit is set in the bufferCongestionDiscardMechanism parameter **received via HiCnfgDu_ConfigParamReq message.** ****



Buffer congestion is detected when number of RLC SDU(s) and PDU(s) stored in buffer exceeds threshold calculated as follow:

"RLC buffer usage threshold" = RoundDown((bufferCongDiscardThreshold * "max RLC buffer size") /100)

Where:

max RLC buffer size - size of buffer in number of packets

When buffer congestion is detected then following actions take place:

If priorityBased bit is set in bufferCongestionDiscardMechanism received via HiCnfgDu_ConfigParamReq message, the priority of a bearer shall be configured during bearer setup via HiUserDu_BearerSetupReq message in case priority is not configured default value shall be applied as per 5G_UP_ALG_31442_replaced([11224057](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kyFF9_vSEe-AqvopbP1qhQ)) (refer Note: 2).

[CB014699-D]

Step A: among the bearers configured to carry non-critical traffic and has data pending to be sent, select a bearer using steps 1 to 5 below.

Step B: in case no bearer is selected using step A, select a bearer among the ones configured to carry critical traffic using steps 1 to 5 below.

[End CB014699-D]

\- step 1: select the non-GBR DRB having priority greater than or equal to 60 and having the highest number of unsent RLC SDU(s)

\- step 2: in case no DRB is selected from step 1, select the GBR DRB having priority greater than or equal to 60 and having the highest number of unsent RLC SDU(s)

\- step 3: in case no DRB is selected until step 2, select the non-GBR DRB having priority greater than 30 and having the highest number of unsent RLC SDU(s)

\- step 4: in case no DRB is selected until step 3, select the GBR DRB having priority greater than 30 and having the highest number of unsent RLC SDU(s)

\- step 5: discards the number “N” (defined as numSduDiscardForCong IE in HiCnfgDu_ConfigParamReq) of RLC SDU(s) of the selected DRB.

\- if less packets is present then all are released and discarding is stopped in this round.

when no bearer is selected (or no packets present in selected bearer) and incoming packet cannot be buffered, then L2-HI discards the incoming non-GBR/GBR packet.

Else

[End CB014699-D]

if incoming PDCP SDU is for DC-GBR DRB

Step 1: select the non-GBR bearer having the highest number of unsent RLC SDU(s). 

Step 2: If only GBR packets are present, select the GBR bearer having highest number of unsent RLC SDU(s). 

else

Step 1: select the non-GBR bearer having the highest number of unsent RLC SDU(s). 

L2-HI DU discards**numSduDiscardForCong of RLC SDU(s) from selected non-GBR** or **GBR (if only GBR RLC SDU(s) are present,** ****

In case when the selected DRB:

\- have M packets and M<N then L2-HI discards M number of packets

\- have 0 packets and the incoming packet cannot be buffered, then L2-HI discards the incoming non-GBR/GBR packet
