## 4.8.4 Overload protection for streaming NR L2 logs

### 4.8.4.0-1 Overload protection for streaming NR L2 logs Description (ID: 11228553)

**4.8.4.0-1.0-3**  (ID: `11228572`)

Overload protection for streaming NR L2 logs CPU OVL state, further referred in this requirement as OVL.

For Cloud deployment L2-HI-DU checks for OVL independently from L2-HI-CU, for Classical deployment L2-HI-DU doesn't have own OVL control mechanism and L2-HI-CU handle it for both parts.

If "actTraceOvlProt" is set to "TRUE" (received in L2LogCu_OvlProtectionForStreamingConfigReq on 5G-L2-HI-CU or Cloud: L2LogDu_OvlProtectionForStreamingConfigReq on 5G-L2-HI-DU) and at least one streaming is enabled (component streaming [Before CNI-115810] or Sherpa [End CNI-115810] ) CPU load is compared with R&D flag rdAvgL2HICPULoadThreshStream and if is above, then OVL is detected.

Action taken when OVL is detected.

OAM is informed about OVL with L2LogCU_OvlStatusInd on 5G-L2-HI-CU or Cloud: L2LogDu_OvlStatusInd on 5G-L2-HI-DU, L2LogCu_SuspendStreamingReq and L2LogDu_SuspendStreamingReq are expected to be received in order to suspend all ongoing streaming.

Guard Timer is started, of duration indicated by R&D parameter rdOvlProtStreamingGuardTimerInterval.

Guard Timer expiration and actions taken.

After Guard Timer expiration CPU load is compared with R&D flag rdAvgL2HICPULoadThreshStream.

If OVL is still present then Guard Timer is restarted,

otherwise OAM is informed about end of OVL with L2LogCU_OvlStatusInd on L2-HI-CU or Cloud: L2LogDu_OvlStatusInd on 5G-L2-HI-DU.

If L2LogCU_OvlStatusAck on L2-HI-CU or Cloud: L2LogDu_OvlStatusAck on 5G-L2-HI-DU, is not received, then message is sent again.

Notes:

Reception of L2LogCu_StreamActivationtReq on L2-HI-CU or Cloud: L2LogDu_StreamActivationtReq on 5G-L2-HI-DU, while OVL is ongoing is treated as end of OVL.

Streams suspended with L2LogCu_SuspendStreamingReq or L2LogDu_SuspendStreamingReq have to be activated again with L2LogCu_StreamActivationtReq or L2LogDu_StreamActivationtReq.

R&D parameter is defined in 5G_L2_14404_replaced([11232435](https://dn-prod.ext.net.nokia.com/rm/resources/BI_3AjMEfvTEe-AqvopbP1qhQ)).
