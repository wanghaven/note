# 3.8 Buffers Management

## 3.8.1 HARQ Buffer

### 3.8.1.0-2 HARQ Buffer 5G-L2-LO shall implement the DL HARQ Buffer for HARQ operation. (ID: 11217544)

**3.8.1.0-2.0-2**  (ID: `11217555`)

HARQ Buffer 5G-L2-LO shall store any assembled 5G-MAC PDU to provide it for possible HARQ retransmissions later on.

Note, in future the DL HARQ buffer management (i.e. storing the 5G-MAC PDUs) shall be moved to 5G-L1-DL. Thus, addressing transport blocks on the 5G-L2-PS/5G-L2-LO interface shall be designed in such a way that it can be applied on 5G-L2-PS/5G-L1-DL interface without changing the logic within 5G-L2-PS and independent of the underlying memory management. For DL HARQ buffer management a similar design as for the UL counterpart has been selected.

When requesting the 5G-MAC PDU generation for an initial HARQ transmission 5G-L2-PS indicates an dlHarqBufferIndex associated with the transport block. This dlHarqBufferIndex is used to address the same transport block for possibly following HARQ retransmissions.

The dlHarqBufferIndex has sub-cell scope and a range from zero to the maximum value calculated as follows:

[PR670454]

max_index = SIZE_HARQ_BUFFER_INDEX_POOL * MAX_NUM_SUB_CELL_POOLED

where

SIZE_HARQ_BUFFER_INDEX_POOL = number of HARQ buffers per subcell. Current value is 224

MAX_NUM_SUB_CELL_POOLED = maximum number of subcells managed in an L2-PS instance. Current value is 16.

[end PR670454]

It is left to implementation to decide on the memory management itself, i.e. wrt. the memory allocation for transport blocks and the mapping from the dlHarqBufferIndex to the memory address of the associated transport block. 

Note, the max. possible size of a transport block for RRC connected UEs can be derived from V5G.213 chapter 8.1.5 assuming max. number of PRBs (100), highest MCS (14), two-layer spatial multiplexing and only single UE is served per sub-cell and 5GTF-subframe. The size of a RA Msg4 / SRB0 transport block is defined in 5G_RAN_UP_4789.

5G-L2-PS is expected to manage the dlHarqBufferIndex in such a way that L2-LO does not need to care about consistency. 5G-L2-PS does explicitly not inform L2-LO about released HARQ processes but it may reuse the associated dlHarqBufferIndex immediately for same or other UE and HARQ process.

For the artificial PDSCH load, there is no HARQ buffer needed. After generating the TB, the data is sent to L1 by DlData_PdschPayloadTbSendReq without buffering.
