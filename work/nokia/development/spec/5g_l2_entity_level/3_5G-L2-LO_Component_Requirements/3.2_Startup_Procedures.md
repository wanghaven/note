**3.2.2.0-3.0-2**  (ID: `12558294`)

EO Deployment If new framework is not activated, as mentioned in 5G_L2_6210_replaced([11140235](https://dn-prod.ext.net.nokia.com/rm/resources/BI_23iaAfvTEe-AqvopbP1qhQ)), then board config is fetched, as mentioned in 5G_L2_1910_replaced([11215287](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2-3xEPvTEe-AqvopbP1qhQ)).

  1. RT_CORE_AFFINITIES indicates total number of cores for 5G-L2-RT(5G-L2-LO and 5G-L2-PS). Refer to Link mentioned in 5G_L2_6210_replaced([11140235](https://dn-prod.ext.net.nokia.com/rm/resources/BI_23iaAfvTEe-AqvopbP1qhQ))(tab ListFlavorID in excel for corelation between FlavorID and no of RTcores)

  2. CONTAINER_PRIVATE_PARAMETER indicates bit position of 5G-L2-LO core.




3 5G-L2-LO computes start core and number of cores

StartCore(within assigned RTcores) = Relative position of first set bit in value extracted from CONTAINER_PRIVATE_PARAMETER + 1

Number of 5G-L2-LO cores = Total no of set bits in CONTAINER_PRIVATE_PARAMETER

Example

  1. RT_CORE_AFFINITIES is represented as string of total 5G-L2-RT cores "1,2,3,4,5,6,7,8" indicates 8 cores(no of elelments in string with seperator , ) and starting with physical core 1

  2. CONTAINER_PRIVATE_PARAMETER represents 5G-L2-LO number and position of cores as a bit map. 0x3 indicates 0000 0011 indicates 2 5G-L2-LO cores at positon 1 and 2




StartCore = first set bit in bit map + 1 = 0+1 = 1

Number of 5G-L2-PS cores = 2 set bits = 2 cores

#### 3.2.2.0-5 Exception handling Determining pool placement in two cores TDD deployment (ID: 11215476)

**3.2.2.0-5.0-1**  (ID: `11215480`)

Exception handling [CNI-138433-A]

When the two-pools L2-LO component starts up on a two-cores deployment then L2-LO shall create L2RT pools instances containing requested number of cells based on the LoCnfg_AddressDistributionReq.L2RtPoolDeploymentInformations.maxNumberOfCells parameter.

When maxNumberOfCells=2 then L2-LO TB Builder component supporting L2RT pool containing 2 cells shall share the core with the L2-LO DL/UL Receiver components.

When maxNumberOfCells=4 then L2-LO TB Builder component supporting L2RT pool containing 4 cells shall not share core with any other L2-LO component

5G-L2-LO shall send LoCnfg_AddressDistributionResp response according to 5G_L2_IF_737_replaced([10983059](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2V1kjfu6Ee-AqvopbP1qhQ)) with parameter status set to EStatus5G_NotOK and cause = InvalidParameter when the value of LoCnfg_AddressDistributionReq.L2RtPoolDeploymentInformations.maxNumberOfCells is not in (2,4)

[End CNI-138433-A]
