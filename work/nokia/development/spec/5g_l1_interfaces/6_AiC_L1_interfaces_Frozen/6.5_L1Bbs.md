## 6.5.3 API2_BBB_CREATE_EAXC_REQ_MSG

**6.5.3.0-6**  (ID: `11519085`)

API2\_BBB\_CREATE\_EAXC\_REQ\_MSG
Structure SUlDelayConfig. This structure is used in UDelayConfig union, which is used in API2\_BBB\_CREATE\_EAXC\_REQ\_MSG:

|  |  |  |
| --- | --- | --- |
| tUlAdvanceCp | u32 | Advance for Uplink control data. Unit is UTU. Parameter defines how much earlier application must send Uplink control-plane data that it arrives in time to RU. Application must calculate exact offset to secondary BCN from  AirtimeOffset and tUlAdvance parameters. Note: client doesn't take account baseband capacity unit  internal path or BCN delay.   Offset to secondary BCN = AirtimeOffset - tUlAdvanceCp - internal processing / path delays  -  internal BCN delays |
| receptionWindowOpen | u32 | Delay how much later reception window is opened. Unit UTU. Application must calculate exact secondary BCN offset for Reception window opening from  AirtimeOffset and receptionWindowOpening parameter. Note: client doesn't take account baseband capacity unit  internal path or BCN delay.   Offset to secondary BCN = AirtimeOffset + receptionWindowOpen  +  internal processing / path delays - internal BCN delay   For Raw SRS streams(EPhyChannelType\_Srs),  if occasionDelay or receiverDelay of the stream is configured with non-zero value,  then receptionWindowOpen specifies delay (occasionDelay  + receiverDelay + T34\_min),  where  occasionDelay and receiverDelay are indicated in API\_BBB\_SET\_SRS\_PATTERN\_CONFIG\_REQ\_MSG. |
| receptionWindowClose | u32 | Delay how much later reception window is closed. Unit UTU. Application must calculate exact secondary BCN offset for Reception window opening from  AirtimeOffset and receptionWindowOpening parameter. Note: client doesn't take account baseband capacity unit  internal path or BCN delay.   Offset to secondary BCN = AirtimeOffset + receptionWindowClose  +  internal processing / path delays - internal BCN delay   For Raw SRS streams (EPhyChannelType\_Srs), if occasionDelay or receiverDelay of the stream is configured with non-zero value,  then receptionWindowClose specifies delay (occasionDelay  + receiverDelay + T34\_max),  where  occasionDelay and receiverDelay are indicated in API\_BBB\_SET\_SRS\_PATTERN\_CONFIG\_REQ\_MSG. |
| nTaOffset | u32 | Uplink airtime advance compared downlink airtime. Unit is UTU. Valid for ABIC, ABIL and ASOD. New product must use AirtimeOffset. |

API\_BBB\_SET\_SRS\_PATTERN\_CONFIG\_REQ\_MSG

|  |  |  |
| --- | --- | --- |
