# 3.3 UlPool

## 3.3.0-1 UlPool Document Owner Jiang Bingzhong (ID: 11515328)

## 3.3.1 General

**3.3.1.0-2**  (ID: `11515339`)

General
UlPool interface is an independent interface entity which facilitates L1 pooling resource information between 5G-L1 and 5G-L2.  Resources in the form of PRB processing capacity (for instance, shared memories) are pooled within L1 pool level. In order to support PRB pooling among the cells within L1 subpool, BBRM (BaseBand Resource Manager), which is part of L2RT, allocates the amount of usable PRB available to the cell scheduler. Such allocations ensure that L1 does not get overloaded. These available PRBs are then pooled among the cells within each L1 Subpools. Cell schedulers in L2RT then consume the available PRB allocations from BBRM to schedule UL and DL traffics (for mor information refer to 5G\_UP\_ALG\_5265).

BBRM re-calculates the amount of PRB allocation for cells mapped to L1 subpools within every pooling period interval. The pooling interval also called slow pooling window. The granted PRB allocation is valid for the entire duration of pooling interval of the next slow pooling window for cell schedulers and L1 as a consumer of these resources.

PRB pooling for UL is performed separately for:

1) UL stream PRBs per symbol

2) UL layer PRBs per symbol (PUSCH layer PRBs).

5G-L2 notifies 5G-L1 regarding pooling resources to be used in l1 Pool level.

5G-L2 sends an UlPool address and message exchanges per L1 pool.

## 3.3.4 Timing

**3.3.4.0-1**  (ID: `11515393`)

Timing
Timing requirement of UlPool\_BbResourceReconfReq message:

The latest time which UlPool\_BbResourceReconfReq shall be received by L1 is 2 SFN before new reconfiguration is to be applied.

Timing requirement of UlPool\_BbResourceReconfResp message:

UlPool\_BbResourceReconfResp shall be sent by L1 within 1000 us (2 slots) after receiving UlPool\_BbResourceReconfReq message.
