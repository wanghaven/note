**Message: UlDataFH\_SrsRtBfReceiveRespPs**

For **FR1 TDD** 7.2e, the following procedure takes place:\*\*\*\*

1. 5G-L1-UL RU generates *UlDataFH\_SrsRtBfReceiveRespPs message, divided in M fragments, from related measurements and hands over to L1-SW Common FH RU.*
2. There is a separate response message **per symbol** per slot **for each UL subcell where SRS signals are scheduled (see 5G\_L1\_IF\_236\_replaced([11514323](https://dn-prod.ext.net.nokia.com/rm/resources/BI_FR2oXfvxEe-AqvopbP1qhQ))). This could be revisited in future.**\*\*\*\*
3. Fragmentation of *UlDataFH\_SrsRtBfReceiveRespPs is handled by 5G-L1-UL DU and 5G-L1-UL RU. This step is necessary when* UlDataFH\_SrsRtBfReceiveRespPs payload exceeds maximum eCPRI payload. Each fragment is limited to a maximum size of 1500 bytes (8 bytes are used for the eCPRI Header).\*\*

2. L1-SW Common FH RU sends fragments, as encapsulated in M eCPRI packets, to DU.

3. L1-SW Common FH DU performs disassembly of the M ecPRI packets and hands fragments of *UlDataFH\_SrsRtBfReceiveRespPs to 5G-L1-UL DU.*

4.  5G L1-UL DU reorganizes the content of the fragments from *UlDataFH\_SrsRtBfReceiveRespPs to comply with content in* UlData\_SrsRtBfReceiveRespPs.\*\*

**Note: Within CB009527 scope, fragementation of *UlDataFH\_SrsRtBfReceiveRespPs\*is needed as message size is larger than 1500 bytes. Each fragementation can contain 12 UE grants that activate RT BF (See 5G\_L1\_IF\_15296\_replaced([11517530](https://dn-prod.ext.net.nokia.com/rm/resources/BI_FSL_jPvxEe-AqvopbP1qhQ))).***\*

**Note: Current assumption for fragmentation (i.e., number of responses per fragmentation, see *srsRtBfRuResp IE in 5G\_L1\_IF\_15284\_replaced([11517503](https://dn-prod.ext.net.nokia.com/rm/resources/BI_FSL_g_vxEe-AqvopbP1qhQ))) MTU is 1500 bytes (8 bytes for eCPRI header + 8 bytes for APP header) and message defined as in 5G\_L1\_IF\_15284\_replaced([11517503](https://dn-prod.ext.net.nokia.com/rm/resources/BI_FSL_g_vxEe-AqvopbP1qhQ)).***

When 5G-L1-UL generates *UlDataFH\_SrsRtBfReceiveRespPs message (see 5G\_L1\_IF\_15282\_replaced([11517500](https://dn-prod.ext.net.nokia.com/rm/resources/BI_FSL_gvvxEe-AqvopbP1qhQ))), 5G-L1-UL proceeds as follows:*

Define *respPerFragment=12 (**Note2),*\*\*

1. Calculate total number of *UlDataFH\_SrsRtBfReceiveRespPs fragments:*

a. *totalFragmentNum is the sum of the* srsRtBfResources of each fragment of *UlDataFH\_SrsRtBfReceiveReq (5G\_L1\_IF\_15283\_replaced([11517458](https://dn-prod.ext.net.nokia.com/rm/resources/BI_FSLYfPvxEe-AqvopbP1qhQ))) message during ReceiveReq window.*\*\*

b. Obtain *NumOfFragments=ceil(*totalFragmentNum / *respPerFragment)*\*\*

c. Index fragments with *i={0,...,NumOfFragments-1}*

1. For *i=0,...,NumOfFragments-2, create the* i-th fragment of *UlDataFH\_SrsRtBfReceiveRespPs by*\*\*

a. Fill in the message common parameters.

b. Fill in the SRS RT BF UE level responses in *UlDataFH\_SrsRtBfReceiveRespPs  for the UEs at positions [*i*respPerFragment...*(i+1)*respPerFragment-1] of the* UlDataFH\_SrsRtBfReceiveReq **message (5G\_L1\_IF\_15285\_replaced([11517462](https://dn-prod.ext.net.nokia.com/rm/resources/BI_FSLYffvxEe-AqvopbP1qhQ))).**\*\*\*\*

1. For last fragment, *i=NumOfFragments-1,*

a. Fill in the message common parameters.

b. Fill in the SRS RT BF UE level responses in *UlDataFH\_SrsRtBfReceiveRespPs for the UEs at positions [*i*respPerFragment...*totalNumOfSrsBmResource -1] of the *UlDataFH\_SrsRtBfReceiveReq message (5G\_L1\_IF\_15285\_replaced([11517462](https://dn-prod.ext.net.nokia.com/rm/resources/BI_FSLYffvxEe-AqvopbP1qhQ))).*\*\*\*

**Note:**

1. Last fragment might contain less SRS RT BF responses than *respPerFragment.*
2. The number of response per fragment of *UlDataFH\_SrsRtBfReceiveRespPs and the number of fragments per slot may change based on message version.* *In case of MULTI message updates, refer to latest document in [12739732](https://dn-prod.ext.net.nokia.com/rm/resources/BI_4LvzkHm5EfCiUemk-K-C4g) for detailed information on number of UE grants per fragment and max number of fragments needed per FH message.*\*\*
