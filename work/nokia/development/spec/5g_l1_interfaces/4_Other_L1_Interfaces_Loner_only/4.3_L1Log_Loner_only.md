### 4.3.2.2 Antenna Snapshot for eCPRI

#### 4.3.2.2.0-4 Antenna Snapshot for eCPRI The parameters usage for the eCPRI capture based on the request using 5G_L1_5G21A_489 are as follows. For the SFN and slot based capture with captureMode = "0", the cutoff time for the last packet capture will allow the delay margin for the eCPRI packet reception window closing from the end of the frame/slot in L1Log_AntennaSnapshotReq (See 5G_L1_5G21A_450 for eCPRI timing parameters). For the SFN and slot based capture with captureMode = "1", the start time for the first packet capture will allow the advance margin for the eCPRI control packets from the start of the frame/slot in L1Log_AntennaSnapshotReq (See 5G_L1_5G21A_450 for eCPRI timing parameters). Immediate and scheduled capture modes will be supported. The scheduling of a synchronous stop triggered by a suspicious event is handled in the same way as for CPRI. eCPRI capture supports only a single file mode. oneFilePerPath = 0. (ID: 11516354)

### 4.3.2.4 Antenna Snapshot Stop Indication

**4.3.2.4.0-1**  (ID: `11516378`)

Antenna Snapshot Stop Indication
Messages:

L1Log\_AntennaSnapshotStopInd

Purpose:

To indicate that antenna snapshot collection has been stopped for cell(s)/subcell(s) at L1 side.

Precondition:

cell/subCell has been configured.

Addressing:

The message is sent to the address from which the related L1Log\_AntennaSnapshotConfigurationReq was received.

---

**4.3.2.4.0-1.0-1**  (ID: `11516382`)

Antenna Snapshot Stop Indication
Description:

This message is sent by L1 to indicate that that the antenna snapshot collection for given cell(s)/subcell(s) has been stopped at L1 side.

This message shall not be sent if the antenna snapshot collection for given cell(s)/subcell(s) has been stopped as a result of L1Log\_AntennaSnapshotConfigurationReq reception.
