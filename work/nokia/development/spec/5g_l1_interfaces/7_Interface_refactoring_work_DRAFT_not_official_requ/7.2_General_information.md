**7.2.1.2.1.2.1.1.0-1**  (ID: `12578164`)

Overview
**Messages:**

* Request messages:

[UlData\_SrsReceiveReq](https://dn-prod.ext.net.nokia.com/rm/resources/BI_6u5lQCqOEfCNjfqfyUlvmw?oslc_config.context=https%3A%2F%2Fdn-prod.ext.net.nokia.com%2Fgc%2Fconfiguration%2F9)

*UlData\_SrsPosReceiveReq*

* Response messages:

*UlData\_SrsReceiveRespPs*

*UlData\_SrsReceiveRespBmPs*

*UlData\_SrsReceiveRespBwvPs*

*UlData\_SrsReceiveRespRtBfPs*

**Purpose:**

The procedure is used to request 5G-L1-UL to receive one or multiple SRS for one or multiple SRS users for different usages.

The procedure shall be done per subcell­­. The number of subcells is always set to 1, and 5G-L1-UL sends response messages for each subcell separately.

The procedure for the response messages is done per usage, indicated by the request messages.

**Precondition:**

Subcell exists and L1 uplink addresses have been negotiated for it ([11513672](https://dn-prod.ext.net.nokia.com/rm/resources/BI_FRyW5vvxEe-AqvopbP1qhQ)).

**Addressing:**

* Request Messages:

The UlData\_SrsReceiveReq message is received at the address indicated by IE srsReceiveReqAdress contained in UlData\_AddressResp message.

The UlData\_SrsPosReceiveReq message is received at the address indicated by IE srsPosReceiveReqAdress contained in UlData\_AddressResp message.

* Response Messages:

The UlData\_SrsReceiveRespPs is sent to the address indicated by IE addrSrsReceiveResp contained in UlData\_SrsReceiveReq message.

The UlData\_SrsReceiveRespBmPs is sent to the address indicated by IE addrSrsReceiveRespBm contained in UlData\_SrsReceiveReq message.

The *UlData\_SrsReceiveRespBwvPs is sent to the address indicated by IE addrSrsReceiveRespBwv contained in UlData\_SrsReceiveReq message.*

The *UlData\_SrsReceiveRespRtBfPs is sent to the address indicated by IE addrSrsReceiveRespBwv contained in UlData\_SrsReceiveReq message.*

The *UlData\_SrsPosReceiveRespPs is sent to the address indicated by IE srsReceiverespPosClientAddress contained in UlData\_SrsPosReceiveReq message.*

###### 7.2.1.2.1.2.1.3 General Message Validation

**7.2.1.2.1.2.1.3.0-1**  (ID: `12562800`)

General Message Validation
Upon reception of message [UlData\_SrsReceiveReq](https://dn-prod.ext.net.nokia.com/rm/resources/BI_6u5lQCqOEfCNjfqfyUlvmw?oslc_config.context=https%3A%2F%2Fdn-prod.ext.net.nokia.com%2Fgc%2Fconfiguration%2F9), 5G-L1-UL shall validate the message  by evaluating each IE in the message against its respective valid value or range:

1. If an IE is contained within its valid range is considered PASSED.
2. If an IE is not contained within its valid range is considered NOT PASSED.

Valid values and ranges are listed in [12562804](https://dn-prod.ext.net.nokia.com/rm/resources/BI_6u4-GCqOEfCNjfqfyUlvmw?oslc_config.context=https%3A%2F%2Fdn-prod.ext.net.nokia.com%2Fgc%2Fconfiguration%2F9).

When an IE is defined as "unconstrained", validation of the IE shall be skipped.

When an IE validation is not passed, 5G-L1-UL shall write an ERROR with description (at least containing IE name and reason of negative validation: "out of range")  in the logging system.

**7.2.1.2.1.2.2.2.6.12.0-1**  (ID: `12578335`)

srsPowerThresholdDl
Description:

Secondary threshold for validation/DTX of SRS CoMa. The stepsize is 0.1 dBm and the value in dBm is srsPowerThresholdDl/10. Default value is -1320 (132 dBm). Only used with eCPRI 7.2e split.

Rationale: This Threshold is meant for SRS resource used for DL BF. Since 5G-L1-UL can not discriminate dedicated SRS resources for UL or DL BF, 5G-L1-UL computes CoMa DTX using both srsPowerThreshold and srsPowerThresholdDl. L2-PS is able to discriminate SRS resources for UL and DL BF.

Support:

Loner/Loki: not supported.

Thor: int16 [-1500,0]
