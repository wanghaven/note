#### 5.2.1.1.0-4 Data collection procedure Calculation of unfiltered pdsch load for each serving cell (ID: 11229781)

**5.2.1.1.0-4.0-1**  (ID: `11229787`)

Data collection procedure For each Serving Cell irrespective of if the cell is DSS or not, corresponding 5G-L2-PS shall calculate unfiltered pdsch load of the cell in case of data collection procedure is activated.

The calculation of the estimated unfiltered pdsch load(LNR(t)) is as follows:

t is defined as the time instantaneous, i.e. radio frame.

LNR_used (t) = number of PRBs assigned by L2-PS for NR PDSCH at time instant t

LNR_avail (t) = number of PRBs available for L2-PS for NR PDSCH at time instant t

LNR (t) = LNR_used (t) / LNR_avail (t)

The PRBs shall be weighted equally, not taking into account the number of PDSCH symbols in different slots. Slots with SSB and CSI-RS are not available for PDSCH. Radio frames without any PDSCH available shall be skipped. LNR shall be calculated per t allocated to NR and per cell.

Note: The communication between 5G-L2-PS instances (SCell -> P(S)Cell) shall be specified in EFS phase.

#### 5.2.1.2.0-3 Inference procedure Sending the inference requests to ML inference engine (ID: 11229850)

**5.2.1.2.0-3.0-1**  (ID: `11229861`)

Inference procedure The inference request is per UE per {PCell, SCell} pair. For each added SCell of each UE, gNB-DU shall perform inference enquiry to ML inference engine according to the following procedures:

  * Prepare inference request data for the UE
  * Inference request data shall contain all the data marked with “Input” in the table defined in 5G_UP_7259_replaced([11365648](https://dn-prod.ext.net.nokia.com/rm/resources/BI_udUXDvvkEe-AqvopbP1qhQ)).
  * 5G-L2-PS shall use the latest instantanous value specified in 5G_UP_ALG_18187_replaced([11229890](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kyq71PvSEe-AqvopbP1qhQ)) as the inference request data.
  * If any of the instantaneous value is missing, 5G-L2-PS shall not perform the inference requests. In this case, legacy SCell activation criteria is used.

  * Send inference requests to ML inference engine

  * 5G-L2-PS shall send the inference requests when UE is configurd with SCells sucessfully and shall send the inference requests periodically. The period length shall be defined in EFS.

  * Receive inference responses from ML inference engine.

  * 5G-L2-PS shall maintain a guard timer to monitor the inference responses from ML inference engine, in case timer is expired before the inference response is received from ML inference engine, the inference responses shall be recorded as error "time-out".



Note: according to the BBP benchmark, the time of single inference requests shall be less than 5ms.

#### 5.2.1.2.0-5 Inference procedure Applying the inference results to SCell Activation procedure (ID: 11229878)

**5.2.1.2.0-5.0-1**  (ID: `11229883`)

Inference procedure Only if there is available inference response for each SCell in the SCell activation candidate list

(5G_UP_ALG_8840_replaced([11121235](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kqWtXfvSEe-AqvopbP1qhQ))), AI/ML based SCell activation selection algorithm can be used.

For one SCell:

1\. Before receiving any inference response, it’s considered the inference response of the SCell is unavailable

2\. Every time inference response is received, inference response if marked as available. That is, it’s assumed that received inference response is always valid. If the SE value is out of range used inside L2PS, simply shaping the value into the range.

3\. Every time timeout occurs while waiting for the inference response, inference response is marked as unavailable.

The algorithm of SCell selection for activation see 5G_UP_ALG_2724_replaced([11121256](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kqXUZfvSEe-AqvopbP1qhQ)).
