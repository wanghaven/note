## 3.19.1 Modification CSI-RS or CSI report config and activation

### 3.19.1.0-1 Modification CSI-RS or CSI report config and activation Modification of CSI-RS configuration and its activation (ID: 11158984)

**3.19.1.0-1.0-1**  (ID: `11158989`)

Modification CSI-RS or CSI report config and activation The modification of the CSI-RS configuration is triggered by the C-Plane. The new configuration is activated only after the indication that the UE is expected to use this new setting. Before this indication the current configuration is applied.

The U-Plane activates and uses the new configuration, as soon as, one of the following condition is met:

 The RRC message is delivered to the UE from the gNB MAC layer point of view and the processing is starting in the UE (see 5G_UP_ALG_17940_replaced([11153286](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kslh8vvSEe-AqvopbP1qhQ))).

 The C-Plane provides the RRC Reconfiguration Complete Indication to the U-Plane, if the MAC layer was able to receive the UL-SCH transmission(s), handling the RRC Confirmation message.

In downlink, the UE will applies the new configuration as soon as the RRC message prossessing is completed, the gNB does not know exactly when it will be the case, and must not activate the new configuration before at least the delivery of message to the UE from the MAC layer point of view.

**3.19.1.0-2.0-2**  (ID: `11159004`)

Modification CSI-RS or CSI report config and activation The modification of the CSI report configuration is triggered by the C-Plane. The new configuration is activated only after the indication that the UE is expected to use this new setting. Before this indication the current configuration is applied.

### 3.19.2.0-1 Periodic CSI handling General principles regarding periodic CSI-RS and periodic CSI report handling (ID: 11159017)

**3.19.2.0-1.0-5**  (ID: `11159033`)

Periodic CSI handling L2 PS shall determine for each slot if periodic CSI-RS needs to be transmitted (i.e., request for transmission needs to be sent to L1 DL). Thus, the periodicity and slot offset for each periodic CSI-RS configuration as received with the user setup message, see 5G_L2_134_replaced([11143879](https://dn-prod.ext.net.nokia.com/rm/resources/BI_232jHvvTEe-AqvopbP1qhQ)), needs to be evaluated.

Furthermore, L2 PS shall determine for each slot if CSI report needs to be received (i.e., request for reception needs to be sent to L1 UL).

Thus, the periodicity and slot offset for each periodic CSI report configuration as received with the user setup message, see 5G_L2_134_replaced([11143879](https://dn-prod.ext.net.nokia.com/rm/resources/BI_232jHvvTEe-AqvopbP1qhQ)), needs to be evaluated.

Thus, the CSI configuration of each RRC connected UE needs to be checked. The CSI configuration is received with the user setup message, see 5G_L2_134_replaced([11143879](https://dn-prod.ext.net.nokia.com/rm/resources/BI_232jHvvTEe-AqvopbP1qhQ)).

Several periodic CSI-RS as well as several periodic CSI reports can be configured for a single UE.

For a cell on FDD carrier w/o beamforming **[CB013442-D] or rdFddBfL3Call = true (refer to 5G_CP_ARC_ALG_16401_replaced([11329923](https://dn-prod.ext.net.nokia.com/rm/resources/BI_TtGWePvcEe-AqvopbP1qhQ))) & NRBTS.NRCELL.dlMimoMode = 40 or 60, **[End CB013442-D] the CSI-RS are sent according 5G_UP_ALG_2507_replaced([11116680](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kp6BbvvSEe-AqvopbP1qhQ)). ~~The periodicity and slot offset for each periodic CSI report configuration are received with the Psuser_UserSetupReq message see 5G_L2_134_replaced([11143879](https://dn-prod.ext.net.nokia.com/rm/resources/BI_232jHvvTEe-AqvopbP1qhQ)).~~****

For a cell on FDD carrier CSI-RS for CSI acquisition can be located in data slots, slots containing SSB, or slots containing CSI-RS for tracking.

Periodic CSI-RS transmission and CSI report reception shall be started for each UE after the random access procedure was finished successfully (i.e., UL contention resolution grant was acknowledged).

L2 PS shall not send request for the reception of CSI for the UE if current slot is DRX off time even if it is the CSI report sending chance based on the configuration, considering the grants/assignments UE received and Scheduling Request, UE sent 4 ms prior to current slot.

L2 PS shall send request for the reception of CSI only or (CSI along with SR) as per the reporting chance based on the configuration for the UE if current slot is DRX active time but corresponding CSI-RS for this report sending was not transmitted because of UE was in DRX off time then.

### 3.19.3.0-16 Aperiodic CSI handling CSI CPU consumption for periodic measurements. (ID: 11159519)

**3.19.3.0-16.0-1**  (ID: `11159525`)

Aperiodic CSI handling In case that periodic CSI reporting for channel feedback (CSI acquisition) or for beam management is enabled, it shall be assumed that in each slot UE consumes NP-CSI CSI processing CPUs, where:

NP-CSI = NCSI_ACQ + NCSI_BM where, for every slot, NCSI_ACQ and NCSI_BM are described below:

In case that periodic CSI reporting for channel feedback (CSI acquisition) is enabled, UE is consuming CSI processing CPUs as follows:

\- NCSI_ACQ CPUs are allocated in each slot during the period PP-CSI_ACQ_CPU slots where:

o NCSI_ACQ = number of CSI-RS resources in the CSI-RS set included by the CSI resource setting mapped by the periodic CSI report.

o PP-CSI_ACQ_CPU = period, as number of slots, between the CSI-RS slot SlotLast-P-CSI-RS_ACQ (included) and SlotP-CSI_REP_ACQ (excluded), where:

 SlotP-CSI_REP_ACQ is the P-CSI for acquisition reporting slot.

 SlotLast_P-CSI-RS_ACQ is the first slot of the last P-CSI RS occasion for acquisition (seen as the entire CSI-RS set occasion) that completes entirely no later than the CSI reference resource (see 5G_UP_ALG_3525_replaced([11159451](https://dn-prod.ext.net.nokia.com/rm/resources/BI_ktJirPvSEe-AqvopbP1qhQ)))

 So, for instance, if the CSI-RS set contains 4 CSI resources, 2 symbols each, then SlotLast_P-CSI-RS_ACQ is that slot that contains the first OFDM symbol of the earliest CSI-RS resource in the set but when the last slot that contains the last OFDM symbol of the 4’th CSI-RS resource is still no later than the reference resource).

\- 0 CPU otherwise

In case that periodic CSI reporting for beam management is enabled, UE is consuming CSI processing CPUs as follows:

\- NCSI_BM CPUs are allocate in each slot during the period PCSI_BM_CPU slots where:

o NCSI_BM = 1

o PCSI_BM_CPU = period as number of slots between the CSI-RS slot SlotLast-P-CSI-RS_BM (included) and SlotP-CSI_REP_BM (excluded), where:

 SlotP-CSI_REP_BM is the P-CSI for beam management reporting slot.

 SlotLast_P-CSI-RS_BM is the first slot of the last P-CSI RS occasion (or SSB, when BM is based on ssb-Index-RSRP) for BM (seen as the entire set occasion) that completes entirely no later than the CSI reference resource (see 5G_UP_ALG_3525_replaced([11159451](https://dn-prod.ext.net.nokia.com/rm/resources/BI_ktJirPvSEe-AqvopbP1qhQ)))

 Note that CSI-RS for beam management is cell specific and it is transmitted in the SSB slots so, for n SSB, the CSI-RS set spawns over the SSB slots containing the SSB0 to SSBn-1

\- 0 CPU otherwise

All the input parameters considered for the algorithm above, i.e. periodicity and starting offset of CSI report for acquisition/ beam management and periodicity and offset of each CSI RS for CSI acquisition in the CSI RS set are received in the user setup/modify message as follows:

\- for the periodic channel state acquisition CSI report, configuration in user setup/modify.

o periodicity, slotOffset

\- for periodic CSI-RS (for each CSI-RS in the set) of the CSI-RS for acquisition configuration in user setup/modify.

o periodicity, slotOffset per CSI RS resource set

### 3.19.7.0-6 CSI Report on PUSCH Computing number coded modulation symbols for UCI on PUSCH (ID: 11160631)

### 3.19.7.0-10 CSI Report on PUSCH Rank Indicator (RI) usage map of an aperiodic CSI report (ID: 11160689)
