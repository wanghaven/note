### 3.24.1.0-2 Paging requests management Paging request reception from C-Plane (ID: 11167495)

**3.24.1.0-2.0-3**  (ID: `13133291`)

Paging requests management After the configuration of the Paging Control Channel (PCCH), the C-Plane aggregates paging requests, if possible, and sends the message to the 5G-L2-PS, when at least one of the following condition is met (5G_L3_31615):

 The C-Plane aggregates enough paging requests to send the message to the U-Plane.

 The number of aggregated paging request did not reach the maximum number allowed and the maximum waiting timer to fill the message is elapsed.

Each paging request contains:

 UE-Specific DRX cycle, if configured by the upper layer. The 5G-L2-PS uses the value defined by the default paging cycle parameter (NRBTS.drxDefaultPaging), if the UE-Specific DRX cycle configured by the upper layer is greater than the default paging cycle or is not specified.

 The list of target Cell identities where the paging must be broacasted and the associated priority for each cell (mandatory). The range is between 0 and 8. the value 0 is the highest priority and 8 means no priority.

Each paging request can contain:

  * Ue Identity Index value (ueIdIndex),
  * S-Temporary Mobile Subscriber Identity (S-TMSI).



**Note:**

1.S-TMSI and ueIdIndex are mutually exclusive. S-TMSI can be used only for CN paging, ueIdIndex can be used only for RAN paging. The ueIdIndex is used when sTmsi is unavailable (RRC_INACTIVE). In that case, instead of the sTmsi value, ueIdIndex shall be used. The ueIdIndex is equivalent to sTmsi mod 1024, and sTmsi and ueIdIndex shall be processed similarly.

Upon the reception of the message, the 5G-L2-PS manages each paging request received as specified hereafter.

Target Paging Frame and Paging Occasion selection for a paging request For each target cell, the 5G-L2-PS uses the following information to determine whether a given paging request should be added in the FIFO PF/PO buffer or discarded:

 The paging priority requested for the target cell.

 The number of PF/PO candidate(s) for the paging request.

 The previous paging records already stored with their priority, in the FIFO buffer for the same PF/PO.

The 5G-L2-PS applies one of both algorithms according to the Idle DRX cycle used for the paging request:

 Paging request with the default paging cycle (5G_UP_ALG_11731_replaced([11167584](https://dn-prod.ext.net.nokia.com/rm/resources/BI_ktqgDfvSEe-AqvopbP1qhQ))).

Paging Frames and Paging Occasions management 11168065: Paging Frames and Paging Occasions management Paging Time Window (PTW) concept is used when UE is configured with eDRX and TeDRX_CN is set to value higher than 'hf1' (TeDRX_CN is longer than 1024 radio frames).

>= ‘hf4’, then value ‘s32’ (L=40.96 sec) shall be used

 Paging Hyperframe (PH):

The PH is equal to the H-SFN (Hyper SFN) which is derived from the following equation:

H-SFN mod TeDRX_CN= (UE_ID_H mod TeDRX_CN)

 Paging Time Window Start (PTW_start):

PTW_start is the first radio frame of the PH that is part the PTW and has SFN satisfying the following equation:

SFN = 128 * ieDRX_CN

where:

\- ieDRX_CN = floor(UE_ID_H /TeDRX_CN) mod 8

 Paging Time Window End (PTW_end):

PTW_end is the last radio frame of the PTW and has SFN satisfying the following equation:

SFN = (PTW_start + L*100 - 1) mod 1024

where:

\- L = Paging Time Window length (Paging Time Window IE in seconds)

Note1: PTW_start can begin only in SFN=0,128,256,384,512,640,768,896 of Paging Hyperframe.
