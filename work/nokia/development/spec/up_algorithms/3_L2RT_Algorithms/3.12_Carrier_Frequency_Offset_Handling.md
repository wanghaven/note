# 3.12 Carrier Frequency Offset Handling

## 3.12.0-4 Carrier Frequency Offset Handling Overview (ID: 11150537)

**3.12.0-4.0-2**  (ID: `11150551`)

Carrier Frequency Offset Handling
CFO (Carrier frequency offset) handing in 5G-L2-PS is providing the filtered LT FOE (long-term frequency offset estimation) metric to L1 for channel estimation.  The LT FOE metric filtering is based on L1 ST FOE (short-term frequency offset estimation) metric reported from PUSCH or PUCCH F3. Refer to 5G\_UP\_ALG\_883\_replaced([11102472](https://dn-prod.ext.net.nokia.com/rm/resources/BI_knip5vvSEe-AqvopbP1qhQ))/5G\_UP\_ALG\_1357\_replaced([11105296](https://dn-prod.ext.net.nokia.com/rm/resources/BI_koJG1fvSEe-AqvopbP1qhQ)) for L1 PUSCH/PUCCH frequency offset estimation and compensation. Refer to 5GMAX algorithm specification(5G\_UP\_ALG\_695\_replaced([11101310](https://dn-prod.ext.net.nokia.com/rm/resources/BI_knWcovvSEe-AqvopbP1qhQ))) section 5.1 for the FOE filtering.

## 3.12.0-5 Carrier Frequency Offset Handling CFO metric - Average mode (ID: 11150556)

**3.12.0-5.0-4**  (ID: `13456978`)

Carrier Frequency Offset Handling
CFO metric is managed on 5G-L2-PS with different methods based on different configurations, the methods shall be maintained per UE per component carrier.

*rrmCfoAverageMode represent the average method, it could be set as: “PUSCH\_Only”, “Both”, "Separate" or**"PUCCH\_Only", meaning of the supported modes is listed below,*\*\*

“*PUSCH\_Only”: PUCCH CFO metric management is not activated on 5G-L2-PS, CFO metric average is based on PUSCH measurement only.*

“*Both”: CFO metric average is based on both PUSCH and PUCCH F3, both PUSCH and PUCCH F3 measurement are used for CFO metric calculation and recorded on the same UE context.*

“*Separate”: CFO metric average is separately handled for PUSCH and PUCCH F3, different UE contexts are used for PUSCH and PUCCH F3.*

"PUCCH\_Only": PUSCH CFO metric management is not activated on 5G-L2-PS, CFO metric average is based on PUCCH measurement only.

*rrmCfoAverageMode is deduced from cell and UE configurations. For specific UE component carrier:*

If (PUCCH F3 additional DMRS is not configured (see [11600334](https://dn-prod.ext.net.nokia.com/rm/resources/BI_Y3V7BfwOEe-AqvopbP1qhQ?oslc_config.context=https://dn-prod.ext.net.nokia.com/gc/configuration/9))) or (*rdActPucchF3Foe (see [11151445](https://dn-prod.ext.net.nokia.com/rm/resources/BI_ksZUuvvSEe-AqvopbP1qhQ)) set to false) or (UE CFO metric average over FR1 SCell),*

*rrmCfoAverageMode = “PUSCH\_Only”*

else

*rrmCfoAverageMode = “Both”*

*else*

## 3.12.0-14 Carrier Frequency Offset Handling Frequency Offset (Hz) calculation (ID: 11151008)

**3.12.0-14.0-1**  (ID: `11151014`)

Carrier Frequency Offset Handling
5G-L2-PS shall covert CFO metric to FO\_Hz and AbsoluteFO\_Hz according to the following equations:

[Before PR697336]

[Before PR656997]

AbsoluteFO\_Hz = FO\_Hz = angle(cfo ) / (2*pi*4\*(FFTlength + CPlen)/FFTlength)  \* CarrierSp

[PR656997]

FO\_Hz = angle(cfo ) / (2*pi*4\*(FFTlength + CPlen)/FFTlength)  \* CarrierSp

AbsoluteFO\_Hz = abs(FO\_Hz)

[End PR656997]

[PR697336]

FO\_Hz = angle(cfo ) / (2*pi*4\*(4096 + 288)/4096)  \* CarrierSp

AbsoluteFO\_Hz = abs(FO\_Hz)

[End PR697336]

where,

* cfo represents the CFO metric, either long term or short term.
* angle(cfo ) represents the angle of the CFO metric,  the output range is (-pi, pi].
* CarrierSp is [Before PR697336]equal to 30e3 [PR697336] the PUSCH sub-carrier space, i.e. 15e3 for FDD, 30e3 for TDD FR1, 120e3 for TDD FR2 [End PR697336].

[Before PR697336]

* FFTlength and CPlen configuration is as following:

|  |  |  |  |
| --- | --- | --- | --- |
| ulBandwidth | 20MHZ~25MHZ | 30MHZ~50MHZ | 60MHZ~100MHZ |
| Subcarrier spacing | 30kHz |  |  |
| FFTlength | 1024 | 2048 | 4096 |
| CPlen | 72 | 144 | 288 |

[End PR697336]

For C2 subcells (2RX cases), only 1 XPol is available, and only first UL beam should be used.

*rrmAvgCfo is obtained from:*

the averaged PUSCH and PUCCH CFO metrics jointly

Else if (*rrmCfoAverageMode =* PUCCH\_Only')\*\*

averaged PUCCH CFO metric

The UL beam mentioned above is PUCCH F3 beam

Else // *rrmCfoAverageMode =* 'PUSCH\_Only' or 'Separate'\*\*

averaged PUSCH CFO metric

The UL beam mentioned above is PUSCH beam
