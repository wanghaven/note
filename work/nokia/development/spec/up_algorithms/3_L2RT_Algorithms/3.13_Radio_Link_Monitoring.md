## 3.13.2 Radio link problem detection

### 3.13.2.0-1 Radio link problem detection Radio link problem detection based on DL HARQ feedback (ID: 11151610)

**3.13.2.0-1.0-2**  (ID: `12978463`)

Radio link problem detection For each UE, DL HARQ feedback is evaluated to detect radio link problems. If the number of consecutively received DTX for DL HARQ feedback exceeds the threshold given by the

NIDD parameter rlpDetDlHarqThreshold

a radio link problem is detected at threshold+1 DTXs.

In the case that rdRlpDetHarq =0: RLF detection based on Harq feedback is switched off and no indications are sent to higher layers.

Note: The DL-HARQ feedback evaluation process for RLF might be 'stretched' in time by the DL scheduling pause triggered by feature functionality NACK-FLOOD-DEFENSE (517-C / 1796-C).

Radio link problem detection is done per carrier. Thus, only DL HARQ feedback received for the current carrier shall be evaluated even if DL HARQ feedback is sent jointly for several DL carriers via single PUCCH.

If long PUCCH is configured (IE pucchModeSelect in PsCell_CellSetupReq) the DL HARQ feedback, which is received via PUSCH when the UCI is added to the shared channel, shall be evaluated in addition to the HARQ feedback received via PUCCH.

[CB014330-B-UPD001] The parameter rlpDetDlHarqThreshold can be online modified and provided during cell setup/cell reconfiguration procedure. The new value shall be applied immediately. [End CB014330-B-UPD001]

rlpDetDlHarqThreshold: MRBTS/NRBTS/NRCELL/rlpDetDlHarqThreshold

---

### 3.13.2.0-2 Radio link problem detection Radio link problem detection based on CSI reports (ID: 11151624)

**3.13.2.0-2.0-3**  (ID: `12978859`)

Radio link problem detection In the case that rdRlpDetCsiBsi>0: For each UE, CSI reports are evaluated to detect radio link problems if the number of consecutively received DTX for CSI reports received via PUCCH or via PUSCH if long PUCCH is configured (IE pucchModeSelect in PsCell_CellSetupReq), exceeds the threshold given by the NIDD parameter rlpDetCsiBsiThreshold.**If a pCSI report reception is skipped for the UE to reduce pCSI processing load (see 5G_UP_ALG_29748_replaced([11159309](https://dn-prod.ext.net.nokia.com/rm/resources/BI_ktI7kPvSEe-AqvopbP1qhQ))), the corresponding pCSI report shall be considered DTX/nonDTX same as the previous received CSI report with the same report type and the same serving cell for the purpose of RLF detection. For example, if the previous received CSI report (A-CSI/p-CSI) is DTxed, then the skipped pCSI occasion shall also be considered DTX and vice versa. The rationale is that if we do not count the skipped pCSI occasions, then L2-PS might take longer time (twice the period) to detect the RLF failure based on pCSI reports which is undesirable.**

DTX for SR only reception [SR along with CSI report chance, see 5G_UP_ALG_3473_replaced([11159017](https://dn-prod.ext.net.nokia.com/rm/resources/BI_ktGfW_vSEe-AqvopbP1qhQ))] during DRX off time for an UE shall be ignored.

A radio link problem is detected at threshold+1 DTXs. Radio link problem detection is done in the P(S)Cel . Thus, only CSI reports for the**P(S)Cell shall be evaluated.**

In the case that rdRlpDetCsiBsi=0: RLF detection based on CSI reports is switched off and no indications are sent to higher layers.

[CB014330-B-UPD001] The parameter rlpDetCsiBsiThreshold can be online modified and provided during cell setup/cell reconfiguration procedure. The new value shall be applied immediately. [End CB014330-B-UPD001]

rlpDetCsiBsiThreshold:MRBTS/NRBTS/NRCELL/rlpDetCsiBsiThreshold.

### 3.13.3.0-3 Recovery from radio link problem Handling at recovery of radio link in P(S)Cell (ID: 11151691)

**3.13.3.0-3.0-2**  (ID: `11151704`)

Recovery from radio link problem If a recovery of the radio link is detected in the [Before PR812471] PCell [PR812471] P(S)Cell [End PR812471] of a UE, L2 PS shall send an indication of the radio link recovery to higher layers. This indication is only sent if an indication of a radio link problem was sent earlier for this UE and no indication of recovery from this problem was sent yet.

Note that indication of radio link recovery is sent as soon as it is detected based on either DL HARQ feedback or CSI reports independent of the cause of the radio link problem indication. That means that also an RLF indicated to higher layers based on HARQ DTX is cancelled by CSI report recovery on one hand and indicated RLF based on CSI report DTX is cancelled by HARQ recovery on the other hand.

## 3.13.5 UL Radio link problem detection

### 3.13.5.0-1 UL Radio link problem detection Radio link outage (RLO) overview (ID: 11151760)

**3.13.5.0-1.0-2**  (ID: `12575180`)

UL Radio link problem detection _numOfPucchDtx: number of PUCCH DTX detected represents the number of DTXed UCI received via PUCCH or via PUCCH and PUSCH if long PUCCH is configured (IE pucchModeSelect in PsCell_CellSetupReq)_

_numOfPucchSample: number of PUCCH report received represents the number of UCI reports received via PUCCH or via PUCCH and PUSCH if long PUCCH is configured (IE pucchModeSelect in PsCell_CellSetupReq)_

_numOfPuschDtx: number of PUSCH DTX detect_

_numOfPuschSample: number of PUSCH scheduled_

Both numOfPuschDtx and numOfPuschSample do not include DTX indications associated any of the following if UE is configured for skippable grants (5G_UP_ALG_4839_replaced([11195286](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kv7J1fvSEe-AqvopbP1qhQ))):

\- TA Grant

**\- fake SR grant**

\- UL grant after a zero BSR was received

_numOfTTTDtx: number of continuously DTX ratio exceed Transmit Termination Threshold. Its value range is [0...]._

wait timer: timer start when RLO or RL resume indication sent to L2-HI, which is used to avoid RL Outage or RL Resume indication message send in a high frequency. Refer to 5G_UP_ALG_3368_replaced([11151886](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kscYCfvSEe-AqvopbP1qhQ)).

Flag_RLOR: a flag used to save the UL radio link problem detection result for the UE per carrier.

  * 0 means no (UL_RL_OUTAGE/UL_RL_RESUME) request

  * 1 means UL_RL_OUTAGE requested.

  * 2 means UL_RL_RESUME request.




UlSplit_workmode: L2-PS workmode for UL data split functionality. It has 3 states includes none, NR, LTEOnly.

  * none means this functionality is disabled.

  * NR means UE works in NROnly or NR+LTE. NR also is the initial state when this functionality is enabled and UE is configured with at least one DRB.

  * LTEOnly means UE work in LTEOnly. In this workmode, the UL user data is sent only on LTE leg, when the configuration is still NSA optoin 3X.

### 3.13.5.0-13 UL Radio link problem detection Handling at wait timer expire (ID: 11151924)

**3.13.5.0-13.0-1**  (ID: `11151931`)

UL Radio link problem detection Handling at wait timer expire

increase NUM_UL_RL_OUTAGE_RAISE by 1 on sending PsCtrl_UlRadioLinkStatusInd with ulRadioLinkStatus=UL_RL_OUTAGE.

If the Flag_RLOR is set as 2 and L2-PS is working in LTEOnly mode, L2 PS shall send an indication(message PsCtrl_UlRadioLinkStatusInd with ulRadioLinkStatus=UL_RL_RESUME, see more in 5G_UP_ALG_5850_replaced([11222555](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kx78B_vSEe-AqvopbP1qhQ))) to 5G-L2-HI.

increase NUM_UL_RL_RESUME_RAISE by 1 on sending PsCtrl_UlRadioLinkStatusInd with ulRadioLinkStatus=UL_RL_RESUME.

L2-PS shall clear the Flag_RLOR after the request is sent out. If the UE is configured multiple bearers, L2-PS shall send one messages PsCtrl_UlRadioLinkStatusInd to L2-HI for each bearer.
