## 3.33.0-4 RTWP measurements PUSCH request in unallocated slots to measure RTWP value (ID: 11215719)

## 3.33.0-5 RTWP measurements Aggregation logic to calculate RTWP value (ID: 11215790)

**3.33.0-5.0-4**  (ID: `11215820`)

RTWP measurements If MRBTS/NRBTS/actRtwpMeasurement is set to true for a NRCELL, or independantly to any feature activation for remote RU diagnostics requests **5G-L2-PS shall receive measurement reports from L1 containing RSSI values and I+N values per spatial stream for each UL grant (UE) on all the subcells of a NRCELL where UE data was scheduled. The RSSI and I+N values received from L1 are normalized to a single PRB, and are provided in linear scale.**

For the calculation of RTWP value, 5G-L2-PS shall consider following measurement values:

1\. If fake UE was scheduled in the slot (i.e. there is no real UE in the slot, and fake UE is scheduled only on primary subcell as per 5G_UP_ALG_17489_replaced([11215719](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kxU4CvvSEe-AqvopbP1qhQ))), 5G-L2-PS shall receive RSSI and I+N measurements for all the spatial streams of primary subcell.

2\. If real UE was scheduled on the primary subcell in the slot, then

a. If a paired UE data was also scheduled on the secondary subcell, 5G-L2-PS shall receive independent reports for each subcell, containing RSSI and I+N values per grant per spatial stream.

b. If there is no real UE scheduled on secondary subcell, 5G-L2-PS, shall receive RSSI and I+N measurement report only for the spatial stream of the primary subcell.

[Before CB008235-D-UPD001]

3\. If paired UEs (MU-MIMO within a subcell) were scheduled on a subcell in the slot, and the maximum number of UEs in each MU-MIMO layer is one, then RSSI and I+N measurements for UE with more scheduled PRBs shall be considered for calculation of RTWP value per spatial stream.

[CB008235-D-UPD001]

3\. If paired UEs (MU-MIMO within a subcell) were scheduled on a subcell in the slot, 5G-L2-PS shall decide the MU-MIMO layer j with the maximum number of PRBs firstly, then 5G-L2-PS shall consider for calculation of RTWP value per spatial stream by accumulating RSSI and I+N measurements over all of the PRBs for the UEs in the same MU-MIMO layer j.

[End CB008235-D-UPD001]

4\. If real UE was scheduled only on the secondary subcell in the slot (no UE is scheduled on the primary subcell), then ignore this slot (i.e. the RSSI and I+N measurement report for this slot) in the calculation of RTWP measurement counters.

RATIONALE: Normally whenever there is a real UE scheduled on secondary subcell, there should be a corresponding paired UE also scheduled on primary subcell. But in case, when retransmission is only requested for the UE scheduled on secondary subcell, then it may happen that no real UE is scheduled on primary subcell, but a real UE is scheduled for retransmission on secondary subcell. Such case may not happen very frequently in real deployment, so it is decided to ignore the reports coming from such slots for RTWP calculations.

When DSS is not active (NRBTS.actLteNrDss = ‘false’ or NRBTS/NRCELL.lteNrDssMode = none):

While accumulating rrmReceivedRssiPerSpatialStream, rrmReceivedINPerSpatialStream, rrmNumOfPRBWithINMeasurement, and rrmNumOfPRBWithoutINMeasurement, 5G-L2-PS shall consider the **number of PRBs for all configured PUSCH regions ([11199501](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kwOExvvSEe-AqvopbP1qhQ?oslc_config.context=https%3A%2F%2Fdn-prod.ext.net.nokia.com%2Fgc%2Fconfiguration%2F9)),**

withing the ranges defined by:

longPucchStartPuschPrb and longPucchStartPuschPrb + longPucchNumPuschPrb

[CB012299-A-UPD002]

and

longPucchStartPuschPrb2 and longPucchStartPuschPrb2 + longPucchNumPuschPrb2

[End CB012299-A-UPD002]

where

  * longPucchStartPuschPrb and longPucchNumPuschPrb



[CB012299-A-UPD002]

and

  * longPucchStartPuschPrb2 and longPucchNumPuschPrb2



[End CB012299-A-UPD002]

are given in (5G_CP_ARC_ALG_2876_replaced( [11310329](https://dn-prod.ext.net.nokia.com/rm/resources/BI_TrPVSPvcEe-AqvopbP1qhQ?oslc_config.context=https%3A%2F%2Fdn-prod.ext.net.nokia.com%2Fgc%2Fconfiguration%2F9))) for the largest UL Bandwidth Part [CB012299-B-UPD003] or when limited bandwidth mode is active( [12505866: Nominal and limited bandwidth modes](https://dn-prod.ext.net.nokia.com/rm/resources/BI_6-kc8RVSEfCV0v_Ffc8sJw)) for non-largest dedicated bandwidth [End CB012299-B-UPD003 ].

For example, if longPucchStartPuschPrb is 20, longPucchNumPuschPrb is 50 for largest UL BWP, and a UE is scheduled such that startPRB is 50, and numOfPrbs is 30, then while calculating the impact on rrmReceivedRssiPerSpatialStream, rrmReceivedINPerSpatialStream, rrmNumOfPRBWithINMeasurement, and rrmNumOfPRBWithoutINMeasurement variables because of the scheduled UE, only PRBs from 50 to 70, shall be considered.

When DSS is active (NRBTS.actLteNrDss = ‘true’ and NRBTS/NRCELL.lteNrDssMode <> none):

While accumulating rrmReceivedRssiPerSpatialStream, rrmReceivedINPerSpatialStream, rrmNumberOfPRBWithINMeasurement, and rrmNumberOfPRBWithoutINMeasurement, 5G-L2-PS shall consider the number of PRBs as received from CRM in “EN DSS UL SUBCARRIER RESOURCE ALLOCATION REQUEST” message.

For all the UL slots, with in a sampling period (1 sec), L2-PS shall maintain following (cumulative for all the UL slots in a sampling period):

1\. If UE is scheduled on secondary subcell, and there is no real UE scheduled on primary subcell, ignore the slot for RTWP calculations and skip further processing for the current slot.

2\. Handling per spatial stream of a primary subcell:

a. rrmReceivedRssiPerSpatialStream = Cumulative RSSI received for all the scheduled PRBs (real + fake UE) per spatial stream.

i. For all the scheduled UE, follow below steps

1\. Calculate RSSI value of a UE = reported RSSI value (normalized to 1 PRB) * number of PRBs (as calculated above)

2\. Add it to rrmReceivedRssiPerSpatialStream

b. rrmReceivedINPerSpatialStream = Cumulative I+N value received for all the scheduled PRBs (real + fake UE) per spatial stream.

i. For all the scheduled UE, follow below steps

1\. Calculate I+N value of a UE = reported I+N value (normalized to 1 PRB) * number of PRBs (as calculated above)

2\. Add it to rrmReceivedINPerSpatialStream

NOTE: For a fake UE, it is expected that 5G-L2-PS shall receive correct I+N value from L1, and it’s value should be same as RSSI value for a the spatial streams, it is left to the CP3 phase to decide the IE to be used to read I+N value for a fake UE.

c. rrmNumOfPRBWithINMeasurement = Cumulative number of PRBs where I+N measurements are received (PRBs allocated to real UE + fake UE).

i. For all the scheduled UEs

1\. Check above how to calculate number of PRBs for a UE.

2\. Add it to rrmNumOfPRBWithINMeasurement

d. rrmNumOfULSlotsInSamplingPeriod = Number of UL slots in a sampling period except special slot and the PRACH slot without any real UE scheduled on PUSCH channel.

e. rrmNumOfPRBWithoutINMeasurement = Cumulative number of PRBS for which I+N measurements are not received (PRBs ouside longPucchStartPuschPrb and longPucchStartPuschPrb + longPucchNumPuschPrb range shall not be considered for this variable).

3\. Handling per spatial stream of a secondary subcell:

a. For every UE scheduled in the primary subcell,

i. If there is a paired UE scheduled in secondary subcell (RSSI and I+N measurements are received for secondary subcell spatial streams)

1\. rrmReceivedRssiPerSpatialStream = rrmReceivedRssiPerSpatialStream + reported RSSI value (normalized to 1 PRB) * number of PRBs (as calculated above)

2\. rrmReceivedINPerSpatialStream = rrmReceivedINPerSpatialStream + reported I+N value (normalized to 1 PRB) * number of PRBs (as calculated above)

ii. If there is no real UE scheduled in secondary subcell (RSSI and I+N measurements are not received for secondary subcell spatial streams)

1\. Consider RSSI and I+N value from primary subcell as corresponding RSSI and I+N value in secondary subcell ((e)AxC index of secondary subcell shall read the value from same (e)AxC index of primary subcell)

2\. rrmReceivedRssiPerSpatialStream = rrmReceivedRssiPerSpatialStream + reported RSSI value (normalized to 1 PRB) * number of PRBs (as calculated above)

3\. rrmReceivedINPerSpatialStream = rrmReceivedINPerSpatialStream + reported I+N value (normalized to 1 PRB) * number of PRBs (as calculated above)

iii. rrmNumOfPRBWithINMeasurement is same as primary subcell.

iv. rrmNumOfULSlotsInSamplingPeriod is same as primary subcell.

v. rrmNumOfPRBWithoutINMeasurement is same as primary subcell.

## 3.33.0-6 RTWP measurements Period of processing and RTWP counter updates (ID: 11215828)

**3.33.0-6.0-1**  (ID: `11215834`)

RTWP measurements On expiry of sampling period (1 sec), 5G-L2-PS shall perform following steps per spatial stream:

1\. rrmAvgINPerPRB = rrmReceivedINPerSpatialStream/ rrmNumOfPRBWithINMeasurement

2\. rrmTotalReceivedRssiPerSpatialStream = rrmReceivedRssiPerSpatialStream + rrmAvgINPerPRB * rrmNumOfPRBWithoutINMeasurement

3\. rrmRTWP = rrmTotalReceivedRssiPerSpatialStream / rrmNumOfULSlotsInSamplingPeriod

4\. rrmRTWP shall be converted to logarithmic scale and multiplied by 10 before reporting to the higher layer.

If MRBTS/NRBTS/actRtwpMeasurement is set to true, RTWP is measured and all PM counters under the measurement (“NRRTP Counters”) shall be updated every sampling period.

NOTE: Since RTWP value is calculated only once in a sampling period, so the AVG, MIN and MAX value shall be same for the sampling period, and number of samples shall be 1.
