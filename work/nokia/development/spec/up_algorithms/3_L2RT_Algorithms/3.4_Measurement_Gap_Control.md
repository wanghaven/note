# 3.4 Measurement Gap Control

## 3.4.0-4 Measurement Gap Control Scheduling opportunities for E-UTRA-NR dual connected UEs/SA UE when Measurement Gaps are configurated from MeNB/gNB (ID: 11114624)

**3.4.0-4.0-2**  (ID: `11114630`)

Measurement Gap Control NSA and SA Measurement Gap can be configured to E-UTRA-NR dual connected (NSA) UE and the Stand Alone (SA) NR UE. If Measurement Gap configurations are included in PsUser_UserSetupReq/ PsUser_UserModifyReq for E-UTRA-NR dual connectivity UE or PsUser_UserModifyReq for SA UE, and the configuration of measGapConfig is valid for the UE, 5G-L2-PS shall store these configurations in the UE context and calculate the MG interrupted slots for this UE as the following procedure.

For each MG object, gapOffset, MGRP, MGTA, MGL can be achieved from measGapConfig from PsUser_UserSetupReq/ PsUser_UserModifyReq,

the first slot of each gap occurs at an SFN and slot meeting the following condition (SFN and slot of the serving cell):

startSlot = {FLOOR[(gapOffset+10-MGTA) _2^μ]} mod (10_ 2^μ) ,

SFN mod T = {FLOOR[(MGRP+gapOffset-MGTA)/10]} mod T,

with T = MGRP/10;

Where startSlot is the slot number within a frame for subcarrier spacing configuration μ, (plus 10 in the 1st function and plus MGRP in the 2nd function here in case that gapOffset is configured to 0);

For measurement gaps configuration (either gapFR1 or gapUE configuration, gapFR2 configuration is not in 5GC001013's scope) to E-UTRA-NR dual connected UEs, if LTE and NR's SFN0 is not aligned by known offset, 5G-L2-PS shall use this offset to calculate gap slot at NR side.

gapOffset in the below two formula shall minus to lteToNrTimeOffset. Note that NRCELL/lteToNrTimeOffset has range of [-10,10] ms , and positive value means LTE's SFN0 is earlier than NR's.

For each MG object, gapOffset, MGRP, MGTA, MGL can be achieved from measGapConfig from PsUser_UserSetupReq/ PsUser_UserModifyReq,

the first slot of each gap occurs at an SFN and slot meeting the following condition (SFN and slot of the serving cell):

startSlot = {FLOOR[(gapOffset-lteToNrTimeOffset+20-MGTA) _2^μ]} mod (10_ 2^μ) ,

SFN mod T = {FLOOR[(MGRP+gapOffset-lteToNrTimeOffset-MGTA)/10]} mod T,

with T = MGRP/10;

Where startSlot is the slot number within a frame for subcarrier spacing configuration μ, (plus 20 in the 1st function and plus MGRP in the 2nd function here in case that gapOffset is configured to 0 and lteToNrTimeOffset is configured to 10);

The number of total interrupted slots is:

Check table using MGL, MGTA and scs of current serving cell:

IF MG configuration type is per-UE or per-FR for FR1(the latter means the frequency range of current serving cell is FR1):

If the EN-DC is synchronous or SA:

Check Table 9.1.2-4 in 3GPP 38133-f70;

Else if the EN-DC is asynchronous:

Check Table 9.1.2-4a in 3GPP 38133-f70;

End

where EN-DC is synchronous status can be achieved from NIDD parameter.

5G-L2-PS shall schedule this UE taking MG interrupted slots into account after the Random Access procedure has been finished, which means the MG has been setup in L2.

[CB008124-D] If the UE has an active Configured grant Type2 in the Uplink, 5G-L2-PS shall consider the CG occasion if any during the measurement period as unavailable for decode.[End CB008124-D]

NSA and SA

For DL: 5G-L2-PS shall not schedule UEs whose PDCCH or PDSCH or PUCCH for DL harq report drop in the interrupted slots by MG;

For UL: 5G-L2-PS shall not schedule UEs whose PDCCH for PUSCH/SRS or PUSCH or SRS drop in the interrupted slots by MG. [CB008124-D] For users with an active Configured grant Type2 in the Uplink , 5G-L2-PS shall not consider to receive any UL PUSCH data during the CG occasions in the interrupted slots by MG [End CB008124-D] ;

5G-L2-PS shall not send decode request of UEs' periodic signals which are influenced by the interrupted slots to L1, e.g. CSI, SR, SRS(may be supported by 5GC001258);

5G-L2-PS can send or not influenced periodic DL reference signals send request to L1 since the checking logic will bring complexity and UE will not decode them anyway.

The detailed handling of periodic signals interrupted by MG will be described in 5G_UP_ALG_2292_replaced([11114634](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kpmfZvvSEe-AqvopbP1qhQ))

Note: 1. For E-UTRA-NR dual connected UEs, per-RF for FR2 MG configuration is not the scope of 5GC001013.

2\. For SA UEs, only per UE MG is available for 5GC000707 & 5GC000706.

  1. When the UE with MG already setup in L2 receives another MG configuration from PsUser_UserSetupReq/ PsUser_UserModifyReq, L2 will handle the second configuration in the same way with the first one. This may lead to dtx in the transient period for the slots which are in the gap of the first MG configuration while is in the available slots of the second MG configuration. The transient period begins from L2 receives the new MG configuration to UE's high layer receives the new configuration. It may not be the case for SA.

## 3.4.0-5 Measurement Gap Control Interrupted Periodic signals for E-UTRA-NR dual connected / SA UEs when Measurement Gaps are configurated from MeNB/gNB (ID: 11114634)

**3.4.0-5.0-1**  (ID: `11114638`)

Measurement Gap Control If Measurement Gap configurations are setup for E-UTRA-NR dual connectivity or SA UE as defined in 5G_UP_ALG_2291_replaced([11114624](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kpmfY_vSEe-AqvopbP1qhQ)), 5G-L2-PS shall handle periodic signals interrupted by MG as bellow.

Currently, the timeRestrictionForChannelMeasurements is configured and the non time restriction will be handled later if needed. For periodic CSI, there are two conditions that can lead to invalid CSI report:

1). the corresponding CSI-RS slot is dropped in the gap;

2). The CSI report slot is dropped in the gap.

L2 shall skip sending the CSI report decode request to L1 if it is invalid.

The corresponding CSI-RS means UE will use this CSI-RS for CSI calculation. Refer to 38.214 for this definition.

5G-L2-PS can request or not influenced periodic DL reference signals to L1 since the checking logic will bring complexity and UE will not decode them anyway.

Note: 1. Currently, all the periodic SR is configurated at the same slot with periodic CSI, refer to 5G_L3_10576, 5GC000523-E.

  1. Currently, periodic SRS is supported by 5GC001258 but is only exist in 19B.
