# 3.23 Public Warning System Management

## 3.23.0-4 Public Warning System Management 11166969: Public Warning System Management The Public Warning System allows the 5G-L2-PS to broadcast PWS messages (PWS-Msg) though specific PWS System Information Block (SIB) embedded in a SI message transmitted on the radio interface. The 5G-L2-PS receives from the C-Plane to broadcast the PWS-Msg(s):  The characteristics of one or several SIB1 and SI-Msg. The SIB1 provides System Information about the SI-Msgs. Among the set of SI-Msgs broacasted, one or several SI-Msgs are handling PWS-SIB(s). The whole SI configuration is  done through the Broadcast Control Channel Management (5G_UP_ALG_3673_replaced( 11163879 ) and followings).  The characteristics of one or several sets of PWS-SIB that the 5G-L2-PS has to broadcast and which contain the PWS message (PWS-Msg) for the end user. For each PWS-Msg to broadcast, the C-Plane provides the U-Plane with up to 32 PWS-SIB segment(s), the type of the PWS-Msg, the number of expected broadcasts and its repetition time. The PWS-Msg session represents the duration of the whole transmissions of a given PWS-Msg. At each repetition interval, the PWS-Msg window represents the broadcasting of one occurence of a given PWS-Msg. ## 3.23.1 PWS configuration

### 3.23.1.0-1 PWS configuration 11166985: PWS configuration The conditions to activate the PWS broadcast processing are the followings:  The 5G-L2-PS must have in the SI active contexts, the SIB1 and SI-Msg description with and without the expected PWS-SIB information.  The active PWS-Msg list, managed by the 5G-L2-PS, must not be empty. The list is managed through the specific C-Plane messages and accroding to the content of this list. Otherwise, the 5G-L2-PS uses only the active SIB1 and SI-Msg(s) contexts without any PWS information (SIB1 and SI-Msg contexts with the pwsType field set to 0). ## 3.23.2 PWS messages management

### 3.23.2.0-1 PWS messages management 11167069: PWS messages management The 5G-L2-PS manages a list of active PWS-Msg to broadcast. Each PWS-Msg in the active list is identified by a unique "pwsMsgId" managed between the C-Plane and the U-Plane. This list is used to find the PWS-Msg to broacast according to the data associated to the message. The C-Plane has the capability to add, delete or replace a PWS-Msg in the list by sending the PsCell_PwsMsg[Add/Del/Replace]Req message requests. The 5G-L2-PS answers to the message with the PsCell_PwsMsg[Add/Del/Replace]Resp. For a delete or replace request, the 5G-L2-PS sends a PsCell_PwsReportInd message with PWS cell broadcast information and dedicated PWS-Msg feedback. When the expected number of broadcast of a given PWS-Msg is done, the 5G-L2-PS sends back to the C-Plane a PsCell_PwsReportInd message with the same level of information. When several messages are present in the active PWS-Msg list, the 5G-L2-PS selects the PWS-Msg to broadcast per PWS type according to:  The PWS type of the message(s) (CMAS, ETWS primary notification, ETWS secondary notification).  The previous  broadcast time, by selecting first the PWS-Msg never transmitted on the radio interface. One First-In First-Out (FIFO) buffer per PWS type is used to maintain the list of the PWS-Msg never broadcasted. ## 3.23.3 PWS broadcast session management

### 3.23.3.0-1 PWS broadcast session management Overview (ID: 11167083)

**3.23.3.0-1.0-1**  (ID: `11167096`)

PWS broadcast session management 11167096: PWS broadcast session management The 5G-L2-PS manages the global PWS broadcast session. The session is active during one or several PWS-Msg broadcasts which are enough close together to avoid useless broacast session deactivation (gNB resource saving). To reduce the UE power draining and to reduce the gNB resources usage, the 5G-L2-PS stops the SIB1 and SI-Msg associated to the PWS as soon as the PWS inactivity duration is enough large compare to the PWS-SI type activation period and if the active SI confguration (all SIB1 and all SI-msg) done by the C-Plane allows to deactive the PWS SI-Msg without any impact on the other SI-Message scheduling.
