## 3.32.2 UL Cell Test Measurements for remote RU-Diagnostics

### 3.32.2.0-1 UL Cell Test Measurements for remote RU-Diagnostics UL Cell Test Measurements Mechanism (ID: 11215591)

**3.32.2.0-1.0-2**  (ID: `11215604`)

UL Cell Test Measurements for remote RU-Diagnostics [Before CB009556-A]When 5G-L2-PS receives an UL cell test measurement request in PsCell_TestMeasurementReq message, 5G-L2-PS shall acknowledge this request by sending PsCell_TestMeasurementResp message.

5G-L2-PS shall use a UlData_PuschReceiveReq message that is sent to 5G-L1-UL to request cell test measurements for concerned slots, until processing period is completed.

[End CB009556-A]

gNB-DU shall elaborate RTWP measurements [CB009556-A] per spatial stream [End CB009556-A] using a similar algorithm as described in 5G_UP_ALG_17492_replaced([11215790](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kxVfFPvSEe-AqvopbP1qhQ)), but independently from any other feature activation.

### 3.32.2.0-5 UL Cell Test Measurements for remote RU-Diagnostics UL scheduler Power Measurements Aggregation Algorithm (ID: 11215657)

**3.32.2.0-5.0-2**  (ID: `11215669`)

UL Cell Test Measurements for remote RU-Diagnostics Power measurements received from 5G-L1-UL are in logarithmic scale, aggregation method over the period shall make a logarithmic average of those values (see 5G_UP_ALG_5286_replaced([11215675](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kxU4BPvSEe-AqvopbP1qhQ))) for each subCell to evaluate the average test measurements to be reported for the requested Cell.

RTWP case:

5G-L2-PS average is calculated on the basis of the number of all measurements for the period of processing (see 5G_UP_ALG_5278_replaced([11215610](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kxUQ-PvSEe-AqvopbP1qhQ))), including [PR694211] (for CPRI) [end PR694211] values received for unallocated slots.

RSSI case:

5G-L2-PS average is calculated on the basis of the number of valid measurements for the period of processing (see 5G_UP_ALG_5278_replaced([11215610](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kxUQ-PvSEe-AqvopbP1qhQ))), meaning that the default values received for unallocated slots [PR694211] (for CPRI) [end PR694211] are not taken for the average calculation. If all RSSI measurements received during the period of processing are invalid then 5G-L2-PS should report default value -140dBm.

SINR case:

The SINR measurements are received from 5G-L1-UL by sinrOfAnt within UlData_PuschReceiveRespPs message, see 5G_L1_IF_200_replaced([11513831](https://dn-prod.ext.net.nokia.com/rm/resources/BI_FRzlAPvxEe-AqvopbP1qhQ)).

5G-L2-PS average is calculated on the basis of the number of valid measurements for the period of processing (see 5G_UP_ALG_5286_replaced([11215675](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kxU4BPvSEe-AqvopbP1qhQ)), 5G_UP_ALG_5278_replaced([11215610](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kxUQ-PvSEe-AqvopbP1qhQ))), meaning that the default values received for unallocated slots ([PR694211]for CPRI [end PR694211] , value: -20) are not taken for the average calculation. If all SINR measurements received during the period of processing are invalid, then 5G-L2-PS should report special value -20dB.
