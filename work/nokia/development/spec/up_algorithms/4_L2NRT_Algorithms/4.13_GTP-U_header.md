# 4.13 GTP-U header

## 4.13.0-2 GTP-U header GTP-U Sequence Number (ID: 11225210)

**4.13.0-2.0-1**  (ID: `11225217`)

GTP-U header Transmission

---

**4.13.0-2.0-1.0-1**  (ID: `11225224`)

GTP-U header This requirement describes the part when NIDD parameter NRBTS.actGtpUpSeqNum is activated and configured to gNB-CU-UP.

GTP-U Sequence Number shall be added in GTP-U header at source gNB-CU-UP when DL PDCP PDUs are transmitted on the following interfaces:

  1. F1-U using NRUP protocol



\- Different GTP-SN sequence shall be maintained and attached per DRB.

  1. NRDC Xn-U using NRUP protocol



\- Different GTP-SN sequence shall be maintained and attached per DRB.

  1. Data forwarding during SA Inter-gNB handover



\- Different GTP-U SN sequence shall be maintained and attached for forwarded packets per QoS Flow when PDU session level data forwarding is triggered.

\- Different GTP-U SN sequence shall be maintained and attached for forwarded packets per DRB when DRB level data forwarding is triggered.

[CB009884-D-UPD001]

  1. Data forwarding using Xn interface during Inter-gNB NR-DC SNODE Addition



\- MN gNB shall maintain different GTP-U SN sequence and attach GTP-U SN for forwarded packets per DRB when DRB level data forwarding is triggered.

Note: NR-DC SNODE Addition bearer change scenario with PDU session level data forwarding is not supported currently.

  1. Data forwarding using Xn interface during Inter-gNB NR-DC bearer takeback



\- SN gNB shall maintain different GTP-U SN sequence and attach GTP-U SN for forwarded packets per DRB when DRB level data forwarding is triggered.

Note: NR-DC bearer takeback scenario with PDU session level data forwarding is not supported currently.

[End CB009884-D-UPD001]

Increasing sequence number is used for GTP-U SN while transmission order must be preserved.

Retransmitted PDCP PDUs shall be attached with next GTP-U SN and older SNs shall not be reused. No retransmittion of PDCP SDUs in case of data forwarding shall be supported in this feature.

From 3GPP TS 29.281 specifications:

To add GTP-U SN, Sequence number flag/bit (S) is set in the first octet.

When Sequence number flag is set to '0', the Sequence Number field either is not present or, if present, shall not be interpreted.

When Sequence number flag is set to '1', the Sequence Number field is present, and shall be interpreted.

Hence, in case NRBTS.actGtpUpSeqNum is activated, then Sequence number flag (S) shall be set to '1' and GTP-U Sequence Number is 16-bit number added in the ninth and tenth octet of the GTP-U header.

GTP-U SN shall not be attached to Echo Response, End marker and Tunnel Status messages on the listed interfaces.

Note:

  1. IRAT (both Direct and Indirect) data forwarding are not in the scope of this feature.

**4.13.0-2.0-2**  (ID: `11225230`)

GTP-U header Reception

---

**4.13.0-2.0-2.0-1**  (ID: `11225238`)

GTP-U header This requirement describes the handling when packets attached with GTP-U SN is received on the following interfaces:

  1. NG-U at source gNB-CU-UP (N3 interface)



GTP-SN sequence shall be detected as missing/received per QoS flow when received on NG-U interface. For the default DRB, PM counters shall be incremented only for the mapped QoS flow.

  1. F1-U at gNB-DU



GTP-SN sequence shall be detected as missing/received per DRB.

  1. NRDC Xn-U at gNB-DU of Secondary Node



GTP-SN sequence shall be detected as missing/received per DRB.

  1. Target gNB on Xn-U data forwarding tunnel during SA Inter-gNB handover



GTP-U SN shall be received in GTP-U header for both PDU session and DRB level data forwarding.

Target gNB entity shall detect GTP-U SN sequence as missing/received per QoS Flow when PDU session level forwarded packets are received.

Target gNB entity shall detect GTP-U SN sequence as missing/received per DRB when DRB level forwarded packets are received.

  1. Target gNB on NG-U data forwarding tunnel using indirect data forwarding



GTP-U SN if received shall be ignored.

[CB009884-D-UPD001]

  1. Data forwarding using Xn interface during Inter-gNB NR-DC SNODE Addition



SN gNB shall receive GTP-U SN embedded in GTP-U header for DRB level data forwarding.

SN gNB entity shall detect GTP-U SN sequence as missing/received per DRB when DRB level forwarded packets are received.

Note: NR-DC SNODE Addition bearer change scenario with PDU session level data forwarding is not supported currently.

  1. Data forwarding using Xn interface during Inter-gNB NR-DC bearer takeback



MN gNB shall receive GTP-U SN embedded in GTP-U header for DRB level data forwarding.

MN gNB entity shall detect GTP-U SN sequence as missing/received per DRB when DRB level forwarded packets are received.

Note: NR-DC bearer takeback scenario with PDU session level data forwarding is not supported currently.

[End CB009884-D-UPD001]

Note:

  1. UPF shall assign the GTP-U SN per QoS flow.
