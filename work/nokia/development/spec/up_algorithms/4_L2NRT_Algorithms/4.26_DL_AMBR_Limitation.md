### 4.26.1-1.0-1 Common Dynamic distribution of DL UE AMBR to the non-GBR DRBs (ID: 11228072)

**4.26.1-1.0-1.0-1**  (ID: `11228079`)

Common The dynamic distribution of AMBR to the non-GBR DRBs of the PDU session shall be performed as below.

For the non-GBR DRB(s) of a UE setup in SA mode, in case when a non-GBR DRB is added/deleted for a UE.

 The dlPduSesDynAmbr is split to the non-GBR DRBs of the PDU session based on the scheduling weight “dlDrbWeightedAmbr” (refer to 5G_UP_ALG_19315_replaced([11227917](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kyfVp_vSEe-AqvopbP1qhQ))) such that, for each of the non-GBR DRB of the UE, set dlDrbDynAmbr = dlDrbWeightedAmbr for a duration of dlAmbrSampInterval.

[Rationale : When the DRB is added, the input rate is unknown. Hence the bearer is assigned AMBR limit (output rate) equivalent to its rate scaled as per its scheduling weight, considering the PDU session AMBR limit to which the Bearer belongs to. This AMBR limit based on scheduling weight “dlDrbWeightedAmbr” is retained for atleast one dlAmbrSampInterval duration for the input rate to build up for the bearer(s)].

 Skip the below procedure.

For a PDU session of the UE for which the dlPduSesDynAmbr has increased :

Redistribute the increase of dlPduSesDynAmbr to the non-GBR bearers of the PDU session for which the dlDrbAvgInBitRate is higher than dlDrbDynAmbr as defined in 5G_UP_ALG_19336_replaced([11228170](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kygjzPvSEe-AqvopbP1qhQ)), where the tmpTotalRecoveredAmbr and totalRequiredDlAmbr are initialised as below:

 tmpTotalRecoveredAmbr = delta increase in dlPduSesDynAmbr compared to the previous value.

 totalRequiredDlAmbr = amount of AMBR required by the PDU session.

Rationale:

When the dynamic PDU session AMBR (internally computed at PDCP) is to be increased/decreased –

The PDU sessions for which the AMBR is reduced – the AMBR values of the non-GBR DRBs are reduced according to the Input rates of the non-GBR DRBs and do not require further re-distribution of the AMBR. While for the PDU session(s) for which the AMBR is increased, the increased AMBR needs to be distributed to the non-GBR DRBs for which the input rate is higher than the dynamic AMBR.

In a case when dlDrbAvgInBitRate for one or more of the non-GBR DRBs of the PDU session is less than “dlDrbDynAmbr”, AND If dlDrbAvgInBitRate for one or more of the non-GBR DRBs of the PDU-session is greater than “dlDrbDynAmbr” [PR794626] AND If there is no non-GBR DRB which has dlDrbAvgInBitRate (Note 5) greater than “dlDrbDynAmbr” (Rationale 2) AND “dlDrbDynAmbr” is less than “dlDrbWeightedAmbr” [End PR794626]

 Perform the steps defined in 5G_UP_ALG_19330_replaced([11228114](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kygjxPvSEe-AqvopbP1qhQ)).

In a case when dlDrbAvgInBitRate for [Before PR794626] all the [PR794626] one or more [END PR794626] non-GBR DRBs of the PDU session is greater than [PR794626] (Note 5) [End PR794626] [Before PR794626] or equal to [End PR794626] “dlDrbDynAmbr”, AND If “dlDrbDynAmbr” is less than “dlDrbWeightedAmbr” for one or more of the non-GBR DRBs of the PDU session with dlDrbAvgInBitRate(Note 5) greater than “dlDrbDynAmbr” (Rationale 2).

OR

In case there is one or more non-GBR DRB(s) which has “dlDrbDynAmbr” [PR794626] is less than or [End PR794626] equal to rdDlAmbrMinLimit AND dlDrbAvgInBitRate is greater than 0 (refer Rationale),

 Perform the steps defined in 5G_UP_ALG_19350_replaced([11228232](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kyhK1fvSEe-AqvopbP1qhQ)).

In a case when there is only one non-GBR bearer for the PDU session – there is no AMBR distribution within the PDU session.

dlDrbDynAmbr for the non-GBR DRB = dlPduSesDynAmbr.

[PR794626]

If rdAmbrDistImbalanceDetect is set to True,

If all the non-GBR DRBs for which dlDrbAvgInBitRate > 0, has dlDrbAvgInBitRate > dlDrbDynAmbr AND if dlDrbDynAmbr is greater than or equal to dlDrbWeightedAmbr,

 For the non-GBR DRBs with dlDrbAvgInBitRate > 0,

If the dlDrbDynAmbr of the non-GBR DRB with lower or equal weight is greater than the dlDrbDynAmbr of any other non-GBR DRB with higher or equal weight.

 Reset the dlDrbDynAmbr for all the non-GBR DRBs to dlDrbWeightedAmbr

[End PR794626]

Rationale:

1\. In case when the non-GBR DRB's dlDrbDynAmbr is set to rdDlAmbrMinLimit, and if the DRB was not able get any allocation for DL AMBR as part of steps defined in 5G_UP_ALG_19330_replaced([11228114](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kygjxPvSEe-AqvopbP1qhQ)), then dlPduSesDynAmbr shall be re-distributed across the non-GBR DRBs within the PDU session as per the steps defined in 5G_UP_ALG_19350_replaced([11228232](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kyhK1fvSEe-AqvopbP1qhQ)) to get the AMBR allocation.

2\. When the “dlDrbDynAmbr” of a non-GBR DRB is reduced to match its dlDrbAvgInBitRate, Due to the slight variations in the average computation the dlDrbAvgInBitRate might seem marginally greater than the “dlDrbDynAmbr” – and such computational variance should not trigger “dlDrbDynAmbr” redistribution among the non-GBR DRBs of the PDU session.

Note:

1\. dlDrbWeightedAmbr, dlDrbDynAmbr, dlDrbAvgInBitRate, dlPduSesDynAmbr are defined in 5G_UP_ALG_19313_replaced([11227904](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kyfVpfvSEe-AqvopbP1qhQ)).

2\. rdDlAmbrMinLimit is a RAD parameter defined in 5G_UP_ALG_27181_replaced([11228316](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kyhx4_vSEe-AqvopbP1qhQ)).

3\. NRBTS/dlAmbrSampInterval, is a NIDD parameter received from C-plane.

4\. rdDlAmbrInBitRateMarginPercent is a RAD parameter defined in 5G_UP_ALG_29835_replaced([11228354](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kyhx6fvSEe-AqvopbP1qhQ)).

5\. As mentioned in Rationale 2 – the dlDrbAvgInBitRate is considered to have exceeded the dlDrbDynAmbr when the dlDrbAvgInBitRate is higher than dlDrbDynAmbr by “rdDlAmbrInBitRateMarginPercent”.

[PR794626]

6\. rdAmbrDistImbalanceDetect is a RAD parameter defined in 5G_UP_ALG_31036_replaced([11228369](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kyhx6_vSEe-AqvopbP1qhQ)).

[End PR794626]

## 4.26.1-2 Common

### 4.26.1-2.0-1 Common Dynamic distribution of DL UE AMBR to the PDU sessions (ID: 11228092)

**4.26.1-2.0-1.0-1**  (ID: `11228098`)

Common The dynamic distribution of DL UE AMBR to the PDU sessions of the UE shall be performed as below.

For the UE setup in SA mode, in case when a PDU session is added/deleted for a UE OR a non-GBR bearer is added/deleted for any of the PDU session of the UE OR If the DL UE AMBR is modified.

 The “DL UE AMBR limit” is split to the PDU Sessions of the UE based on the scheduling weight “dlPduSesWeightedAmbr” (refer to 5G_UP_ALG_26977_replaced([11227931](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kyfVqfvSEe-AqvopbP1qhQ))) such that, for each of the PDU session of the UE, set dlPduSesDynAmbr = dlPduSesWeightedAmbr for a duration of dlAmbrSampInterval.

 Restart the monitoring for dlAmbrSampInterval and dlAmbrMonitorPeriod (Refer Note 3).

 dlPduSesDynAmbr is then distributed to the non-GBR DRBs of the respective PDU session as defined in 5G_UP_ALG_19365_replaced([11228072](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kyf8vvvSEe-AqvopbP1qhQ)).

 Skip the below procedure.

In a case when dlPduSesAvgInBitRate for one or more of the PDU sessions of the UE is less than “dlPduSesDynAmbr”, AND If dlPduSesAvgInBitRate for one or more of the PDU sessions of the UE is greater than “dlPduSesDynAmbr” [PR794626] AND If there is no PDU session which has dlPduSesAvgInBitRate (Note 5) greater than “dlPduSesDynAmbr” (Rationale 2) AND “dlPduSesDynAmbr” is less than “dlPduSesWeightedAmbr” [End PR794626]

 Perform the steps defined in 5G_UP_ALG_26990_replaced([11228183](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kygjzvvSEe-AqvopbP1qhQ)).

In a case when dlPduSesAvgInBitRate for [Before PR794626] all the [PR794626] one or more [END PR794626] PDU sessions of the UE is greater than [PR794626] (Note 4) [End PR794626] [Before PR794626] or equal to [End PR794626] “dlPduSesDynAmbr”, AND If “dlPduSesDynAmbr” is less than “dlPduSesWeightedAmbr” for one or more of the PDU sessions of the UE with dlPduSesAvgInBitRate greater than (Note 4) “dlPduSesDynAmbr” (Rationale 2).

OR

In case there is one or more PDU sessions which has “dlPduSesDynAmbr” [PR794626] is less than or [End PR794626] equal to minDlPduSesAmbr(5G_UP_ALG_27189_replaced([11228052](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kyf8u_vSEe-AqvopbP1qhQ))) AND dlPduSesAvgInBitRate is greater than 0 for those PDU sessions (refer Rationale),

 The AMBR limit for all the PDU Sessions shall be reset to the AMBR limit based on scheduling weight:

dlPduSesDynAmbr = dlPduSesWeightedAmbr.

 Restart the monitoring for dlAmbrSampInterval and dlAmbrMonitorPeriod (Refer Note 3).

 dlPduSesDynAmbr is then distributed to the non-GBR DRBs of the respective PDU session as defined in 5G_UP_ALG_19365_replaced([11228072](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kyf8vvvSEe-AqvopbP1qhQ)).

In a case when above conditions does not change the "dlPduSesDynAmbr" of a PDU session, execute the dynamic DL DRB AMBR calculation as defined in 5G_UP_ALG_19365_replaced([11228072](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kyf8vvvSEe-AqvopbP1qhQ)) for the non-GBR DRBs of the corresponding PDU session.

Rationale:

1\. In case when the non-GBR DRB's dlDrbDynAmbr is set to rdDlAmbrMinLimit or lower for all the bearers of the PDU session (minDlPduSesAmbr) due to lack of input rate, and the PDU session dynamic AMBR limit was reduced to minDlPduSesAmbr, dlDrbDynAmbr must be updated when the input data starts arriving for such non-GBR bearers of the PDU session.

2\. When the “dlPduSesDynAmbr” of a PDU session is reduced to match its dlPduSesAvgInBitRate, Due to the slight variations in the average computation the dlPduSesAvgInBitRate might seem marginally greater than the “dlPduSesDynAmbr” – and such computational variance should not trigger “dlPduSesDynAmbr” redistribution among the non-GBR DRBs of the PDU session.

Note:

1\. dlPduSesAvgInBitRate, dlPduSesWeightedAmbr, dlPduSesDynAmbr, dlDrbDynAmbr, dlDrbAvgInBitRate are defined in 5G_UP_ALG_19313_replaced([11227904](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kyfVpfvSEe-AqvopbP1qhQ)).

2\. rdDlAmbrMinLimit is a RAD parameter defined in 5G_UP_ALG_27181_replaced([11228316](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kyhx4_vSEe-AqvopbP1qhQ)).

3\. NRBTS/dlAmbrSampInterval, NRBTS/dlAmbrMonitorPeriod are NIDD parameters received from C-plane.

### 4.26.1-3.0-1 Common Low input DL bitrate triggered re-distribution of the DL DRB AMBR limit (ID: 11228114)

**4.26.1-3.0-1.0-1**  (ID: `11228121`)

Common In case when the total dlPduSesDynAmbr is not achieved due to one or more non-GBR DRBs not having enough input DL data, the dlPduSesDynAmbr is re-distributed to the non-GBR DRBs (not directly aligned based on scheduling weights of the bearers) of the PDU session as below:

 For the non-GBR DRBs of a PDU Session which has dlDrbAvgInBitRate greater than dlDrbDynAmbr perform the following actions:

 Initialize the total reduction in AMBR limit (totalRequiredDlAmbr) to 0 (refer Note 3).

 For each of the non-GBR DRB of the PDU session, calculate the total amount of increase in dlDrbDynAmbr required for the non-GBR bearers of the PDU session (totalRequiredDlAmbr) as defined in 5G_UP_ALG_19744_replaced([11228128](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kygjxvvSEe-AqvopbP1qhQ)).

 For the non-GBR DRBs of a PDU session which has dlDrbAvgInBitRate less than dlDrbDynAmbr perform the following actions:

 Initialize the total recovered AMBR limit (tmpTotalRecoveredAmbr) to 0 (refer Note 2).

 Execute the below steps in iteration once for each of the non-GBR DRB of the PDU session in the order of scheduling weights starting with the lowest priority to highest priority.

If tmpTotalRecoveredAmbr < totalRequiredDlAmbr (Rationale),

1\. In case when there is no input DL data received for the non-GBR DRB:

 Decrease the dlDrbDynAmbr and update the tmpTotalRecoveredAmbr as defined in 5G_UP_ALG_19332_replaced([11228140](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kygjyPvSEe-AqvopbP1qhQ)).

2\. In case there is input DL data received for the non-GBR DRB:

 Decrease the dlDrbDynAmbr and update the tmpTotalRecoveredAmbr as defined in 5G_UP_ALG_19334_replaced([11228156](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kygjyvvSEe-AqvopbP1qhQ)).

 Distribute the recovered AMBR for each of the non-GBR DRBs which has dlDrbAvgInBitRate greater than dlDrbDynAmbr of the UE, as defined in 5G_UP_ALG_19336_replaced([11228170](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kygjzPvSEe-AqvopbP1qhQ)).

Rationale:

If the required amount of AMBR is recovered from the non-GBR bearers of the UE, there is no need to reduce the AMBR limit for the other non-GBR DRBs.

Note:

1\. dlDrbDynAmbr, dlDrbAvgInBitRate are defined in 5G_UP_ALG_19313_replaced([11227904](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kyfVpfvSEe-AqvopbP1qhQ)).

2\. tmpTotalRecoveredAmbr is updated in 5G_UP_ALG_19332_replaced([11228140](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kygjyPvSEe-AqvopbP1qhQ)) and 5G_UP_ALG_19334_replaced([11228156](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kygjyvvSEe-AqvopbP1qhQ)), and it is used in 5G_UP_ALG_19336_replaced([11228170](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kygjzPvSEe-AqvopbP1qhQ)).

3\. totalRequiredDlAmbr is computed in 5G_UP_ALG_19744_replaced([11228128](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kygjxvvSEe-AqvopbP1qhQ)).

### 4.26.1-3.0-2 Common Calculate the amount of increase in dynamic AMBR for the non GBR DRBs (ID: 11228128)

**4.26.1-3.0-2.0-1**  (ID: `11228134`)

Common  Calculate the amount of increase in AMBR limit required for the non-GBR DRB based on the average DL input bit rate.

dlDrbDeltaIncAmbr = dlDrbAvgInBitRate – dlDrbDynAmbr.

 Update the total increment in DL AMBR required across all the non-GBR DRBs of the UE

totalRequiredDlAmbr = totalRequiredDlAmbr + dlDrbDeltaIncAmbr

### 4.26.1-3.0-3 Common Reduction in DL DRB dynamic AMBR due to zero input bitrate (ID: 11228140)

**4.26.1-3.0-3.0-1**  (ID: `11228147`)

Common  If there is no DL input data received for the non-GBR DRB for the last "dlAmbrZeroRateMonitorDuration" period, AND

non-GBR DRB buffer is empty (Note 9) AND

the corresponding dlDrbDynAmbr is greater than rdDlAmbrMinLimit,

  * Calculate amount of AMBR reduction possible:



tmpDrbAmbrReduc = dlDrbDynAmbr – rdDlAmbrMinLimit.

  * If the required AMBR is higher than or equal to AMBR recoverable from the non-GBR DRB,



[IF (tmpTotalRecoveredAmbr + tmpDrbAmbrReduc) <= totalRequiredDlAmbr]

  * update the total reduction in AMBR limit for the corresponding PDU session (tmpTotalRecoveredAmbr) as below:



tmpTotalRecoveredAmbr = tmpTotalRecoveredAmbr + tmpDrbAmbrReduc.

  * Decrease the dlDrbDynAmbr for the non-GBR DRB to the minimum AMBR for DRB,



dlDrbDynAmbr = rdDlAmbrMinLimit.

 Skip the below steps.

  * If the required AMBR is less than AMBR recoverable from the non-GBR DRB,



[IF (tmpTotalRecoveredAmbr + tmpDrbAmbrReduc) > totalRequiredDlAmbr]

  * Calculate the required reduction in AMBR for the non-GBR DRB as below:



drbRequiredAmbrReduct = (totalRequiredDlAmbr – tmpTotalRecoveredAmbr)

  * Decrease the dlDrbDynAmbr for the non-GBR DRB based on the required AMBR for the other non-GBR DRBs of the PDU session:



dlDrbDynAmbr = dlDrbDynAmbr - drbRequiredAmbrReduct.

  * Update the total reduction in AMBR limit for the corresponding UE (tmpTotalRecoveredAmbr) as below:



tmpTotalRecoveredAmbr= tmpTotalRecoveredAmbr+ drbRequiredAmbrReduct

In case the DL DRB dynamic AMBR reduction is initiated due to PDU session AMBR distribution (5G_UP_ALG_26990_replaced([11228183](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kygjzvvSEe-AqvopbP1qhQ))) totalRequiredDlAmbr = totalRequiredDlPduSesAmbr and tmpTotalRecoveredPduSessAmbr = tmpTotalRecoveredAmbr.

Note:

1\. NRBTS/dlAmbrZeroRateMonitorDuration is a NIDD parameter received from C-plane.

2\. NRBTS/minDlDrbAmbr is configured as a percentage of DL UE AMBR limit.

3\. tmpTotalRecoveredAmbr is initialized to 0 at the beginning of dynamic DL AMBR calculation for the non-GBR DRBs for each UE as defined in 5G_UP_ALG_19330_replaced([11228114](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kygjxPvSEe-AqvopbP1qhQ)).

4\. tmpTotalRecoveredAmbr is computed in 5G_UP_ALG_19332_replaced([11228140](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kygjyPvSEe-AqvopbP1qhQ)) and 5G_UP_ALG_19334_replaced([11228156](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kygjyvvSEe-AqvopbP1qhQ)), and it is used in 5G_UP_ALG_19336_replaced([11228170](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kygjzPvSEe-AqvopbP1qhQ)).

5\. totalRequiredDlAmbr is computed in 5G_UP_ALG_19744_replaced([11228128](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kygjxvvSEe-AqvopbP1qhQ)).

6\. totalRequiredDlPduSesAmbr is computed in 5G_UP_ALG_26992_replaced([11228196](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kyhK0PvSEe-AqvopbP1qhQ)).

7\. tmpTotalRecoveredPduSessAmbr is used in 5G_UP_ALG_26994_replaced([11228211](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kyhK0vvSEe-AqvopbP1qhQ)).

8\. rdDlAmbrMinLimit is a RAD parameter defined in 5G_UP_ALG_27181_replaced([11228316](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kyhx4_vSEe-AqvopbP1qhQ)).

### 4.26.1-3.0-4 Common Reduction in DL DRB dynamic AMBR due to low input bitrate (ID: 11228156)

**4.26.1-3.0-4.0-1**  (ID: `11228162`)

Common If there is DL input data received for the non-GBR DRB in the last dlAmbrZeroRateMonitorDuration period, Calculate and reduce the dynamic DL DRB AMBR limit gradually as below:

 Calculate the delta difference between the AMBR limit of the non-GBR DRB “dlDrbDynAmbr” and the dlDrbAvgInBitRate.

deltaDiffAmbrVsInputRate = dlDrbDynAmbr - dlDrbAvgInBitRate

 Calculate the amount of reduction in AMBR limit for the bearer as a percentage of rdDlAmbrReductMargin.

deltaReductAmbr = MIN (deltaDiffAmbrVsInputRate * rdDlAmbrReductMargin, totalRecoverableDlPduSesAmbrOfPdusession)

  * If the required AMBR is higher than or equal to AMBR recoverable from the non-GBR DRB,



[IF (tmpTotalRecoveredAmbr + deltaReductAmbr) <= totalRequiredDlAmbr]

 Update the dynamic DL DRB AMBR limit with the reduced AMBR limit

dlDrbDynAmbr = dlDrbDynAmbr – deltaReductAmbr

  * update the total reduction in AMBR limit for the corresponding PDU session (tmpTotalRecoveredAmbr).



tmpTotalRecoveredAmbr = tmpTotalRecoveredAmbr + deltaReductAmbr.

 Skip the below steps.

  * If the required AMBR is less than AMBR recoverable from the non-GBR DRB,



[IF (tmpTotalRecoveredAmbr + deltaReductAmbr) > totalRequiredDlAmbr]

  * Calculate the required reduction in AMBR for the non-GBR DRB as below:



drbRequiredAmbrReduct = (totalRequiredDlAmbr – tmpTotalRecoveredAmbr)

 Reduce the dynamic DL DRB AMBR limit of the non-GBR DRB based on the required AMBR limit to be recovered

dlDrbDynAmbr = dlDrbDynAmbr – drbRequiredAmbrReduct

  * update the total reduction in AMBR limit for the corresponding PDU session (tmpTotalRecoveredAmbr).



tmpTotalRecoveredAmbr= tmpTotalRecoveredAmbr+ drbRequiredAmbrReduct

In case the DL DRB dynamic AMBR reduction is initiated due to PDU session AMBR distribution (5G_UP_ALG_26994_replaced([11228211](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kyhK0vvSEe-AqvopbP1qhQ))), totalRequiredDlAmbr = totalRequiredDlPduSesAmbr and tmpTotalRecoveredPduSessAmbr = tmpTotalRecoveredAmbr.

Note:

 dlDrbWeightedAmbr, dlDrbDynAmbr, dlDrbAvgInBitRate are defined in 5G_UP_ALG_19313_replaced([11227904](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kyfVpfvSEe-AqvopbP1qhQ)).

### 4.26.1.0-14 Common dlDrbAvgOutBitRate (ID: 11228027)

**4.26.1.0-14.0-2**  (ID: `13260836`)

Common The average DL data rate (in bits per second) at the PDCP entity for the non-GBR DRB shall be updated as below,

This metric is calculated and updated for every _outRateUpdatePeriod (Rationale 2) or when a non-GBR bearer is added /deleted for the lifetime of the non-GBR DRB._

The average DL bit rate per second is calculated as a sliding average (updated every _outRateUpdatePeriod ) as defined below:_

 Calculate the current output bitrate as bits per second for the bit rate sampling interval (dlAmbrSampInterval),

  * dlDrbAvgOutBitRate =



(Total amount of data sent (in bits) from gNB-CU-UP(PDCP) excluding the PDCP PDU retransmissions (refer Rationale) in the last dlAmbrSampInterval duration) * 1000/ dlAmbrSampInterval.

The _outRateUpdatePeriod is computed as:_

**= dlAmbrSampInterval / rdNumOfDataRecords**

[CB012554-A]

Same method shall also be used to calculate average DL throughput for each nonGBR bearer reusing NIDD parameter dlAmbrSampInterval.

[End CB012554-A]

Note:

1\. NRBTS/dlAmbrSampInterval is a NIDD parameter received from C-plane. 

2\. "rdNumOfDataRecords" is a RAD parameter defined in 5G_UP_ALG_27302_replaced([11228344](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kyhx5_vSEe-AqvopbP1qhQ)).

3\. It is up to implementation how the metric (dlDrbAvgOutBitRate) update is carried out at each update period (Per PDU or Per update period) to ensure that the overall data sent out for the bearer does not exceed the dlDrbDynAmbr.

Rationale:

1\. When the PDCP PDU is reported as lost over F1-U and gets re-transmitted, these retransmissions are not accounted towards the calculation of dlDrbAvgOutBitRate (as such PDUs are already accounted as transmitted once for the original transmission).

2\. The update of the “dlDrbAvgOutBitRate” due to data sending between two consecutive _outRateUpdatePeriod’s is left to the implementation and is defined in more detail in EFS._

### 4.26.1.0-16 Common minDlPduSesAmbr (ID: 11228052)

**4.26.1.0-16.0-1**  (ID: `11228059`)

Common The minimum possible assigned AMBR limit of the PDU session.

The minimum possible AMBR limit for a PDU session is computed as:

minDlPduSesAmbr = Sum of "min(rdDlAmbrMinLimit,dlDrbDynAmbr)" of all the non-GBR DRBs of the PDU session.

Note:

1\. rdDlAmbrMinLimit is a RAD parameter defined in 5G_UP_ALG_27181_replaced([11228316](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kyhx4_vSEe-AqvopbP1qhQ)).
