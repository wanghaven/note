# 4.33 Supported Deployments

## 4.33.0-2 Supported Deployments 5G-L2-HI shall support the following deployments. (ID: 11228915)

**4.33.0-2.0-3**  (ID: `11228939`)

Supported Deployments Data delivery between PDCP (5G-L2-HI in CU) and RLC (5G-L2-HI in DU) is handled with F1-U interface.

5G-L2-HI (CU) processes only PDCP and uses F1-U interface if it receives non-zero F1-U parameters.remoteIpAddress from CP at Bearer Setup.

5G-L2-HI (DU) processes only RLC and uses F1-U interface if it receives non-zero F1-U parameters.remoteIpAddress from CP in Bearer Setup Request for DU.

Otherwise 5G-L2-HI processes both PDCP/RLC and handles data delivery between PDCP/RLC internally. In such case CP shall provide equal bearer index in both Bearer Setup Request for CU and Bearer Setup Request for DU.

[CB008717-E]

In case of NR DC Split Bearers, bearer index allocation for 5G-L2-HI CU and the 5G-L2-HI DU (SCG) can be different. In this case, the TEID can be used for unique identification.

[End CB008717-E]

Rationale:

Support for co-located PDCP/RLC with optimized internal interface (PDCP/RLC in CU) is needed in SA call demos and in future with traditional BTS feature.

Support for PDCP in CU and RLC in DU with F1-U interface is needed in NSA call.
