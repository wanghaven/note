## 4.25.2 Downlink Data Transmission on Target based on PDCP Status Report

### 4.25.2.0-1 Downlink Data Transmission on Target based on PDCP Status Report Overview (ID: 11226724)

**4.25.2.0-1.0-1**  (ID: `11226733`)

Downlink Data Transmission on Target based on PDCP Status Report During UE mobility, gNB can request feedback on the reception of PDCP PDUs via the RRC Reconfiguration. The PDCP Status Report from the UE can help gNB to retransmit the unacknowledged PDCP PDUs on the target cell.

Prior to this solution (i.e, CNI-86916), 5G-L2-HI (PDCP) initiates the DL PDU transmission towards DL RLC on successful reception of the first UL GTP-U PDU. The last received DDDS information from source is used to determine the PDCP PDUs to be transmitted on the target cell. Although, this allows for faster resumption of DL transmission towards UE post handover, it can result in unnecessary duplicate transfer of PDCP PDUs towards UE, resulting in longer data interruption for the UE.

5G-L2-HI (PDCP) can initiate DL transmission towards RLC after successful reception of DL PDCP Status report to accurately determine the PDCP PDUs to be retransmitted thus avoiding data interruption for the UE. The solution is applicable for all mobility scenarios for SA and NSA bearers.

### 4.25.2.0-2 Downlink Data Transmission on Target based on PDCP Status Report Configuration (ID: 11226739)

**4.25.2.0-2.0-1**  (ID: `11226745`)

Downlink Data Transmission on Target based on PDCP Status Report The PDCP Status report based DL transmission functionality is configured via the following NIDD parameters:

  * NRBTS.NRDRB.actDlTxResumeOnPdcpStatRep: To activate functionality related to waiting for PDCP status report before resuming DL data transfer at target
  * NRBTS.NRDRB.pdcpStatRepWaitTimer: To configure a wait timer for the reception of PDCP Status Report
  * NRBTS.NRDRB.pdcpStatRepWaitTimerOffset: To configure an additional offset for the wait timer for PS-Cell / PCell change scenario.



NOTE: The above NIDD parameters shall be configured to 5G-L2-HI during the appropriate bearer procedures.

3) The last received F1-U DDDS provides the information about

a. The SN of the last in-order PDCP PDU which have been successfully received at the UE.

b. The SN of the last PDCP PDU which have been sent to the UE. 

The PDCP PDU(s) between a) and b) are not yet acked by the UE and could be retransmitted over the target F1-U tunnel.

**4.25.9.0-3.0-4.0-1**  (ID: `13136256`)

SA Inter RAT HO The gNB-CU-UP (PDCP) is responsible for evaluating the monitored DL UE throughput aggregated over all PDU sessions and comparing against the configured threshold (refer Parameters 1) and reporting status changes to the gNB-CU-UP(CP-NRT).

The gNB-CU-UP (PDCP) shall send an indication to the gNB-CU-UP (CP-NRT) when there is a change in throughput status as given below:

  * Low Throughput indication: When gNB-CU-UP (PDCP) detects that the average DL UE throughput is below the threshold (refer Parameters 1), it shall start a timer for a duration defined by rdLowTputHysteresisPeriodMSec (refer Parameters 2) and if the average DL UE throughput is lower than the threshold for the entire duration of the timer, gNB-CU-UP (PDCP) shall send Low throughput indication to the gNB-CU-UP (CP-NRT). This notification is the primary trigger for the gNB-CU-UP (CP-NRT) to proceed with the next steps of the Inter-RAT HO procedure.
  * High Throughput indication: If low throughput indication has been sent to gNB-CU-UP (CP-NRT) and subsequently if the average DL UE throughput increases above the threshold (refer Parameters 1), the gNB-CU-UP (PDCP) shall send the high throughput indication to gNB-CU-UP (CP-NRT). This informs the gNB-CU-UP (CP-NRT) that the performance has recovered.



Parameters:

  1. thresholdDlUeTputIratHo is NIDD parameter which is the throughput threshold configured by CPLANE.



2\. rdLowTputHysteresisPeriodMSec is RAD parameter defined in [13253846: rdLowTputHysteresisPeriodMSec](https://dn-prod.ext.net.nokia.com/rm/resources/BI_9UD4QcbIEfCYBIz6acpPHw?oslc_config.context=https://dn-prod.ext.net.nokia.com/gc/configuration/9)
