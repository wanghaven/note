# 4.14 RLC STATUS Polling Procedure

## 4.14.0-2 RLC STATUS Polling Procedure Setting Poll bit and Poll parameter (ID: 11225257)

**4.14.0-2.0-2**  (ID: `11225268`)

RLC STATUS Polling Procedure 5G-L2-HI shall determine the need for requesting RLC status (I.e. setting the poll -flag) based on RLC rules (parameters PDU_WITHOUT_POLL, BYTE_WITHOUT_POLL, t-PollRetransmit or transmitting the last data in buffer or transmission window stalling). If the rules indicate a status request needs to be triggered, 5G-L2-HI shall set a poll -flag according to following options:

  1. If there is RLC PDU transmission taking place (new or retransmission), then 5G-L2-HI shall set the poll -flag of that RLC PDU (for one PDU).

  2. If there is no RLC PDU transmission taking place (e.g. new RLC PDU cannot be transmitted due transmission window stalling or due no data to be transmitted, new/retransmission), 5G-L2-HI shall select the RLC PDU with sequence number TX_Next -1 and retransmit it with poll -flag set.

  3. If there is no RLC PDU transmission taking place (e.g. new RLC PDU cannot be transmitted due transmission window stalling or due no data to be transmitted, new/retransmission), 5G-L2-HI shall consider highest RLC SN submitted to lower layer which has not been acknowledged(either not acked or has been nacked) and retransmit it with poll -flag set.




3.For DRB, 5G-L2-HI shall update the poll parameters accordingly (PDU_WITHOUT_POLL and BYTE_WITHOUT_POLL), but shall not start the t-PollRetransmit timer and not update the POLL_SN

For SRB, 5G-L2-HI shall update the poll parameters accordingly (PDU_WITHOUT_POLL and BYTE_WITHOUT_POLL), but shall not start the t-PollRetransmit timer and not update the POLL_SN. [PR795277]In addition, when an SRB PDU is marked as final, the 5G-L2-Hi shall stop any DL polling timer that is already running and shall not start another DL polling timer for that SRB.[End PR795277]

  1. 5G-L2-HI shall increase the retransmission counter of that RLC PDU.

  2. If RLC PDU cannot be transmitted due flow control limitation, 5G-L2-HI keep poll parameters at current values (I.e. not reset them) and wait for next possible transmission possibility.




Rationale:

3GPP is not clear on how to handle retransmission counter if the PDU is retransmitted only to be a 'transporter' for poll-flag (step 2 above). It was decided to increase the corresponding retransmission counter to have some control on how long the situation can continue. Without increasing the counter it is possible that step 2 situation can continue forever without higher layers getting any indication about it. I.e. t-PollRetransmit keeps getting timeouts, PDU is transmitted to carry poll-flag, no status reports and t-PollRetransmit gets again timeout. Rinse and repeat.

POLL_SN update and t-PollRetransmit timer is started based on feedback from 5G-L2-LO on the transmission status of the RLC PDU with poll bit.

When UE send RLC STATUS with ACK_SN = TX_NEXT, RLC PDU wiht SN = TX_NEXT - 1 would be released. Hence L2-HI choose the highest RLC SN submitted to lower layer which has not been acknowledged for retransmission.
