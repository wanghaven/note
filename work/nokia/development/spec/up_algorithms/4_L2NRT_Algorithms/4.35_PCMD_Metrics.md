**4.35.0-2.0-1.0-1.0-2**  (ID: `11229524`)

PCMD Metrics **[Before CB013789-E] If NRBTS.actIpThpt set to TRUE**[CB013789-E] When requested by the PCMD Tracing Framework **[End CB013789-E], gNB-CU-UP shall collect the following DL PCMD metrics related to volume and time of data bursts for the DL UE throughput.** ****

[Before CNI-134962-C]

  * Downlink PDCP SDU Tx Burst Bytes (refer 5G_UP_ALG_19873_replaced([11229531](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kyofm_vSEe-AqvopbP1qhQ)))
  * Downlink Tx Burst Duration (refer 5G_UP_ALG_19874_replaced([11229544](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kyofnfvSEe-AqvopbP1qhQ)))



The start of a burst is the point in time the first portion of the data is sent over air to UE, i.e. the burst time shall exclude the initial scheduling / retention delay.

The end of burst is identified when last portion of the data sent over air to UE which empties the buffer for the given DRB.

Small volume and short session bursts are excluded from the counting. After end of each burst if the Volume or Time count value is less than the corresponding thresholds given below then the volume and time count value is not added to increment the corresponding measurement.

_NRBTS.NRDRB.volThresDiscardDl - data burst volume threshold to discard the counting results._

_NRBTS.NRDRB.durationThresDiscardDl - data burst duration threshold to discard the counting results._

[CNI-134962-C]

Burst handling is as described in [12726202: Burst handling](https://dn-prod.ext.net.nokia.com/rm/resources/BI_WXPUYXD_EfCRE7eh4Ndkrw?oslc_config.context=https://dn-prod.ext.net.nokia.com/gc/configuration/9)

Volume trace field description for the same is given in 5G_TC_2874_replaced([12455939](https://dn-prod.ext.net.nokia.com/rm/resources/BI_FLJO5wpAEfCT35inaBGo7g?oslc_config.context=https%3A%2F%2Fdn-prod.ext.net.nokia.com%2Fgc%2Fconfiguration%2F9)) and shall be counted as specified in [12726199: Volume counter](https://dn-prod.ext.net.nokia.com/rm/resources/BI_Pk_kgXD_EfCRE7eh4Ndkrw?oslc_config.context=https://dn-prod.ext.net.nokia.com/gc/configuration/9)

Time (Burst duration) trace field description for the same is given in 5G_TC_2875_replaced([12454834](https://dn-prod.ext.net.nokia.com/rm/resources/BI_FLE9mQpAEfCT35inaBGo7g?oslc_config.context=https%3A%2F%2Fdn-prod.ext.net.nokia.com%2Fgc%2Fconfiguration%2F9)) and shall be counted as specified in [12726201: Time counter](https://dn-prod.ext.net.nokia.com/rm/resources/BI_TIrGonD_EfCRE7eh4Ndkrw?oslc_config.context=https://dn-prod.ext.net.nokia.com/gc/configuration/9)

[End CNI-134962-C]

_NOTE:_

1) The measurement is combined for both LTE and 5G in case of NSA.

**4.35.0-2.0-1.0-3**  (ID: `11229544`)

PCMD Metrics Downlink Tx Burst Duration

---

**4.35.0-2.0-1.0-3.0-1**  (ID: `11229551`)

PCMD Metrics The time taken for each downlink PDCP SDU level data burst (refer _NOTE 1) to be transmitted to gNB-DU, is accumulated over the measurement period for the bearer as indicated in 5G_TC_2254_replaced ([12455983](https://dn-prod.ext.net.nokia.com/rm/resources/BI_FLAsFQpAEfCT35inaBGo7g?oslc_config.context=https%3A%2F%2Fdn-prod.ext.net.nokia.com%2Fgc%2Fconfiguration%2F9))._

The estimation of the time for the transmission of a burst is based on 3GPP based input information of DDDS message (last recevied), such as Desired Data Rate (DDR). In case the DDR is not received in the DDDS message, 5G-L2-HI can use the instantaneous estimated throughput based on the data outflow and the duration between DDDS messages, as DDR.

Time needed to transmit (ms) = Total size of PDCP PDUs of the data burst (Kbits) / Desired Data Rate (Kbits / ms)

_NOTE:_

1) Only new PDCP SDUs shall be considered (i.e. **PDCP retransmitted PDUs/ PDCP PDUs marked for retransmission are excluded)**.****

**4.35.0-3.0-2.0-1.0-2**  (ID: `11229599`)

PCMD Metrics **[Before CB013789-E] If _NRBTS.actIpThpt set to TRUE**[CB013789-E] When requested by the PCMD Tracing Framework**[End CB013789-E], 5G-L2-HI shall collect the following metrics of the volume and time for data bursts for the computation of average UL UE throughput._******

[Before CNI-134962-C]

  * Uplink RLC SDU Rx Burst Bytes [5G_UP_ALG_19899_replaced([11229606](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kypGp_vSEe-AqvopbP1qhQ))]

The UL data burst is defined with the start of the sending of first RLC PDU after the reception a packet with BSR>0 is received by 5G-L2-HI and ends with when a packet with BSR=0 is received.

Small volume and short session bursts are excluded from the counting. After end of each burst if the Volume or Time count value is less than the corresponding thresholds given below then the volume and time count value is not added to increment the corresponding measurement.

_NRBTS.NRDRB.volThresDiscardUl - data burst volume threshold to discard the counting results._

_NRBTS.NRDRB.durationThresDiscardUl - data burst duration threshold to discard the counting results._

[CNI-134962-C]

Burst handling shall be handled as described in [12727309: Burst handling](https://dn-prod.ext.net.nokia.com/rm/resources/BI_VTEXcHFIEfCRE7eh4Ndkrw?oslc_config.context=https://dn-prod.ext.net.nokia.com/gc/configuration/9)
