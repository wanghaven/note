## 4.32.1 Counting Secondary RAT DATA Usage

**4.32.1.0-1**  (ID: `11228884`)

Counting Secondary RAT DATA Usage
If activated in the request, 5G-L2-HI shall count the number of bytes of extracted user data packet from uplink RLC SDUs received at PDCP level over F1-U and count the number of bytes of downlink PDCP SDUs transmitted at PDCP level when a PDCP PDU is sent over F1-U.

The start time is initialized with the current time when the functionality is enabled.

The UL data Usage is initialized with zero and increments with the number of bytes of extracted user data packet from uplink RLC SDUs received at PDCP level over F1-U when the functionality is enabled.  It is equal to the number of bytes of UL PDCP SDUs which are successfully sent to TRSW towards S1U.

The DL Data Usage initialized with zero and increments with the number of bytes of DL PDCP SDUs transmitted at PDCP level every time a PDCP PDU is sent over F1-U when the functionality is enabled.  It is equal to the number of bytes of DL PDCP SDUs received by 5G-L2-HI(PDCP layer) when the corresponding PDCP SNs are less or equal to highest delivered NR PDCP SN provided in the F1-U DDDS message.

Both RLC mode AM and UM are supported.

Note 1: During the suspension 5G-L2-HI shall not reset the values of these counters and shall continue counting.

Note 2: The extracted user data packet from uplink RLC SDUs to be counted for the uplink are the PDCP SDUs which are ready to be sent to the higher layer.

## 4.32.2 Reporting Secondary RAT Data Usage

**4.32.2.0-1**  (ID: `11228899`)

Reporting Secondary RAT Data Usage
5G-L2-HI shall do the following when it receives request for reporting the Secondary RAT Data Usage

1.   Include Data Usage Report in the response message with following details:

a.   Data Usage Start time

b.   Data Usage End time as the current time

c.   UL Data usage

d.   DL Data usage

2.   Reset the start time to current time

3.   Set the values of UL and DL data usage to 0.
