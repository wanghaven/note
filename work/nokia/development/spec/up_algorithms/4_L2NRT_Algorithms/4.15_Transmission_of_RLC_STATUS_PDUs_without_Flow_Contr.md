# 4.15 Transmission of RLC STATUS PDUs without Flow Control

## 4.15.0-2 Transmission of RLC STATUS PDUs without Flow Control Transmission of RLC Status PDUs (ID: 11225288)

**4.15.0-2.0-1**  (ID: `11225294`)

Transmission of RLC STATUS PDUs without Flow Control
When RLC STATUS PDU transmission is triggered

1) Due to reception of Poll bit = 1 in the RLC PDU received from UE OR

2) Due to t-reassembly expiration

and the transmission is allowed, 5G-L2-HI shall generate the RLC STATUS PDU according to 5G\_UP\_IF\_3310\_replaced([11529309](https://dn-prod.ext.net.nokia.com/rm/resources/BI_tWIcz_v1Ee-AqvopbP1qhQ))

For the t-reassembly exipry case, he RLC STATUS PDU is allowed to be sent when

* t-StatusProhibit is not running.

For poll triggered case the transmission is allowed when

* t-StatusProhibit is not running AND
* The sequence number(SN) of received RLC PDU fulfils rule [SN < RX\_Highest\_Status or PDU SN>= RX\_Next + AM\_WINDOW\_SIZE].

The status PDU shall indicate the failed PDUs and segments and the sequence number up to which everything else is correctly receive. See 5G\_UP\_IF\_3310\_replaced([11529309](https://dn-prod.ext.net.nokia.com/rm/resources/BI_tWIcz_v1Ee-AqvopbP1qhQ)) for status PDU construction. The maximum size for status PDU shall be fixed to 1864 bits.

5G-L2-HI shall send the RLC STATUS PDU directly to 5G-L2-LO regardless of flow control or any RLC data PDU transmission.

5G-L2-HI shall select 5G-L2-LO according to 5G\_UP\_ALG\_5740\_replaced([11221207](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kx0APvvSEe-AqvopbP1qhQ)), in case there is more than one instance configured.

Upon status PDU transmission,

* 5G-L2-HI shall (re-)start t-StatusProhibit timer.
* Any Pending RLC status PDU transmissions shall be cleared.

If the STATUS PDU transmission is not allowed,

* 5G-L2-HI shall keep the RLC STATUS PDU transmission as pending and it shall generate & send it in the first possible occasion when the criteria allow transmission.

Rationale:

5G-L2-HI is not aware of the actual transport block size determined by packet scheduler so a fixed maximum limit is defined. Status PDU max size is defined as number of bits on MCS 0 per 100 PRBs on one component carrier. This is considered the minimum allocation from a scheduler.

STATUS PDU transmission for DRB shall be supported.

STATUS PDU transmission for SRB shall be supported.
