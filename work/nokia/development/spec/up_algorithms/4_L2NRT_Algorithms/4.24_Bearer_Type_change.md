# 4.24 Bearer Type change

## 4.24.0-2 Bearer Type change NR DC Glossary (ID: 11226009)

**4.24.0-2.0-2**  (ID: `11226023`)

Bearer Type change
[Before CB008490-D]

* For NR-NR DC, the following terminology is applicable:
* F1-U reference in all L2 NRT requirements refer to the Master Cell Group (MCG) F1-U.
* Secondary Cell Group (SCG) F1-U is specific to NR-DC configuration where the SCG F1-U is the secondary link.
* NIDD Parameters for Flow Control:

All the NIDD parameters for Flow Control applicable for EN-DC are extended to NR DC as well. Specifically X2 related parameters are applicable for the SCG F1-U, where it is not applicable has been explicitly stated.

* The flow control parameter names for secondary link are generalized with the Xx nomenclature. In case of NR-NR DC, Xx refers to the SCG F1-U.

[CB008490-D]

For NR DC bearers, the following terminology is applicable based on the type of the NR DC:

* MN Terminated Intra-gNB NR DC
* F1-U reference in all L2 NRT requirements refer to the Master Cell Group (MCG) F1-U. The MCG F1-U is the default Primary link.
* Secondary Cell Group (SCG) F1-U is specific to NR-DC configuration where the SCG F1-U is the secondary link. The flow control parameter names for secondary link are generalized with the Xx nomenclature.
* SN Terminated Inter-gNB NR DC
* F1-U reference in all L2 NRT requirements refer to the Secondary Cell Group (SCG) F1-U. The SCG F1-U is the default Primary link.
* Master Cell Group (MCG) Xn-U is the default Secondary link. The flow control parameter names for secondary link are generalized with the Xx nomenclature.
* NIDD Parameters for Flow Control

All the NIDD parameters for Flow Control applicable for EN-DC are extended to NR DC as well. Specifically X2 related parameters are generalized and made applicable for the SCG F1-U / MCG Xn-U.

[End CB008490-D]

## 4.24.0-5 Bearer Type change Intra gNB NR DC Split Bearer type setup (ID: 11226402)

**4.24.0-5.0-3**  (ID: `11226422`)

Bearer Type change
In the case of Bearer Type change from SA Bearer to NR DC Split Bearer,

Downlink:

* Upon reception of Bearer Modification with SCG F1-U information:

- IF DL SCG F1-U TEID is provided:

o 5G-L2-HI shall store the DL F1-U TEID

-    Flow Control Parameter reinitialization:

[Before CB008717-DA]

* In case when dlDataSplitMode = dlOverF1U , set initDataTrafficRouting = initDLoverF1U
* In case when dlDataSplitMode = dlOverX2U , set initDataTrafficRouting = initDLoverX2U

[CB008717-DA]

* In case when dlDataSplitModeNrDc = dlOverF1U , set initialDLTrafficRoutingNrDc = initDLoverF1U
* In case when dlDataSplitModeNrDc = dlOverXxU , set initialDLTrafficRoutingNrDc = initDLoverXxU

[End CB008717-DA]

* DL DATA Transmission:

[Before CB008717-DA]

* IF dlDataSplitMode = dlOverF1UXxU and initDLDataTrafficRouting is enabled (set to dlOverF1U or dlOverX2U)

[CB008717-DA]

* IF dlDataSplitModeNrDc = dlOverF1UXxU and initialDLTrafficRoutingNrDc is enabled (set to dlOverF1U or dlOverXxU)

[End CB008717-DA]

* If DL data transmission (refer Note) has not started before Bearer Type change, the initial DL Traffic Routing shall be enabled and routing applied as per the configured value of initDLDataTrafficRouting
* If DL data transmission (refer Note) was ongoing before Bearer Type change, the initial DL traffic Routing shall be disabled, set initDLDataTrafficRoutingAct to FALSE

NOTE: DL Data transmission here refers to atleast one PDCP PDU sent over the MCG F1-U prior to bearer type change.

* Set xxAvailable = FALSE until the reception of first UL GTP-U PDU. (See Rationale)
* Execute the steps defined in 5G\_UP\_ALG\_5599\_replaced([11219850](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kxsrfPvSEe-AqvopbP1qhQ))

[CB008717-DA]

NOTE:

1. dlDataSplitModeNrDc and initialDLTrafficRoutingNrDc are algorithm parameters. Refer to "NIDD parameter for flow control operation" in 5G\_UP\_ALG\_5295\_replaced([11217671](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kxf3KvvSEe-AqvopbP1qhQ)).

[End CB008717-DA]

Uplink:

* Upon reception of Bearer Modification with SCG F1-U information:

- 5G-L2-HI shall allocate UL SCG F1-U TEID if requested

* DDDS handling for SCG F1-U is done as defined in 5G\_UP\_ALG\_6474\_replaced([11217803](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kxgePPvSEe-AqvopbP1qhQ))

Rationale:

* In case of NR DC, secondary F1-U link shall be used for DL data transmission only upon successful RACH procedure completion (at SCG). The completion of RACH is known by the reception of the first UL GTP-U PDU at 5G-L2-HI PDCP.

NOTE:

1. UL SCG F1-U TEID allocation request and DL SCG F1-U TEID are not requested/provided in the same Bearer Modification.
