**4.29.0-2.0-1.0-2**  (ID: `12771702`)

Handling of RLC Initial Transmission Delay for Signalling Bearers Defines a timeout for notification of RLC SRB delivery to scheduler

Value range: 0...999

[Before PR849638]

Default value: 170

[PR849638]

Default value: 320

[End PR849638]

Unit: ms

Online modifiable: No

Step Size: 1

Specified in: 5G_UP_ALG_23097

**Note:**

Default value proposed is selected heuristically based on assumption that first attempt of transmitting the SRB message should happened in the time significantly smaller than full MAC transmission including the retransmissions. This is because SRB has high priority in scheduling and notification about the error comes only if the message is not chosen for scheduling for sufficiently long time. Longest time of maximum MAC transmission is estimated assuming (very conservatively) 10 ms for initial processing and maximum number of 8 transmissions (1 initial transmission and 7 retransmission) with conservative assumption about 20 ms for single cycle, i.e. 160 ms.

[PR849638]

Additional 160 ms added to this R&D parameter to accommodate DRX long cycle timer value when scheduling is not done during DRX period.

[End PR849638]
