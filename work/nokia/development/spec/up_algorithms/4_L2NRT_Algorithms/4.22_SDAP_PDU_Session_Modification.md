# 4.22 SDAP PDU Session Modification

## 4.22.0-2 SDAP PDU Session Modification Modification of 5QI for non-GBR QoS Flows (ID: 11225760)

**4.22.0-2.0-1**  (ID: `11225768`)

SDAP PDU Session Modification
gNB-CU-UP shall support the modification of the 5QI for non-GBR QoS flows only, and perform the following actions:

* On request for release of existing bearer context:
* gNB-CU-UP shall release the bearer context including the SDUs and buffered PDUs, as specified in 5G\_UP\_ALG\_7866\_replaced([11226587](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kyWLufvSEe-AqvopbP1qhQ)).
* On request for setup of new bearer context:
* Allocation of UL F1-U TEID if requested.
* Flow control parameters shall be initialized as defined in 5G\_UP\_ALG\_5456\_replaced([11218965](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kxny8PvSEe-AqvopbP1qhQ)).
* On request for modification of bearer context:
* Configuration of DL F1-U TEID for the new bearer context.
* DL Data transmission towards gNB-DU can start once the DL F1-U TEIID is configured.
* The flow control actions shall be performed as defined in 5G\_UP\_ALG\_5622\_replaced([11220178](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kxugr_vSEe-AqvopbP1qhQ)).
* On request for PDU Session Modification with updated QFI to DRB mapping:
* Update of the existing DRB to QoS flow mapping with the new configured mapping.

NOTE:

1. The PDU Session modification with the updated QoS Flow to DRB mapping shall be done after the new bearer context is setup. Since the traffic flow continues from UPF during the PDU Session Modification procedure, there will be PDU discards at SDAP if the new DRB context does not exist.
2. At gNB-DU (RLC), when the old bearer context is released, there shall be discard of RLC PDUs and SDUs in the buffer. The loss of data is expected during a bearer release and add procedure.

## 4.22.0-3 SDAP PDU Session Modification Modification of Uplink Ng-U Tunnel Information (ID: 11225776)

**4.22.0-3.0-1**  (ID: `11225782`)

SDAP PDU Session Modification
\* On request for update of the Ng-U Uplink Tunnel information as part of PDU Session Modification procedure, gNB-CU-UP shall:
\* Update the current Ng-U UL TEID with the configured TEID, and shall transfer the SDAP SDUs to the new Ng-U UL TEID thereafter.
\* gNB-CU-UP shall provide the Downlink Ng-U tunnel information in the response if requested.
