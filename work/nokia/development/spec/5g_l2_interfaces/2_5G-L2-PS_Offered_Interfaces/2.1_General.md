# 2.1 General

## 2.1.0-1 General L2-PS Variant Specific Parameters Robustness (ID: 13350878)

**2.1.0-1.0-1**  (ID: `13350884`)

General
When L2-PS receives an interface Information Element (IE) that is not supported by the receiving variant (as documented in the "L2 Variant Limitation" column of the interface specification), then L2-PS shall either:

* Accept the message and proceed without applying the unsupported IE, or
* Reject the message by sending a NOK response with the following IEs:
* status: EStatus5G\_NotOk
* cause: ECause5G\_InvalidParameter (2)

Depending on the IE description.

NOTE: If for a IE, there is a dedicated error handling, then it shall take precedence over this one. For detailed guidelines on documenting L2 Variant parameter support in interface specifications, refer to <https://confluence.ext.net.nokia.com/x/PNaXgQ>.
