## 2.12.1 Procedures

### 2.12.1.1 Machine Learning Use Cases Activation

#### 2.12.1.1.0-1 Machine Learning Use Cases Activation Overview (ID: 10981830)

**2.12.1.1.0-1.0-2**  (ID: `10981840`)

Machine Learning Use Cases Activation
Messages:  
PsMl\_ActivateUseCaseReq

PsMl\_ActivateUseCaseResp

Purpose:  
Activation of machine learning use cases.

For online machine learning, the model is not always available, so the activation procedure includes 2 service:

1.activate data collection procedure of the user case

2.activate inference procedure of the user case

Precondition:  
1.Data collection activation: Cell for which use case is activated is set up.

2.Inference activation: ML model is available or accuracy of the model change from NOK to OK.

Detailed description of activation scenarios refer to 5G\_L3\_31235\_replaced([11069807](https://dn-prod.ext.net.nokia.com/rm/resources/BI_BDSjnf_KEe-D3seUwaI0tg)).

Addressing:  
Sent to PsMl address.

Relevant MSCs:  
None

#### 2.12.1.1.0-3 Machine Learning Use Cases Activation Message validation (ID: 10981854)

**2.12.1.1.0-3.0-2**  (ID: `10981863`)

Machine Learning Use Cases Activation
Upon reception of the PsMl\_ActivateUseCaseReq message, 5G-L2-PS shall check the validity of the request message.

[Before PR805810] 5G-L2-PS shall reply with PsMl\_ActivateUseCaseResp message and set the status IE = Ok if the request message is valid and machine learning use case was activated successfully.

[PR805810]

For AI/ML based Scell activation use case, actMlScellAct is always true, useCaseId is always 1.

DataCollectionRequest and InferenceRequest are separatly activated, it depends on different scenarios.

Please refer to 5G\_L3\_31235\_replaced([11069807](https://dn-prod.ext.net.nokia.com/rm/resources/BI_BDSjnf_KEe-D3seUwaI0tg)) to get more configuration scenarios.

5G-L2-PS shall reply status of DataCollectionRequest and/or status of InferenceRequest.

[End PR805810]

For exception handling, see 5G\_L2\_IF\_3012\_replaced([10981869](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2VuPxPu6Ee-AqvopbP1qhQ)).

#### 2.12.1.1.0-4 Machine Learning Use Cases Activation Exceptions (ID: 10981869)

**2.12.1.1.0-4.0-2**  (ID: `10981881`)

Unexpected Exceptions:

In case of an unexpected exception, 5G-L2-PS shall not send response message.

Error is logged for both expected and unexpected exception.

#### 2.12.1.2.0-3 Machine Learning Use Cases Deactivation Message validation (ID: 10981959)

**2.12.1.2.0-3.0-1**  (ID: `10981965`)

Machine Learning Use Cases Deactivation
Upon reception of the PsMl\_DeactivateUseCaseReq message, 5G-L2-PS shall check the validity of the request message.

[PR805810] For AI/ML based Scell activation use case, actMlScellAct could be true or false.

 Deactivate data collection and/or Inference: actMlScellAct is true in this message.

 Deactivate AI/ML Scell activation or cell enter energy saving: actMlScellAct is false without DataCollectionRequest and InferenceRequest in this message.

Please refer to 5G\_L3\_31237\_replaced([11069817](https://dn-prod.ext.net.nokia.com/rm/resources/BI_BDSjn__KEe-D3seUwaI0tg)) to get more configuration scenarios.[End PR805810]

[Before PR805810]5G-L2-PS shall reply with PsMl\_DeactivateUseCaseResp message and set the status IE = Ok if the request message is valid and machine learning use case was deactivated successfully.

[PR805810] 5G-L2-PS shall reply with PsMl\_DeactivateUseCaseResp message and set the status of DataCollectionRequest and/or status of InferenceRequest.

 Deactivate data collection and/or Inference: 5G-L2-PS shall reply with PsMl\_DeactivateUseCaseResp message and set the status of DataCollectionRequest and/or status of InferenceRequest.

 Deactivate AI/ML Scell activation or cell enter energy saving: 5G-L2-PS shall reply with PsMl\_DeactivateUseCaseResp message without any status.

[End PR805810]

For exception handling, see 5G\_L2\_IF\_3012\_replaced([10981869](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2VuPxPu6Ee-AqvopbP1qhQ)).
