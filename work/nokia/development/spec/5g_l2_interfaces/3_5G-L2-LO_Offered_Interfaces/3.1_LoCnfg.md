## 3.1.2 Procedures

### 3.1.2.1 Address Distribution

#### 3.1.2.1.0-1 Address Distribution Overview (ID: 10983032)

**3.1.2.1.0-1.0-1**  (ID: `13238341`)

Address Distribution
Messages:

LoCnfg\_AddressDistributionReq/Resp

Purpose:

To distribute addresses to 5G-L2-LO and to enquire 5G-L2-LO addresses.

Precondition:

5G-L2-LO has been started.

Addressing:

The request is sent to the LoCnfg address.

The response is sent to the address where the request was sent from.

Relevant UP MSCs:

5G\_L1L2\_FLOW\_72\_replaced([11177073](https://dn-prod.ext.net.nokia.com/rm/resources/BI_BeRPmfvREe-AqvopbP1qhQ)): L2RT and L2NRT Instance Configuration

#### 3.1.2.1.0-2 Address Distribution Description (ID: 10983035)

**3.1.2.1.0-2.0-3**  (ID: `10983050`)

Address Distribution
Upon reception of the request, 5G-L2-LO shall store the given addresses, and then return its own dynamic addresses in the related LoCnfg\_AddressDistributionResp.

The request also contains elements to inform L2-LO [Before CB009851-F]instance[CB009851-F] pool[End CB009851-F] about possible type of call. See 5G\_L2\_IF\_14\_replaced([10966124](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2UMlxPu6Ee-AqvopbP1qhQ)) for more details.

---

#### 3.1.2.1.0-3 Address Distribution Message validation (ID: 10983054)

**3.1.2.1.0-3.0-1**  (ID: `13238342`)

Address Distribution
Upon reception of the LoCnfg\_AddressDistributionReq message, 5G-L2-LO shall check the validity of the request message.

5G-L2-LO shall reply with LoCnfg\_AddressDistributionResp message and set the status IE = Ok if the request message is valid and the addresses and pools were configured successfully.

**Note: 5G-L2-LO cannot validate that distributed address/parameter values are correct. Only ranges can be checked where applicable.**

For exception handling, see 5G\_L2\_IF\_737\_replaced([10983059](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2V1kjfu6Ee-AqvopbP1qhQ)).

---

#### 3.1.2.1.0-4 Address Distribution Exceptions (ID: 10983059)

**3.1.2.1.0-4.0-1**  (ID: `13238344`)

Address Distribution
Expected Exceptions:

Generic exception handling rules can be found in the chapter 5G\_UP\_1795\_replaced([11374912](https://dn-prod.ext.net.nokia.com/rm/resources/BI_ueKrmfvkEe-AqvopbP1qhQ)).

5G-L2-LO shall reply with LoCnfg\_AddressDistributionResp message, set the status IE = NotOk, and set an appropriate cause value if:

* the request message is not valid or;
* the addresses were not configured successfully.

For available cause values, see 5G\_UP\_1846\_replaced([11375790](https://dn-prod.ext.net.nokia.com/rm/resources/BI_uePkGPvkEe-AqvopbP1qhQ)).

Unexpected Exceptions:

In case of an unexpected exception, 5G-L2-LO shall not send response message.

Error is logged for both expected and unexpected exception.
