#### 3.4.2.1.0-2 LoCtrl Address Exchange Description (ID: 10984438)

**3.4.2.1.0-2.0-1**  (ID: `10984443`)

LoCtrl Address Exchange
5G-L2-LO receives LoCtrl Client addresses in the LoCtrl\_AddressReq message.

5G-L2-LO provides LoCtrl Server addresses and the address where 5G-L1-UL shall send UlData\_PuschReceiveRespLo messages in the LoCtrl\_AddressResp message.

ulDataLoClientAddress EQID client address provided to 5G-L2-PS and after to 5G-L1-UL (with parameter name addrPuschReceiveRespLo)  shall correspond to the event queue attached to backplane PEP of the core. Corresponding "EM queue create" API pointing to backplane PEP ( First\_Device\_id = default) shall be used when EM queue is created.

For L2-RT, connectivity is reversed at HWAPI side. ulDataLoClientAddress EQID client address provided to 5G-L2-PS shall still correspond to API em\_queue\_create with first\_Device\_id parameter pointing now to L1 instance to communicate with 5G-L1-UL.

5G-L2-LO provides LoCtrl Server address for DL Receiver (loCtrlDlReceiverAddress) in the LoCtrl\_AddressResp message.

#### 3.4.2.1.0-3 LoCtrl Address Exchange Message Validation (ID: 10984448)

**3.4.2.1.0-3.0-1**  (ID: `10984452`)

LoCtrl Address Exchange
Upon reception of the LoCtrl\_AddressReq message, 5G-L2-LO shall check the validity of the request message.

5G-L2-LO shall reply with LoCtrl\_AddressResp message and set the status IE = Ok if the request message is valid and addresses were configured successfully.

For exception handling, see 5G\_L2\_IF\_860\_replaced([10984458](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2WAjpPu6Ee-AqvopbP1qhQ)).

#### 3.4.2.1.0-4 LoCtrl Address Exchange Exceptions (ID: 10984458)

**3.4.2.1.0-4.0-2**  (ID: `10984468`)

LoCtrl Address Exchange
Expected Exceptions:

Generic exception handling rules can be found in the chapter 5G\_UP\_1795\_replaced([11374912](https://dn-prod.ext.net.nokia.com/rm/resources/BI_ueKrmfvkEe-AqvopbP1qhQ)).

5G-L2-LO shall reply with LoCtrl\_AddressResp message, set the status IE = NotOk, and set an appropriate cause value if:

* the request message is not valid or;[CP3] 5G-L2-LO shall check the value range of parameters: nrCellGrpId IE. If out of range, message is invalid. refer to 5G\_L2\_IF\_3469\_replaced([10984472](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2WAjp_u6Ee-AqvopbP1qhQ)).
* the addresses were not configured successfully.

[CP3]In message LoCtrl\_AddressReq handling, If 5G-L2-LO failed in saving the entry of nrCellGrpId, loCtrlDlClientAddress and loCtrlUlClientAddress, 5G-L2-LO respond LoCtrl\_AddressResp message with status NotOk and the coresponding cause value.

For available cause values, see 5G\_UP\_1846\_replaced([11375790](https://dn-prod.ext.net.nokia.com/rm/resources/BI_uePkGPvkEe-AqvopbP1qhQ)).

[5GC001868-B]

Upon reception of a LoCtrl\_AddressReq, 5G-L2-LO shall check if the received loCtrlDlClientAddress and loCtrlUlClientAddress are already allocated to another nrCellGrpId. If so, the association of all parameters with the other nrCellGrpId shall be removed. Then the nrCellGrpId received in LoCtrl\_AddressReq shall be linked to the loCtrlDlClientAddress and loCtrlUlClientAddress in the message. 5G-L2-LO shall reply with a LoCtrl\_AddressResp message with status = EStatus5G\_Ok.

[End 5GC001868-B]

Unexpected Exceptions:

In case of an unexpected exception, 5G-L2-LO shall not send response message.

Error is logged for both expected and unexpected exception.

5G-L2-PS robustness functions related to the LoCtrl\_AddressResp message are implemented.

#### 3.4.2.19.0-4 Release Temporary UE Request Exceptions (ID: 10985741)

**3.4.2.19.0-4.0-1**  (ID: `10985746`)

Release Temporary UE Request
**Expected Exceptions:**

Generic exception handling rules can be found in the chapter 5G\_UP\_1795\_replaced([11374912](https://dn-prod.ext.net.nokia.com/rm/resources/BI_ueKrmfvkEe-AqvopbP1qhQ)).

5G-L2-LO shall discard the message if:

• nrCellIdentity IE is not known (cell does not exist) or;

• crnti is unknown or invalid.

**Unexpected Exceptions:**

In case of an unexpected exception, 5G-L2-LO shall discard the message.

Error is logged for both expected and unexpected exception.
