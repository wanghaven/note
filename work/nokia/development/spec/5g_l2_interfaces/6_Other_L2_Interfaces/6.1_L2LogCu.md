#### 6.1.2.5.0-2 CU Overload Status Indication to Client. Description (ID: 10993900)

**6.1.2.5.0-2.0-1**  (ID: `10993906`)

CU Overload Status Indication to Client.
L2-HI CU informs when CPU overload state is detected.

When message is sent with OvlStatus "ON" suspend of streaming, in CU and DU, is expected to be triggered by client.

And L2-HI CU starts guard timer of 5 min. after which overload status is checked again:

- if CPU overload is still present then guard timer is started again.

If L2LogCu\_SuspendStreamingReq has not been received,

then L2LogCU\_OvlStatusInd is sent again with OvlStatus "ON".

- if CPU overload is over then L2LogCU\_OvlStatusInd is sent with OvlStatus "OFF".

If after sending msg actTraceOvlProt is set to "FALSE", then all above action are immediately aborted.

Suspended streamings must be resumed with L2LogCu\_StreamActivationtReq.

#### 6.1.2.8.0-2 CU Bearer Monitoring Description (ID: 10994091)

**6.1.2.8.0-2.0-1**  (ID: `10994097`)

CU Bearer Monitoring
Upon reception of the request, 5G-L2-HI (CU) instance shall start or stop monitoring of requested bearers. TraceFilter structure (5G\_L2\_IF\_5182\_replaced([10994173](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2XDFfPu6Ee-AqvopbP1qhQ))) given in the request defines which bearers are to be monitored per bearer group.

When monitoring session is active 5G-L2-HI (CU) provides copy of payload of PDCP SDUs which are carried by bearer with requested QCI/5QI as per 5G\_UP\_8356\_replaced([11400856](https://dn-prod.ext.net.nokia.com/rm/resources/BI_ugGlT_vkEe-AqvopbP1qhQ)).

Copy of part or whole PDCP SDU payload is sent as udpPayload via L2LogCu\_BearerMonitoringInd message towards AaTPServer.

---

#### 6.1.2.8.0-3 CU Bearer Monitoring Exceptions (ID: 10994103)

**6.1.2.8.0-3.0-2**  (ID: `10994116`)

CU Bearer Monitoring
Expected Exceptions:

5G-L2-HI shall reply with L2LogCu\_BearerMonitoringResp message with messageResult = NOK\_ValidationFailure in case L2LogCu\_BearerMonitoringReq message is received with IE actBearerMonitoring = True AND:

 qiValue,

 udpPort (allowed values for Bearer Monitoring are 4096..65535)

is out of range, or:

 traceFilter.size(),

 numBearers,

 tailCutter

are = 0.

5G-L2-HI shall reply with L2LogCu\_BearerMonitoringResp message with messageResult = NOK\_MonitoringOngoing in case L2LogCu\_BearerMonitoringReq was send with actBearerMonitoring = True and there is already Bearer Monitoring session ongoing (previous L2LogCu\_BearerMonitoringReq was received with actBearerMonitoring = True [Before CNI-107353] OR rdImsPacketsTracingEnabled = true) [End CNI-107353].

Unexpected Exceptions:

In case of an unexpected exception, 5G-L2-HI shall not send response message.

CU VoNR Mute Detection

L2LogCu\_VonrMuteBinUpdateInd

Purpose:  
To simplify the usage of collected logs to identify a call that has experienced a VoNR mute.

Precondition:

Addressing:  
L2-HI sends the message according to L2LogCU address table that can be found here [11177003](https://dn-prod.ext.net.nokia.com/rm/resources/TX_WQS0UvvOEe-AqvopbP1qhQ).

#### 6.1.2.9.0-2 CU VoNR Mute Detection Description (ID: 12656141)

**6.1.2.9.0-2.0-1**  (ID: `12656142`)

CU VoNR Mute Detection
The message will be sent only if mute occurs and it contains UeId, Mute type and Bin information values to simplify the identification process and troubleshooting of calls that experienced VoNR mute. 5G-L2-HI updates counters and reports the UL/DL mute duration for VoNR bearer. Every time the counter gets updated, 5G-L2-HI (CU) shall send a message to drain sicad when debug tracing is configured.
