#### 4.3.2.1.1-2 Parameter Details

**4.3.2.1.1-2.0-3**  (ID: `10987355`)

Parameter Details
[CNI-109641-L] Upon reception of the request, Perform actions defined in 5G\_L2\_16204\_replaced([11221514](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2_fcLvvTEe-AqvopbP1qhQ)) [End CNI-109641-L].

[Before CNI-109641-L] Upon reception of the request, 5G-L2-HI shall create PDCP entities and configure them with parameter values given in the message if the bearer context exists.

If the new bearer is DRB, then 5G-L2-HI shall also configure the PDCP entity with the security key and security algorithm to be used as indicated in the bearer creation.

In case flowControlActive = True, 5G-L2-HI shall initialise 3GPP F1-U, SCG flow control parameter and common flow control parameter as defined in 5G\_UP\_ALG\_5446.

In case flowControlActive = False, 5G-L2-HI shall initialise common flow control parameters as defined in 5G\_UP\_ALG\_5733.

5G-L2-HI shall create a new NRDU object instance for PM, if the bearer has non-existing nrDuId. See 5G\_L2\_2776\_replaced([11233420](https://dn-prod.ext.net.nokia.com/rm/resources/BI_3Aqg0PvTEe-AqvopbP1qhQ))/5G\_L2\_2780\_replaced([11235879](https://dn-prod.ext.net.nokia.com/rm/resources/BI_3A8NpfvTEe-AqvopbP1qhQ)).

5G-L2-HI shall allocate new UL F1-U TEID and send this in the response message. The TEID allocation is defined in 5G\_L2\_IF\_970\_replaced([10986733](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2WRCX_u6Ee-AqvopbP1qhQ)).

If bearerType = ENDC bearer:

- 5G-L2-HI shall allocate new DL S1-U TEID and send this in the response message

- 5G-L2-HI shall allocate new X2-U TEID and send this in the response message

- if actDlDataForwarding = 1, 5G-L2-HI shall allocate new X2-U forwarding TEID and send this in the response message

- [CB009266-SR-A]If ip under localIpAddress[0] and localIpAddress[1] are non-zero under forwardingTunnelParams, then 5G-L2-HI shall assign both these(IPv4 and IPv6) addresses to the same forwarding tunnel.[End CB009266-SR-A]

- if inactPeriodX2PdcpDuplication is present, 5G-L2-HI shall activate the duplicate PDCP PDU transmission over X2-U interface.

* if reTxPrioritization is present, 5G-L2-HI shall activate the fast retransmission of lost PDCP PDU.

Note: If reTxPrioritization is not present, 5G-L2-HI shall ignore the value of reTxPrioritizationType and continue the bearer setup procedure.

* If pdcpDelayThreshRetxMode = DelayThresh1Enabled, 5G-L2-HI shall check whether all parameters listed below are set to one of values of their specified range (refer to 5G\_L2\_IF\_1020\_replaced([10987413](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2WWh6Pu6Ee-AqvopbP1qhQ))):

 pdcpPduRetxDelayThresh1

 pdcpMaxNumRetxPdu

 pdcpRetxPduSendMaxDelay

 pdcpDelayThresh1TrigType

* If pdcpDelayThreshRetxMode = DelayThresh1and2Enabled, 5G-L2-HI shall check whether all parameters listed below are set to one of values of their specified range (refer to 5G\_L2\_IF\_1020\_replaced([10987413](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2WWh6Pu6Ee-AqvopbP1qhQ))):

 pdcpPduRetxDelayThresh1

 pdcpPduRetxDelayThresh2

 pdcpMaxNumRetxPdu

 pdcpRetxPduSendMaxDelay

 pdcpDelayThresh1TrigType

The TEID allocation is defined in 5G\_L2\_IF\_970\_replaced([10986733](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2WRCX_u6Ee-AqvopbP1qhQ)).

For bearerType = SA bearer:

- 5G-L2-HI shall use the PDU Session index, PDU Session Id and PDU Session Group to identify the PDU session.

- if actDlDataForwading is True, 5G-L2-HI shall allocate a Xn-U DL data forwarding tunnel. The TEID allocation is defined in 5G\_L2\_IF\_970\_replaced([10986733](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2WRCX_u6Ee-AqvopbP1qhQ)).

If RohcParams is filled, L2-HI shall reserve:

* rohcMaxCid + 1 ROHC contexts for decompressor

to be used for this DRB.

The TEID allocation is defined in 5G\_L2\_IF\_1020\_replaced([10987413](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2WWh6Pu6Ee-AqvopbP1qhQ)).

Bearer contexts in CU and DU  are assigned to same bearer group for co-deployed PDCP/RLC.

If cookie information is present, perform actions required for PCMD as defined in 5G\_L2\_9479\_replaced([11231373](https://dn-prod.ext.net.nokia.com/rm/resources/BI_3AbQQPvTEe-AqvopbP1qhQ)).

For sending the periodic PCDM trace report messages to PCDM Trace Framework (Drain Address) refer 5G\_L2\_9508\_replaced([11231780](https://dn-prod.ext.net.nokia.com/rm/resources/BI_3AeTkfvTEe-AqvopbP1qhQ)).

[End CNI-109641-L]

#### 4.3.2.1.1-3 Parameter Details

**4.3.2.1.1-3.0-1**  (ID: `10987364`)

Parameter Details
Message validation:

Upon reception of the HiUserCu\_BearerSetupReq message, 5G-L2-HI shall check the validity of the request message.

5G-L2-HI shall reply with HiUserCu\_BearerSetupResp message and set the status IE = Ok if the request message is valid and the bearer was setup successfully.

For exception handling, see 5G\_L2\_IF\_3721\_replaced([10987369](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2WWh4Pu6Ee-AqvopbP1qhQ)).

#### 4.3.2.1.1-4 Parameter Details

**4.3.2.1.1-4.0-8**  (ID: `12680524`)

Parameter Details
Generic exception handling rules can be found in the chapter [11374912](https://dn-prod.ext.net.nokia.com/rm/resources/BI_ueKrmfvkEe-AqvopbP1qhQ).

Exception handling rules specifically for object setup can be found in the requirements [11375008](https://dn-prod.ext.net.nokia.com/rm/resources/BI_ueLSpvvkEe-AqvopbP1qhQ) and [11375035](https://dn-prod.ext.net.nokia.com/rm/resources/BI_ueLSqfvkEe-AqvopbP1qhQ).

5G-L2-HI shall reply with HiUserCu\_BearerSetupResp message, set the status IE = NotOk, and set an appropriate cause value (see [11375790](https://dn-prod.ext.net.nokia.com/rm/resources/BI_uePkGPvkEe-AqvopbP1qhQ)) if:

* bearerGroup IE out of range or;
* bearerIndex IE out of range or;
* bearerIndex IE with same index exists already or;
* the request message is not valid or;
* the resources were not allocated successfully or;
* plmnId given in the message is not known (from HiCnfgCu\_ConfigParamReq received earlier) or;
* snFieldLength IE is set to 12 bits for AM DRB and typeOfBearer = NSA.

Set cause InvalidParameter (2) if:

* pdcpStatusReportWaitTimer > 0 is out of range;
* reTxPrioritizationType is present AND:
* resourceType**= GBR/dcGBR OR**
* resourceType**= nonGBR AND rlcMode = UM;**
* when actDlDataForwarding is True and one of the below conditions is satisfied:
* ip in both localIpAddress[0] and localIpAddress[1] under forwardingTunnelParams is zero
* version is set as IPv6 for both localIpAddress[0] and localIpAddress[1] (See Note 1)
* version is set as IPv4 for both localIpAddress[0] and localIpAddress[1] (See Note 1)
* version is set as IPv6 and ip is set as zero for the same localIpAddress (See Note 2)
* version is set as IPv4 and ip is set as zero for the same localIpAddress (See Note 2)
* IE dlPreferredPath:
* is not present and dlDataSplitMode = overF1UXxU;
* is present and dlDataSplitMode set to other than overF1UXxU;
* IE ulPreferredPath:
* is not present and ulDataSplitMode = overF1UXxU;
* is present and ulDataSplitMode set to other than overF1UXxU;
* IE qosFlowId (QFI):
* is not present for typeOfBearer = SA;
* is present for typeOfBearer = NSA;
* IE DcGbrPmCounterParams:
* is present for Mapping/resourceType = nonGbr or GBR;
* [CB013978-A] IE [DpiMeasurements](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2WXwBPu6Ee-AqvopbP1qhQ):
* is present AND DPI is not configured ([DpiMeasurementsParams](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2WUFpvu6Ee-AqvopbP1qhQ) structure was not received via HiCnfgCu\_ConfigParamReq message);
* contains both actDpiGoodput and actDpiRtt IEs set to False. [End CB013978-A]

Set cause NoUserPlaneResourcesAvailable (1) if:

* there is more than 12 L2-LO’s DL connection configured per L2Hi instance.

Rationale: This limitation is due to L2-HI needs to create a bundle to send DL RLC PDUs for each L2-LO local connection and 12 L2-LO’s instances is currently the maximum supported number and results from SW deployment.

In case of failure, if 5G-L2-HI is not able to do a proper internal rollback for all sub-systems, and the state of the 5G-L2-HI is inconsistent, 5G-L2-HI shall indicate this in the response message, set status IE = "NotOk" and set the needForRollback IE = TRUE. This is to indicate that clean-up is expected from 5G-CP-NRT, and that 5G-CP-NRT should not presume that no resources were allocated by the 5G-L2-HI.

Note:

1. Local IP address for the tunnel can be either IPv4 or IPv6 or both. But receiving either IPv6 or IPv4 address in both localIpAddress[0] and localIpAddress[1] is invalid.
2. If version is set as either IPv4 or IPv6 and the corresponding ip filed is zero, then it is invalid.

#### 4.3.2.2.0-4 Bearer Modify Expected exceptions: (ID: 10987965)

**4.3.2.2.0-4.0-9**  (ID: `13433713`)

Bearer Modify
5G-L2-HI shall reply with HiUserCu\_BearerModifyResp message, set the status IE = NotOk, and set an appropriate cause value if:

* bearerIndex IE is not known or;
* bearerGroup IE is not known or;
* the request message is not valid or;
* the resources were not modified successfully.

Set cause InvalidParameter (2) if:

* pdcpStatusReportWaitTimer > 0 is received with:
* pdcpReset = True;
* IE cause set to other than Suspend;
* out of range.
* L2TrafficShapingXxFwdParams **structure is present AND:**
* IE cause set to other than BearerRelease;
* **GtpuInfo - ForwardingTunnels - drb not filled**;\*\*\*\*
* maxForwardTimeLimitXx and minForwardTputXx **out of range;**
* **Traffic Shaping functionality is disabled (actL2TrafficShapingXxParams equals to True is not stored in L2-HI CU context)**
* dscp IE is out of range**;**
* dscp IE is not populated when cause = DscpUpdate;
* IE cause = SGWRelocation AND s1u is NOT filled.
* IE cause = ReleaseSecondaryLink and:
* typeOfBearer = NSA;
* there are resources allocated for SCG link.
* **IE cause =BearerRelease and:**
* typeOfBearer = NSA;
* stopOngoingForwarding IE ([10988084](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2Wbaafu6Ee-AqvopbP1qhQ)) is present
* **IE cause = Suspend AND:**
* GtpuInfo - ForwardingTunnel (5G\_L2\_IF\_3316\_replaced([10988084](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2Wbaafu6Ee-AqvopbP1qhQ))) structure filled;
* pdcpCountRequired = True.
* **IE cause = Suspend AND:**
* stopOngoingForwarding IE ([10988084](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2Wbaafu6Ee-AqvopbP1qhQ)) is present
* **IE interRatHoWithDirectFwd = True (During Inter-RAT HO from NR to LTE with direct data forwarding) AND**
* forwarding tunnel parameter for PDU session forwarding (5G\_L2\_IF\_3316\_replaced([10988084](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2Wbaafu6Ee-AqvopbP1qhQ)), pduSession) are filled

OR

* forwarding tunnel parameter for DRB forwarding (5G\_L2\_IF\_3316\_replaced([10988084](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2Wbaafu6Ee-AqvopbP1qhQ)), drb) are not filled
* [CB014699-D] criticalDrbSchedEnabled IE is not populated when cause = TrafficTypeUpdate is used; [End CB014699-D]

Set cause NoUserPlaneResourcesAvailable (1) if:

 there is more than 12 L2-Lo’s DL connection configured per L2Hi instance.

Rationale: This limitation is due to L2-HI needs to create a bundle to send DL RLC PDUs for each L2-LO local connection and 12 L2-Lo’s instances is currently the maximum supported number and results from SW deployment.

In case of failure, if 5G-L2-HI is not able to do a proper internal rollback for all sub-systems, and the state of the 5G-L2-HI is inconsistent, 5G-L2-HI shall indicate this in the response message, set status IE = "NotOk" and set the needForRollback IE = TRUE. During RRC Reestablishment or (RRC Reestablishment during Inter-gNB Handover via Xn or NG) when (XnAP: SN Status Transfer or NGAP:Downlink RAN Status Transfer is received after RRC Reestablishment request)):

* if 5G-L2-HI receives HiUserCu\_BearerModifyReq with cause= **BearerRelease OR cause=BearerSetup AND with securityParams is filled AND crypto session is created with failure, 5G-L2-HI shall reply HiUserCu\_BearerModifyResp message with:**
* status IE: EStatus5G\_NotOk
* cause IE: NoUserPlaneResourcesAvailable (1)
* set needForRollback = TRUE

During Intra-gNB, Intra-DU NR-DC Bearer takeback (deleting NR-DC split bearer):

* if 5G-L2-HI receives HiUserCu\_BearerModifyReq with cause = SuspendSecondaryLink and 5G-L2-HI  has already suspended secondary link SCG F1-U (see: Note 2),

5G-L2-HI shall reply with HiUserCu\_BearerModifyResp message with:

* status IE: EStatus5G\_NotOk
* cause IE: ObjectDoesNotExist.

In case if 5G-L2-HI receives HiUserCu\_BearerModifyReq with cause=FetchResource AND

allocateUlSplitTeid = True

OR allocateF1uTeid = True

AND Teid allocation failed, 5G-L2-HI shall reply HiUserCu\_BearerModifyResp message with:

status IE: EStatus5G\_NotOk

cause IE: NoUserPlaneResourcesAvailable

#### 4.3.2.9.0-2 PDU Session Setup Description (ID: 10988512)

**4.3.2.9.0-2.0-1**  (ID: `13383016`)

PDU Session Setup
Upon reception of the request, 5G-L2-HI shall store PDU session related information

5G-L2-HI shall allocate new NG-U TEID and send this in the response message. The TEID format is defined in 5G\_L2\_2200\_replaced([11221946](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2_h4bPvTEe-AqvopbP1qhQ)).

**[5GC000704-C] In case when “AllocatePduSessionXnForwardingTEID” IE is True,**

* 5G-L2-HI shall allocate “PduSessionXnUForwardingTEID” and, send this in the response message. The TEID format is defined in 5G\_L2\_2200\_replaced([11221946](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2_h4bPvTEe-AqvopbP1qhQ)).
* 5G-L2-HI shall start the pdu session endmarkerTimer with the duration set to "pduSessionEndMarkerTimer".

5G-L2-HI shall support creation of PDU session with multiple DRB. Each of the DRB could be present in different Bearer Group.

5G-L2-HI shall maintain mapping between the "Mapping Index" of the NG-U TEID and (Bearer Index, Bearer Group, QoS Flow Id).

Since NG-U TEID (PDU session) maps to multiple Bearers and multiple QoS Flow Id's, the multiple mappings are needed to be maintained.

**[5GC000752-K]**

5G-L2-HI shall support PDU session setup in case of absence of default DRB (i.e. Default DRB set to „False” and SDAP UL Header Presence set to “Absent”). See also 5G\_L2\_2911\_replaced([11222893](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2_nX8fvTEe-AqvopbP1qhQ)).

**[End 5GC000752-K]**

### 4.3.2.18 Throughput Monitoring

#### 4.3.2.18.0-1 Throughput Monitoring Overview (ID: 13247288)

**4.3.2.18.0-1.0-1**  (ID: `13247222`)

Throughput Monitoring
Messages:

HiUserCu\_ThroughputMonitoringReq

HiUserCu\_ThroughputMonitoringResp

HiUserCu\_ThroughputMonitoringInd

Purpose:

Provide UE-level DL throughput monitoring control at 5G-L2-HI (CU) and notify when the downlink throughput crosses a configured threshold.

Precondition:

* A valid UE context exists on 5G-L2-HI (CU) for given ueIdCu.
* Valid values of dlTputMonitorPeriod and dlTputSampInterval has been received for any DRB related to monitored ueIdCu (via HiUserCu\_BearerSetupReq under [DlUeTputMonitoringParams](https://dn-prod.ext.net.nokia.com/rm/resources/BI_-nSPk9T-EfCnq6aYkXj8Pw) structure.)

Addressing:

The request is sent to the HiUserCu address.

The response is sent to the address where the request was sent from.

Relevant UP MSCs:

[13362535](https://dn-prod.ext.net.nokia.com/rm/resources/BI_HIDSIvU9EfC0Z6kKit-MzQ): DL UE throughput monitoring (5G-L2-HI CU)

---

#### 4.3.2.18.0-2 Throughput Monitoring Description (ID: 13247365)

**4.3.2.18.0-2.0-1**  (ID: `13247366`)

Throughput Monitoring
Upon reception of the request, 5G-L2-HI (CU) shall:

1. Validate the request as per [13247367](https://dn-prod.ext.net.nokia.com/rm/resources/BI_9ZUYecRnEfCnuMJFWtMTpQ).
2. Process the tputMonitorAction:
3. startDl: Initiate DL throughput monitoring for the specified ueIdCu using the provided thresholdDlUeTput.
4. stopDl: Stop the ongoing DL throughput monitoring for the specified ueIdCu and clear any associated monitoring context.
5. modifyThreshold: Update the existing thresholdDlUeTput value for the ueIdCu without interrupting the monitoring.
6. Monitor throughput (for startDl/modifyThreshold):
7. Continuously monitor the DL throughput for the ueIdCu
8. Detect threshold events:

   1. dlLow - when UE DL throughput drops below thresholdDlUeTput
   2. dlHigh - when UE DL throughput rises to or above thresholdDlUeTput, but only after dlLow event was previously indicated
   3. Send HiUserCu\_ThroughputMonitoringInd:
9. When a threshold event is detected, send an indication to 5G-CP-NRT as defined in [13372401](https://dn-prod.ext.net.nokia.com/rm/resources/BI_E9SCQfqkEfCPDo8cGBoqIw).
