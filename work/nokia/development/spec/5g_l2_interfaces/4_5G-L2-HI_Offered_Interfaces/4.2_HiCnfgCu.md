#### 4.2.2.1.0-2 Address Distribution Description (ID: 10986818)

**4.2.2.1.0-2.0-3**  (ID: `10986833`)

Address Distribution
[CB007078-E]5G-L2-HI shall process the request as idempotent message.[End CB007078-E]

Upon reception of the request, 5G-L2-HI shall store the given addresses, and then return its own dynamic addresses in the related HiCnfgCu\_AddressDistributionResp.

[CB007078-E]

Upon reception of the request with parameters already configured, 5G-L2-HI shall process it, recognize that nothing has changed and send OK response with its own dynamic addresses in the related HiCnfgCu\_AddressDistributionResp.

Upon reception of the request with different parameters than already configured, 5G-L2-HI shall send “NOK” (EStatus5G\_NotOk) response.

[End CB007078-E]

5G-L2-HI shall create a new NRBTS object instance for PM [CB007078-E] only for initial 5G-L2-HI instance creation[End CB007078-E], see 5G\_L2\_2776\_replaced([11233420](https://dn-prod.ext.net.nokia.com/rm/resources/BI_3Aqg0PvTEe-AqvopbP1qhQ))/5G\_L2\_2780\_replaced([11235879](https://dn-prod.ext.net.nokia.com/rm/resources/BI_3A8NpfvTEe-AqvopbP1qhQ)).

Due to common code for Classical and Cloud, rd1514TeIdMappingLogicFlag is removed (there is one common TEID format that can be found under 5G\_L2\_15625\_replaced([11221929](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2_h4avvTEe-AqvopbP1qhQ))).

[End CB007431-A]

#### 4.2.2.2.0-2 Configuration Parameters Description (ID: 10986958)

**4.2.2.2.0-2.0-5**  (ID: `10986982`)

Configuration Parameters
Upon reception of the request, 5G-L2-HI shall store the configuration parameters.

 5G-L2-HI (CU) shall use the given PLMN and PLMN Slice (SNSSAI) configuration for PM object management as described in 5G\_L2\_2776\_replaced([11233420](https://dn-prod.ext.net.nokia.com/rm/resources/BI_3Aqg0PvTEe-AqvopbP1qhQ))/5G\_L2\_2780\_replaced([11235879](https://dn-prod.ext.net.nokia.com/rm/resources/BI_3A8NpfvTEe-AqvopbP1qhQ)).

 When bufferCongestionDiscardMechanism is available, L2-HI activates PDCP buffer congestion mechanism.

 Some Optional IEs are introduced due to HiCnfgCu\_ConfigParamReq can be obtained from CPlane in different contexts. If they are not set means that nothing has changed according to the previous settings and L2Hi should not change it.

 5G-L2-HI shall activate/deactivate HFN Desynchronization Detection mechanism and use threshold parameters specified in 5G\_L2\_IF\_5942\_replaced([10987252](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2WVTx_u6Ee-AqvopbP1qhQ))

* [CB008490-DT] 5G-L2-HI shall activate/deactivate traffic shaping functionality (actL2TrafficShapingXx) or use modified parameters (l2TrafficShapingXxParams structure - 5G\_L2\_IF\_6879\_replaced([10987140](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2WUssfu6Ee-AqvopbP1qhQ))) immediately for all existing and new bearers. (see Note) [End CB008490-DT]

5G-L2-HI shall process the request as idempotent message. It means that upon reception of the request with parameters already configured, 5G-L2-HI shall process it and recognize that nothing has changed, in other cases 5G-L2-HI shall override the values and start using the new ones.

In both situations, 5G-L2-HI shall send the related response message with OK status.

[CB008490-DT] Note:

During transition period (new trsBurstSize and/or trsBurstPeriod are received in l2TrafficShapingXxParams structure - 5G\_L2\_IF\_6879\_replaced([10987140](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2WUssfu6Ee-AqvopbP1qhQ))) there might be data burst increase as current traffic shaping process is interrupted and immediately started with new parameters.

[End CB008490-DT]

#### 4.2.2.2.0-4 Configuration Parameters Exceptions (ID: 10986997)

**4.2.2.2.0-4.0-7**  (ID: `13419563`)

Configuration Parameters
**Expected Exceptions:**

Generic exception handling rules can be found in the chapter [11374912](https://dn-prod.ext.net.nokia.com/rm/resources/BI_ueKrmfvkEe-AqvopbP1qhQ).

If a HiCnfgCu\_ConfigParamReq requested to add one or more plmnId which already exists, 5G-L2-HI shall write notification to error log for each such plmnId, and return a HiCnfgCu\_ConfigParamResp message with status IE = Ok.

For any other expected exceptions,

5G-L2-HI shall reply with HiCnfgCu\_ConfigParamResp message, set the status IE = NotOk, and set an appropriate cause value if:

* the request message is not valid or;
* the parameters were not configured successfully.

Set status IE: EStatus5G\_NotOk with cause IE: ECause5G\_InvalidParameter (2) if:

* trsBurstSize, trsBurstPeriod or nrCellIdsForCountersReporting IE is out of range.
* actL2TrafficShapingXx is equal to True AND:
* l2TrafficShapingXxParams structure is not present**,**
* **content of l2TrafficShapingXxParams is not changed since previous valid HiUserCu\_ConfigParamReq message,**
* actL2TrafficShapingXx is equal to False or not present AND l2TrafficShapingXxParams structure ([10987140](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2WUssfu6Ee-AqvopbP1qhQ)) is present.
* The number of PMQAP profiles configured are more than 40 or out of range.
* The number of PMQAP profiles for DC-GBR configured are more than 7.
* For DC-GBR bearer ulRanDelayHistRange IE is filled but actPerDelayCounters is set to FALSE.
* PmCountersInfo--[DpiMeasurementsParams](https://dn-prod.ext.net.nokia.com/rm/resources/BI_2WUFpvu6Ee-AqvopbP1qhQ) structure:
* not present in initial HiCnfgCu\_ConfigParamReq message,
* contains parameters out of range.
* **[CNI-150183] PmCountersInfo--downlinkThptHistParams structure not present  AND IE cfgThptDistUpdate is present.**
* PmCountersInfo--downlinkThptHistParams structure is present  AND IE cfgThptDistUpdate is not present.
* IE minRange is present AND IE scale set to linear.
* IE minRange is not present AND IE scale set to logarithmic.
* nrPmqapId provided in [HistogramPmqap](https://dn-prod.ext.net.nokia.com/rm/resources/BI_JTaEQvrAEfCPDo8cGBoqIw) does not exist in the pmqapProfileList. **[End CNI-150183]**

For available cause values, see [11375790](https://dn-prod.ext.net.nokia.com/rm/resources/BI_uePkGPvkEe-AqvopbP1qhQ).
