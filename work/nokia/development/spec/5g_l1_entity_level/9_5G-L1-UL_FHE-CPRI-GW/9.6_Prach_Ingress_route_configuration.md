###### 10.1.1.1.1 Phase compensation LUT index calculation

**10.1.1.1.1.0-1**  (ID: `13400058`)

Phase compensation LUT index calculation
Note: The default values for testing (when specific carrier frequency is not relevant or can be set freely within Frequency Range) are the following:

FR1: 145600 (FDD), 636666 (TDD)

FR2: 2081665 (TDD)

End Note

Input:

* ARFCN: provided in API\_BBB\_CREATE\_EAXC\_AXC\_ROUTING\_REQ\_MSG message with direction = "EBbbEaxcDirection\_Uplink".
* μ = log2(scs/15)

Definition:

* p(𝑙): phaseCompensationLutIndex, with OFDM symbol  l=0..14\*2μ-1

Algorithm:

Calculate PUL based on:

If ARFCN< 600000:  PUL = ARFCN

If 600000 <= ARFCN< 2016667:  PUL = 3\*(ARFCN- 400000)

If ARFCN>= 2016667:  PUL = 12\*(ARFCN– 1612499)

END

offset = 0

FOR  𝑙 = 0 to (14\*2μ)-1

IF 𝑙 =0 OR 𝑙 =7\*2μ:

NCP = 32+288/2μ

ELSE

NCP = 288/2μ

END

offset = offset + NCP

p(𝑙) = (PUL x offset) mod 12288

offset = offset + 4096

END

Output:

* The phaseCompensationLutIndex p(𝑙) with  l=0..14\*2μ-1

##### 10.1.1.3 Dependencies

**10.1.1.3.0-1**  (ID: `13409563`)

Dependencies
These are dependencies between UlCell\_SetupReq parameters related to PRACH and other parameters inside UlCell\_SetupReq and L1Config\_SwConfigurationReq. These dependencies show allowed values for each parameter.

Dependency follow logical operators - IF/ELSE/ELSEIF/AND/OR. GT/LT/GTEQ/LTEQ stand for Greater Than, Lesser Than, Greater Than or Equal, Lesser Than or Equal. Word THEN indicate end of condition and the valid values of a given parameter. Round braces () denotes interval of values. Curly braces {} denotes set of values. Indentation denotes nested conditions. Value "ignore" after word THEN means validation to be skipped.

Below there is a list of used abbreviations:  
freqRange1 = L1Config.frequencyRange == {FR1, FR1\_NB, FR1\_WB}  
freqRange2 = L1Config.frequencyRange == FR2  
fddDuplex = L1Config.duplexMode == FDD AND freqRange1  
tddDuplex = L1Config.duplexMode == TDD  
fh72a = L1Config.fronthaulMode == eCPRI  
fh72e = L1Config.fronthaulMode == eCPRI\_7\_2e

fhCPRI = L1Config.fronthaulMode == {CPRI, OBSAI}

##### 10.1.2.2 Dependencies for Thor

**10.1.2.2.0-1**  (ID: `13360659`)

Dependencies for Thor
These are dependencies between UlCell\_SetupReq parameters related to PRACH and other parameters inside UlCell\_SetupReq and L1Config\_SwConfigurationReq. These dependencies are only supposed to show what are allowed values for each parameter. They do not state what setting a parameter to a particular value mean, nor what should it result in.

Dependency follow simple and widely recognized logical operators - IF/ELSE/ELSEIF/AND/OR and so on. GT/LT/GTEQ/LTEQ stand for Greater Than, Lesser Than, Greater Than or Equal, Lesser Than or Equal. Word THEN indicate the end of condition and then the available value of a given paramter. Round braces () represent range potential of values. Curly braces {} represent list of potential values. Identations mean nested conditions. Value "ignore" after word THEN mean that the validation for a given parameter should be skipped entirely.

Below there is a list of commonly used abbreviations, to make dependencies be more readable  
freqRange1 = L1Config.frequencyRange == {FR1, FR1\_NB, FR1\_WB}  
freqRange2 = L1Config.frequencyRange == FR2  
fddDuplex = L1Config.duplexMode == FDD AND freqRange1  
tddDuplex = L1Config.duplexMode == TDD  
fh72a = L1Config.fronthaulMode == eCPRI  
fh72e = L1Config.fronthaulMode == eCPRI\_7\_2e
