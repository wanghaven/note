### 7.6.3.1 RIM requests

#### 7.6.3.1.1 Remote interference (RI) detection

**7.6.3.1.1.0-1**  (ID: `11470522`)

Remote interference (RI) detection
When 5G-L1-UL (Radio) receives message *UlDataFH\_RimReceiveReq from L1-BB over FH interface (see [11517626](https://dn-prod.ext.net.nokia.com/rm/resources/BI_FSMml_vxEe-AqvopbP1qhQ)) with parameter* rimRssiRequest set to TRUE (see [11517567](https://dn-prod.ext.net.nokia.com/rm/resources/BI_FSL_lPvxEe-AqvopbP1qhQ)), it shall perform RSSI estimation for remote interference management (RIM) for the cell indicated in parameter *sucellId and the slot specified by parameters* sfn and *slot in the message.*\*\*\*\*

RSSI estimation for RIM is supported on special and UL slots.

RSSI estimation shall be performed for the symbols and sub-bands specified in the message.

---

#### 7.6.3.1.2 RIM reference signal (RIM-RS) detection

**7.6.3.1.2.0-1**  (ID: `11470531`)

RIM reference signal (RIM-RS) detection
When 5G-L1-UL (Radio) receives message *UlDataFH\_RimReceiveReq from L1-BB over FH interface (see [11517626](https://dn-prod.ext.net.nokia.com/rm/resources/BI_FSMml_vxEe-AqvopbP1qhQ)) with parameter* rimRsDetectionRequest set to TRUE (see [11517567](https://dn-prod.ext.net.nokia.com/rm/resources/BI_FSL_lPvxEe-AqvopbP1qhQ)), it shall perform detection of RIM reference signal (RIM-RS) for the cell indicated in parameter *sucellId and the slot specified by parameters* sfn and *slot in the message.*\*\*\*\*

5G-L1-UL (Radio) shall shall support the detection of up to 8 candidate RIM-RS sequence per subcell and per slot. The detection of each of these candidate RIM-RS sequence can be requested on up to 14 symbols and up to 4 frequency resources.

All parameters for RIM-RS sequence detection are given in message *UlDataFH\_RimReceiveReq.*

5G-L1-UL (Radio) shall support both remote interference detection (RSSI measurement) and RIM-RS detection for the same subcell and the same slot if both parameters *rimRssiRequest and* rimRsDetectionRequest are set to 'true'.\*\*

### 7.6.3.2 Resource allocation

#### 7.6.3.2.1 Resource allocation for RI detection

**7.6.3.2.1.0-1**  (ID: `11470545`)

Resource allocation for RI detection
The symbols and sub-bands on which RSSI shall be estimated are independent from the multiplexed channels of the slot and on the scheduling of PRBs. These resources may overlap with PUSCH and periodic or aperiodic SRS.

The message *UlDataFH\_RimReceiveReq contains the definition of time and frequency domain regions on which RSSI shall be measured (see [11517567](https://dn-prod.ext.net.nokia.com/rm/resources/BI_FSL_lPvxEe-AqvopbP1qhQ)):*

* The set of consecutive symbols is defined by parameters *riStartSymbol and* riNumOfSymbols.\*\*
* The number of sub-bands is given by the *length of dynamic array* riStartPrbPerSubBand.\*\*
* The PRBs of each sub-band is given by the content of array *riStartPrbPerSubBand.*
* The common number of PRBs for all sub-band is given by parameter *riNumOfPrbPerSubBand. Only 6-PRB sub-bands are supported.*

##### 7.6.3.2.2.4 Overlapping with other channels

**7.6.3.2.2.4.0-2**  (ID: `12706375`)

Overlapping with other channels
5G-L1-UL (Radio) shall perform RIM-RS detection operations on top of uplink channel reception procedures in the same slot.

Frequency resources for RIM-RS detection may overlap with scheduled PUSCH, aperiodic or periodic SRS.

[Before CB008235-SR-X]If subcell type is C4'2, overlapping with long PUCCH or PRACH is also supported but not recommended. In this case, L1 shall apply zeroing of data overlapping with PUCCH or PRACH before performing RIM-RS detection (see [11470751](https://dn-prod.ext.net.nokia.com/rm/resources/BI_fCBssPvvEe-AqvopbP1qhQ)).[End CB008235-SR-X]

### 7.6.3.3 Configuration of spatial streams

**7.6.3.3.0-4**  (ID: `12706377`)

Configuration of spatial streams
5G-L1-UL (Radio) shall use in case of :

* Only RIM-RSSI request in the slot (rimRsDetectionRequest=FALSE), the first two consecutive spatial streams defined for the cell for RI detection. Those streams are statically determined from the subcell type given by HLAPI at combining link creation (see L1L\_SW\_4847): stream 0 and 1 for subcell type [Before CB008235-SR-X]C4'2 and[End CB008235-SR-X] C16'y.
* RIM-RS detection request is present in the same slot (rimRsDetectionRequest=TRUE) , the last two consecutive spatial streams defined for the cell for RIM-RS detection. Those streams are statically determined from the subcell type given by HLAPI at combining link creation (see L1L\_SW\_4847): [Before CB008235-SR-X]stream 2 and 3 for subcell type C4'2[End CB008235-SR-X] stream 14 and 15 for subcell type C16'y.

The default beam allocated for RI detection **and RIM-RS detection is given by parameter *patternId of message *UlDataFH\_RimReceiveReq* \*(see [11517567](https://dn-prod.ext.net.nokia.com/rm/resources/BI_FSL_lPvxEe-AqvopbP1qhQ)).***\*\*\*

Note: The default RIM beam is the sector beam but L1 shall support any beam ID.

###### 7.6.3.4.1.2 Loss of message UlDataFH_RimReceiveReq

**7.6.3.4.1.2.0-1**  (ID: `11470653`)

Loss of message UlDataFH\_RimReceiveReq
If no message *UlDataFH\_RimReceiveReq is received at the end of FH reception window [*T2a\_max\_cp\_ul..*T2a\_min\_cp\_ul], 5G-L1-UL (Radio) shall consider that no such message was sent from L1-BB or that the message is lost.*\*\*

In this case RIM processing is not done and no response message is sent to L1-BB.

### 7.6.3.5 Management of overlapping resources for beamforming

**7.6.3.5.0-2**  (ID: `11470667`)

Management of overlapping resources for beamforming
On uplink slots, 5G-L1-UL (Radio) shall make the beamforming of time, frequency and spatial resources required for RIM while managing the overlapping with resources allocated to other channels. For each symbol, PRB and pair of spatial streams, only one beamforming operation is made.

On special slots, management of resources overlapping with periodic SRS is not needed as beamforming for RIM and periodic SRS are using separate HW resources.

Uplink and special slots are determined by class *TddSwitchingConf (see L1L\_SW\_1133).*

Details of the procedure are given in [11628896](https://dn-prod.ext.net.nokia.com/rm/resources/BI_aYWFrfwMEe-AqvopbP1qhQ)**[CB012564-B] (case of RI detection only), [11628899](https://dn-prod.ext.net.nokia.com/rm/resources/BI_aYWFrvwMEe-AqvopbP1qhQ?oslc_config.context=https%3A%2F%2Fdn-prod.ext.net.nokia.com%2Frm%2Fcm%2Fstream%2F_sTQh5tnCEe26EtvmdlGL5w%3Foslc_config.context%3Dhttps%3A%2F%2Fdn-prod.ext.net.nokia.com%2Fgc%2Fconfiguration%2F9)(case of RIM-RS detection or RI+RIM-RS detection) and [11628901](https://dn-prod.ext.net.nokia.com/rm/resources/BI_aYWFr_wMEe-AqvopbP1qhQ?oslc_config.context=https%3A%2F%2Fdn-prod.ext.net.nokia.com%2Frm%2Fcm%2Fstream%2F_sTQh5tnCEe26EtvmdlGL5w%3Foslc_config.context%3Dhttps%3A%2F%2Fdn-prod.ext.net.nokia.com%2Fgc%2Fconfiguration%2F9)(case of simultaneous RI and RIM-RS detection).** [End CB012564-B]\*\*\*\*

### 7.6.4.4 RIM-RS detection algorithm

#### 7.6.4.4.1 Overview

**7.6.4.4.1.0-2**  (ID: `12713555`)

Overview
RIM-RS detection algorithm is described in section 2.10.2.6 of "5GMax algorithm specification" (see [11470492](https://dn-prod.ext.net.nokia.com/rm/resources/BI_fB_3hfvvEe-AqvopbP1qhQ)).

This algorithm shall be executed for each candidate RIM-RS sequence on each associated detection resource unit.

The main steps of RIM-RS detection algorithm are:

* Combination of the received data over both spatial streams,
* Correlation of the combined signal with the candidate root sequence in frequency domain,
* Conversion of the correlated sequence in time domain by IFFT,
* Determination of the power delay profile of the correlated signal in time domain, and search for the peak value.

The peak value is then compared to the threshold[Before CB014153-C] defined by R&D parameter *rdRimRsDetectionThreshold [CB014153-C] defined by ri*mRsDetectionThreshold which is set by M-Plane**. [End CB014153-C].**\*\*

* If the peak value is greater or equal to the threshold, the candidate RIM-RS sequence is considered as detected on this resource unit.
* If the peak is below the threshold, the candidate RIM-RS sequence is considered as not detected on this resource unit.

[CB014153-C-UPD001]

Note:

L1(Radio) shall get value of the threshold from parameter rim\_rs\_detection\_threshold of CombiningLinkConf class from HLAPI interface.

The value of threshold could change during carrier component lifetime.

For details please see: [11630915: CombiningLinkConf (in L1 Radio and L1Low module)](https://dn-prod.ext.net.nokia.com/rm/resources/BI_aYugJPwMEe-AqvopbP1qhQ?oslc_config.context=https%3A%2F%2Fdn-prod.ext.net.nokia.com%2Fgc%2Fconfiguration%2F9)

[End CB014153-C-UPD001]

### 7.6.4.5 RIM-RS detection report

**7.6.4.5.0-1**  (ID: `11470796`)

RIM-RS detection report
5G-L1-UL (Radio) shall create a detection report for each RIM-RS sequence and each detection resource unit if the candidate sequence is positively detected on this resource unit. Each detection report contains the following information (see [11517617](https://dn-prod.ext.net.nokia.com/rm/resources/BI_FSMmlfvxEe-AqvopbP1qhQ)):

* sequence identifier *n*SCID*,*\*\*
* symbol index,
* starting PRB of the frequency resource,
* average signal power in dBm,
* amplitude of the peak value of the correlated signal in time domain,
* position of the peak value.

All RIM-RS detection reports for a subcell and a slot are transmitted to L1-BB over FH interface in message *UlDataFH\_RimReceiveRespPs (see [11517603](https://dn-prod.ext.net.nokia.com/rm/resources/BI_FSMmk_vxEe-AqvopbP1qhQ)).*

### 7.6.4.6 Interworking wth periodic SRS

#### 7.6.4.6.0-1 Interworking wth periodic SRS The scheduling of periodic SRS on special slot is not supported if RIM-RS detection is requested on the same slot (see 11162042 ). If both periodic SRS and RIM-RS are requested in the same special slot, 5G-L1-UL (Radio) will attempt to perform both operations but will log a warning. (ID: 11470806)

## 7.6.5 RIM-RS capacity

### 7.6.5.0-1 RIM-RS capacity On 5G-L1-UL (Radio), RIM-RS capacity is defined by the number of RIM-RS detection operations that 5G-L1-UL can perform per DL/UL switching period and per component carrier. A RIM-RS detection operation is the detection of one sequence over one detection resource unit. RIM-RS capacity for L1(Radio) is detailed in 5G_UP_12766. (ID: 11470816)
