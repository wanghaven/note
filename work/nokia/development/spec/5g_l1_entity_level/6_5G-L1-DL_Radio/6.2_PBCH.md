### 6.2.3 Dynamic Configuration/RE mapping

**6.2.3.0-2**  (ID: `12707839`)

Dynamic Configuration/RE mapping
When eCPRI-7-2e FH is used, after received *DlData\_SsBlockSendReq message from L2, 5G\_L1\_DL(BB) will encapsule all information in this message to* DlDataFH\_SsBlockSendReq message and send it to 5G\_L1\_DL(RU) via L1sw\_common\_FH. For more detail, refer to 5G\_L1\_IF\_15031\_replaced([11516633](https://dn-prod.ext.net.nokia.com/rm/resources/BI_FSFR3PvxEe-AqvopbP1qhQ)).\*\*

Upon reception of the *DlDataFH\_SsBlockSendReq message, 5G-L1-DL(RU) shall generate and transmit SS blocks in the* sfn and *slot indicated in the* DlDataFH\_SsBlockSendReq message (5G\_L1\_IF\_15029\_replaced([11516645](https://dn-prod.ext.net.nokia.com/rm/resources/BI_FSFR4PvxEe-AqvopbP1qhQ))).\*\*\*\*

Which SS blocks are transmitted is indicated via a 4-bit bitmap under parameter *activeSsBlocks that is part of the* DlDataFH\_SsBlockSendReq message. The parameter *threeLsbSsbIndex in the* DlDataFH\_SsBlockSendReq message is three least significant bits (38.213 section 4.1) used to indicate the SS block index together with the three most significant 3 bits conveyed in the MIB (38.212 section 7.1.1).\*\*\*\*

Notes:

1. For the contents of MIB, refer to 5G\_L1\_2437\_replaced([11485174](https://dn-prod.ext.net.nokia.com/rm/resources/BI_fD1qkfvvEe-AqvopbP1qhQ)), 5G\_L1\_4369\_replaced([11428865](https://dn-prod.ext.net.nokia.com/rm/resources/BI_e-9KlvvvEe-AqvopbP1qhQ)) and 5G\_L1\_4445\_replaced([11428958](https://dn-prod.ext.net.nokia.com/rm/resources/BI_e-9KmPvvEe-AqvopbP1qhQ)).
2. For SsbConfiguration(ssBlockPower, ssBlockPrbOffset, ssBlockSubcarrierOffset and ssBlockConfiguration) which was carried in DlCell\_SetupReq, will be sent to RU by M-Plane within SOAP message ModulationLink:ssbBlock(ssBlockPower, ssBlockPrbOffset, ssBlockSubcarrierOffset and ssBlockConfiguration).
3. For dynamic SsbConfiguration(ssBlockPower) which was carried in DlCell\_ParameterConfigurationReq, will be sent to RU by M-Plane within SOAP message ModulationLink:ssbBlock with new ssBlockPower.

##### 6.2.7.4.3 PSS/SSS

**6.2.7.4.3.0-2**  (ID: `12707853`)

PSS/SSS
For **synchronization signals, logical antenna ports may be mapped to either the one or the other polarization of the X-Pol-Beam or to both. An R&D parameter is required which allows to switch between these three configurations. This R&D parameter *rdSyncSignalsTxMode is defined in requirement  5G\_L1\_8418\_replaced([11445204](https://dn-prod.ext.net.nokia.com/rm/resources/BI_e_vNuPvvEe-AqvopbP1qhQ)).***

Logical antenna port used for synchronization signals is logical antenna port 4000.

The mapping of the logical antenna ports for synchronization signals to polarizations shall be done as given in below table.

|  |  |  |  |
| --- | --- | --- | --- |
| *rdSyncSignalsTxMode* | Logical antenna port(s) for synchronization signals | Polarization |  |
|  |  |  |  |
| 0 | 4000 | polarization +45° |  |
| 1 | 4000 | polarization -45° |  |
| 2 | **4000** | polarizations +/-45° |  |

If *rdSyncSignalsTxMode is set to 2, multiplexing (with a potential sign inversion) should be applied to the synchronization signals. This multiplexing operation is described in the section 3.3 of 5GMax algorithm specification document (5G\_L1\_2411\_replaced([11481180](https://dn-prod.ext.net.nokia.com/rm/resources/BI_fDcB_vvvEe-AqvopbP1qhQ))).*

If *rdSyncSignalsTxMode is set to 2, 5G-L1-DL uses the IE* PrecodingVectorIndex retrieved from *DlData\_SsBlockSendReq message to select the appropriate Precoding Vector (PV) specified in the section 3.3 of 5GMax algorithm specification document (5G\_L1\_2411\_replaced([11481180](https://dn-prod.ext.net.nokia.com/rm/resources/BI_fDcB_vvvEe-AqvopbP1qhQ))).*\*\*

* If *PrecodingVectorIndex = 0; 5G-L1-DL selects its corresponding PV*
* If *PrecodingVectorIndex = 1; 5G-L1-DL selects the other PV*

The mapping between the *PrecodingVectorIndex and the respective precoding vector is handled internally in 5G-L1-DL.*

During a given SSB period (let's denote *n), L2-PS fills the IE* PrecodingVectorIndex with the same value (either 0 or 1, see 5G\_UP\_ALG\_4109\_replaced([11173773](https://dn-prod.ext.net.nokia.com/rm/resources/BI_kuV1ffvSEe-AqvopbP1qhQ))) which means 5G-L1-DL uses the same PV during this period. For next SSB period (i.e. *n+1), the value of* PrecodingVectorIndex changes and 5G-L1-DL uses the other PV.\*\*\*\*

The mapping between the PrecodingVectorIndex and the respective precoding vector is handled internally in 5G-L1-DL.

For next SSB period (i.e. *n+1), the value of* PrecodingVectorIndex changes and 5G-L1-DL uses the other PV.\*\*\*\*
