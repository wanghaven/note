## 11.4.3 FCP

### 11.4.3.1 General

#### 11.4.3.1.0-13 General 5G-L1-HI shall support Fast Control Plane UL packets using section type 3 for PRACH. (ID: 11476408)

### 11.4.3.2 Common Requirements

#### 11.4.3.2.0-7 Common Requirements 5G-L1-HI shall use value of DlData_CsiRsSendReq-&gt;CsiRsResource-&gt;patternId  to set beamId  field of eCPRI FCP message corresponding to resources in which SSB transmission is scheduled. (ID: 11476442)

### 11.4.3.3 Single Section Operation Mode

**11.4.3.3.0-7**  (ID: `11476476`)

Single Section Operation Mode
5G-L1-HI for non-BF for PRACH shall fill FCP Section Type 3 parameters as follows:

- payloadVersion=001b

- filterIndex=1

- startSymbolid =0

- numberOfsections=1

- timeOffset =3168

- frameStructure =10101100b

- cpLength =0

- udCompHdr =0

- sectionId=4095

- rb=0

- symInc=0

- startPrbc=0

- numPrbc= 70

- reMask=0xFFF

- numSymbol  =1

- beamId =0

### 11.4.3.4 FD BF section mode

#### 11.4.3.4.1 Common

##### 11.4.3.4.1.0-5 Common 5G-L1-HI shall fill sectionId field of UL FCP section with value of eCpriSectionId field received in corresponding UlData request entry. (ID: 11476508)

##### 11.4.3.4.1.1 ext6

###### 11.4.3.4.1.1.0-6 ext6 When FD Beamforming is enabled, 5G-L1-HI shall fill symbolMask field of section extension #6 to define set of symbols corresponding to the FCP section. (ID: 11476536)

##### 11.4.3.4.1.2 ext12

###### 11.4.3.4.1.2.0-9 ext12 When FD Beamforming is enabled and DlCell_SetupReq -&gt; actDlEcpriExtType12 = true, the size of DL FCP message constructed by 5G-L1-HI shall not exceed MTU size (1500 bytes, including 8 bytes for transport header). 5G-L2-PS shall schedule L1 resources in a way that all needed FCP sections can fit into this limit. (ID: 11476579)

#### 11.4.3.4.2 PDCCH

##### 11.4.3.4.2.0-1 PDCCH When FD Beamforming is enabled and extType = 12, 5G-L1-HI shall merge PDCCH fragments into one section with the same eCpriSectionId (different DCI in the same beam). (ID: 11476586)

#### 11.4.3.4.3 PDSCH

##### 11.4.3.4.3.0-1 PDSCH When FD beamforming is enabled and extType =12, in case of PDSCH being frequency multiplexed with CSI-RS (TRS) the same usage of priorities overlays applies as in the case of extType 6. (ID: 11476594)

#### 11.4.3.4.4 CSI-RS

##### 11.4.3.4.4.0-3 CSI-RS 5G-L1-HI shall address a pair of CSI-RS for Tracking resources (with DlData_CsiRsSendReq-&gt;startSymbol equal to n and n+4, respectively) in a single FCP section when DL FD Beamforming is enabled. symbolMask field of extension type #6 shall be used to indicate allocated symbols. (ID: 11476610)

#### 11.4.3.4.5 PUCCH

##### 11.4.3.4.5.0-9 PUCCH 5G-L1-HI shall support separate eCPRI FCP section for each PUCCH resource with restriction that there should be one section for all PUCCH resources multiplexed in the same PRB. (ID: 11476650)

## 11.4.4 IQ data processing

### 11.4.4.0-6 IQ data processing 5G-L1-Hi shall use the selected scaling factors for reporting absolute noise and signal power to the higher layers from the fixed precision Receiver outputs. (ID: 11476678)
