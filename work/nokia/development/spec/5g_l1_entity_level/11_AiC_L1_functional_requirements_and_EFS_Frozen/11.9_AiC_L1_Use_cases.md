## 11.9.1 Overview

**11.9.1.0-4**  (ID: `11477815`)

Overview
Use Case formating

Parameters expression/notation rules:

* Use square brackets [] notation to express continue range of values with type eg. INT [1..8], FLOAT [0...100],
* Use curly brackets {} notation to express range as list of values eg. ENUM {Format0, Format1, Format2, Format3}
* Parameter value expression should show full supported range available for described UseCase. Functional requirements documented in other chapters of L1\_requirements should cover specific information about availability for feature and software release.
* Parameter name meaning should be described in functional requirements documented in other chapters of L1\_requirements and should cover specific information about availability for feature and software release

Text formatting rules:

-Style of parameter name in italic

-Style of L1 procedure name in bold

-Style of actor name in bold

| Exception | If 5G-L1-HI doesn’t receive message SymbolEvent or UData\_PrachReceiveReq (Step 2 and/or Step 3) PRACH processing is dropped – no further steps are realized in 5G-L1-HI. |
