# 2.1 SM-RM IDT Requirements

## 2.1.1 CPRI IDT Requirements

### 2.1.1.1 Requirements for CPRI-N

**2.1.1.1.0-1**  (ID: `11415912`)

Requirements for CPRI-N
For Nokia radios, SM-RM CPRI L1L2 interface specification (Reference: [13349466](https://dn-prod.ext.net.nokia.com/rm/resources/TX_2aGsAOqlEfCnwLcpJWh_Zw)) chapter 4.1.4 IQ power scaling states below requirements for 5G CPRI:

DL-direction (IQ sample width: 30 bit (15 bit for I and 15 bit for Q))

Relationship between RMS-level of the DL IQ-data on the CPRI interface and maximum 5G carrier power at the antenna connector shall be SW-configurable. Required configuration:

VRMS = sqrt[mean(I2+Q2)] = 2012[dec] @ configured maximum 5G/LTE carrier power

This RMS IQ-level would be ~ 18.2dB below full scale (**-18.2dBFS) as adopted from OBSAI [3] and would thus not produce any noticeable interface clipping effects (avoidance of EVM degradation). This value is stored in the BTS Property File (BPF).**

UL-direction (IQ sample width: 30 bit (15 bit for I and 15 bit for Q))

For 5G AxC signals no digital AGC-functionality is required since 15bit per I and Q sample data is available and capable of covering the expected UL-power dynamic range. Instead, a fixed mapping between actual UL input power at the antenna connector and the RMS IQ-level on the CPRI-interface shall be used that shall be SW configurable. Following Table 4‑1 lists the Rx Scaling Factors for the relevant 5G bandwidth cases. In total four 5G bandwidth groups are defined with one Rx Scaling Factor per group being calculated as follows:

*Rx Scaling Factor* = -174 dBm/Hz – 21.425 dB + 10dB*log*10*(Fs\_min) + 10dB*log*10*(6)**\***\*

(*Fs\_min = smallest sample rate of the respective bandwidth group in samples/s, e.g. 30.72·106 samples/s in case of the second group starting with a 5G BW of 25 MHz). By means of this mapping, it is guaranteed, that – even in case of a noiseless receiver (NF = 0dB) – the interface quantization noise is at least 21.425 dB below thermal noise power, meaning a receiver desensitization of below 0.04 dB.*

Required *Rx Scaling Factor values for 5G as below table:*

|  |  |  |
| --- | --- | --- |
| RAT | BW [MHz] | *Rx Scaling Factor* |
| 5G | 5 / 10 / 15 / 20 | -118.79 dBm |
| 5G | 25 / 30 / 40 / 50 | -112.77 dBm |
| 5G | 60 / 70 / 80 / 90 / 100 | -108.00 dBm |
| 5G | 200 / 400 | -103.73 dBm |

### 2.1.1.2 Requirements for CPRI-A - 5G FDD only

**2.1.1.2.0-1**  (ID: `11415922`)

Requirements for CPRI-A - 5G FDD only
For ALU radios, the IQ sample width for both DL & UL are 30 bit (15 bit for I and 15 bit for Q) which is same as the one for Nokia radios, while there are still differences as below according to CPRI-A Power Scaling definition (Reference: [13349469](https://dn-prod.ext.net.nokia.com/rm/resources/TX_dvBncOqnEfCnwLcpJWh_Zw))

The required CPRI-A downlink RMS IQ level is VRMS = sqrt[mean(I2+Q2)] = 4024[dec] @ configured maximum 5G/LTE carrier power. This RMS IQ-level would be ~ 12.2dB below full scale (**-12.2dBFS).**

CPRI-A uplink has also different digital power level representations, it uses thermal noise at the RU antenna to define the power level (based on average power of IQ samples) and is set to -**64.42 dBFs. This level is fixed regardless of the 5G/LTE carrier bandwidth.**

Since thermal noise power at the RU antenna input scales with carrier bandwidth:

*Thermal noise power* = -174 dBm/Hz + 10dB*log*10*(carrier bandwidth in Hz) + noise figure*\*\*\*

That means it is RU’s responsibility to provide proper scaling based on carrier bandwidth to reach the specific fixed power level at the CPRI port.

Note that -64.42 dBFs is calculated based on the agreement to associate 2.8 bits of the 15-bit IQ dynamic range as thermal noise.

With the same rx scaling factor as CPRI-N, the noise level between CPRI-N and CPRI-A can be calculated as the table below.

|  |  |  |  |  |
| --- | --- | --- | --- | --- |
| RAT | BW [MHz] | *Rx Scaling Factor* | *Noise level (Nokia)* | *Noise level (ALU)* |
| 5G | 5 | -118.79 dBm | -69.52 dBFs | -64.42 dBFs |
| 5G | 10 | -118.79 dBm | -66.51 dBFs | -64.42 dBFs |
| 5G | 15 | -118.79 dBm | -64.75 dBFs | -64.42 dBFs |
| 5G | 20 | -118.79 dBm | -63.50 dBFs | -64.42 dBFs |

The difference shall be compensated at L1-SW in case of CPRI-A configuration and rx Scaling factor is kept same for both CPRI-N and CPRI-A.

### 2.1.2.0-1 eCPRI IDT Requirements ORAN-WG4.CUS specification (Reference: 13349472 ) chapter 6.1.3 digital power scaling states below requirement for 5G eCPRI. (ID: 11415933)

### 2.1.2.1 Definition of IQ Power in dBFs

**2.1.2.1.0-1**  (ID: `11415942`)

Definition of IQ Power in dBFs
IQ power level in dBFS (dB full scale) is a logarithmic representation of the power level for an IQ sample carried over the digital interface. IQ power level in dBFS is proportional to logarithm of I²+Q²:

IQ power level [dBFS] = 10·log10( I²+Q² ) - 10·log10(FS)

where FS = max(I²) = max(Q²) = max(I²+Q²) with max over all IQ values that can be represented by interface.

For frequency domain IQ data, 0 dBFS is the maximum power level which can be carried by one subcarrier. The smallest non-zero IQ power level is defined by the interface resolution.

It is expected that an RU will normalize any received DL value to its internal representation of full scale so that a 0 dBFS can be properly handled.

Example:

I=min I, Q=0

With 9bit mantissa 2’s complement + 4bit exponent compression:

min I = -256·2^15 = -2^23 

0 dBFS  average(I²+Q²) = 2^46

Interface resolution ** 1/2^46 ** -138.47dBFs\*\*\*\*

### 2.1.2.2 DL Gain Guideline

#### 2.1.2.2.0-1 DL Gain Guideline The configured DL gain of a TX array carrier is equivalent to mapping 0dBFS is carried over any subcarrier (of the carrier) on the digital interface to the maximum RMS power the TX array carrier element is supposed to output + 10·log10 (the number of array elements). (ID: 11415952)

### 2.1.2.3 UL Gain Definiton

#### 2.1.2.3.0-1 UL Gain Definiton The UL gain or scaling of an rx-array carrier element (i.e. ”rx-array carrier element’’ refers to the rx-array element serving the respective carrier configured on the respective rx-array) is defined by mapping -152dBm at the assumed-lossless antenna port to the smallest power level an IQ sample can carry over the digital interface (i.e., average(I²+Q²) = 1) while considering a configured gain_correction value and the IQ compression properties to be used; valid for each individual spatial stream received by an rx-array carrier element and for any carrier. Note that the -152dBm is considered as the smallest level that can be measured by a narrow subcarrier of 1.25kHz (i.e., CW tone) for a system with 3dB of equivalent noise figure and without being impacted by the interface noise (i.e. 20dB margin considered). (ID: 11415962)

## 2.1.3 OBSAI IDT Requirements

**2.1.3.0-1**  (ID: `11415972`)

OBSAI IDT Requirements
SM-RM\_RP301\_L1L2\_interface\_specification (Reference: [13349474](https://dn-prod.ext.net.nokia.com/rm/resources/TX_2yblEOqoEfCnwLcpJWh_Zw)) chapter 4.1.4 IQ power scaling states below requirements fro 5G NR interface.

DL-direction

Relationship between RMS-level of the DL IQ-data on the OBSAI interface and maximum 5G NR carrier power at the antenna connector shall be configurable by module SW.

Required configuration:

VRMS = 4024 [dec] @ maximum 5G NR carrier power.

This RMS IQ-level would be ~ 18.2dB below full scale (-18.2dBFS) and would thus not produce any noticeable interface clipping effects (avoidance of EVM degradation). This value is stored in the Basestation Property File (BPF).

UL-direction

For 5G NR AxC signals no digital AGC-functionality is required since 16bit per I and Q sample data is available and capable of covering the expected UL-power dynamic range. Instead, a module-SW configurable mapping between the actual UL input power at the antenna connector PRX and the RMS IQ-level on the OBSAI-interface shall be implemented.

UL IQ-data mapping shall be done according to an *RxPowerScaling value that is defined as follows:*

VRMS,UL(PRX = *RxPowerScaling) = 1*

Required Target value for RxPowerScaling: RxPowerScaling = -126.31dBm

Exception: 900 MHz RM frequency variant:  RxPowerScaling = -120.31dBm

Note: the mentioned "no digital AGC-functionality" is for RRU, do not mix it with the T-AGC/ISS in BBU.
